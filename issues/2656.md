#### <img src="https://avatars.githubusercontent.com/u/275142?v=4" width="50">[benblank](https://github.com/benblank) commented at [2023-01-12 21:45](https://github.com/koalaman/shellcheck/issues/2656#issuecomment-1381029140):

I do believe there is value in having the CLI properly documented online, particularly for users of environments which give access to customizing ShellCheck's command-line arguments, but not to the `shellcheck` executable itself.

For example, I use the [ShellCheck extension for Visual Studio Code](https://marketplace.visualstudio.com/items?itemName=timonwong.shellcheck), which offers the ability to add arbitrary CLI flags via settings, but doesn't actually add the `shellcheck` binary to the path, so running `shellcheck --help` isn't trivial. Worse, while that extension does a good job at staying up-to-date with ShellCheck releases, distros' packages may not, meaning that even though I have installed ShellCheck through `apt`, it remains at v0.8.0, whereas my editor is using the more recent v0.9.0 and I have no guarantee that the CLI hasn't changed between versions.

What holds me back from simply creating a wiki page is that this seems like a slam-dunk for automation rather than something which must be manually updated if/when changes are made to ShellCheck.
