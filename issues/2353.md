#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2021-10-08 19:50](https://github.com/koalaman/shellcheck/issues/2353#issuecomment-939079499):

Like you say, ShellCheck thinks you maybe wanted the `rm` alias used in your `xargs` command, and therefore emits a set of companion warnings: one where `rm` is defined, and one where it's used by `xargs`.

```sh
#!/bin/bash
alias rm='rm --one-file-system'
echo foo | xargs rm
```

Results in

```
Line 2:
alias rm='rm --one-file-system'
      ^-- SC2032: Use own script or sh -c '..' to run this from xargs.
 
Line 3:
echo foo | xargs rm
                 ^-- SC2033: Shell functions can't be passed to external commands.
```

It's very odd that it says `su` instead of `xargs` though. Can you find the line with the associated SC2033 and see what it is?
