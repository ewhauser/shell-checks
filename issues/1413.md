# [\#1413 Issue](https://github.com/koalaman/shellcheck/issues/1413) `closed`: shellcheck is unresponsive on ">/dev/null"

#### <img src="https://avatars.githubusercontent.com/u/952007?v=4" width="50">[szepeviktor](https://github.com/szepeviktor) opened issue at [2018-12-13T14:19:57Z](https://github.com/koalaman/shellcheck/issues/1413):

```bash
#!/bin/bash

Info()
{
    # Informational messages displayed only during `set -x`
    # shellcheck disable=SC2188
    >/dev/null
}

set -e -x

Info "List what boot packages are installed"
```

(strace shows SIGVTALRM)

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2018-12-16T22:50:25Z](https://github.com/koalaman/shellcheck/issues/1413#issuecomment-447684048):

Wow, this was indeed an infinite loop. ShellCheck tried to look up the tree to see whether the command-less redirection's parent was a `$()`, such as `$(< file)` . When it found an annotation, it was supposed to look one more step up the tree, but instead it went one step down. That took it to the command-less redirection again, and the cycle started over.

Excellent catch, thanks!
