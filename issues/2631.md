- [x] https://www.shellcheck.net/ (i.e. the latest commit) currently gives no useful warnings about this

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash
cleanup() {
    echo Time to clean up!
}
trap cleanup EXIT
exec sleep 1
```

#### Here's what shellcheck currently says:

```
No issues detected!
```

#### Here's what I wanted or expected to see:

I'd like an error on the exec, noting that using exec from a script with a `trap ... EXIT` will bypass that exit trap, which is very likely not intended by the script author.

#### <img src="https://avatars.githubusercontent.com/u/5316?u=8763bdb9ba5cbe23084e8249dcbd21bbb23150f2&v=4" width="50">[mcandre](https://github.com/mcandre) commented at [2023-01-29 18:15](https://github.com/koalaman/shellcheck/issues/2631#issuecomment-1407733199):

Please warn on the use of any non-function-local traps, *regardless* of the particular trap signal(s) (e.g., exit, error, interruption, ...) with `exec`.

The script author is more likely to have made a mistake and assumed that the traps would persist to safeguard the control flow of the exec'ed command.
