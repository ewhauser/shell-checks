But the `SC1089` seems to be a false positive.

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/sh

# shellcheck disable=SC2006,SC2039,SC2046,SC1117
if [ .`echo -n` = .-n ]
then
    echon() { echo "$1\c"; }
    echoe() { echo "$1\c"; }
else
    echon() { echo -n "$1"; }
    if [ .`echo -e` = .-e ]
    then echoe() { echo -n "$1"; }
    else if [ .`echo -e '\070\c'` = .8 ]
         then echoe() { echo -e "$1\c"; }
         else echoe() { echo -n -e "$1"; }
         fi
    fi
fi
if [ .`echoe '\070'` != .8 ]
then echoe(){ printf "%b" "$*" 2>/dev/null ; }
fi
if [ .`echoe '\171'` = .y ]
then o(){ echoe "`printf '\\\\%03o' $1`" ; }
else o(){ echoe "`printf '\\\\%04o' $1`" ; }
fi

```

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2017-07-08T23:34:01Z](https://github.com/koalaman/shellcheck/issues/948#issuecomment-313887296):

You're right, of course. Your use of `else if` is perfectly correct, and ShellCheck is wrong.

Empirically, most of the time when people write `else if`, they think it's the same as `elif`. ShellCheck goes along with that -- even though it's incorrect -- because it gives these users better help. They clearly need it since they can't write `if` statements.

I know it's awful to ask anyone to change a perfectly correct script, but I hope you'll consider putting the `if` on a separate line as suggested to allow ShellCheck to be more helpful to people who don't know the syntax as well as you do.
