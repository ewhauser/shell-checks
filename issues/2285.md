#### Here's a snippet or screenshot that shows the problem:

```sh
#!/opt/brew/bin/bash
declare -A commands

commands[aa]=3

[[ -v commands[a] ]] && echo a
[[ -v commands[aa] ]] && echo aa
[[ -v commands[aaa] ]] && echo aaa
```

**Output**

```
$ ./myscript
aa
```

#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2021-07-25 20:06](https://github.com/koalaman/shellcheck/issues/2285#issuecomment-886251254):

ShellCheck parsed the arguments to `-v` (and also other dereferencing operators like `-gt` and `-eq`) as regular shell words, causing the `[..]` to be considered a glob range. The warning is now suppressed whenever the word is used as an argument to such an operator. Thanks!
