#### Here's a snippet or screenshot that shows the problem:

```sh

#!/bin/bash

set -e

eval "$(some-command-which-could-fail)"
```

#### Here's what shellcheck currently says:

Nothing

#### Here's what I wanted or expected to see:

Instead of using eval directly on result of `"$(some-command-which-could-fail)"`, it would be better if it was written like this:

```sh

#!/bin/bash

set -e

foo="$(some-command-which-could-fail)"
eval "${foo}"
```
