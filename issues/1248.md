# [\#1248 Issue](https://github.com/koalaman/shellcheck/issues/1248) `open`:  feature: check for options (-e etc) in shebang

#### <img src="https://avatars.githubusercontent.com/u/328602?v=4" width="50">[pabs3](https://github.com/pabs3) opened issue at [2018-06-04T04:03:23Z](https://github.com/koalaman/shellcheck/issues/1248):

#### For new checks and feature suggestions

- [x] shellcheck.net (i.e. the latest commit) currently gives no useful warnings about this
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related

#### Here's a snippet that shellcheck should flag:

```sh

#!/bin/sh -e
false
echo true
```

#### Here's a snippet that illustrates the problem:

```
$ ./foo.sh 
$ sh ./foo.sh 
true
$ sh -x ./foo.sh 
+ false
+ echo true
true
```

#### Here's what shellcheck currently says:

```
$ shellcheck foo.sh
```

#### Here's what I wanted or expected to see:

```
$ shellcheck foo.sh
In foo.sh line 1:
#!/bin/sh -e
          ^-- SCXXXX: Options in shebang lines are lost when passing the script to sh, use 'set -e' instead.
```
