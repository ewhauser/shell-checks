# [\#1738 Issue](https://github.com/koalaman/shellcheck/issues/1738) `open`: Provide a bash4 compatibility mode

#### <img src="https://avatars.githubusercontent.com/u/24410343?v=4" width="50">[vvug](https://github.com/vvug) opened issue at [2019-10-27T11:54:01Z](https://github.com/koalaman/shellcheck/issues/1738):

It would be nice if shellcheck supported a bash4 compatibility mode. I recently wrote a script on my system using bash5 and using the `EPOCHSECONDS` variable; no problems were spotted by shellcheck, the script works fine. The problem is that I'm going to run this script on systems where only bash4 is available. With a bash4 compatibility mode I could have used shellcheck to be sure that my script was bash4-safe.

#### <img src="https://avatars.githubusercontent.com/u/19507282?v=4" width="50">[automorphism88](https://github.com/automorphism88) commented at [2019-12-24T18:01:52Z](https://github.com/koalaman/shellcheck/issues/1738#issuecomment-568785915):

Every version of bash has introduced new features though. Do you want a bash 4.4, 4.3, 4.2, 4.1, 4.0, 3.x (still in use on Mac OS) mode? Bash 5 actually introduced relatively little in new features compared to many of the 4.x point releases (e.g. namerefs in 4.3).

If you need a script to work on old bash versions, you can check [here](https://mywiki.wooledge.org/BashFAQ/061) for a summary of what's new. Or if your script relies on a newer feature, you can make it check `$BASH_VERSION`/`$BASH_VERSINFO` at runtime and exit with an error message if the bash version is incompatible.
