#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2017-09-09T22:41:28Z](https://github.com/koalaman/shellcheck/issues/995#issuecomment-328307721):

ShellCheck is right. `array` is not used here.

You can do `doSomething "${array[@]}"` to actually reference the array, otherwise it'll pass the string `array[@]`  literally (or glob expanded).

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2017-09-10T01:28:07Z](https://github.com/koalaman/shellcheck/issues/995#issuecomment-328313541):

It's shellcheck's intention not to try to figure out whether or not a variable is accessed through indirection.

The canonical way of doing this is to pass arrays as multiple positional parameters after optional flags, e.g. `myfunction -a foo -b bar "${args[@]}"`

If you still want to do it through indirection, ignore this warning.
