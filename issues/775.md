#### <img src="https://avatars.githubusercontent.com/u/197253?v=4" width="50">[kurahaupo](https://github.com/kurahaupo) commented at [2016-11-14T03:59:34Z](https://github.com/koalaman/shellcheck/issues/775#issuecomment-260243655):

It depends on which version of Bash you're using.

The %(...)X format control was introduced relatively recently (in 4.2 IIRC).
For any older version of Bash the warning is correct, but in addition %( was
not a valid conversion.


#### <img src="https://avatars.githubusercontent.com/u/3155552?v=4" width="50">[slodki](https://github.com/slodki) commented at [2016-11-27T19:46:41Z](https://github.com/koalaman/shellcheck/issues/775#issuecomment-263142738):

%()T was added in [bash 4.2](http://git.savannah.gnu.org/cgit/bash.git/commit/builtins/printf.def?id=495aee441b75276e38c75694ccb455bb6463fdb9), 5 years ago.

% chars inside () cannot be count as required parameters (empty format string, %()T is valid too). [checkPrintfVar](https://github.com/koalaman/shellcheck/blob/08f7ff37c590f8aa6dba6d989d707aad4adc5f4c/ShellCheck/Checks/Commands.hs#L473) should be rewritten to skip them.

If %()T is last format variable the corresponding printf parameter could be skipped (default will be used), so shellcheck shouldn't raise en error in this case.
