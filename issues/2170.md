# [\#2170 Issue](https://github.com/koalaman/shellcheck/issues/2170) `open`: Content of files included by "source" are ignored

#### For bugs
- Rule Id SC2154
- My shellcheck version (`shellcheck --version` or "online"): Bundled with PHPStorm 2020.3.2

- [ ] The rule's wiki page does not already cover this (e.g. https://shellcheck.net/wiki/SC2086)

- [ ] I tried on https://www.shellcheck.net/ and verified that this is still a problem on the latest commit


#### Here's a snippet or screenshot that shows the problem:

_File config.sh_
```
#!/bin/bash
podName="traefik"
```

_File init.sh_
```
#!/bin/bash
source config.sh
podman pod create --name "${podName}" -p80:80 -p443:443
```

#### Here's what shellcheck currently says:
podName is referenced but not assigned.
See SC2154.


#### Here's what I wanted or expected to see:
Nothing. "podName" is defined in config.sh and included by _source config.sh_
