# [\#2859 Issue](https://github.com/koalaman/shellcheck/issues/2859) `open`: Possible to globally exclude patterns or literal strings somehow?

#### <img src="https://avatars.githubusercontent.com/u/1992842?u=44655cad67cb71582979c4c15c12285d751dae99&v=4" width="50">[luckman212](https://github.com/luckman212) opened issue at [2023-11-09 22:01](https://github.com/koalaman/shellcheck/issues/2859):

I am wondering if there's any way to exclude specific patterns (fixed strings, globs or regexes) from triggering [SC2154](https://github.com/koalaman/shellcheck/wiki/SC2154)

#### Situation

I export certain variables for formatting ANSI bold, colors etc in my `.bash_profile`
But, most of my scripts trigger SC2154 because shellcheck doesn't know about these declared vars. I don't want to completely disable the check, because it's very useful! But, I just want to exclude a few specific vars or strings from triggering it.

E.g. in `.bash_profile`:
```bash
export Fg=$(printf '\x01\e[1;32m\x02')   # green
export Fn=$(printf '\x01\e[0m\x02')      # normal
```

And dumb example script
```bash
#!/bin/bash
echo "Hello ${Fg}green${Fn}"  # <== triggers SC2154 warning
```

In that case I would love if there was a `~/.shellcheckignore` file or something where I could put literal strings

```
${Fg}
${Fn}
```

Thank you 🙏
