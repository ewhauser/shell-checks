#### Here's a snippet or screenshot that shows the problem:

```sh
#! /bin/bash

func () {
   eval "$1"
   if [[ $? -ne 0 ]]; then
      echo -e "\nAn error occured running: $1"
      exit 1
   fi
}

func2 () {
   if ! $1; then
      echo -e "\nAn error occured running: $1"
      exit 1
   fi
}

func "pwd && ls"  # executes both commands
func2 "pwd && ls" # only executes first command
```

#### Here's what shellcheck currently says:
```
Line 5:
   if [[ $? -ne 0 ]]; then
         ^-- SC2181: Check exit code directly with e.g. 'if mycmd;', not indirectly with $?.
```


#### Here's what I wanted or expected to see:
Shellcheck should not suggest changes that aren't equivalent. If there is a better way of accomplishing `func()`, I'd love to learn it. Perhaps this caveat with logical operators could be amended to this rule or the wiki entry.

#### <img src="https://avatars.githubusercontent.com/u/25598447?u=73c3f3a041be0ccabd72bb816d5782b30aefe822&v=4" width="50">[TinCanTech](https://github.com/TinCanTech) commented at [2021-04-02 16:18](https://github.com/koalaman/shellcheck/issues/2188#issuecomment-812599157):

You have two non-equivalent commands:
1. `eval` `"$1"`
2. `if` `!` `$1`

`shellcheck` is correct, your code is not.

```
#!/bin/bash

# https://github.com/koalaman/shellcheck/issues/2188

func_Orig () {
   eval "$1"
   if [[ $? -ne 0 ]]; then
      echo -e "\nAn error occured running: $1"
      exit 1
   else
      echo
   fi
}

func_Equiv () {
   if ! eval "$1"; then
      echo -e "\nAn error occured running: $1"
      exit 1
   else
      echo
   fi
}

func_notEq () {
   if ! $1; then
      echo -e "\nAn error occured running: $1"
      exit 1
   else
      echo
   fi
}

func_Orig   "echo '*func_Orig:' && pwd && ls"
func_Equiv  "echo '*func_Equiv:' && pwd && ls"
func_notEq  "echo '*func_notEq:' && pwd && ls"
```
