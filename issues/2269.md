# [\#2269 Issue](https://github.com/koalaman/shellcheck/issues/2269) `closed`: MaxOS shellcheck binary is not notarized

#### <img src="https://avatars.githubusercontent.com/u/290311?u=ebcd893f02ee419d3b7b6f63018d422b25dfec34&v=4" width="50">[adamawolf](https://github.com/adamawolf) opened issue at [2021-07-01 16:11](https://github.com/koalaman/shellcheck/issues/2269):

I wanted to incorporate shellcheck into my organization's tool chain to run as a precommit on each developer's machine. I planned to include the prebuilt binary as an artifact that gets downloaded when the developer initializes their working copy of our source code. However, when first running the executable we're greeted with:

<img width="428" alt="Screen Shot 2021-07-01 at 9 06 49 AM" src="https://user-images.githubusercontent.com/290311/124155811-bb9d9080-da4b-11eb-9da2-603468bc6b58.png">

This is because shellcheck is not notarized. 

It would be amazing if future versions could follow [these steps](https://developer.apple.com/documentation/security/notarizing_macos_software_before_distribution) to notarize the pre-built binaries.

Either way, thanks for this great tool!

#### <img src="https://avatars.githubusercontent.com/u/290311?u=ebcd893f02ee419d3b7b6f63018d422b25dfec34&v=4" width="50">[adamawolf](https://github.com/adamawolf) commented at [2021-07-14 19:59](https://github.com/koalaman/shellcheck/issues/2269#issuecomment-880169717):

I discovered this is just the "quarantine" file attribute that gets set when the file originates from a web browser. I can unset it on the binary using `sudo xattr -r -d com.apple.quarantine shellcheck` and this allows me to distribute it to the rest of my team as desired. Closing.
