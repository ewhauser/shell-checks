# [\#3189 Issue](https://github.com/koalaman/shellcheck/issues/3189) `open`: SC2086 with parameter expansion ':-' but not with ':+'

#### <img src="https://avatars.githubusercontent.com/u/33481653?u=8ce2ad088e5e364b427184fb7bc5d53b9dc906c6&v=4" width="50">[thranur](https://github.com/thranur) opened issue at [2025-04-15 14:56](https://github.com/koalaman/shellcheck/issues/3189):

In the following script, the first echo gets an SC2086, but not the second one:

```sh
#!/bin/bash
echo ${VAR:-}
echo ${VAR:+$VAR}
```

I fail to see why that is the case; I thought that either both should be warned about, or none should.

In particular, if I run it with `VAR="a    b"` (with several spaces), in both cases, without double quotes, they echo `a b` (single space), and with double quotes, both echo `a    b`. So, for me, `${VAR:+$VAR}` should also deserve a SC2086 (but `${VAR:+"$VAR"}` is fine).
