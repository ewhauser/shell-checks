# [\#972 Issue](https://github.com/koalaman/shellcheck/issues/972) `open`: Enhancement: Exclude non-shell scripts from linting

#### <img src="https://avatars.githubusercontent.com/u/5316?v=4" width="50">[mcandre](https://github.com/mcandre) opened issue at [2017-08-16T16:03:57Z](https://github.com/koalaman/shellcheck/issues/972):

For large projects with many shell and non-shell files, it can be difficult to find the right syntax to get ShellCheck to lint all shell scripts, that is, *just* the shell scripts and nothing else. One can construct a UNIX find command, such as `find lib -name "*.sh" -exec shellcheck {} \;`, though this is not a one-size-fits-all, if your shell scripts sometimes lack extensions or are housed in different directory structures. Accordingly, it would be helpful if ShellCheck automatically ignored file paths that are not actually shell scripts.

As a workaround, I'm releasing [stank](https://github.com/mcandre/stank), so users can `stank . | xargs shellcheck`. It's not a Haskell application, but there are binary ports for macOS, Windows, Linux, and other platforms so at least it's relatively easy to obtain, even if you're not a hardcore Go developer.

#### <img src="https://avatars.githubusercontent.com/u/4790123?v=4" width="50">[ArturKlauser](https://github.com/ArturKlauser) commented at [2020-01-14T18:59:12Z](https://github.com/koalaman/shellcheck/issues/972#issuecomment-574321956):

You can also find shell scripts with [identify](https://github.com/chriskuehl/identify).

```shell
for f in $(find "$dir" -type f); do identify-cli "$f" | grep -q '"shell"' && shellcheck "$f"; done
```
