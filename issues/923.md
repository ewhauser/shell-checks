#### <img src="https://avatars.githubusercontent.com/u/2199511?v=4" width="50">[Nightfirecat](https://github.com/Nightfirecat) commented at [2017-06-02T07:25:13Z](https://github.com/koalaman/shellcheck/issues/923#issuecomment-305710879):

I don't think this is a false positive. Realistically, your code is functionally the same as writing the following, which generates no such notice. (note the added space)

```
#!/usr/bin/env bash
declare \
        verbose='0' \
        debug='0'

echo "${verbose} ${debug}"
```

For code style purposes, it makes more sense to me to add the extra space following the `verbose='0'` line, as there's no functional impact. Typically, placing backslashes immediately following a quoted string is used to continue long strings.
