# [\#1995 Issue](https://github.com/koalaman/shellcheck/issues/1995) `open`: Report superfluous "return" uses

#### <img src="https://avatars.githubusercontent.com/u/191204?v=4" width="50">[vpelletier](https://github.com/vpelletier) opened issue at [2020-06-25 00:47](https://github.com/koalaman/shellcheck/issues/1995):

- [x] shellcheck.net (i.e. the latest commit) currently gives no useful warnings about this
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related


#### Here's a snippet or screenshot that shows the problem:

```sh

#!/bin/sh
foo () {
  true
  return $?
}

```

#### Here's what shellcheck currently says:

```
No issues detected!
```

#### Here's what I wanted or expected to see:

Possibly with some optional checker(s) enabled:
- a warning about `$?` being superfluous, as it is the implicit default (or is it just in bash ? I could not locate posix shell specification on return statement)
- a warning about `return` being superfluous, as it is the last statement before function end (ideally, it would also be reported in any nested bloc when there is no other statement until function end)

I think the former is probably good to enable by default. The latter is probably best left to be explicitly enabled.
