# [\#339 Issue](https://github.com/koalaman/shellcheck/issues/339) `closed`: shellcheck: user error (Text.Regex.Posix.String died: (ReturnCode 17,"illegal byte sequence"))

#### <img src="https://avatars.githubusercontent.com/u/10622042?v=4" width="50">[apennebaker-ni](https://github.com/apennebaker-ni) opened issue at [2015-04-02T15:46:28Z](https://github.com/koalaman/shellcheck/issues/339):

When I try to shellcheck my wget-memegenerator script, ShellCheck fails on a regex error.

**Trace**

```
$ shellcheck lib/wget-memegenerator 
shellcheck: user error (Text.Regex.Posix.String died: (ReturnCode 17,"illegal byte sequence"))
```

Update:

I narrowed down the problem to the ASCII `»` symbol in:

```
PAGES=$(wget -q -O- "http://memegenerator.net/$TAG/images/new/alltime/page/1" | grep » -B 6 | grep page | head -n 1 | tr "[<>]" " " | awk '{ print $3 }')
```

That's where shellcheck's regex fails.

https://github.com/mcandre/politerips/blob/shellcheck-339/lib/wget-memegenerator#L13
