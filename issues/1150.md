```sh
#!/bin/bash
f () (( x > 0 ))
```
this shouldn't trigger SC2210

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2018-03-21T17:30:41Z](https://github.com/koalaman/shellcheck/issues/1150#issuecomment-375029349):

Totally agree. However, as a current design decision, ShellCheck only accepts `{ .. }` and `( .. )` for function definitions. This is to improve error diagnostics when people use `()` in strange places and unintentionally start a function definition.
