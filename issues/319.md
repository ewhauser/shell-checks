# [\#319 Issue](https://github.com/koalaman/shellcheck/issues/319) `closed`: SC2143: Possible wrong suggestion

#### <img src="https://avatars.githubusercontent.com/u/864213?v=4" width="50">[bayandin](https://github.com/bayandin) opened issue at [2015-03-01T16:48:29Z](https://github.com/koalaman/shellcheck/issues/319):

In my code I used this condition:

``` bash
if [ -z "$(grep 'Europe/Moscow' /etc/timezone)" ]; then
    echo Europe/Moscow > /etc/timezone
    dpkg-reconfigure --frontend noninteractive tzdata
fi
```

Shellcheck catches this with the following suggestion:

``` bash
if [ -z "$(grep 'Europe/Moscow' /etc/timezone)" ]; then
     ^-- SC2143: Instead of [ -z $(foo | grep bar) ], use ! foo | grep -q bar .
```

My first reaction was fix my condition by something like this:

``` bash
if ! /etc/timezone | grep -q 'Europe/Moscow'; then
...
```

But the right way it is a

```bash
if ! grep -q 'Europe/Moscow' /etc/timezone; then
...
```

It would be better if shellcheck can split cases like `cmd | grep smth` and `grep smth filename`.
