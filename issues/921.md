# [\#921 Issue](https://github.com/koalaman/shellcheck/issues/921) `closed`: Add error for passing stdin to builtins which do not expect it

#### <img src="https://avatars.githubusercontent.com/u/379509?v=4" width="50">[alexjurkiewicz](https://github.com/alexjurkiewicz) opened issue at [2017-05-31T02:30:39Z](https://github.com/koalaman/shellcheck/issues/921):

I just spent an embarrassing amount of time troubleshooting the following snippet:

```sh

#!/bin/bash

echo >foo <<EOF
hello
EOF

```

Of course, I really meant to use `cat` rather than `echo`. shellcheck doesn't give any error for this file, but I think it could detect if you are passing something on stdin to a builtin that doesn't expect it. Example builtins would be `echo`, `source`, `alias`, etc etc.

#### Here's what shellcheck currently says:

No error.

#### Here's what I wanted or expected to see:

SCxxxx: 'echo' builtin does not expect stdin - is this a mistake?
