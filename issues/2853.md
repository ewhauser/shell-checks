You need to specify both -a and -x to make it follow the source.

```
brother@janmayen:/tmp$ cat test.sh
#!/bin/bash

source setup_config.in
brother@janmayen:/tmp$ shellcheck test.sh
brother@janmayen:/tmp$ shellcheck -a test.sh
brother@janmayen:/tmp$ shellcheck -x test.sh
brother@janmayen:/tmp$ shellcheck -ax test.sh

In setup_config.in line 3:
	while str=$(caller $frame) ; do
                           ^----^ SC2086 (info): Double quote to prevent globbing and word splitting.

Did you mean:
	while str=$(caller "$frame") ; do

For more information:
  https://www.shellcheck.net/wiki/SC2086 -- Double quote to prevent globbing ...
```

#### <img src="https://avatars.githubusercontent.com/u/243351?v=4" width="50">[brother](https://github.com/brother) commented at [2023-10-31 21:25](https://github.com/koalaman/shellcheck/issues/2853#issuecomment-1788059394):

That plugin is separate from shellcheck and have them offer a way to set command line option is probably a good idea. Having both a and x enabled for all might not be acceptable. 

The shellcheck help about the command line options.
```
Usage: shellcheck [OPTIONS...] FILES...
  -a                  --check-sourced            Include warnings from sourced files
...
  -x                  --external-sources         Allow 'source' outside of FILES
```

To have your case working as desired you need to have both active as I show earlier.
