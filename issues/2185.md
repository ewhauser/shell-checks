# [\#2185 Issue](https://github.com/koalaman/shellcheck/issues/2185) `open`: SC2086 quoting wiki suggestion and -u not identical for empty array

#### <img src="https://avatars.githubusercontent.com/u/529963?v=4" width="50">[ndevenish](https://github.com/ndevenish) opened issue at [2021-03-24 16:36](https://github.com/koalaman/shellcheck/issues/2185):

The Wiki for SC2086 "Double quote to prevent globbing and word splitting." [suggests](https://github.com/koalaman/shellcheck/wiki/SC2086#exceptions):

> Just quoting this doesn't work. Instead, you should have used an array (bash, ksh, zsh):
> ```
> options=(-j 5 -B) # ksh: set -A options -- -j 5 -B
> make "${options[@]}" file
> ```

However, in the case where this is potentially empty and `-u` is turned on e.g. 
```
set -u
args=()
echo "${args[@]}"
```
This is _unsafe_, depending on bash version - as described [here](https://stackoverflow.com/questions/7577052/bash-empty-array-expansion-with-set-u) - it works on my local bash 5, but broke on the CI system with bash 4 (and a plain mac bash 3).

I'm not sure whether it'd be better to note this and advise disabling the shellcheck if it could potentially be empty, or to suggest some alternate approach, although the `${arr[@]+"${arr[@]}"}` from the StackOverflow thread seems particularly distasteful....
