#### <img src="https://avatars.githubusercontent.com/u/84648683?v=4" width="50">[wileyhy](https://github.com/wileyhy) commented at [2025-07-31 15:55](https://github.com/koalaman/shellcheck/issues/3268#issuecomment-3140494708):

Hello,

It's a little unclear from your post just what exactly you wanted for the
example code to have accomplished.

As-is, the `alias` command would create two separate and functionally
equivalent aliases, `eee` and `fff`. However by enclosing the entire
definition string in single quotes, it's also possible to create the alias
`{eee,fff}`, although this is rarely intended.

In either case, aliases are usually disabled by default in shell scripts,
so it would be necessary to enable them via `shopt -s expand_aliases`.

~ $ set -x; shopt | grep alias; alias {eee,fff}='echo hi'
+ grep alias
+ shopt
expand_aliases on
progcomp_alias off
+ alias 'eee=echo hi' 'fff=echo hi'
~ $ alias '{eee,fff}=echo hi' + alias '{eee,fff}=echo hi'
~ $ alias
+ alias
alias eee='echo hi'
alias fff='echo hi'
alias {eee,fff}='echo hi'
~ $ {eee,fff}
+ echo hi
hi
~ $

Some would discourage use of aliases in shell scripts, as for nearly all
uses aliases have been superceded by functions.

While both of these cases, re `alias`, would be valid code, it's much more
common for people to want for any expansions to occur when an alias is
called, rather than when an alias is defined.

In terms of brace expansion on the left side of any assignment statement,
there are numerous cases where this would produce an error.

~ $ declare "{foo,bar}=quux"
+ declare '{foo,bar}=quux'
bash: declare: `{foo,bar}=quux': not a valid identifier
~ $ {a,b}=( c d e )
bash: syntax error near unexpected token `c'

Any shellcheck tests can be disabled by including a "disable statement" in
your script:

# shellcheck disable=SC2139
