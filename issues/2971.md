# [\#2971 Issue](https://github.com/koalaman/shellcheck/issues/2971) `open`: Feature Request: Provide an option to specify the fallback shell type

#### <img src="https://avatars.githubusercontent.com/u/1070090?v=4" width="50">[moonfruit](https://github.com/moonfruit) opened issue at [2024-04-24 03:35](https://github.com/koalaman/shellcheck/issues/2971):

#### For new checks and feature suggestions
- [x] https://www.shellcheck.net/ (i.e. the latest commit) currently gives no useful warnings about this
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related

#### Description

Some tools like [bash-language-server](https://github.com/bash-lsp/bash-language-server) detect the shell type themselves and add the option `--shell=SHELLNAME` to `shellcheck`.
But this option will overwrite `shellcheck` directive for type in file or `.shellcheckrc`.

I think a new option should be provided to specify a fallback shell type.
