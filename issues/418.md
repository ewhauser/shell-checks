# [\#418 Issue](https://github.com/koalaman/shellcheck/issues/418) `closed`: Erroneous SC2140 when using alias + quotes

#### <img src="https://avatars.githubusercontent.com/u/660663?v=4" width="50">[cdown](https://github.com/cdown) opened issue at [2015-07-25T20:28:00Z](https://github.com/koalaman/shellcheck/issues/418):

Unless I'm misunderstanding, I think this instance of SC2140 is mistaken:

```
alias "$cmd"="$cmd -s"
            ^-- SC2140: The double quotes around this do nothing. Remove or escape them.
```

In both of the double quoted sections, not quoting affects word splitting.

This is with ShellCheck-0.3.8.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2015-07-25T20:56:11Z](https://github.com/koalaman/shellcheck/issues/418#issuecomment-124892760):

It's pointing to the `=` sign and wondering why you decided to add no-op quotes instead of doing 

```
alias "$cmd=$cmd -s"
```

In your case they're just superfluous, but in cases like `echo "<img src="foo.png" />" > file.html` this indicates a problem. 

This message needs a rephrasing and should probably be suppressed when the quoted part is `=` or maybe even non-alphabetical
