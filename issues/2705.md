#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash

command
errors=$((errors + ?))

exit "$errors"
```

#### Here's what shellcheck currently says:

```
In - line 4:
errors=$((errors + ?))
       ^-- SC1102 (error): Shells disambiguate $(( differently or not at all. For $(command substitution), add space after $( . For $((arithmetics)), fix parsing errors.
                   ^-- SC2035 (info): Use ./*glob* or -- *glob* so names with dashes won't become options.
```

#### Here's what I wanted or expected to see:

No errors.


#### Debug

I think SC2035 is mistaking `?` for a glob when it is only an arithmetic operator and subsequently causing SC1102 to fire.

The actual practice of accumulating status codes is probably bad, but for proper uses of the ? variable in arithmetic this should be remedied.
