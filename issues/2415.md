#### For bugs
- Rule Id (if any, e.g. SC1000): SC2057
- My shellcheck version (`shellcheck --version` or "online"): 0.7.2 and online
- [X] The rule's wiki page does not already cover this (e.g. https://shellcheck.net/wiki/SC2086)
- [X] I tried on https://www.shellcheck.net/ and verified that this is still a problem on the latest commit

#### Here's a snippet or screenshot that shows the problem:

```sh
$ cat test.sh 
#!/bin/sh
if [ \! -x "$1" ]; then
	echo "Not executable"
fi
```

#### Here's what shellcheck currently says:

```sh
$ shellcheck test.sh 

In test.sh line 2:
if [ \! -x "$1" ]; then
        ^-- SC2057: Unknown binary operator.

For more information:
  https://www.shellcheck.net/wiki/SC2057 -- Unknown binary operator.
```

#### Here's what I wanted or expected to see:

Shellcheck not being that confused by quoting.
