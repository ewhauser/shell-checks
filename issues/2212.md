#### Tags for stable VS latest seem to differ
Using the PKGBUILD in AUR(Arch Linux) https://aur.archlinux.org/cgit/aur.git/tree/PKGBUILD?h=shellcheck-git-static

Using this PKGBUILD downloads the latest 'shellcheck-latest.linux.x86_64.tar.xz'

Am I correct to assume this downloads shellcheck-git-static 0.7.1?
Used URL: https://github.com/koalaman/shellcheck/releases/download/latest/shellcheck-latest.linux.x86_64.tar.xz

If I manually change the version to 0.7.2(pkgver) the version check(pkgver function) puts the changed version back to 0.7.1 in PKGBUILD

Using a PKGBUILD explicitly pointing to stable 0.7.2, the correct version is downloaded and installed. 

Used (working correct)changed PKGBUILD:
```
pkgname=shellcheck-git-static
pkgver=0.7.2
pkgrel=1
pkgdesc="shellcheck-static, but using the latest-commit builds maintained by the author"
license=("AGPL3")
arch=("x86_64")
depends=('gmp')
conflicts=("shellcheck")
provides=("shellcheck=${pkgver}")

source=(
    "https://github.com/koalaman/shellcheck/releases/download/v"$pkgver"/shellcheck-v"$pkgver".linux.x86_64.tar.xz"
)
sha512sums=('067e2b8ee1910218de1e62068f7cc86ed7048e97b2a9d7c475ea29ae81c17a944376ce5c240d5c783ef3251d9bee7d7d010351958314eadd0fc88b5decfd8328')

#}

    #cd "${srcdir}/shellcheck-latest"
    cd "${srcdir}/shellcheck-v"$pkgver""

    install -Dt "${pkgdir}/usr/share/shellcheck" "LICENSE.txt" "README.txt"
}
