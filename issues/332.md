# [\#332 Issue](https://github.com/koalaman/shellcheck/issues/332) `closed`: catastrophic rm check does not check for `rm -rf $FOO/*`

#### <img src="https://avatars.githubusercontent.com/u/22718?v=4" width="50">[egh](https://github.com/egh) opened issue at [2015-03-26T18:37:51Z](https://github.com/koalaman/shellcheck/issues/332):

This works

``` sh
$ echo "rm -rf \$FOO/\$BAR" | shellcheck -

In - line 1:
rm -rf $FOO/$BAR
       ^-- SC2115: Make sure this never accidentally expands to '/'.
       ^-- SC2086: Double quote to prevent globbing and word splitting.
            ^-- SC2086: Double quote to prevent globbing and word splitting.
```

But this doesn't

``` sh
$ echo "rm -rf \$FOO/*" | shellcheck -

In - line 1:
rm -rf $FOO/*
       ^-- SC2086: Double quote to prevent globbing and word splitting.
```
