#### Here's a snippet or screenshot that shows the problem:

```sh
#!/usr/bin/sh

fn() {
	cd folder
	return
}

fn() {
	cd folder
	exit
}

fn() {
	cd folder; return
}

fn() {
	cd folder; exit
}

```

#### Here's what shellcheck currently says:
![image](https://user-images.githubusercontent.com/24364012/106073198-7e7e4c80-6101-11eb-81c0-fc1ecbc8cc5e.png)




#### Here's what I wanted or expected to see:
In case cd fails, the very next statement will either return/exit, which should be enough for `SC2164` not to come up, since it performs the action return/exit, no matter the exit code
