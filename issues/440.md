# [\#440 Issue](https://github.com/koalaman/shellcheck/issues/440) `closed`: disabling SC2129 check should disable all connected lines

#### <img src="https://avatars.githubusercontent.com/u/115592?v=4" width="50">[mika](https://github.com/mika) opened issue at [2015-08-13T11:11:53Z](https://github.com/koalaman/shellcheck/issues/440):

Given a file `foo` containing:

```
reprepro removematched bar '*foo' 2>>errors.txt                   
reprepro removematched bar '*foo' 2>>errors.txt
reprepro removematched bar '*foo' 2>>errors.txt
reprepro removematched bar '*foo' 2>>errors.txt
reprepro removematched bar '*foo' 2>>errors.txt
```

shellcheck complains regarding SC2129:

```
In foo line 1:
reprepro removematched bar '*foo' 2>>errors.txt
^-- SC2129: Consider using { cmd1; cmd2; } >> file instead of individual redirects.
```

When disabling the SC2129 check via:

```
# shellcheck disable=SC2129
reprepro removematched bar '*foo' 2>>errors.txt
reprepro removematched bar '*foo' 2>>errors.txt
reprepro removematched bar '*foo' 2>>errors.txt
reprepro removematched bar '*foo' 2>>errors.txt
reprepro removematched bar '*foo' 2>>errors.txt
```

it complains about the next line(s) though:

```
In foo line 3:
reprepro removematched bar '*foo' 2>>errors.txt
^-- SC2129: Consider using { cmd1; cmd2; } >> file instead of individual redirects.
```

It would be nice if `# shellcheck disable=SC2129` could disable those checks for all connected lines, so you don't have to add it before every single line.

Thanks for shellcheck! :+1: 

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2015-08-13T17:00:42Z](https://github.com/koalaman/shellcheck/issues/440#issuecomment-130761029):

That is indeed annoying. As a workaround, you could ignore them as a block:

```
# shellcheck disable=SC2129
{
  reprepro removematched bar '*foo' 2>>errors.txt                   
  reprepro removematched bar '*foo' 2>>errors.txt
  reprepro removematched bar '*foo' 2>>errors.txt
  reprepro removematched bar '*foo' 2>>errors.txt
  reprepro removematched bar '*foo' 2>>errors.txt
}
```
