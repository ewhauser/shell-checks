# [\#357 Issue](https://github.com/koalaman/shellcheck/issues/357) `closed`: $@: Double quote array expansions, otherwise they're like $* and break

#### <img src="https://avatars.githubusercontent.com/u/5316?v=4" width="50">[mcandre](https://github.com/mcandre) opened issue at [2015-04-23T23:31:59Z](https://github.com/koalaman/shellcheck/issues/357):

I often write shell scripts like:

whizzbang <host> <port>

```
#!/bin/sh
java -jar whizzbang.jar $@
```

that act as shell scripts, invoking Java classes behind the scenes. But ShellCheck wants me to surround `$@` with double quotes, which would actually be a mistake--I _want_ to pass arguments expanded, not try to fit them into a single string.

How should I write this script, or could we tweak shellcheck to ignore globs for `$@` and `$*`?

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2015-04-23T23:46:53Z](https://github.com/koalaman/shellcheck/issues/357#issuecomment-95751109):

Shellcheck is right: you want `"$@"` here. It will be equivalent to `"$1" "$2" "$3" ..`

Fitting them in a single string, I.e. `"$1 $2 $3.."` would be `"$*"`, and is not ShellCheck's recommendation.
