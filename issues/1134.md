# [\#1134 Issue](https://github.com/koalaman/shellcheck/issues/1134) `closed`: Repeat-only for loop without SC2034

#### <img src="https://avatars.githubusercontent.com/u/952007?v=4" width="50">[szepeviktor](https://github.com/szepeviktor) opened issue at [2018-03-03T07:10:34Z](https://github.com/koalaman/shellcheck/issues/1134):

I'd like to repeat something N times:
`for UNUSED in {1..16}; do echo 1; done` 

but shellcheck tells me `^-- SC2034: UNUSED appears unused. Verify it or export it.`

What could I do?

#### <img src="https://avatars.githubusercontent.com/u/952007?v=4" width="50">[szepeviktor](https://github.com/szepeviktor) commented at [2018-03-03T07:10:59Z](https://github.com/koalaman/shellcheck/issues/1134#issuecomment-370125893):

I know-I know: `# shellcheck disable=SC2034`

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2018-03-03T07:20:55Z](https://github.com/koalaman/shellcheck/issues/1134#issuecomment-370126418):

You can also use the name `_`. This is a feature in JS Python, Haskell and others, and semi-commonly used in sh in e.g. `read foo _ bar`

#### <img src="https://avatars.githubusercontent.com/u/315003?v=4" width="50">[adsr](https://github.com/adsr) commented at [2026-03-31T17:23:37Z](https://github.com/koalaman/shellcheck/issues/1134#issuecomment-4164211550):

Note `_` is a special built-in variable, at least in Bash.

```
   Shell Variables
       The following variables are set by the shell:

       _      At shell startup, set to the pathname used to invoke the shell or shell script being ex‐
              ecuted as passed in the environment or argument list.  Subsequently, expands to the last
              argument  to  the  previous  simple command executed in the foreground, after expansion.
              Also set to the full pathname used to invoke each command executed and placed in the en‐
              vironment exported to that command.  When checking mail, this parameter holds  the  name
              of the mail file currently being checked.
```
