# [\#1930 Issue](https://github.com/koalaman/shellcheck/issues/1930) `open`: Duplicate filenames in the input file list are scanned twice

#### <img src="https://avatars.githubusercontent.com/u/501702?u=cb840aeb51c307330d6c735b51689569a92a9f96&v=4" width="50">[edmorley](https://github.com/edmorley) opened issue at [2020-04-27 12:50](https://github.com/koalaman/shellcheck/issues/1930):

#### For bugs
- My shellcheck version (`shellcheck --version` or "online"): 0.7.1 (brew installed)

#### Here's a snippet or screenshot that shows the problem:

```sh
$ mkdir -p shellcheck-test/bin && cd shellcheck-test
$ touch bin/script.sh
$ shellcheck **/bin/* **/*.sh
```

#### Here's what shellcheck currently says:
```
In bin/script.sh line 1:

^-- SC2148: Tips depend on target shell and yours is unknown. Add a shebang or a 'shell' directive.


In bin/script.sh line 1:

^-- SC2148: Tips depend on target shell and yours is unknown. Add a shebang or a 'shell' directive.

For more information:
  https://www.shellcheck.net/wiki/SC2148 -- Tips depend on target shell and y...
```


#### Here's what I wanted or expected to see:
```
In bin/script.sh line 1:

^-- SC2148: Tips depend on target shell and yours is unknown. Add a shebang or a 'shell' directive.

For more information:
  https://www.shellcheck.net/wiki/SC2148 -- Tips depend on target shell and y...
```

ie: The results for `bin/script.sh` are shown twice, rather than just once.

#### Context
I'm wanting to run shellcheck in CI against a repository that inconsistently locates scripts (some in `bin/` directories, some not) and doesn't always use an extension of `.sh`. Whilst I can come up with a way to deduplicate the filenames myself, it would be a lot simpler if shellcheck could do that, so as to allow using bash globbing as-is.

Many thanks :-)
