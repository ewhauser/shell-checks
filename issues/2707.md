#### Problem/suggestion description

I think it would be nice if Shellcheck suggested explicitly specifying `--` (end of options) for utilities that support it if the last argument is a variable acting as a positional argument. This would avoid issues with filenames beginning with 1/2 dashes. This may seems like a stupid way to name files, but with [systemd-escape](https://www.freedesktop.org/software/systemd/man/systemd-escape.html) replacing `/` with `-`, files with names generated from escaped absolute paths could get interpreted as options.

A malicious actor could introduce additional options into scripts that don't use `--` by creating weirdly-named files like that. 

This would probably require keeping track of commonly used utilities that support `--` for signalling end-of-options. From the top of my head,  `rm`, `cat`, `cp` and `mv` come to mind, but there's many more.
