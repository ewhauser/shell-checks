# [\#3244 Issue](https://github.com/koalaman/shellcheck/issues/3244) `open`: SC2034 not detected when variable as same name as in function?

Should `SC2034` be emitted in the following example?
```
#!/usr/bin/env bash

foo() {
  impl() {
    local bar
    bar=$(printf '%s' bar)
    printf '%s\n' "$bar"
  }
  local bar
  bar=$(impl)  # ← missing SC2034?
}
```

Hm. Perhaps the existing test doesn't account for instances of parameter
name collisions. On shellcheck.net, changing the second 'bar' to 'baz'
triggers SC2034.

Name collisions can be confusing.
