# [\#2219 Issue](https://github.com/koalaman/shellcheck/issues/2219) `open`: SC2009: ignore / whitelist usage of "ps e" (with grep)

#### <img src="https://avatars.githubusercontent.com/u/9766?v=4" width="50">[blueyed](https://github.com/blueyed) opened issue at [2021-04-26 17:49](https://github.com/koalaman/shellcheck/issues/2219):

I've found the pattern of using `ps e … | grep …` useful to match/find environment variables of processes, e.g. `pid_window_id=$(ps e -o command= -p "$pid" | grep -Po '\bWINDOWID=\S+')`.

While https://github.com/koalaman/shellcheck/wiki/SC2009#exceptions mentions that it is valid/normal to ignore SC2009 in such cases I think that detecting the usage of `ps e` here explicitly could be supported and whitelisted automatically.

`ShellCheck version: 0.7.2`
