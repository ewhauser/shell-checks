# [\#3247 Issue](https://github.com/koalaman/shellcheck/issues/3247) `open`: SC2317 flagged for functions declared in bats-core's `setup`

#### <img src="https://avatars.githubusercontent.com/u/11473063?u=5e4318223971036462166a25a5e61386bfbd414a&v=4" width="50">[jansorg](https://github.com/jansorg) opened issue at [2025-07-14 08:19](https://github.com/koalaman/shellcheck/issues/3247):

ShellCheck flags code in functions, which are declared inside a bats-core setup function, as "unreachable" (SC2317).
Apparently, its data-flow engine doesn't know that `setup` and `setup_file` are called before each test function.

Sample script:

```bash
#!/usr/bin/env bats

setup() {
  # executed before each test
  echo "setup" >&3

  function foo() {
    echo hello >&3
  }
}

@test "test_name" {
  foo
}
```

ShellCheck 0.10's output (shellcheck.net is similar):
```text
$ shellcheck /home/user/temp/bash-test/unused.bats

In /home/user/temp/bash-test/unused.bats line 8:
    echo hello >&3
    ^------------^ SC2317 (info): Command appears to be unreachable. Check usage (or ignore if invoked indirectly).

For more information:
  https://www.shellcheck.net/wiki/SC2317 -- Command appears to be unreachable...

```
