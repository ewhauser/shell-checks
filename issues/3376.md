- Rule Id (if any, e.g. SC1000): SC3037
- My shellcheck version (`shellcheck --version` or "online"): 0.11.0

#### Here's a snippet or screenshot that shows the problem:

```sh
echo -e "builtin"
/usr/bin/echo -e "not builtin"
env -S echo -e "not builtin"
```

#### Here's what shellcheck currently says:

--shell=sh:

```
-:1:6: warning: In POSIX sh, echo flags are undefined. [SC3037]
-:2:15: warning: In POSIX sh, echo flags are undefined. [SC3037]
```

--shell=bash: nothing

#### Here's what I wanted or expected to see:

- the first line calls a shell builtin, and shellcheck works as expected
- the second line calls an external program, therefore SC3037 should be issued independently of shell
    - GNU coreutils supports `-n`, `-e` and `-E`, while BSD supports only `-n`
- the third line calls an external program via `env`, which is not checked
