# [\#2627 Issue](https://github.com/koalaman/shellcheck/issues/2627) `open`: Feature: Detect when getopts is called in function with no arguments

#### <img src="https://avatars.githubusercontent.com/u/6753988?u=3db322492df8cc56359ef7a38ac89efd19b59c15&v=4" width="50">[samherrmann](https://github.com/samherrmann) opened issue at [2022-11-20 22:30](https://github.com/koalaman/shellcheck/issues/2627):

ShellCheck does an excellent job at detecting when a function references arguments that are never passed in ([SC2120](https://www.shellcheck.net/wiki/SC2120)). This issue is a feature request to add the same behavior as SC2120 when `getopts` is used inside of a function to which no arguments are passed.

#### Here's a snippet or screenshot that shows the problem:

```sh

#!/bin/sh

a=""

parse_flags() {
  while getopts ":a:" opt; do
    case "$opt" in
      a)
        a="$OPTARG";;
      *)
        exit 1
      ;;
    esac
  done
}

parse_flags # Should be parse_flags "$@".

echo "a: $a" # Value is always empty because no arguments were passed to parse_flags.
```

#### Here's what shellcheck currently says:

Nothing.


#### Here's what I wanted or expected to see:

parse_flags references option arguments, but none are ever passed.
