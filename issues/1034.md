#### For new checks and feature suggestions
- [x] shellcheck.net (i.e. the latest commit) currently gives no useful warnings about this
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related


#### Here's a snippet or screenshot that shows the problem:

```sh

#!/bin/sh

foo=bar

. foo.sh
. bar.sh

echo "$bar"
```

#### Here's what shellcheck currently says:

```
Line 3:
foo=bar
^-- SC2034: foo appears unused. Verify it or export it.
 
Line 5:
. foo.sh
^-- SC1091: Not following: foo.sh was not specified as input (see shellcheck -x).
 
Line 6:
. bar.sh
^-- SC1091: Not following: bar.sh was not specified as input (see shellcheck -x).
 
Line 8:
echo "$bar"
      ^-- SC2154: bar is referenced but not assigned.
```

#### Here's what I wanted or expected to see:

It is possible that `$foo` is being used and that `$bar` is being set in either `./foo.sh` or `./bar.sh`, but shellcheck does not recursively look into these files and print warnings for them. It is understandable to see these warnings at https://www.shellcheck.net/, but when a user is using a local copy of shellcheck with `-x` it should be possible even if as an optional feature to recursively search these scripts as `./foo.sh` may not source any files itself and its warnings may not be correct as a result.

The alternative is to compile these scripts into a single file while may not fit the specific project and is time consuming for mere purpose of testing.

#### <img src="https://avatars.githubusercontent.com/u/3999221?v=4" width="50">[pawamoy](https://github.com/pawamoy) commented at [2017-10-26T17:38:53Z](https://github.com/koalaman/shellcheck/issues/1034#issuecomment-339742559):

I think the `-x` tells shellcheck not to trigger warning if you source a file different than the ones specified on the command-line. To actually do what you want, you should use both `-x` and `source` directives: https://github.com/koalaman/shellcheck/wiki/Directive.

For instance:

```bash
#!/bin/sh

foo=bar

# shellcheck source=foo.sh
. foo.sh
# shellcheck source=bar.sh
. bar.sh

echo "$bar"
```

Then `shellcheck -x YOUR_SCRIPT`

```
Usage: shellcheck [OPTIONS...] FILES...

```
I don't know when this was implemented as I have not been following here, but I would recommend closing this issue.
