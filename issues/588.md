# [\#588 Issue](https://github.com/koalaman/shellcheck/issues/588) `closed`: let expressions do not require dollars

#### <img src="https://avatars.githubusercontent.com/u/1219667?v=4" width="50">[lyokha](https://github.com/lyokha) opened issue at [2016-01-28T14:19:20Z](https://github.com/koalaman/shellcheck/issues/588):

Like `(( ... ))` `let` does not require dollar signs, 

``` sh
let "a = b + c"
```

is quite valid, so using dollars for variable expansions like `let "a = $b + $c"` should symmetrically give warning `SC2004` like in `(( ... ))` case.

Accordingly, `let` expressions should also collect all variables written without dollars to avoid warning `SC2034` about unused variables if they were defined on top of a script and further used only in `let` expressions.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2016-02-03T04:57:33Z](https://github.com/koalaman/shellcheck/issues/588#issuecomment-179009240):

`let` should preferably be deprecated. There's `$((..))` in posix and `((..))` in bash.


#### <img src="https://avatars.githubusercontent.com/u/198379?v=4" width="50">[bittorf](https://github.com/bittorf) commented at [2016-02-03T14:14:35Z](https://github.com/koalaman/shellcheck/issues/588#issuecomment-179258856):

let has it's own goodies, e.g. the endcondition:

```
J=10
while let J-=1; do echo $J; done
```


#### <img src="https://avatars.githubusercontent.com/u/302055?v=4" width="50">[deryni](https://github.com/deryni) commented at [2016-02-03T14:44:09Z](https://github.com/koalaman/shellcheck/issues/588#issuecomment-179271710):

`((..))` in bash is documented as being exactly equivalent to `let` so for bash (and ksh) at least `let` isn't necessary and `let` appears (from the little I can find right now) to be a bash-ism itself... unless someone knows better.


#### <img src="https://avatars.githubusercontent.com/u/198379?v=4" width="50">[bittorf](https://github.com/bittorf) commented at [2016-02-03T14:57:28Z](https://github.com/koalaman/shellcheck/issues/588#issuecomment-179276588):
