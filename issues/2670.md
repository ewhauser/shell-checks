#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/sh

YQ_VERSION="$(yq --version)"
if [ $? -ne 0 ]; then
  # yq is not installed
  exit 1
elif echo "$YQ_VERSION" | grep -q mikefarah; then
  echo thing
else
  echo other thing
fi
```

#### Here's what shellcheck currently says:
```
if [ $? -ne 0 ]; then
     ^-- [SC2181](https://www.shellcheck.net/wiki/SC2181) (style): Check exit code directly with e.g. 'if ! mycmd;', not indirectly with $?.
```

#### Following the advice from https://www.shellcheck.net/wiki/SC2181:

![image](https://user-images.githubusercontent.com/35791147/213964488-55cd7817-b743-4058-99cc-166222f35bb7.png)

```sh
#!/bin/sh

if [ ! YQ_VERSION="$(yq --version)" ]; then
  # yq is not installed
  exit 1
elif echo "$YQ_VERSION" | grep -q mikefarah; then
  echo thing
else
  echo other thing
fi
```

```
if [ ! YQ_VERSION="$(yq --version)" ]; then
       ^-- [SC2077](https://www.shellcheck.net/wiki/SC2077) (error): You need spaces around the comparison operator.
```

Isn't that a very different piece of code?
  - `[ ! YQ_VERSION="$(yq --version)" ]` captures the output of `yq --version` in the variable `YQ_VERSION` and checks if the exit code was non-zero
  - `[ ! YQ_VERSION = "$(yq --version)" ]` checks that the output of `yq --version` is not equal to the string `YQ_VERSION`
