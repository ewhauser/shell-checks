# [\#673 Issue](https://github.com/koalaman/shellcheck/issues/673) `open`: feature: Prescribe an explicit "--" to mark end of command line options

#### <img src="https://avatars.githubusercontent.com/u/3374183?v=4" width="50">[sanmai-NL](https://github.com/sanmai-NL) opened issue at [2016-05-21T09:32:51Z](https://github.com/koalaman/shellcheck/issues/673):

`--` should always be used before operands in statements that invoke some utility, at least when providing variable operands.

``` sh
my_file='-some_unexpected_file_name'
rm "$my_file"
```

Will result in an error:

```
rm: invalid option -- 's'
```

Suggestion:

``` sh
my_file='-some_unexpected_file_name'
rm -- "$my_file"
```

See guideline 10 of [The Open Group Base Specifications Issue 7](http://pubs.opengroup.org/onlinepubs/9699919799/basedefs/V1_chap12.html).
