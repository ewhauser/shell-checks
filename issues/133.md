#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2014-04-08T17:42:33Z](https://github.com/koalaman/shellcheck/issues/133#issuecomment-39878796):

ShellCheck is technically correct. `$res` in `$(ls $res)` is unquoted, and this causes ls to list all files in the current directory.

Since xargs doesn't support running shell commands at all, ShellCheck doesn't try to detect premature expansion like you have here. If you'd done the same with ssh, you'd get a "SC2029: Note that, unescaped, this expands on the client side."

Try a while read loop instead.
