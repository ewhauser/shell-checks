# [\#3202 Issue](https://github.com/koalaman/shellcheck/issues/3202) `open`: Shellcheck fails to warn about this syntax error

#### <img src="https://avatars.githubusercontent.com/u/1731537?u=c6e688a9e3785d292579c177b41175bc848f3836&v=4" width="50">[shelvacu](https://github.com/shelvacu) opened issue at [2025-05-06 04:31](https://github.com/koalaman/shellcheck/issues/3202):

```bash
#!/usr/bin/env bash
foo=(1 2 3)
fooLength=$(( #foo + 1 ))
```

This is a syntax error in bash (``bad substitution: no closing `)' in $(( #foo + 1 ))``) but shellcheck says everything is fine (just warns about unused variables)

The corrected code would be:
```bash
fooLength=$(( ${#foo[@]} + 1 ))
```

Tested with version 0.10.0
