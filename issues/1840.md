# [\#1840 Issue](https://github.com/koalaman/shellcheck/issues/1840) `open`: Allow defining directives inline, ie. after a command

#### <img src="https://avatars.githubusercontent.com/u/37372069?u=9d1c8fb51a7f991968c5dad0bc6d954e93ccb803&v=4" width="50">[setpill](https://github.com/setpill) opened issue at [2020-02-18 18:57](https://github.com/koalaman/shellcheck/issues/1840):

This is useful especially in Dockerfiles (ie. when using shellcheck through hadolint). In Dockerfiles you cannot have true multiline shell scripts without really ugly hacks, so having to put the directive *before* the affected command is impossible as shell scripts do not have comments that end before the line ends.

#### For new checks and feature suggestions
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related

#### Here's a snippet or screenshot that shows the problem:

```Dockerfile

RUN cp $(ls | grep "something" | sort | uniq) somewhere # shellcheck disable=SC2046

```

#### Here's what shellcheck currently says:

```
SC2046 Quote this to prevent word splitting.
```

#### Here's what I wanted or expected to see:

Nothing
