# [\#3227 Issue](https://github.com/koalaman/shellcheck/issues/3227) `closed`: Documentation feature request: using "ignore" directive with line continuations

#### <img src="https://avatars.githubusercontent.com/u/119972?v=4" width="50">[jasonkarns](https://github.com/jasonkarns) opened issue at [2025-06-20 15:45](https://github.com/koalaman/shellcheck/issues/3227):

Could someone add an example for how the ignore directive should be used with line continuations?

Say:

```bash
          # shellcheck disable=SC2086
          gh release view $tag || \
            gh release create ${tag/*-*/"$tag" --prerelease} --generate-notes
```

The directive in this case should apply to the gh-release-create line.

Also, for extra confusion, the issue in question is being caught by the github-actions linter in SuperLinter. Which means this is linting the shell scripts that are embedded in github actions workflow yaml, (not a plain shell script file). So any suggestions for "whole-file" directives aren't really helpful here (unless each shell block in the yaml file is treated like a separate file?)

#### <img src="https://avatars.githubusercontent.com/u/84648683?v=4" width="50">[wileyhy](https://github.com/wileyhy) commented at [2025-07-16 03:49](https://github.com/koalaman/shellcheck/issues/3227#issuecomment-3076633023):

OP's example:

  <sh>
  # shellcheck disable=SC2086
  gh release view $tag || \
      gh release create ${tag/*-*/"$tag" --prerelease} --generate-notes
  <\sh>

On shellcheck.net, in sh and dash the only issues detected from this
version of the script, below, are SC2060 (String replacement...). All five
shell crash bangs, however, pass in terms of the (amended) line
continuation syntax between lines 4 and 5.

  <sh>
  #!/bin/SHELL
  tag=1
  # shellcheck disable=SC2086
  gh release view $tag ||
    gh release create ${tag/*-*/"$tag" --prerelease} --generate-notes
  <\sh>
