# [\#625 Issue](https://github.com/koalaman/shellcheck/issues/625) `open`: Suggest (( over [[ for arithmetic comparisons?

#### <img src="https://avatars.githubusercontent.com/u/1174486?v=4" width="50">[curby](https://github.com/curby) opened issue at [2016-03-08T19:53:38Z](https://github.com/koalaman/shellcheck/issues/625):

GreyCat's wiki and #bash on freenode suggest that (( is preferred over [[ for arithmetic comparisons.  For example:

```
if ((a > 5)); then 
  echo "a is more than 5";
fi
```

is preferred over:

```
if [[ $a -gt 5 ]]; then 
  echo "a is more than 5";
fi
```

The rationale is better readability, more explicit definition of an arithmetic context, and additional arithmetic capabilities within (( as opposed to [[.

If this is indeed a generally-accepted best practice, can shellcheck be updated to suggest (( over [[ for arithmetic comparisons?

#### <img src="https://avatars.githubusercontent.com/u/1823914?v=4" width="50">[arth1](https://github.com/arth1) commented at [2016-09-04T05:39:51Z](https://github.com/koalaman/shellcheck/issues/625#issuecomment-244585475):

It could be argued that many of us prefer

```
if [ "$a" -gt 5 ]; then
    echo "a is more than 5"
fi
```

Simply because that will work in all shells.  No need to use [[ nor (( in this case, and certainly not a command separator at the end of the echo line (which I think shellcheck should throw an info on).
