#### <img src="https://avatars.githubusercontent.com/u/24546770?u=42da8658469bc072e1b61e2956e7468162b4ac29&v=4" width="50">[rockandska](https://github.com/rockandska) commented at [2021-12-11 11:22](https://github.com/koalaman/shellcheck/issues/1965#issuecomment-991606094):

Allowing "disable" on a whole block will be also useful for this kind of cases where local variables are shared between internal functions

```
test() {
  local a
  local b
  local c
}

a() {
  echo $a
}
```

Right now, could be rewritten as below but prefer the version with local on independent lines
```
test() {
  local a b c
}
```

A kind of `# shellcheck start_disable=SC2034` `# shellcheck end_disable=SC2034` markers will be neat

#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2021-12-12 01:42](https://github.com/koalaman/shellcheck/issues/1965#issuecomment-991818398):

Since directives are scoped to commands and not to lines, you can use them before functions to apply them only to that entire function. You can also make creative use of brace groups to define an arbitrary block to ignore in:

```
#!/bin/bash
: # this is a dummy command so that the directive isn't the first thing in the file and therefore filewide

# This directive only applies to the `test` function (all of it)
# shellcheck disable=SC2034
test() {
  local a
  local b
  local c
}

test2() {
  # Still triggers
  local d 
  # This directive only applies to `e` and `f`
  # shellcheck disable=SC2034
  {
    local e
    local f
  }
}

test3() {
  # Still triggers
  local g
}
```

It's not entirely the same thing, but generally close enough.

@XaaT Note that ShellCheck is right in this case: the here doc delimiter is not quoted so variables in the contents will be expanded by the current shell and result in an output of `if [ "" = "efi" ]`
