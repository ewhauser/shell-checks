# [\#131 Issue](https://github.com/koalaman/shellcheck/issues/131) `closed`: SC2016 in PS1 

#### <img src="https://avatars.githubusercontent.com/u/1235108?v=4" width="50">[myint](https://github.com/myint) opened issue at [2014-04-05T23:50:58Z](https://github.com/koalaman/shellcheck/issues/131):

Is this intended behavior for shellcheck? Using double quotes would break the git prompt.

```
$ shellcheck bashrc

In bashrc line 58:
export PS1='\[\e[01;32m\]\u@\h\[\e[00m\]:\[\e[01;34m\]\w\[\e[00m\]$(__git_ps1 " \[\e[01;31m\](%s)\[\e[00m\]")\n\$ '
           ^-- SC2016: Expressions don't expand in single quotes, use double quotes for that.
```

For some time now, I've used `# shellcheck disable=SC2016` to suppress this (so that Syntastic won't complain via shellcheck).

Thanks

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2014-04-06T00:10:51Z](https://github.com/koalaman/shellcheck/issues/131#issuecomment-39654838):

You're right, when assigning PS1 you'll usually want single quotes. There should probably be a warning when you _don't_ single quote it. 

ShellCheck no longer warns about this for PROMPT_COMMAND and PS[1-4] in 81956d3
