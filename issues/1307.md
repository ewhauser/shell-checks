I'm trying to bring shellcheck into our CI system, but running into some discrepancies between the Docker image and the Atom version of the linter.

Both OSX (through brew) and Docker (through docker hub's koalaman/shellcheck:stable) are on shellcheck version 0.5.0, and Atom's linter plugin is on 1.4.7.

For a given file, Atom is showing a green light, but running the same file through Docker throws a dozen errors. This is severely slowing down the attempt to get these tests to green.

What can I try to get these tests in sync?

@indietravel The functionality for enabling/disabling notice messages has been added to linter-shellcheck in [this commit](https://github.com/AtomLinter/linter-shellcheck/commit/d3346769de6f21b78a83402a272a0b347be39695).
