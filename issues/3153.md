# [\#3153 Issue](https://github.com/koalaman/shellcheck/issues/3153) `open`: $HOME gets resolved as . (current directory)

#### <img src="https://avatars.githubusercontent.com/u/61058728?u=c268ca9a13254a203d87024aa3db2776dbf34e90&v=4" width="50">[palapapa](https://github.com/palapapa) opened issue at [2025-03-02 06:54](https://github.com/koalaman/shellcheck/issues/3153):

I'm trying to set my [session environment variables](https://userbase.kde.org/Session_Environment_Variables) in KDE. You put in those files what you would normally put in `~/.profile` or `~/.bash_profile`, but KDE doesn't read those on login, so you need to put them under `$HOME/.config/plasma-workspace/env/`.

I have the following line in `$HOME/.config/plasma-workspace/env/environment-variables.sh`:

```bash
. "${HOME}/.bash_environment_variables"
```

and I have a file named `.bash_environment_variables` in my home directory. This script is executed correctly by KDE and my environment variables are indeed set on login, but shellcheck complains:

```
Not following: ./.bash_environment_variables: openBinaryFile: does not exist (No such file or directory)
```

It seems like shellcheck thinks that `${HOME}` is `.`. Is this normal behavior?

#### <img src="https://avatars.githubusercontent.com/u/6365158?u=2dc52a787f75eb2f88b87e5258a80295b4821a2e&v=4" width="50">[kg4zow](https://github.com/kg4zow) commented at [2026-03-23 17:45](https://github.com/koalaman/shellcheck/issues/3153#issuecomment-4112544718):

+1 ... having it expand `$HOME` would be a HUGE help. I have several scripts that use something like this:

```
# shellcheck source=/home/jms1/.config/MYNAME/config.txt
source "${XDG_CONFIG_HOME:-$HOME/.config}/MYNAME/config.txt"
```

The problem is, `$HOME` is `/home/jms1` on my Linux machines, or `/Users/jms1` on my macOS machines, which means I can only use `shellcheck` on one type of machine (Linux or macOS) without editing it. This also means that when other people use my scripts on their own machines, they would need to edit the `# shellcheck source=` lines when they check any other edits for errors.

Ideally I'd like to see `HOME` be usable at the beginning of a `source=` directive's value, similar to how `SCRIPTDIR` can be used. For example ...

```
# shellcheck source=HOME/.config/MYNAME/config.txt
source "${XDG_CONFIG_HOME:-$HOME/.config}/MYNAME/config.txt"
```
