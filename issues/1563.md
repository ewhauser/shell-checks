# [\#1563 Issue](https://github.com/koalaman/shellcheck/issues/1563) `closed`: Feature Request: disable by red (error), yellow (warning), and green (info)

#### <img src="https://avatars.githubusercontent.com/u/17078368?v=4" width="50">[EricLuehrsen](https://github.com/EricLuehrsen) opened issue at [2019-05-01T03:39:07Z](https://github.com/koalaman/shellcheck/issues/1563):

It appears shellcheck has priority or classes of checks although they are not numbered in such order. However it is not possible to disable by classification. It would be a good option and common with other lint checkers to to be able to disable all green. This would probably also need a `+e` option to add few back (for example note "not following").

`$ shellcheck -egreen +eSC1091 myscript.sh`

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2019-07-05T01:51:01Z](https://github.com/koalaman/shellcheck/issues/1563#issuecomment-508603970):

You can already specify `--severity=warning` to only show warnings and higher.

You can't add certain warnings back in, but if you want especially fine grained filtering logic, you can e.g. use `jq` on the JSON output.
