# [\#668 Issue](https://github.com/koalaman/shellcheck/issues/668) `open`: Assign default values 

#### <img src="https://avatars.githubusercontent.com/u/19393173?v=4" width="50">[zhatt](https://github.com/zhatt) opened issue at [2016-05-16T16:59:39Z](https://github.com/koalaman/shellcheck/issues/668):

When I use := or = in parameter expansion to define default values I get a referenced but not assigned warning.  I don't think shellcheck should warn for this case as it is an assignment.

For this script.
# !/bin/bash

: "${loc_bindir=/usr/local/bin}"

I get this output.
In ./x.sh line 2:
: "${loc_bindir:=/usr/local/bin}"
   ^-- SC2154: loc_bindir is referenced but not assigned.
