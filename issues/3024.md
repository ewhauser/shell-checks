# [\#3024 Issue](https://github.com/koalaman/shellcheck/issues/3024) `open`: SC2001 should not be recommended when shebang is /bin/ksh

#### <img src="https://avatars.githubusercontent.com/u/430807?v=4" width="50">[mbbh](https://github.com/mbbh) opened issue at [2024-07-17 09:10](https://github.com/koalaman/shellcheck/issues/3024):

#### For bugs
- SC2001: 
- My shellcheck version (`shellcheck --version` or "online"): 0.9.0
- [X] The rule's wiki page does not already cover this (e.g. https://shellcheck.net/wiki/SC2086)
- [X] I tried on https://www.shellcheck.net/ and verified that this is still a problem on the latest commit

#### Here's a snippet or screenshot that shows the problem:

```sh

#!/bin/ksh
x="foo"
foo=$(echo "$x" | sed -e 's,foo,bar,')
echo "$foo"

```

#### Here's what shellcheck currently says:
^-- [SC2001](https://www.shellcheck.net/wiki/SC2001) (style): See if you can use ${variable//search/replace} instead.


#### Here's what I wanted or expected to see:

Shellcheck should not be suggesting this as this is a bash/zsh feature and not present in ksh

in ksh using ${a//b/c} or ${a//b/c} will produce 'bad  substitution' when used in the way suggested by shellcheck.


#### <img src="https://avatars.githubusercontent.com/u/67505529?u=7c2b2aa731dbe27dc2a731f0790b2d4ea01b4645&v=4" width="50">[dumitrache-adrian92](https://github.com/dumitrache-adrian92) commented at [2024-07-27 11:11](https://github.com/koalaman/shellcheck/issues/3024#issuecomment-2254117290):

Hey, this version of the script outputs `bar` for me on ksh:
```
#!/bin/ksh
x="foo"

foo=${x//foo/bar}
echo "$foo"
```

I looked a bit into this, it seems that ksh93 does have this kind of string replacement according to this [documentation](https://www.ibm.com/docs/ro/aix/7.3?topic=shell-enhanced-korn-ksh93) (described in the "Parameter expansions" section), but ksh88 does not. Can you check your version with `ksh --version`?

Not sure how this situation should be handled since distros like Ubuntu install ksh93 by default under `/bin/ksh` when you ask for ksh. I can't even find instructions for how to install ksh88.

#### <img src="https://avatars.githubusercontent.com/u/430807?v=4" width="50">[mbbh](https://github.com/mbbh) commented at [2024-07-28 08:15](https://github.com/koalaman/shellcheck/issues/3024#issuecomment-2254389152):

Hey,

thanks for looking into this. As it turns out the OpenBSD ksh is not a direct fork of the upstream ksh, but apparently some sort of rewrite, if I am understanding https://github.com/openbsd/src/blob/master/bin/ksh/README correctly. As such, the version offered by this ksh is @(#)PD KSH v5.2.14 99/07/13.2, which likely doesn't map to the version of ksh shellcheck is supporting.

the conversion isn't mentioned in OpenBSDs ksh manpage https://man.openbsd.org/ksh.1.

I should've expected this, apologies for not checking that it may be OpenBSD related in the beginning, I did not think of it.
