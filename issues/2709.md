# [\#2709 Issue](https://github.com/koalaman/shellcheck/issues/2709) `closed`: SC2181 conflicts with SC2310

#### For bugs
- Rule Ids: SC2181 and SC2310
- My shellcheck version: 0.9.0
- [x] The rule's wiki page does not already cover this
- [x] I tried on https://www.shellcheck.net/ and verified that this is still a problem on the latest commit

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash
#shellcheck enable=check-set-e-suppressed

if OUT=$(
    set -euo pipefail

    success() {
        echo success;
    }

    fail() {
        echo fail
        return 1
    }

    out=$(fail)
    out+=$(success)
    out+=$(fail)

    printf '%s' "$out"
); then
    export OUT
fi
```

#### Here's what shellcheck currently says:

> Line 16:
>     out=$(fail)
>           ^-- [SC2310](https://www.shellcheck.net/wiki/SC2310) (info): This function is invoked in an 'if' condition so set -e will be disabled. Invoke separately if failures should cause the script to exit.

### Rewriting as suggested:
```sh
#!/bin/bash
#shellcheck enable=check-set-e-suppressed

OUT=$(
    set -euo pipefail

    success() {
        echo success;
    }

    fail() {
        echo fail
        return 1
    }

    out=$(fail)
    out+=$(success)
    out+=$(fail)

    printf '%s' "$out"
)

if (( $? == 0 )); then
    export OUT
fi
```

gives  SC2181:

> Line 23:
> if (( $? == 0 )); then
>       ^-- [SC2181](https://www.shellcheck.net/wiki/SC2181) (style): Check exit code directly with e.g. 'if mycmd;', not indirectly with $?.

#### Here's what I wanted or expected to see:

SC2181 is not issued if the last command is prone to SC2310 (or any other warning).

#### <img src="https://avatars.githubusercontent.com/u/615684?u=10b3a58cd92f636bec2d46385a52273490743197&v=4" width="50">[h3xx](https://github.com/h3xx) commented at [2023-04-15 16:46](https://github.com/koalaman/shellcheck/issues/2709#issuecomment-1509890738):

Because you have `set -e` on, the `if` statement becomes unnecessary. The script dies after it finishes executing line 15, so I think the suggestion was actually to re-write the script as:

```sh
#!/bin/bash
#shellcheck enable=check-set-e-suppressed

set -euo pipefail

success() {
    echo success;
}

fail() {
    echo fail
    return 1
}

out=$(fail; success; fail)
echo "$out"
```
