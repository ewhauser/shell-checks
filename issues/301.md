# [\#301 Issue](https://github.com/koalaman/shellcheck/issues/301) `closed`: Warn about invalid return values

#### <img src="https://avatars.githubusercontent.com/u/174451?v=4" width="50">[omajid](https://github.com/omajid) opened issue at [2015-02-06T00:05:24Z](https://github.com/koalaman/shellcheck/issues/301):

Bash complains about negative return values:

```
$ return -1
bash: return: -1: invalid option
return: usage: return [n]
```

shellsheck should warn about this if encounters it in scripts.

A simple test case:

```
$ cat test.sh 
return -1
$ shellcheck test.sh 
$ bash test.sh 
test.sh: line 1: return: -1: invalid option
return: usage: return [n]
```

Great suggestion! Some support for this is included in 61b4b65.

It gives you the benefit of the doubt regarding variables and substitutions, but if will catch simple things like `return -1` and `return "hello world"`
