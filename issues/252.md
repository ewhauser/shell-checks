Hm, I don't see why this is necessary. ShellCheck is a static analysis tool. It's impossible to predict if a given program will fork-bomb you, with static analysis. You have two options:

1) Don't put that code in your scripts.
2) If it's not your script, and you don't trust the author, then don't execute the script.

This applies to all programs though, not just bash scripts.
