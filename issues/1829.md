# [\#1829 Issue](https://github.com/koalaman/shellcheck/issues/1829) `closed`: Eq and Ord instances for Replacement are incompatible

#### <img src="https://avatars.githubusercontent.com/u/16061266?v=4" width="50">[josephcsible](https://github.com/josephcsible) opened issue at [2020-02-10 04:45](https://github.com/koalaman/shellcheck/issues/1829):

https://github.com/koalaman/shellcheck/blob/61b073d507bc2282f36995518b1b4ee818d2548c/src/ShellCheck/Interface.hs#L246-L260

Given two `Replacement`s `x` and `y`, if their `repStartPos` fields are equal but other fields are not, then you'll have both `x /= y` and  `compare x y == EQ`. However, [the documentation for `Ord`](https://hackage.haskell.org/package/base-4.12.0.0/docs/Prelude.html#t:Ord) says:

> Note that the following operator interactions are expected to hold:
> [...]
> 6. `x == y` = `compare x y == EQ`

Which instance is correct? The derived `Eq` one or the custom `Ord` one?

#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2020-02-17 21:16](https://github.com/koalaman/shellcheck/issues/1829#issuecomment-587161611):

The Eq one was correct and the Ord one wasn't even used. I've removed it. Thanks!
