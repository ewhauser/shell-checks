#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2019-03-04T18:29:04Z](https://github.com/koalaman/shellcheck/issues/1184#issuecomment-469362555):

@krry The issue here is that `%*s` takes two parameter: a length and a string to pad to that length:

    $ printf '%*s' 16 "hello world"
         hello world
    ^------16------^

Your snippet leaves out the second argument, so Bash just inserts a type specific default: the empty string. If you just want N spaces, that happens to work out, but it's arguably better to be explicit about it:

    printf -v _hr "%*s" "$(tput cols)" ""

It is easy to see what is the issue, if you try to break the chaining:
```bash
#!/bin/bash

printf -v _hr "%*s" "$(tput cols)"
#             ^-- SC2183: This format string has 2 variables, but is passed 1 arguments.
echo -en "${_hr// /${2-$}}"
echo -e "\r\033[20C $1 "

```

So, you are trying to do "some" kind of magic, that involves the width of your terminal - some kind of "measurement" specifically (I am familiar with some `printf` "snippets", and they all look hacky to the untrained eye). What `shellcheck` is telling you, is that you asked `printf` to do "something" with 2 arguments (`%*`: min arg width, [...]`s`: of-type: string), and you gave one (min arg width)

Maybe your extra code gives some hints about what `printf` (and the rest of the `echo`s) is/are supposed to do. Warning (could be) easy to fix, and it might be better to do so: you will then be explicit.

You might also understand better the job the tool is performing 😄
