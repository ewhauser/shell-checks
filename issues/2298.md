#### Here's a snippet or screenshot that shows the problem:

```sh
#!/usr/bin/env bash

set -euo pipefail

for _ in $(seq 10); do
    for _ in $(seq 10); do
        :
    done
done
```

#### Here's what shellcheck currently says:
```
In ./solutions/uri/2168/generate_in.sh line 5:
for _ in $(seq 10); do
^-^ SC2167: This parent loop has its index variable overridden.


In ./solutions/uri/2168/generate_in.sh line 6:
    for _ in $(seq 10); do
    ^-^ SC2165: This nested loop overrides the index variable of its parent.
```
#### Here's what I wanted or expected to see:

AS [SC2034](https://github.com/koalaman/shellcheck/wiki/SC2034) says, intentionally unused variables should be declared as `_`. I had to declare nested loops, but I didn't want to use the variables created by them, so I did as SC2034 says. But I expected get no warnings, but I got errors above.
