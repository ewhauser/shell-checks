- Rule Id SC2086
- My shellcheck version 0.9.0
- Wiki https://www.shellcheck.net/wiki/SC2086 covers it but there is possible alternative way.

#### Feature request

Some variables are meant to be expanded with spaces, for example to pass multiple arguments to some call. Currently there are 3 ways: 
- use array --  con: bash(3 and above) only and requires recode to array syntax)
- use function (with `set --` inside) --  con: requires heavy recode
- annotate a call with `disable=SC2086` --con: hides other possible argument expansions that need to be quoted.

I suggest there is way to annotate variable with `disable=SC2086` when it's first defined, so later when it's used in a call it does not require to be quoted.

With this we would only annotate single place in script and not need to recode and don't hide other errors.
