Hi,

I threw some functional code at it and it breaks parsing.
I don't know if you wanna hurt yourself as much as fixing that, but for reference, this example code only parsed half-way.

It's proof of concept code to a generic logging / action handler, doesn't really do much besides giving a warm fuzzy feeling once you put a state machine around it.
Execution of shellcheck breaks at the end of the first function.

I'm not sure if bash can even do this, maybe bash4 would.

```
#!/usr/bin/ksh

f_wget()
{

  unset x y ret
  x=${ { y=$(wget -O- $1); } 2>&1; }
  ret=$?
  #typeset -p x y
  export x y ret

}

f_handle()
{

  if [ $ret != 0  ]; then
     print "Error"
  else
     print "OK"
  fi
  [[ -z $x ]] || print "Fehlerdaten vorhanden"
  [[ -z $y ]] || print "Nutzdaten vorhanden"

}

f_wget /dev/null
f_handle
f_wget http://www.heise.de/
f_handle

```
