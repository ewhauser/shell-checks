#### <img src="https://avatars.githubusercontent.com/u/243351?v=4" width="50">[brother](https://github.com/brother) commented at [2022-04-28 08:36](https://github.com/koalaman/shellcheck/issues/2486#issuecomment-1111904311):

You can specify severity level to avoid info messages.

```
-S SEVERITY         --severity=SEVERITY        Minimum severity of errors to consider (error, warning, info, style)
```
Example

```
brother ~$ cat /tmp/test.sh
#!/bin/bash
echo "\\t"
brother ~$ shellcheck -S warning /tmp/test.sh
brother ~$ echo $?
0
brother ~$ shellcheck -S info /tmp/test.sh

In /tmp/test.sh line 2:
echo "\\t"
     ^---^ SC2028 (info): echo may not expand escape sequences. Use printf.

For more information:
  https://www.shellcheck.net/wiki/SC2028 -- echo may not expand escape sequen...
brother ~$ echo $?
1
```
