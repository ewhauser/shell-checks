# [\#586 Issue](https://github.com/koalaman/shellcheck/issues/586) `closed`: [[ a != b || a != c ]] is not tautologically true when b and c use pattern matching

#### <img src="https://avatars.githubusercontent.com/u/17551?v=4" width="50">[eatnumber1](https://github.com/eatnumber1) opened issue at [2016-01-27T00:35:57Z](https://github.com/koalaman/shellcheck/issues/586):

I've got the following code:

``` bash
[[ "$foo" != */foo || "$foo" != /foo/bar/baz/* ]]
```

This triggers [SC2055](https://github.com/koalaman/shellcheck/wiki/SC2055) which protects against tautological conditionals which are always true. In this case however, bash performs [pattern matching](https://www.gnu.org/software/bash/manual/bashref.html#Pattern-Matching) on the right hand side of `==` and `!=` operators which means that this can be false. See the section 3.2.4.2 of the Bash [reference manual](https://www.gnu.org/software/bash/manual/bashref.html#Conditional-Constructs) for a primary reference.
