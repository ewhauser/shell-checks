# [\#2348 Issue](https://github.com/koalaman/shellcheck/issues/2348) `open`: Support multiple output to file

#### <img src="https://avatars.githubusercontent.com/u/1245341?u=92bc3bae0adb38941124e2f7b3d15e1648fab20a&v=4" width="50">[oliv3r](https://github.com/oliv3r) opened issue at [2021-10-06 20:17](https://github.com/koalaman/shellcheck/issues/2348):

Currently, shellcheck supports various output formats on stdout using the `--format` flag. This is really good in itself, but in a CI you may want to have more. For example a report in XML/json format, that the CI can parse, but still 'human readable' output in the logs.

To accomplish this, I suggest to add a flag for each of the output formats, that take a filename to store their outputs in. For example.
```sh
shellcheck --format tty --output-json reports/shellcheck.json --output-checkstyle reports/shellcheck.checkstyle
```
GitLab for example relies heavily on parsing reports from report-artifacts, which it can do great things with; but users may just want to get the actual messages in the build log.
