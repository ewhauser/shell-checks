# [\#275 Issue](https://github.com/koalaman/shellcheck/issues/275) `closed`: Silencing specific lines with `# noqa` or smth similar

#### <img src="https://avatars.githubusercontent.com/u/109442?v=4" width="50">[teferi](https://github.com/teferi) opened issue at [2015-01-02T19:51:36Z](https://github.com/koalaman/shellcheck/issues/275):

Some errors state that I can verify the line (or at least that's what I imagine that message means) but I couldn't find anything like that in the docs. 

Pep8/pyflakes has a very nice feature, that if you leave a comment `# noqa` or `# NOQA` after problematic line — pep8 would ignore the line completely. This might be usefull, if you do not want to ignore the issue completely (with --exlude option) and know exactly what you're doing in this specific line and want to suppress the error.

It would be nice if you could implement smth similar.

#### <img src="https://avatars.githubusercontent.com/u/255938?v=4" width="50">[ormaaj](https://github.com/ormaaj) commented at [2015-01-02T20:39:14Z](https://github.com/koalaman/shellcheck/issues/275#issuecomment-68560218):

My favorite implementation is clang's [#pragma clang diagnostic push / pop](http://llvm.org/releases/3.4/tools/clang/docs/UsersManual.html#controlling-diagnostics-via-pragmas) because you just define a block region wherein warnings are modified from the surrounding state.

Definitely a nice feature. It makes it explicit in the code that a particular warning is expected.
