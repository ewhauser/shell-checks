# [\#933 Issue](https://github.com/koalaman/shellcheck/issues/933) `open`: names/identifiers in bash must be alphanumeric+underscore only (eg, no dash)

#### <img src="https://avatars.githubusercontent.com/u/987449?v=4" width="50">[draeath](https://github.com/draeath) opened issue at [2017-06-12T21:40:36Z](https://github.com/koalaman/shellcheck/issues/933):

shellcheck 0.3.7
#### For new checks and feature suggestions
- [x] shellcheck.net (i.e. the latest commit) currently gives no useful warnings about this
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related

It seems that shellcheck doesn't catch dashes used in identifier names (eg, functions). The allowable characters are documented [here](https://www.gnu.org/software/bash/manual/bashref.html#index-identifier):

> A `word` consisting solely of letters, numbers, and underscores, and beginning with a letter or underscore. `Name`s are used as shell variable and function names. Also referred to as an `identifier`.

For example, this does not return any warnings from `shellcheck -s bash` despite violating the above with `conda-activate` being an invalid name:

```
conda-activate()
{
    source /opt/"${USER}"/miniconda3/bin/activate "${@}";
}
```
