# [\#2256 Issue](https://github.com/koalaman/shellcheck/issues/2256) `closed`: `shellcheck -x` does not understand conditional includes (`source`)

#### <img src="https://avatars.githubusercontent.com/u/8544900?u=a1b4aae4c226d19eb291c4364daaa941fc67580e&v=4" width="50">[zx8](https://github.com/zx8) opened issue at [2021-06-18 09:14](https://github.com/koalaman/shellcheck/issues/2256):

#### Here's a snippet or screenshot that shows the problem:

```sh

#!/bin/bash

test -f foo.sh && source foo.sh

test -e foo.sh && source foo.sh

if [[ -f foo.sh ]]; then
  source foo.sh
fi

# etc 
```

#### Here's what shellcheck currently says:

```
$ shellcheck -x foo.sh

In 1.sh line 3:
test -f foo.sh && source foo.sh
                         ^----^ SC1091: Not following: foo.sh: openBinaryFile: does not exist (No such file or directory)

In 1.sh line 4:
test -e foo.sh && source foo.sh
                         ^----^ SC1091: Not following: foo.sh: openBinaryFile: does not exist (No such file or directory)

In 1.sh line 6:
  source foo.sh
         ^----^ SC1091: Not following: foo.sh: openBinaryFile: does not exist (No such file or directory)
```

#### Here's what I wanted or expected to see:

Nothing.




#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2021-06-23 09:52](https://github.com/koalaman/shellcheck/issues/2256#issuecomment-866697131):

The warning is not that the file may not be available for the script at runtime, but rather that ShellCheck is unable to account for it at analysis time.

You can use a `# shellcheck source=/dev/null` directive to simply skip the file.
