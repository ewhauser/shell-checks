# [\#2593 Issue](https://github.com/koalaman/shellcheck/issues/2593) `open`: Suggest here-string rather than `echo` + a pipeline

#### <img src="https://avatars.githubusercontent.com/u/168301?u=b824d9e71723b9993b271767e40426de83aa7263&v=4" width="50">[l0b0](https://github.com/l0b0) opened issue at [2022-09-25 16:32](https://github.com/koalaman/shellcheck/issues/2593):

#### For new checks and feature suggestions
- [x] https://www.shellcheck.net/ (i.e. the latest commit) currently gives no useful warnings about this
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related

#### Here's a snippet that shows the problem:

```sh
#!/usr/bin/env bash
echo value | foo
```

#### Here's what shellcheck currently says:

> No issues detected!

#### Here's what I wanted or expected to see:

> Use a here document instead of `echo` + a pipeline.

```sh
#!/usr/bin/env bash
foo <<< value
```

[Rationale](https://paperless.blog/bash-lints-beyond-shellcheck): This is analogous to the [useless cat](https://github.com/koalaman/shellcheck/wiki/SC2002) lint.

Exceptions: If you’re a strong proponent of left-to-right flow of information you can also use `<<< value foo`, but that doesn't seem worth the confusion.
