# [\#1650 Issue](https://github.com/koalaman/shellcheck/issues/1650) `closed`: Heredoc delimiter/marker not parsed correctly

#### <img src="https://avatars.githubusercontent.com/u/18301471?v=4" width="50">[Dabombber](https://github.com/Dabombber) opened issue at [2019-07-18T08:19:24Z](https://github.com/koalaman/shellcheck/issues/1650):

Heredocs made by perverse people where the delimiter contains the `'` or `"` characters are not parsed correctly. I'm not sure if it's valid code, but it works in Busybox's ash and Bash. The [posix docs](http://pubs.opengroup.org/onlinepubs/9699919799/utilities/V3_chap02.html#tag_18_07_04) say
> If any part of word is quoted, the delimiter shall be formed by performing quote removal on word, and the here-document lines shall not be expanded.


#### For bugs
- Rule Ids: SC1073, SC1072, SC1042, SC1041
- version: 0.6.0
- [x] The rule's wiki page does not already cover this (e.g. https://shellcheck.net/wiki/SC1041)
- [x] I tried on shellcheck.net and verified that this is still a problem on the latest commit

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/sh

cat <<'"EOF'
hello
"EOF
```

likewise for 
```sh
#!/bin/sh

cat <<'E'\''OF'
hello
E'OF
```
