```
#!/bin/bash
if [ $(foo)$? -eq 1 ]

```

#### Here's what I wanted or expected to see:
Shellcheck should detect that $? is in use at the end of the command substitution and realize I am checking the return code of the command rather than the output.

```

```

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2016-12-08T18:49:38Z](https://github.com/koalaman/shellcheck/issues/792#issuecomment-265821250):

Actually, you *are* checking the output. Just with the exit code concatenated to the end. 

If `foo` outputs "hello world", you'll be doing `[ hello world1 -eq 1 ]` which will fail with an error. If it outputs "4" and returns 1, you'll be doing `[ 41 -eq 1 ]`, which will just give the wrong result.

It only happens to work if `foo` will never write anything to stdout, which is hard for `shellcheck` to determine.

The canonical way to do this without checking the output of `foo` would be:

```
foo
if [ $? -eq 1 ]
```
