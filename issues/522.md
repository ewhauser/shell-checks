# [\#522 Issue](https://github.com/koalaman/shellcheck/issues/522) `open`: New potential error proposal: piping into eval without operands

#### <img src="https://avatars.githubusercontent.com/u/6459309?v=4" width="50">[Artoria2e5](https://github.com/Artoria2e5) opened issue at [2015-10-26T13:27:03Z](https://github.com/koalaman/shellcheck/issues/522):

There is a common mistake in new users:

``` Bash
echo echo 1 | eval
```

The correct method is:

``` Bash
source <(echo echo 1)
eval "$(echo echo 1)"
```

This is observed at https://unix.stackexchange.com/questions/238582. Some of my friends sometimes make this mistake, too.

This error is serious and almost definite.
