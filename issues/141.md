# [\#141 Issue](https://github.com/koalaman/shellcheck/issues/141) `closed`: Possible false positive for SC2016

#### <img src="https://avatars.githubusercontent.com/u/1235108?v=4" width="50">[myint](https://github.com/myint) opened issue at [2014-04-19T18:13:37Z](https://github.com/koalaman/shellcheck/issues/141):

I didn't write [this code](https://github.com/myint/misspellings/blob/master/utils/fetch_updated_misspellings_dict.sh), so I'm not sure if there is supposed to be a better way of doing it. But as far as I can tell, the below example is a false positive. Though, I'm not sure if `sed` is in the purview of `shellcheck`.

```
$ shellcheck -
sed '$s/,$//'
^D
In - line 1:
sed '$s/,$//'
    ^-- SC2016: Expressions don't expand in single quotes, use double quotes for that.
```

[Here is a reference](http://stackoverflow.com/questions/12390134/remove-comma-from-last-line/12390197#12390197) of similar usage.

Thanks

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2014-04-19T19:30:57Z](https://github.com/koalaman/shellcheck/issues/141#issuecomment-40878412):

This is indeed a false positive.

ShellCheck has some logic to try to differentiate e.g. sed 's/$foo/$bar/' and sed '$d', but it's highly simplistic and in this case it failed. 6ad3f55 fixes this particular case, but some kind of overhaul should probably be considered in the future.
