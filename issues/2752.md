#### Here's a snippet or screenshot that shows the problem:

```sh
#!/usr/bin/env bash

set -euo pipefail

added_files=()
modified_files=()

has_valid_added_files=$(git diff --name-status "${1}" "${2}" | grep -oP '[A]{1}[^A-Za-z]*(.*)' && echo true || echo false)

echo "${has_valid_added_files}"

if [[ ${has_valid_added_files}  ]]; then

  mapfile -t added_files < <(git diff --name-status "${1}" "${2}" | grep -oP '[A]{1}[^A-Za-z]*(.*)' | sed --regexp-extended 's/[AMD]\s+//g')

fi

has_valid_modified_files=$(git diff --name-status "${1}" "${2}" | grep -oP '[M]{1}[^A-Za-z]*(.*)' && echo true || echo false)

echo "${has_valid_modified_files}"

if [[ ${has_valid_modified_files}  ]]; then

  echo "getting modified files"

  mapfile -t modified_files < <(git diff --name-status "${1}" "${2}" | grep -oP '[M]{1}[^A-Za-z]*(.*)' | sed --regexp-extended 's/[AMD]\s+//g')

fi

cspell_ignored_files=$(jq '.ignorePaths | @sh' < .vscode/cspell.json)

# The problematic line
ignored_files=($(echo "${cspell_ignored_files}" | tr -d \'\"))

```

#### Here's what shellcheck currently says:

So in this situation, Shellcheck says:

`Prefer mapfile or read -a to split command output (or quote to avoid splitting)`

Now if I replace the problematic line by: `mapfile -t ignored_files < <(echo "${cspell_ignored_files}" | tr -d \'\")`

The resulting variable `ignored_files` does not have the appropriate structure. It is an array with one huge entry instead of several small entries.

If I use: `read -ar ignored_files <<< $(echo "${cspell_ignored_files}" | tr -d \'\")`

The same problem occurs.

Again, if I quote the command, the same problem occurs.

I'm by no means a BASH wizard and I looked up the rationale and the rule as long as the internet and I am completely puzzled what's going on here.

I tried to use `IFS=" "` but to no end.

Could you enlighten me about how to get a robust line here, please?
