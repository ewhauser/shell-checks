# [\#532 Issue](https://github.com/koalaman/shellcheck/issues/532) `closed`: "Non-exhaustive patterns in function hasNum" error

#### <img src="https://avatars.githubusercontent.com/u/147409?v=4" width="50">[mafredri](https://github.com/mafredri) opened issue at [2015-10-31T14:16:02Z](https://github.com/koalaman/shellcheck/issues/532):

Input:

``` shell
#!/bin/sh

# shellcheck source=source/config/settings.sh
. ${VAR}/config/settings.sh
```

Gives:

```
shellcheck: ShellCheck/Analytics.hs:226:9-46: Non-exhaustive patterns in function hasNum
```

On OS X El Capitan, Shellcheck v. 0.4.1

Only seems to be a problem when `$VAR` is unknown. Can also be worked around by wrapping `$VAR` in quotes, like so: `"${VAR}"`.
