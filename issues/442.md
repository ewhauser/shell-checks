# [\#442 Issue](https://github.com/koalaman/shellcheck/issues/442) `closed`: SC2154 var is referenced but not assigned

#### <img src="https://avatars.githubusercontent.com/u/13869060?v=4" width="50">[valiko-ua](https://github.com/valiko-ua) opened issue at [2015-08-19T12:12:01Z](https://github.com/koalaman/shellcheck/issues/442):

prj_path is assigned using "eval" but shellcheck reports it's not assigned:

``` bash
eval prj_path=\$$prj_var
                ^––SC2086 Double quote to prevent globbing and word splitting.
if [ -z "$prj_path" ]; then
        ^––SC2154 prj_path is referenced but not assigned.
```

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2015-08-19T17:49:38Z](https://github.com/koalaman/shellcheck/issues/442#issuecomment-132718465):

Unfortunately there's no good way for ShellCheck to figure out assignments that happen based on runtime information, like with `eval`. https://github.com/koalaman/shellcheck/wiki/SC2154 contains various workarounds.

Fun fact: the variable is not necessarily assigned. If `prj_var='1 true'` or `prj_var=\'`  then the variable will remain unset.
