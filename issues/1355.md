# [\#1355 Issue](https://github.com/koalaman/shellcheck/issues/1355) `open`: Feature: Shellcheck disable all

#### <img src="https://avatars.githubusercontent.com/u/38833810?v=4" width="50">[andrewmiller1](https://github.com/andrewmiller1) opened issue at [2018-10-07T22:08:39Z](https://github.com/koalaman/shellcheck/issues/1355):

Use case: When transitioning from pseudocode to actual code, you may want the linting disabled for parts of your code in a single file.

1. By dropping a `# shellcheck disable` (following violations are unreported) below what you're currently working on, you can keep linting enabled, without being bombarded by violations you haven't even got to yet.
2. Pairing with a `# shellcheck enable` (reporting of violations is turned back on) can allow you more flexibility in what parts of the file in progress you chose to work on and when.

This will keep the relevant errors that grab your attention to be the only ones that do.
