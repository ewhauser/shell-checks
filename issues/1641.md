# [\#1641 Issue](https://github.com/koalaman/shellcheck/issues/1641) `closed`: SC2203 assumes * is always a globbing attempt, even though it can also be multiplication

#### For new checks and feature suggestions

#### Here's a snippet or screenshot that shows the problem:

```sh

#!/bin/bash

foo=3
bar=4

if [[ 2*$foo -gt $bar ]]; then
  echo yes
else
  echo no
fi

```

#### Here's what shellcheck currently says:
```
$ shellcheck myscript
 
Line 6:
if [[ 2*$foo -gt $bar ]]; then
      ^-- SC2203: Globs are ignored in [[ ]] except right of =/!=. Use a loop.

$ 
```


#### Here's what I wanted or expected to see:

Either no finding, or something that mentions multiplication. Since `*` is valid for multiplication, we shouldn't yell at the user for using it for globbing any time we see it.

#### <img src="https://avatars.githubusercontent.com/u/7482557?v=4" width="50">[Gandalf-](https://github.com/Gandalf-) commented at [2019-08-07T04:42:38Z](https://github.com/koalaman/shellcheck/issues/1641#issuecomment-518937257):

You could work around this and preserve the multiplication behavior with `\*`
```sh
foo=3
bar=4

if [[ 2\*$foo -gt $bar ]]; then
  echo yes
else
  echo no
fi
```
```
$ bash example.sh
yes
```

It would be easy enough to add a comment to this warning explaining "if you want multiplication here, we suggest `\*` for clarity".

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2019-10-13T03:22:47Z](https://github.com/koalaman/shellcheck/issues/1641#issuecomment-541381622):

You're right, of course. You will no longer get this warning, and if you use `[ .. ]` you will get a new SC2255 suggesting that you use `$((..))` to expand your own expressions. Thanks!
