#### <img src="https://avatars.githubusercontent.com/u/433817?v=4" width="50">[grimreaper](https://github.com/grimreaper) commented at [2014-03-10T18:33:56Z](https://github.com/koalaman/shellcheck/issues/106#issuecomment-37217500):

Almost any use of ls followed by a pipe is wrong. Likely they intended to use find(1).


#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2014-03-15T23:30:11Z](https://github.com/koalaman/shellcheck/issues/106#issuecomment-37741874):

@grimreaper makes a good point. Shellcheck already suggests using `find`.

GNU `ls -l` prints a status line, so the count would be off by one anyways. 

Technical Trivia: in obscure cases, `ls` may work better than `find` here. `ls` mangles filenames, including the line feeds that would cause `find ... | wc -l` to count the file twice.
