# [\#1481 Issue](https://github.com/koalaman/shellcheck/issues/1481) `open`: Be more consistent in warning about unnecessary ${} with associative arrays in $((..))

#### Here's a snippet or screenshot that shows the problem:

This is a strange case that ShellCheck actually gets right, but for the wrong reasons. `${..}` *is* required for associative arrays with weird key names:

```sh
#!/bin/bash
declare -A arr
x="]"
echo "$(( ${arr[$x]} ))"     # Works
echo "$(( ${arr[bla]} ))"    # Says to remove the ${..}
echo "$(( arr[$x] ))"        # Fails
```

#### Here's what shellcheck currently says:

```sh
In foo line 5:
echo "$(( ${arr[x]} ))"
          ^-------^ SC2004: $/${} is unnecessary on arithmetic variables.
```

Note in particular that it only warns about the case where it *is* fine to remove. 

#### Here's what I wanted or expected to see:

Probably nothing, since it would be better for consistency if all instances of using arrays were warned about the same way.
