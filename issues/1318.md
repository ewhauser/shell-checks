#### Here's a snippet or screenshot that shows the problem:

```sh
#! /bin/bash

nowarn=(
	abc,def
	ghi,jkl
)

warn=(
	abc,def,ghi
	jkl,mno,pqr
)

# use the arrays to avoid warnings for that
echo "${nowarn[@]}"
echo "${warn[@]}"
```

#### Here's what shellcheck currently says:
```
In test.sh line 8:
warn=(
     ^-- SC2054: Use spaces, not commas, to separate array elements.
```

#### Here's what I wanted or expected to see:
```
In test.sh line 3:
nowarn=(
     ^-- SC2054: Use spaces, not commas, to separate array elements.

In test.sh line 8:
warn=(
     ^-- SC2054: Use spaces, not commas, to separate array elements.
```

It looks like https://github.com/koalaman/shellcheck/blob/master/src/ShellCheck/Analytics.hs#L1212 should say `length (...) > 0`, rather than `length (...) > 1`.
