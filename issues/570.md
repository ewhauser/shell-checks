# [\#570 Issue](https://github.com/koalaman/shellcheck/issues/570) `closed`: False positive on SC1037 with "$$-$foo"

#### <img src="https://avatars.githubusercontent.com/u/217935?v=4" width="50">[AloisMahdal](https://github.com/AloisMahdal) opened issue at [2016-01-05T19:12:05Z](https://github.com/koalaman/shellcheck/issues/570):

With following code:

```
#!/bin/sh
foo=bar
echo "$$-$foo"
```

ShellCheck raises problem 1037 with the last line

```
In sc-doldol line 3:
echo "$$-$foo"
        ^-- SC1037: $$-$ is equivalent to ${$}-$.
```

While the hint seems to be technically true (changing to `"${$}-$foo"` works the same), I don't see what this has to do with "SC1037 - Braces are required for positionals over 9, e.g. ${10}" at least as how it is documented now. With regards to the SC name, "Braces are required" is actually _false_ about `$$`.

Also I'm not convinced it improves the code anyway. Quite contrary: I'd say `${$}` is much less readable and less common. (I've seen it first time after over 2 years of active Bash scripting).

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2016-01-05T19:29:03Z](https://github.com/koalaman/shellcheck/issues/570#issuecomment-169104812):

Agreed. I don't think anyone would expect the `-` to be part of the variable name in this case.
