# [\#2847 Issue](https://github.com/koalaman/shellcheck/issues/2847) `open`: Add rule for checking return code when using negation

#### <img src="https://avatars.githubusercontent.com/u/342823?u=0aa582f1b0658800fd821ac4f9cb0b7b6ed8ad07&v=4" width="50">[andsens](https://github.com/andsens) opened issue at [2023-10-20 07:45](https://github.com/koalaman/shellcheck/issues/2847):

[This one](https://github.com/andsens/homeshick/pull/218/files) just popped up in my own code from years ago and I have run into it a couple of times before that as well.  
I think adding a rule to warn about the following would make a lot of sense:
```
if ! command; then
  ret=$?
fi
...
return $ret
```

Here, `$?` will always be `0` because the `!` negates the return code itself. This is a non-obvious bug and can be hard to find. If you don't know about that behavior you can look at the code for hours and still not get what the hell is going on.  
Alternative working versions:

```
if command; then
  :
else
  ret=$?
fi
```

```
command || ret=$?
```
