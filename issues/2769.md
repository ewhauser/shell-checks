#### <img src="https://avatars.githubusercontent.com/u/25598447?u=73c3f3a041be0ccabd72bb816d5782b30aefe822&v=4" width="50">[TinCanTech](https://github.com/TinCanTech) commented at [2023-06-25 23:37](https://github.com/koalaman/shellcheck/issues/2769#issuecomment-1606328845):

I think this example highlights what `shellcheck` is really doing:
```
#!/bin/sh

# Question - Do line-continuation in comments work? \
  Answer - No.

	echo "$*" |
		# Preceeding pipe
		# comment allowed
		sed -e 's/0/1/g' \
			-e 's/1/0/g' || \
				# Preceeding OR \
				# comment allowed
				return

	exit

$ shellcheck myscript
 
[Line 12:]
                        -e 's/1/0/g' || \
                        ^-- [SC2215](https://www.shellcheck.net/wiki/SC2215) (warning): This flag is used as a command name. Bad line break or missing [ .. ]?
 
[Line 13:]
                                # Preceeding OR \
                                                 ^-- [SC1143](https://www.shellcheck.net/wiki/SC1143) (error): This backslash is part of a comment and does not continue the line.
```

If you remove the unnecessary `\` backslashes after your `|` pipes then `shellcheck` will not complain about the backslash in comments, because the **common error**: _Commenting out a line in a wrapped command_, is not triggered.

`shellcheck` is assuming that a line ending `\` is a wrapped command, even though it is not.

eg:
```
188 -e 's|.*.||g' \
189 -e 's|.+.||g' | \ <- *** REMOVE THIS backslash ***
190 # These two sorts need to be separate commands \
191 sort -u | \
192 sort -d | \
```
