#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash

_ekvstore_nonatomic_set () {
  declare -n ns="$1"
  local key="$2"
  local val="$3"

  ns[$key]=$val
}


```

#### Here's what shellcheck currently says:
```
Line 8	SC2034: ns appears unused. Verify use (or export if used externally).
```
#### Here's what I wanted or expected to see:

Nothing. `declare -n` creates an alias to the variable whose name is in $1. Setting it should be considered "usage" in this case.
