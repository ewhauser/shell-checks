# [\#3326 Issue](https://github.com/koalaman/shellcheck/issues/3326) `closed`: Is OPTARG really a bash specific variable?

#### <img src="https://avatars.githubusercontent.com/u/2486859?u=f870822acdc0d647fcb583b9e2e96efb8e73304d&v=4" width="50">[sstrollo](https://github.com/sstrollo) opened issue at [2025-10-16 09:47](https://github.com/koalaman/shellcheck/issues/3326):

Hi,

This commit https://github.com/koalaman/shellcheck/commit/1901feb5274214fbf3ebbc218631b6de81254d12 introduced `OPTARG` as a bash specific variable. I don't think it is, can't really find any references to why it should be, in fact the opposite seems true:

https://pubs.opengroup.org/onlinepubs/9699919799/utilities/getopts.html
https://en.wikipedia.org/wiki/Getopts

`OPTARG` is how `getopts` communicates the option character found to the caller.

#### Here's what shellcheck currently says:

```
        e)  engine="${OPTARG}"
                    ^-------^ SC3028 (warning): In POSIX sh, OPTARG is undefined.
```

When run as
```
shellcheck --shell=sh my-script
```

On my-script:
```
...
     21 while getopts d:e:hikp:sv: opt; do
     22     case $opt in
...
     39         e)  engine="${OPTARG}"
     40             ;;
```


#### Here's what I wanted or expected to see:
Nothing :)
