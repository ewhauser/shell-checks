# [\#2977 Issue](https://github.com/koalaman/shellcheck/issues/2977) `closed`: Skip SC2015 when it end with: || true

#### For bugs
- Rule Id (if any, e.g. SC1000): SC2015
- My shellcheck version (`shellcheck --version` or "online"): online
- [x] The rule's wiki page does not already cover this (e.g. https://shellcheck.net/wiki/SC2086)
- [x] I tried on https://www.shellcheck.net/ and verified that this is still a problem on the latest commit

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/sh
set -e

foo && bar || true
```

#### Here's what shellcheck currently says:

```
foo && bar || true
    ^-- [SC2015](https://www.shellcheck.net/wiki/SC2015) (info): Note that A && B || C is not if-then-else. C may run when A is true.
```

#### Here's what I wanted or expected to see:

Nothing, the `|| true` is used to ignore return codes so even if it get executed when A is true it doesn't matter.


#### <img src="https://avatars.githubusercontent.com/u/65165943?u=9422f91a6925814463363e7e051da970b1395a74&v=4" width="50">[Jakub-Kapusta](https://github.com/Jakub-Kapusta) commented at [2024-10-04 04:58](https://github.com/koalaman/shellcheck/issues/2977#issuecomment-2392823717):

Workaround: Currently this can be bypassed by using command groupings.
```bash
#!/bin/sh
set -e

# $? will be 0
{ foo && bar; } || :
```
