# [\#1648 Issue](https://github.com/koalaman/shellcheck/issues/1648) `closed`: Warn about subshells not respecting `set -e` ins some situations

#### <img src="https://avatars.githubusercontent.com/u/838250?v=4" width="50">[cdman](https://github.com/cdman) opened issue at [2019-07-17T10:04:25Z](https://github.com/koalaman/shellcheck/issues/1648):

- [x] shellcheck.net (i.e. the latest commit) currently gives no useful warnings about this
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related


#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash
set -e
echo "Foo: $(false)"
echo "After"
```

#### Here's what shellcheck currently says:
No issues detected! :)


#### Here's what I wanted or expected to see:
Warn about the fact that exit values inside of echoed strings are not kept and recommend something like:

```sh
# ...
ouput=$(false)
echo "Foo: $output"
# ...
```

Which would die (as expected) if `set -e` is in effect.
