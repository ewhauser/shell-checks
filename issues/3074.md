# [\#3074 Issue](https://github.com/koalaman/shellcheck/issues/3074) `closed`: SC2154 is triggered on an all caps variable

#### <img src="https://avatars.githubusercontent.com/u/3066697?u=b0881521c5aff3cc9084fc44efc27d042fb6047b&v=4" width="50">[danielcft](https://github.com/danielcft) opened issue at [2024-10-31 22:12](https://github.com/koalaman/shellcheck/issues/3074):

- Rule Id: SC2154
- My shellcheck version: 0.10.0
- It works on shellcheck.net - could not reproduce the issue there.

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/sh

echo "${SRC}"
```

#### Here's what shellcheck currently says:
```
In bug.sh line 3:
echo "${SRC}"
      ^----^ SC2154 (warning): SRC is referenced but not assigned.

For more information:
  https://www.shellcheck.net/wiki/SC2154 -- SRC is referenced but not assigned.
```

#### Here's what I wanted or expected to see:

I expect no warnings. 

The variable uses the all caps convention, so it should not be flagged.
The documentation on wiki states it explicitly.

from https://www.shellcheck.net/wiki/SC2154
> Note: This message only triggers for variables with lowercase characters in their name (foo and kFOO but not FOO) due to the standard convention of using lowercase variable names for unexported, local variables.
