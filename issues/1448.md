```sh
#!/bin/sh
foo() { echo bar; }
```

#### Here's what I wanted or expected to see:
The only acceptable flag under POSIX sh for `readonly` is -p ([see spec](https://pubs.opengroup.org/onlinepubs/9699919799/utilities/V3_chap02.html#readonly)). The same is true for dash, but bash also accepts [-aAf]. I expected to see the usual "In POSIX sh, x flags are undefined".
