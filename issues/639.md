#### <img src="https://avatars.githubusercontent.com/u/197253?v=4" width="50">[kurahaupo](https://github.com/kurahaupo) commented at [2016-04-05T00:35:31Z](https://github.com/koalaman/shellcheck/issues/639#issuecomment-205560564):

> with the exact message being "Found trailing ] outside test. Missing [? [SC2171]

Fair point about the general case, however when Shellcheck highlights code
for some oddball reason, it's worth taking another look anyway.

> `each_node=$(echo $each_node | tr -d "'" | tr -d [ | tr -d ])`

In this case, "tr -d" can take a string of multiple characters, so there's
no need for the isolated ]:

```
tr -d "'[]"
```

And moreover, variable substitution can do what you want better and MUCH
faster:

```
each_node=${each_node//[\'\[\]]/}
```
