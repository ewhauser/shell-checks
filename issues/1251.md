# [\#1251 Issue](https://github.com/koalaman/shellcheck/issues/1251) `closed`: SC2217: mv does read stdin

#### <img src="https://avatars.githubusercontent.com/u/10639154?v=4" width="50">[s-hamann](https://github.com/s-hamann) opened issue at [2018-06-06T11:47:25Z](https://github.com/koalaman/shellcheck/issues/1251):

I have code similar to the following:

```sh
while read -r filename; do
  mv -i $filename . < /dev/tty
done < <(something)
```

I get SC2217: "Redirecting to 'mv', a command that doesn't read stdin."
In this case `mv` does read stdin and I am not aware of any way to get the desired behaviour without redirecting into `mv`.

I think the check should be refined to test if `mv`/`cp`/`rm`/... is used with `-i`.
