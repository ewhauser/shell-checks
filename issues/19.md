Interesting project. 

I am wondering if it would be possible to use the AST to generate correct shell scripts from Haskell. The use case I have in mind is what Ansible is doing: managing remote hosts with SSH and shell scripts.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2013-11-03T21:03:39Z](https://github.com/koalaman/shellcheck/issues/19#issuecomment-27654259):

The AST used by ShellCheck supports a lot of features at great precision, which is good for parsing but terrible for synthesis. In something simple like "echo foo" there are half a dozen nodes dealing with things like how "foo" is not quoted and how we have specified zero redirections and no per-command assignments. 

If you want to generate correct scripts, I would think it would be way easier to make an AST for the logic you want to support and then generating a correct script based on it.
