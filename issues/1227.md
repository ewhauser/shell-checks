#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2018-05-20T04:30:42Z](https://github.com/koalaman/shellcheck/issues/1227#issuecomment-390457007):

Doubling double quotes does not escape them. It just stops one string and starts another, which is a no-op:

```
$ echo "a ""b"" c"
a b c

```

Use `\"` instead:
```
$ echo "a \"b\" c"
a "b" c
```

ShellCheck should probably warn about this though. There's no reason to ever do it.
