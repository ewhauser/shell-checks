#### <img src="https://avatars.githubusercontent.com/u/1823914?v=4" width="50">[arth1](https://github.com/arth1) commented at [2014-08-18T15:05:38Z](https://github.com/koalaman/shellcheck/issues/204#issuecomment-52505136):

I am against this;  .profile is used by all bourne shell family shells, and if you move stuff out of it to .bashrc/.bash_profile, the assignments will no longer be picked up by other shells, even if you want it.  I tend to use .profile, and test the shell for precisely that reason.

As for the PATH="$PATH:..." being called multiple times, here's a quick PATH cleanup that should work in all bourne family shells:

```
PATH=$(echo $PATH | awk '!s[$1]++' RS=: ORS=: | sed 's/:$//')
export PATH
```
