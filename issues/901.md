# [\#901 Issue](https://github.com/koalaman/shellcheck/issues/901) `open`: Feature request: Alert for exporting an array

#### <img src="https://avatars.githubusercontent.com/u/619016?v=4" width="50">[andras-tim](https://github.com/andras-tim) opened issue at [2017-05-11T16:25:14Z](https://github.com/koalaman/shellcheck/issues/901):

#### For new checks and feature suggestions
- [x] shellcheck.net (i.e. the latest commit) currently gives no useful warnings about this
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related


#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash
array=()

function set_array()
{
    export array=(1 2)
}

set_array
echo "${array[*]}"
```

#### Here's what shellcheck currently says:
No issues detected!


#### Here's what I wanted or expected to see:
Bash doesn't support exporting an array


#### Extra info
StackOverflow question about this:
http://stackoverflow.com/questions/5564418/exporting-an-array-in-bash-script

The current maintainer of Bash (Jan 2022) said in 2014 that he had no intention of every supporting the export of Bash arrays:

https://web.archive.org/web/20150530054828/https://www.openwall.com/lists/oss-security/2014/09/30/1

Seems like it would be a great thing for shellcheck to look for and prevent.
