#### <img src="https://avatars.githubusercontent.com/u/129536?v=4" width="50">[dolmen](https://github.com/dolmen) opened issue at [2014-07-09T21:03:25Z](https://github.com/koalaman/shellcheck/issues/188):

Here is [some shell code](/nojhan/liquidprompt/blob/9633ac83cad5f5702c1e853940c0ab2e166961bf/liquidprompt#L719) that triggers a false positive for SC1001:

```
    \git rev-parse --inside-work-tree >/dev/null 2>&1 || return
```

In this code, the backslash is used to tell bash/zsh to bypass any alias.
But shellcheck reports this:

```
SC1001 This \g will be a regular 'g' in this context.
```

This is a false positive.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2014-07-27T00:02:05Z](https://github.com/koalaman/shellcheck/issues/188#issuecomment-50251875):

I've updated [the wiki](https://github.com/koalaman/shellcheck/wiki/SC1001) for this message to suggest using `command` instead.
