#### <img src="https://avatars.githubusercontent.com/u/3999221?v=4" width="50">[pawamoy](https://github.com/pawamoy) commented at [2017-08-26T21:15:24Z](https://github.com/koalaman/shellcheck/issues/982#issuecomment-325162315):

Some of your commands are not semantically valid.
If you want to use the output of a command as a temporary file for input, you should use the following syntax:

```
shellcheck <(cat getMOSPatch.sh)  # note the absence of $
```

Also, `cat getMOSPatch.sh > shellcheck` will only write output of the cat command to a **file** named shellcheck. Il will not pipe it to the **command** named shellcheck.

These are just syntax observations, not ways to achieve what you would like to do with shellcheck.

#### <img src="https://avatars.githubusercontent.com/u/78236?v=4" width="50">[mineo](https://github.com/mineo) commented at [2017-08-27T07:29:36Z](https://github.com/koalaman/shellcheck/issues/982#issuecomment-325182036):

`cat getMOSPatch.sh | shellcheck` works if you change it to `cat getMOSPatch.sh | shellcheck -`.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2017-08-27T19:30:21Z](https://github.com/koalaman/shellcheck/issues/982#issuecomment-325219092):

Specifying `-` as a filename is the canonical way to get programs to read from stdin and shellcheck supports it. Process substitution and `/device/stdin` works too
