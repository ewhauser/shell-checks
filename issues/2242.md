#### <img src="https://avatars.githubusercontent.com/u/25376866?u=9cd835e0f28186baf980e20304525aef6edc2dab&v=4" width="50">[KlzXS](https://github.com/KlzXS) commented at [2021-06-20 22:18](https://github.com/koalaman/shellcheck/issues/2242#issuecomment-864618992):

Put quotes around it (`"[640x480]"`) so that it properly interprets it as an argument and not a range.

#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2021-07-25 19:59](https://github.com/koalaman/shellcheck/issues/2242#issuecomment-886250292):

It's a bit of a false lead but definitely a true error: it is indeed interpreted as a range, and the script will break if you have a file named `x` or `0` in the current directory. Make sure to quote it so that it'll be used literally.
