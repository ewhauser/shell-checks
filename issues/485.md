# [\#485 Issue](https://github.com/koalaman/shellcheck/issues/485) `closed`: Alias suppression is not interpreted properly

#### <img src="https://avatars.githubusercontent.com/u/929932?v=4" width="50">[RedX2501](https://github.com/RedX2501) opened issue at [2015-10-01T09:00:10Z](https://github.com/koalaman/shellcheck/issues/485):

`grep` is an alias in my bashrc. `\grep` forces bash to use the unaliased command. This is not regarded correctly by the checker.

![2015-10-01_12-00-10](https://cloud.githubusercontent.com/assets/929932/10216696/9647ea72-6848-11e5-9f02-6ed51e16f0fc.png)

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2015-10-02T02:00:58Z](https://github.com/koalaman/shellcheck/issues/485#issuecomment-144898200):

Normally in a script, aliases are not expanded at all, but in `.bashrc` and otherwise sourcing from an interactive shell, you're right.

How would you feel about using the more explicit (and POSIX compliant) `command grep ...` to override functions and aliases?


#### <img src="https://avatars.githubusercontent.com/u/929932?v=4" width="50">[RedX2501](https://github.com/RedX2501) commented at [2015-10-03T05:48:48Z](https://github.com/koalaman/shellcheck/issues/485#issuecomment-145204279):

Maybe the best approach to write a notice about the behaviour. Something like:

```
\<command> only suprresses aliases in interactive shell. Alias suppression is not necessary in scripts. \<letter> will be treated as <letter>. Posix compliant is command <command>
```
