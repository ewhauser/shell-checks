#### Here's a snippet or screenshot that shows the problem:

```sh

#!/bin/sh
 : "${myvar="${1:-}"}"
[ -n "${myvar}" ] || { echo "Usage [...]"; exit 0; }
echo "${myvar}"
```

#### Here's what shellcheck currently says:
```
Line 2:
 : "${myvar="${1:-}"}"
    ^-- SC2154: myvar is referenced but not assigned.
```

#### Here's what I wanted or expected to see:
I would expect no error. The statement basically sets `myvar` to `<value>`. This is useful when you want to assign a fallback value to a variable that might be overwritten by an already existing variable.

In the snippet example I can either-or do:
- `myvar=output myscript.sh`
- `myscript.sh output`
- `myvar=output myscript.sh ignored` => results in overruling `$1`.

Maybe not the best example, but useful in certain situations. There are at least two other ways of implementing this behavior, but I think this is one valid way.
