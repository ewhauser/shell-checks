# [\#3289 Issue](https://github.com/koalaman/shellcheck/issues/3289) `closed`: SC3045 does not conform the actual POSIX.1-2024

#### <img src="https://avatars.githubusercontent.com/u/12705661?u=7d7c4a7a8564e173cc8c0f6ac0254707a340ba49&v=4" width="50">[juliyvchirkov](https://github.com/juliyvchirkov) opened issue at [2025-09-05 21:36](https://github.com/koalaman/shellcheck/issues/3289):

Shellcheck flags `ulimit -n` as `SC3045 (warning): In POSIX sh, ulimit -n is undefined.`, but for [the actual POSIX.1-2024 specifications](https://pubs.opengroup.org/onlinepubs/9799919799/nframe.html) that's not true anymore.

> Issue 8
> Austin Group Defect 854 is applied, adding a note to the APPLICATION USAGE section that this utility is required to be intrinsic.
> 
> Austin Group Defect 1122 is applied, changing the description of NLSPATH .
> 
> Austin Group Defect 1418 is applied, adding the -H, -S, -a, -c, -d, -n, -s, -t, and -v options, and relating the -f option to the RLIMIT_FSIZE resource for [setrlimit()](https://pubs.opengroup.org/onlinepubs/9799919799/functions/setrlimit.html).
> 
> Austin Group Defect 1669 is applied, moving the ulimit utility, excluding the -t option, from the XSI option to the Base.

Please examine [The Open Group Base Specifications Issue 8 IEEE Std 1003.1-2024 > Shell & Utilities > Utilities > ulimit](https://pubs.opengroup.org/onlinepubs/9799919799/utilities/ulimit.html) for details.

I can confirm the latest Shellcheck release v0.11.0 still flags `ulimit -n`.
