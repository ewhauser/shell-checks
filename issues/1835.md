# [\#1835 Issue](https://github.com/koalaman/shellcheck/issues/1835) `open`: SC2068 should warn on unquoted :+ parameter expansions

#### <img src="https://avatars.githubusercontent.com/u/1885701?v=4" width="50">[dimo414](https://github.com/dimo414) opened issue at [2020-02-13 06:12](https://github.com/koalaman/shellcheck/issues/1835):

- Rule Id (if any, e.g. SC1000): SC2068
- My shellcheck version (`shellcheck --version` or "online"): 
- [x] The rule's wiki page does not already cover this (e.g. https://shellcheck.net/wiki/SC2086)
- [x] I tried on shellcheck.net and verified that this is still a problem on the latest commit

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash

bar=
foo --bang=$bar
foo ${bar:+--bang=$bar}
foo ${bar:+"--bang=$bar"}
foo "${bar:+--bang=$bar}"
```

#### Here's what shellcheck currently says:

> ```
> Line 4:
> foo --bang=$bar
>            ^-- SC2086: Double quote to prevent globbing and word splitting.
> ```

#### Here's what I wanted or expected to see:

Line 5 should similarly trigger, as it is also subject to word splitting. By contrast line 6 (correctly) does not trigger because the inner expansion is properly quoted. This has the effect of only passing an argument to `foo` if `$bar` is set, and otherwise no arguments are passed.

As @eatnumber1 notes below, line 7 is also safe because the entire expansion is quoted, but will result in an empty-string argument being passed to `foo`, which isn't necessarily desirable.

#### <img src="https://avatars.githubusercontent.com/u/17551?u=0e8d378c2580ea8803024faa7893fc10512c63d5&v=4" width="50">[eatnumber1](https://github.com/eatnumber1) commented at [2020-02-13 17:26](https://github.com/koalaman/shellcheck/issues/1835#issuecomment-585874620):

Worth noting that ShellCheck does the right thing (no warnings) if the entire expansion is quoted. E.g. the following produce no warnings and each `printf` correctly prints `foo bar` with no line breaks.
```bash
foo="baz foobaz"
printf '%s\n' "${foo:+foo bar}"
printf '%s\n' "${foo:+"foo bar"}"
```
