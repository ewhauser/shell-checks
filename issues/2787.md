It's probably a long standing debate, to do integer attribute declaration or not to do, and, for my part, my teachers have been among those who fell on the side of "not to do," arguing against issues of portability, of maintainability, of bash's inner workings, as well as some [bugginess](https://unix.stackexchange.com/questions/381761/what-do-declare-name-and-declare-g-do). More specifically, $((...)) is Posix conformant, any future maintainers would need to know of how, unless I'm mistaken about this syntax again, integer declaration adds arithmetic to the list of expansions that occur during variable assignment, and bash stores data as strings. 

```
 $ bash -c 'f() { declare a; integer a; a=1+1; echo "$a"; }; integer() { typeset -gi "$1"; }; f'
1+1
 $ bash -c 'f() { integer a; a=1+1; echo "$a"; }; integer() { typeset -gi "$1"; }; f'
2
 ```

But now I wonder, what are some of the arguments in favor of integer declaration?
