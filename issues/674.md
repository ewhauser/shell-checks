# [\#674 Issue](https://github.com/koalaman/shellcheck/issues/674) `open`: Deprecated programs with better alternatives?  - readlink -f vs realpath?

#### <img src="https://avatars.githubusercontent.com/u/16256753?v=4" width="50">[mikedlr](https://github.com/mikedlr) opened issue at [2016-05-22T22:06:48Z](https://github.com/koalaman/shellcheck/issues/674):

There are lots of issues where there was an old or temporary way to do something, such as **realpath**, which has a more stably supported way to do the same thing, e.g. **readlink -f**.  Looking at the manual it seems that shellcheck doesn't check for these which meant I was surprised by a broken script porting from fedora to ubuntu 14.04.  

Typical other examples would be gnu extensions that posix later adopted with a different syntax.  There should be a warning if the old syntax is present and a suggestion to use the more portable syntax.  

I'd like to propose implementing warnings in these cases as an improvement for shellcheck.  

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2016-05-23T00:04:27Z](https://github.com/koalaman/shellcheck/issues/674#issuecomment-220864850):

This is definitely something shellcheck should warn about, but it's not always straight forward. `readlink -f` is a good suggestion on GNU and FreeBSD, but is not supported by OS X or POSIX.

ShellCheck currently doesn't know which userspace you're targeting, but it definitely should in the future. At the very least, it should support warning about non-POSIX flags and commands.    

If there are GNU extensions that POSIX now adopts, then please comment with them. ShellCheck already suggests `$((..))` over `$[..]`, and suggests getting rid of `expr`.
