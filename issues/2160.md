According to https://pubs.opengroup.org/onlinepubs/9699919799/utilities/cd.html on "ENVIRONMENT VARIABLES"
section:

```
PWD
This variable shall be set as specified in the DESCRIPTION. If an application sets or unsets the value of PWD, the behavior of cd is unspecified.
```

Not sure if this is a reasonable thing to lint/watch out for, as there might be a lot more "unsafe" env vars in other use cases, but since I didn't saw a precedent for this I am bringing it up (up to shellcheck maintainers to deem this as a non-feature).

#### Here's a snippet or screenshot that shows the problem:

```sh

#!/bin/sh

PWD=""

```

#### Here's what shellcheck currently says:
No issues detected!


#### Here's what I wanted or expected to see:
Warning: Modifying PWD can cause undefined behavior for `cd`.
