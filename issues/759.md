#### <img src="https://avatars.githubusercontent.com/u/197253?v=4" width="50">[kurahaupo](https://github.com/kurahaupo) commented at [2016-11-01T07:34:47Z](https://github.com/koalaman/shellcheck/issues/759#issuecomment-257505635):

This spectrum of behaviour is of course the main reason that printf is
recommended over echo ... and it's advice worth following.

> when checking with busybox and bash, i got the same result:
> 
> ```
> # echo -en "foo\nbar\n"
> # foo
> # bar
> #
> # echo "-enfoo\nbar\n"
> # -enfoo\nbar\n
> ```
> 
> maybe it is "special", because $2 is empty?

In the case of Bash, that observed behaviour is not because $2 is empty, but
because although $1 starts with '-', it contains characters other than 'e',
'E' and 'n'. By the way, this conflicts with the POSIX standard, which
specifies only the exact string '-n' for special treatment.
