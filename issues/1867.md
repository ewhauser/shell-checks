#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash
source "$(dirname "$0")/lib/logging.sh"
source "$(dirname "$0")/lib/config.sh"
```

It might be tempting (and go unnoticed), that you fill the first line, and you only duplicate the comment on the second one.

```sh
#!/bin/bash
# shellcheck source=SCRIPTDIR/lib/logging.sh
source "$(dirname "$0")/lib/logging.sh"
# shellcheck source=SCRIPTDIR/lib/logging.sh
source "$(dirname "$0")/lib/config.sh"
```

#### Here's what shellcheck currently says:

Nothing

#### Here's what I wanted or expected to see:

```
# shellcheck source=SCRIPTDIR/lib/logging.sh
source "$(dirname "$0")/lib/config.sh"
       ^-- SC1090: Are you sure that this is the correct source to follow?
```

I know that this can be highly controversial, so, I just want to open the discussion about it.

My idea is that, in python terms, this check would come:

```python
if os.path.basename(directive)[0] not in os.path.basename(source)[0]:
	emit_notice(...)
```
