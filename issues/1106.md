#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash
x=y
eval "$x=(\"\${$x[@]}\")"
```

#### Here's what shellcheck currently says:

```
eval "$x=(\"\${$x[@]}\""
               ^-- SC1087: Braces are required when expanding arrays, as in ${array[idx]}.
```


#### Here's what I wanted or expected to see:

The error should not be reported, the array expansion is inside braces; `$x` is the name of the array. This may not be possible to avoid.

#### <img src="https://avatars.githubusercontent.com/u/10137?v=4" width="50">[ghost](https://github.com/ghost) commented at [2018-02-05T07:11:51Z](https://github.com/koalaman/shellcheck/issues/1106#issuecomment-362999298):

Incidentally, there's a way to to this without eval.

```sh
#!/bin/bash
x=(1 2 3)
ref=x
declare -a 'y=("${!'"$ref"'[@]}")'
```

Both shellcheck 0.7.4 and the online version object, however:

```
In y line 2:
x=(1 2 3)
^-- SC2034: x appears unused. Verify it or export it.

In y line 4:
declare -a 'y=("${!'"$ref"'[@]}")'
           ^-- SC2016: Expressions don't expand in single quotes, use double quotes for that.
```

Not that it affects me, but I'm just pointing it out, seeing as it pertains to the intent of the OP's sample code.
