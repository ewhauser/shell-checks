However, the process of building ShellCheck via `cabal install` is very slow (despite using almost no CPU and very little RAM).  I am looking for a way to cut down on the edit-compile-run cycle for testing new changes.

I did discover that I can speed up the process a bit by instead running `cabal build -j12 -O0` which skips some packaging steps, however the build is still done sequentially (one file at a time, despite `-j12`) and takes quite a bit of time.

Can you offer any tips for speeding up the build?  `build` seems faster than `install`, which makes sense -- but I've got 12 cores and 32GB of RAM and ...

## `quickrun`

I've also learned that you can run Haskell programs in interpreter mode, skipping the compilation step.  I believe this is what the `quickrun` script is for.

I have been unable to get it to work (despite being able to build / link / run Shellcheck with `cabal` commands).  It seems that the script expects generated objects / compiled libraries to be at the path `./dist/build/autogen`, but they are not.

https://github.com/koalaman/shellcheck/blob/99f6554c9bd97d4815848b3aeba027ce196cbd41/quickrun#L1-L5

Being new to Haskell, Cabal, and GHC in general, I'm unable to diagnose the issues with this.  I would love to be able to use `quickrun` for quick edit-compile-run cycles, but the script does not work as-is.

#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2021-07-21 04:57](https://github.com/koalaman/shellcheck/issues/2286#issuecomment-883888396):

`quickrun` is definitely the way to go for faster iteration.

Cabal keeps changing, but on MacOS I was able to:

1. Go to a directory with no *.cabal files and install the dependencies manually with `cabal install --lib`:

    ( cd /tmp && cabal update && cabal install --lib aeson array bytestring containers deepseq Diff directory mtl filepath parsec QuickCheck regex-tdfa )

2. Generate the necessary files by starting a one-time build (it doesn't even need to finish, you can kill it after "[ 1 of 23] Compiling Paths_ShellCheck")

    cabal build

3. Symlink the "dist-newstyle" directory to the old-style "dist". The path is listed in the `cabal build` output. As of 2020-07-20 it's:

    ln -sf dist-newstyle/build/x86_64-osx/ghc-8.10.5/ShellCheck-0.7.2 dist

4. Now `./quickrun` works the same as `shellcheck`, but runs in ~10 seconds instead of the ~160 sec it takes to compile.

    echo 'echo $1' | ./quickrun -

5. You can similarly use `quicktest` to run the unit tests:

    ./quicktest


The `dist` symlink will go stale when GHC or ShellCheck versions change, but I don't think that's a problem since all you need it for is the `Paths_ShellCheck.hs` module for the sole purpose of providing a version number for `./quickrun --version`

Let me know how it goes!
