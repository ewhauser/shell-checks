# [\#425 Issue](https://github.com/koalaman/shellcheck/issues/425) `open`: Warn about "return" can only in a function or sourced script

#### <img src="https://avatars.githubusercontent.com/u/2398949?v=4" width="50">[rony36](https://github.com/rony36) opened issue at [2015-08-04T09:14:34Z](https://github.com/koalaman/shellcheck/issues/425):

"return" can only be used in a function or sourced script in Bash.
It will causes an execution error.
Here is a simple example.

``` bash
#!/bin/bash
echo "Hello!"
return 18
echo "Hello you!"
```

In bash, it will show "Hello!" and "Hello you!" and return error code 0 with message
**line 3: local: can only be used in a function**.

In zsh, it will show  only "Hello!" and return error code 18.

May shellcheck check for this?
