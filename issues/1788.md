#### <img src="https://avatars.githubusercontent.com/u/215702?v=4" width="50">[gromgit](https://github.com/gromgit) commented at [2020-01-06T10:52:47Z](https://github.com/koalaman/shellcheck/issues/1788#issuecomment-571094728):

> AFAICT this is valid (meaning it works under /bin/sh as expected)

Neither of those assertions are true, and shellcheck is correct in throwing all three errors. Did you actually try running your script?

First of all, the semicolon in your `then` body is syntactically invalid (it should be a colon instead), but even ignoring that, your `stderr` redirection is hiding the main problem in your `if` test:
```
$ cat test.sh
#!/bin/sh
if [ ! type hostname ] ; then
        :
fi

$ sh test.sh
test.sh: 2: [: type: unexpected operator
```
What you wrote is exactly equivalent to:
```
if test ! type hostname ; then
```
which makes no sense per the `test(1)` man page. The correct syntax is in fact:
```
#!/bin/sh
if ! type hostname 2>/dev/null ; then
        :
fi
```
