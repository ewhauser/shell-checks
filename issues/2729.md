# [\#2729 Issue](https://github.com/koalaman/shellcheck/issues/2729) `open`: Not detected return inside subshell

#### <img src="https://avatars.githubusercontent.com/u/15802528?u=19dcd5369f39f05c7d6bd8272da2f356a1456902&v=4" width="50">[mikhailnov](https://github.com/mikhailnov) opened issue at [2023-04-05 22:25](https://github.com/koalaman/shellcheck/issues/2729):

"return 0" in the following code is misused:

```bash
#!/bin/bash
# returns 0 if at least one user exists
_has_local_users(){
	if [ -z "$(ls -A /home)" ]; then
		return 1
	fi
	find /home -maxdepth 1 -type d | grep -v '^/home$' |
	while read -r line
	do
		local user
		user="$(LC_ALL=C stat --printf='%U' "$line")"
		# lost+found on ext4 belongs to root
		# UNKNOWN means that a user with such UID does not exist
		if [ "$user" != UNKNOWN ] && [ "$user" != root ]; then
			return 0
		fi
	done
	return 1
}
```

Its author (me) wanted to make a return from function, but made a return from subshell. that does not make sense.
shellcheck does not give any warnings.
