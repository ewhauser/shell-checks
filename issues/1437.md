#### <img src="https://avatars.githubusercontent.com/u/3129206?v=4" width="50">[matthewpersico](https://github.com/matthewpersico) commented at [2018-12-31T02:35:27Z](https://github.com/koalaman/shellcheck/issues/1437#issuecomment-450602500):

Look again at the comment - it's asking you to quote the `${dir}`, not the whole path. Try this on shellcheck.net:

```
#!/bin/bash

dir=$1
rm -rf ${dir}/results/*
rm -rf "${dir}"/results/*

$ shellcheck myscript
 
Line 4:
rm -rf ${dir}/results/*
       ^-- SC2086: Double quote to prevent globbing and word splitting.
```
Notice, no complaint about Line 5.

But this DOES bring up the point that the location markers can be confusing. @koalaman, @ngzhian : can you use some of the parsing from the upcoming self-correcting stuff to print the END marker for a statement, where applicable; something like:

```
Line 4:
rm -rf ${dir}/results/*
       ^----^ - SC2086: Double quote to prevent globbing and word splitting.
