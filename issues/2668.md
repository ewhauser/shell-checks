#### Here's a snippet or screenshot that shows the problem:
strict-mode.bash (on `$PATH`)
```sh
set -e
```
my script
```sh
#/usr/bin/env bash
source strict-mode.bash

cat foo
```
#### Here's what shellcheck currently says:
```
In /nix/store/fd4dxiajw5lyki2v5x6cya8079wmyd1d-bin/git-bare line 2:
source strict-mode.bash
       ^--------------^ SC1091 (info): Not following: strict-mode.bash: openBinaryFile: does not exist (No such file or directory)

In /nix/store/fd4dxiajw5lyki2v5x6cya8079wmyd1d-bin/git-bare line 27:
cd some/path
^----------^ SC2164 (warning): Use 'cd ... || exit' or 'cd ... || return' in case cd fails.
Did you mean: 
cd some/path || exit
```
#### Here's what I wanted or expected to see:
No warning, no info message.

If I inline `set -e` into my script, I get no warning about `cd`.

Bash finds “bare” sourced files on `$PATH`, so Shellcheck should check the `$PATH` in this case.

[SC1090](https://github.com/koalaman/shellcheck/wiki/SC1090)

---

The issue is that shellcheck only sources from absolute paths.
You also need `-x` to check sourced files.

Given a `my-script.bash` like the following
```sh
#!/usr/bin/env bash
source /usr/local/bin/strict-mode.bash

cd some/path
cat foo
```

`shellcheck -x my-script.bash` no longer complains

---

A workaround could be to, instead of sourcing it, just call strict-mode, tho I don't think that has the desired effect.

So I guess I can reproduce your issue, tho I'm not sure if something can or should be done about it
