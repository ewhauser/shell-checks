# [\#397 Issue](https://github.com/koalaman/shellcheck/issues/397) `closed`: SC2059 does not seem to protect against multiple variables

#### <img src="https://avatars.githubusercontent.com/u/126441?v=4" width="50">[travi](https://github.com/travi) opened issue at [2015-06-25T02:35:54Z](https://github.com/koalaman/shellcheck/issues/397):

If a `printf` statement references multiple variables, replacing one of them with `%s` and passing the variable as a second argument to `printf` satisfies this check. Instead, it should check that all variables are replaced with `%s` and passed as additional arguments to `printf`

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2015-06-25T18:31:24Z](https://github.com/koalaman/shellcheck/issues/397#issuecomment-115356345):

You're right, shellcheck doesn't warn about variables in the format string if the format string already contains format specifiers. 

This is intentional: if you use format specifiers you probably know how printf works, and any decision to mix formatters with variables is well informed. In practice most of these cases are legitimate, using the variables for ansi escapes or variable width formatting. 

ShellCheck's tradeoff was therefore to have a few false negatives over a lot of false positives.
