#### <img src="https://avatars.githubusercontent.com/u/39659394?v=4" width="50">[FriendlyNeighborhoodShane](https://github.com/FriendlyNeighborhoodShane) commented at [2024-03-22 06:50](https://github.com/koalaman/shellcheck/issues/2710#issuecomment-2014463305):

Agreed, it would be great to have _some_ amount of dynamism in the source directive.

For my use case, I have a script that chooses one of a bunch of config files at runtime, and uses variables and functions from it. According to recommendations, I created a static dummy config file and added that as a source directive to the main script, and it lints the flow of the main script great.
But it would be nice if I could run it on all of the various config files as well, by e.g. using:
```sh
# shellcheck source=${config_script}
```
with (the `--allow-source-vars` for obvious security considerations):
```sh
export config_script="./configs/conf-abc.sh";
shellcheck -ax --allow-source-vars=config_script ./main.sh;
```
or more usefully:
```sh
export config_script;
for config_script in ./configs/conf-*.sh; do
    shellcheck -ax --allow-source-vars=config_script ./main.sh;
done;
```
---

RIght now this indirection can be (poorly) emulated using a generated proxy file as a poor man's `export`:
```sh
# shellcheck source=./configs/test-proxy.sh
```
```sh
for config_script in ./configs/conf-*.sh; do
    printf '. "%s";\n' "${config_script}" > ./configs/test-proxy.sh;
    shellcheck -ax ./main.sh;
done;
rm -f ./configs/test-proxy.sh;
```
