#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2023-07-31 02:57](https://github.com/koalaman/shellcheck/issues/2809#issuecomment-1657424906):

ShellCheck is right in this case. You have enabled `check-extra-masked-returns` and this construct does indeed mask the return value of the command. `[[ $(ls /tmp /nosuchfile) ]]` evaluates to true even though the command exits with failure.

#### <img src="https://avatars.githubusercontent.com/u/11234239?v=4" width="50">[bruce-ricard](https://github.com/bruce-ricard) commented at [2023-08-25 18:54](https://github.com/koalaman/shellcheck/issues/2809#issuecomment-1693798833):

Sorry, you are absolutely right. I got confused by the code I was reading.

For potential future readers, do NOT use `[[` mechanically after every `if`, `[[` is only to be able to do boolean manipulation like equals, or greater than etc. 
If you want to test if a function call like `ls` is successful, just run

```
if ls; then ... else ... fi
```
