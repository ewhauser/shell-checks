#### <img src="https://avatars.githubusercontent.com/u/442919?v=4" width="50">[dualbus](https://github.com/dualbus) commented at [2014-08-14T12:39:37Z](https://github.com/koalaman/shellcheck/issues/212#issuecomment-52177752):

Uh, no and no.

Don't claim stuff without testing. See:

```
dualbus@h ~ % bash script
multiple
93.140
single
87.612

dualbus@h ~ % cat script 
#!/bin/bash

a=123
b=987
N=10000000
TIMEFORMAT=%R

echo multiple
time for ((i=0;i<N;i++)); do
    : "$a"..."$b"
done

echo single
time for ((i=0;i<N;i++)); do
    : "$a...$b"
done
```

Pretty clear which one is faster...

Also, don't claim stuff that's clearly subjective. "my version is
easier to analyze". That's evidently subjective. Depends on who is
reading your code. Personally, I find my version easier to read,
because instead of having to parse where the quotes start or end in
my mind, I just know it's a single string. And the quotes have
nothing to do with something being variable or constant.

So, using multiple quotes _IS_ redundant and less efficient.
