#### <img src="https://avatars.githubusercontent.com/u/2327324?v=4" width="50">[newgiin](https://github.com/newgiin) opened issue at [2015-11-17T00:29:36Z](https://github.com/koalaman/shellcheck/issues/549):

From http://tldp.org/LDP/Bash-Beginners-Guide/html/sect_03_02.html#sect_03_02_05, there is a note that says

> $\* vs. $@
> 
> The implementation of "$_" has always been a problem and realistically should have been replaced with the behavior of "$@". In almost every case where coders use "$_", they mean "$@". "$*" Can cause bugs and even security holes in your software.

but this [suggestion](https://github.com/koalaman/shellcheck/wiki/SC2124#rationale) says to use $\* instead of $@.

> If you want to assign N elements as 1 element by concatenating them, use \* instead of @

Which one should I follow?
