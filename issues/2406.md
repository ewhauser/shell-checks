# [\#2406 Issue](https://github.com/koalaman/shellcheck/issues/2406) `open`: find -printf is not defined in POSIX

#### <img src="https://avatars.githubusercontent.com/u/68940237?v=4" width="50">[slycordinator](https://github.com/slycordinator) opened issue at [2021-12-08 06:29](https://github.com/koalaman/shellcheck/issues/2406):

Shellcheck does not warn when one is using "find" with the GNU "-fprint" option.

#### For bugs
- My shellcheck version (`shellcheck --version` or "online"): 0.8.0

#### For new checks and feature suggestions
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related


#### Here's a snippet or screenshot that shows the problem:

```sh

#!/bin/sh
find . -type f -printf '%f\n'

```

#### Here's what shellcheck currently says:
[nothing]


#### Here's what I wanted or expected to see:
In POSIX sh, find -printf is undefined

When I run the command above in dash/bash, I get a listing of all files each on their own line as expected
When I run it in busybox, I get "find: unrecognized: -printf"

https://pubs.opengroup.org/onlinepubs/007904975/utilities/find.html

#### <img src="https://avatars.githubusercontent.com/u/68940237?v=4" width="50">[slycordinator](https://github.com/slycordinator) commented at [2021-12-16 04:24](https://github.com/koalaman/shellcheck/issues/2406#issuecomment-995427715):

I'm aware that find is an external command.

AFAIK, shellcheck checks for arguments to some external commands to warn if they are not posix compliant.

But I could be wrong about this. Either way, it would be a nice addition to warn that the argument given is not portable.

#### <img src="https://avatars.githubusercontent.com/u/68940237?v=4" width="50">[slycordinator](https://github.com/slycordinator) commented at [2021-12-16 19:52](https://github.com/koalaman/shellcheck/issues/2406#issuecomment-996147230):

I mean portable as in being easily moved from one system to another.

A script that uses "find -printf" will run perfectly fine on most Linux
distros, fail if you move it to one that uses posix utilities, and
similarly fail if you were to run it on many other Unixes.
