#### <img src="https://avatars.githubusercontent.com/u/1693206?v=4" width="50">[knirch](https://github.com/knirch) commented at [2014-03-17T15:26:03Z](https://github.com/koalaman/shellcheck/issues/118#issuecomment-37828473):

< and > means something completly different in [/[[/test in bash, STRING1 < STRING2 True if STRING1 sorts before STRING2 lexicographically.

Also you can't use test/arithmetic to test version numbers, you'll need something more advanced that split it into parts and a bunch of logic, or enforce the same number of characters etc, consider this example;

``` bash
[[ 3.32 > 2.9 ]]    # True
[[ 10.32 > 2.9 ]]   # False
expr 3.32 \> 2.9    # True
expr 10.32 \> 2.9   # False
```

(bash 4.2.45(1) / coreutils 8.21-1 on a debian system) Using [[ as SC2039 is right that < is not part of the posix standard.
