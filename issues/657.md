# [\#657 Issue](https://github.com/koalaman/shellcheck/issues/657) `closed`: Enhancement: support ignoring single line

#### <img src="https://avatars.githubusercontent.com/u/3399022?v=4" width="50">[mikeroll](https://github.com/mikeroll) opened issue at [2016-04-21T09:36:05Z](https://github.com/koalaman/shellcheck/issues/657):

It would awesome to be able to ignore just a single line regardless of the errors in it.

``` bash
# this yeilds an SC1083
this_is_a_{template} 

# this will disable SC1083's in the whole file, but I only want one line
# shellcheck disable=SC1083
this_is_a_{template} 

# this is the desired behaviour
this_is_a_{template}  # shellcheck ignore
```

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2016-04-21T17:55:28Z](https://github.com/koalaman/shellcheck/issues/657#issuecomment-213038218):

Easy mistake to make. 

Any directives that appear after the shebang and before any command will apply to the entire file. Therefore, if you try to ignore something on the first line you can easily end up ignoring it in the entire file.
