# [\#2505 Issue](https://github.com/koalaman/shellcheck/issues/2505) `closed`: New check for "grep | head -1" which should inform that "grep -m1" is better

#### <img src="https://avatars.githubusercontent.com/u/18568381?u=520aed8f7e665d9d1caee6358db291489a99ef03&v=4" width="50">[thomasmerz](https://github.com/thomasmerz) opened issue at [2022-05-24 10:38](https://github.com/koalaman/shellcheck/issues/2505):

#### For new checks and feature suggestions
- [x] https://www.shellcheck.net/ (i.e. the latest commit) currently gives no useful warnings about this
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related

#### Here's a snippet or screenshot that shows the problem:
```sh
#!/bin/bash
cmd="grep \"javaExe=\" $sJavaNew | head -1"
```

#### Here's what shellcheck currently says:
It says: nothing.
But it should tell users that there's a "-m NUM, --max-count=NUM" parameter which makes `grep` "Stop reading a file after NUM matching lines."

#### Here's what I wanted or expected to see:
Therefore the example above should be corrected to this:
```
cmd="grep -m1 \"javaExe=\" $sJavaNew"
```
