# [\#2330 Issue](https://github.com/koalaman/shellcheck/issues/2330) `closed`: Add git-describe (with sha1) as version text to Shellcheck

#### <img src="https://avatars.githubusercontent.com/u/133706?u=c19422ae9646a1a58b9c62a57a2143e62c5f8001&v=4" width="50">[stdedos](https://github.com/stdedos) opened issue at [2021-09-16 07:33](https://github.com/koalaman/shellcheck/issues/2330):

Especially with the https://github.com/koalaman/shellcheck/releases/tag/latest build, `shellcheck --version` returning a version string in the format

> v0.7.2-56-g4e703e5

would make it much easier to identify from where is the latest release built out of.

If, for some reason, you prefer the `Version: xxx` field to have the specific parsing (backwards compatible), add an extra line with that information.


#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2021-09-19 04:43](https://github.com/koalaman/shellcheck/issues/2330#issuecomment-922414493):

I've wanted this for a while, but didn't know how to make it happen since the version number is fetched via Cabal to avoid going out of sync. I worked around it by adding a CI kludge that swaps out the source code. `docker run -t koalaman/shellcheck:latest --version` now shows `v0.7.2-64-gdb4701d`. Thanks!
