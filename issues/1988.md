#### Here's a snippet or screenshot that shows the problem:

```sh

#!/bin/busybox sh
true

```

#### Here's what shellcheck currently says:
```
$ shellcheck myscript
 
Line 1:
#!/bin/busybox sh
^-- SC1008: This shebang was unrecognized. ShellCheck only supports sh/bash/dash/ksh. Add a 'shell' directive to specify.
```

#### Here's what I wanted or expected to see:
Nothing.

I noticed this in a real world script: https://github.com/SolidHal/PrawnOS/blob/master/resources/BuildResources/initramfs-init

the script itself is a posix sh script, invoked via busybox. I'm not sure how common it is to call busybox directly, but given that it's a shell script, perhaps shellcheck should accept that shebang.
