# [\#1191 Issue](https://github.com/koalaman/shellcheck/issues/1191) `open`: disable directive does not work properly on lines with multiple statements

#### <img src="https://avatars.githubusercontent.com/u/18865?v=4" width="50">[smoser](https://github.com/smoser) opened issue at [2018-04-27T14:16:00Z](https://github.com/koalaman/shellcheck/issues/1191):

Attempting a local directive to a line that has multiple statements does not have the desired affect. The combining of 3 statements here on a single line is admittedly not necessary, but is by design to keep it clear that IFS is set back and only changed on that line.

#### For bugs
- My shellcheck version: 0.4.6
- [X] I tried on shellcheck.net and verified that this is still a problem on the latest commit
#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/sh
out="$1"; delim="$2"; orig_ifs="$3"

# shellcheck disable=SC2086
IFS="$delim"; set -- $out; IFS="${orig_ifs}"

# spreading it to multiple lines makes it not complain.
IFS="$delim";
# shellcheck disable=SC2086
set -- $out; IFS="${orig_ifs}"
```

#### Here's what shellcheck currently says:
    Line 4:
    IFS="$delim"; set -- $out; IFS="${orig_ifs}"
                         ^-- SC2086: Double quote to prevent globbing and word splitting.

#### Here's what I wanted or expected to see:
I expect / hoped that the local disable would apply to the unquoted (by design) `set -- $out`.



#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2018-04-28T01:14:29Z](https://github.com/koalaman/shellcheck/issues/1191#issuecomment-385128607):

ShellCheck statements always apply to the immediate single command (simple or compound). This was supposed to make it easy to disable warnings for functions and blocks, but has instead turned out to be a constant source of confusion. It should probably be changed to a line based model at some point.

#### <img src="https://avatars.githubusercontent.com/u/18865?v=4" width="50">[smoser](https://github.com/smoser) commented at [2018-04-30T15:26:50Z](https://github.com/koalaman/shellcheck/issues/1191#issuecomment-385433685):

For what its worth, you *can* do this:

    # shellcheck disable=2086
    { IFS=","; set -- $val; IFS="$oifs"; }

Ie, shellcheck does the right thing with the added context {}
