# [\#3208 Issue](https://github.com/koalaman/shellcheck/issues/3208) `open`: `SC2004`: false positive on associative arrays used inside `(())`

#### <img src="https://avatars.githubusercontent.com/u/74512479?u=b358859564e7e40f14500bb78d1dcc974d166d2b&v=4" width="50">[Yrahcaz7](https://github.com/Yrahcaz7) opened issue at [2025-05-20 01:59](https://github.com/koalaman/shellcheck/issues/3208):

#### For bugs
- Rule Id: [SC2004](https://www.shellcheck.net/wiki/SC2004)
- My shellcheck version: online
- [x] I tried on shellcheck.net and verified that this is still a problem on the latest commit

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/usr/bin/env bash

char="-"
value=10

readonly -A CHAR_VALUES=(["+"]=1 ["-"]=2 ["|"]=3)
(( value += CHAR_VALUES[$char] ))
```

#### Here's what shellcheck currently says:
```
[Line 7:](javascript:setPosition(7, 25))
(( value += CHAR_VALUES[$char] ))
                        ^-- [SC2004](https://www.shellcheck.net/wiki/SC2004) (style): $/${} is unnecessary on arithmetic variables.
```
#### Here's what I wanted or expected to see:

Nothing, as removing the `$` changes the meaning of the expression.

Note: If `$char` is changed to `"$char"`, shellcheck no longer complains.
