#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2017-04-14T18:06:12Z](https://github.com/koalaman/shellcheck/issues/876#issuecomment-294204417):

You're right, this is a definitely a fixable false positive.

The minimal case would be e.g. `PAGER=cat mail`.

SC2037 checks for prefix assignments that only contain a common command, thinking that the user wanted to run `cat mail` and assign it, instead of running `mail` with `PAGER=cat` in the environment.

The real annoyance is not the fact that `PAGER` is commonly used in this way, but that `PAGER="cat" mail` still triggers, even though the user's intention is clear. That should be fixed.
