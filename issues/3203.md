# [\#3203 Issue](https://github.com/koalaman/shellcheck/issues/3203) `open`: SC2086: Herestrings don't need quoting

#### <img src="https://avatars.githubusercontent.com/u/1002436?v=4" width="50">[ntrrgc](https://github.com/ntrrgc) opened issue at [2025-05-06 11:40](https://github.com/koalaman/shellcheck/issues/3203):

Here is an example:

```bash
#!/bin/bash
set -eu
foo="12   34
  56"
echo "$foo"
cat <<< $foo

mkdir -p /tmp/glob-test
cd /tmp/glob-test
touch abcdef
foo="a*"
echo "$foo"
cat <<< $foo
```

Running the above script produces the following output:

```
12   34
  56
12   34
  56
a*
a*
```

From that output you can see that neither word splitting or globbing have ocurred. However, shellcheck still warns of both issues:

```
In a.sh line 6:
cat <<< $foo
        ^--^ SC2086 (info): Double quote to prevent globbing and word splitting.

Did you mean: 
cat <<< "$foo"


In a.sh line 13:
cat <<< $foo
        ^--^ SC2086 (info): Double quote to prevent globbing and word splitting.

Did you mean: 
cat <<< "$foo"
```
