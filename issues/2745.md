# [\#2745 Issue](https://github.com/koalaman/shellcheck/issues/2745) `open`: Detect wrong uses of `exec` for programs with arguments 

#### <img src="https://avatars.githubusercontent.com/u/15112080?u=9258327e2b4aace64e41b3af8a8de57331a84bba&v=4" width="50">[jjant](https://github.com/jjant) opened issue at [2023-04-26 10:02](https://github.com/koalaman/shellcheck/issues/2745):

#### For new checks and feature suggestions
- [x] https://www.shellcheck.net/ (i.e. the latest commit) currently gives no useful warnings about this
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related


#### Here's a snippet or screenshot that shows the problem:

```sh
#!/usr/bin/env

exec "echo with arguments"
```

#### Here's what shellcheck currently says:
Nothing


#### Here's what I wanted or expected to see:

`exec` will interpret `"echo with arguments"` as the command name, not `echo` with arguments `with` and `arguments`. This will result in an error like:

```console
bash: line 0: exec: echo with args: not found
```


You probably want:

```sh
#!/usr/bin/env

exec "echo" with arguments
```
