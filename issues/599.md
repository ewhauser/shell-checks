#### <img src="https://avatars.githubusercontent.com/u/197253?v=4" width="50">[kurahaupo](https://github.com/kurahaupo) commented at [2016-02-13T15:27:52Z](https://github.com/koalaman/shellcheck/issues/599#issuecomment-183686294):

tl;dr: there is no easy way out.

It is not correct to suppose that a bash command is terminated by EOL.

Bash commands finish when they are complete, however many lines that takes.
Any sort of compound command can span multiple lines, including function
definitions, if/then/else/fi blocks, for-loops, while-loops, braces,
brackets, and pipelines.

And quoted "words" can span multiple lines too.
