# [\#2484 Issue](https://github.com/koalaman/shellcheck/issues/2484) `closed`: detect ignored exitcode from subshell parameter substitution

#### <img src="https://avatars.githubusercontent.com/u/12697118?v=4" width="50">[emdej](https://github.com/emdej) opened issue at [2022-04-10 14:13](https://github.com/koalaman/shellcheck/issues/2484):

#### Here's a snippet or screenshot that shows the problem:

```sh

#!/bin/bash
set -Eeuo pipefail

printf "%s" "$(false)"
echo "went fine"

cat <<EOF
$(false)
EOF
echo "went fine"


```

#### Here's what shellcheck currently says:
```sh
$ shellcheck myscript
No issues detected!
```


#### Here's what I wanted or expected to see:
```sh
Exit code from subshell ignored.
```


#### Recommended alternative can be to use a temporary variable for that, like:
```sh
X="$(command)"
printf "%s" "$X"
```
or even better
```sh
local X
X="$(command)"
printf "%s" "$X"
```
instead of
```sh
printf "%s" "$(command)"
```
