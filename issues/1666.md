#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2022-07-22T23:41:13Z](https://github.com/koalaman/shellcheck/issues/1666#issuecomment-1193007045):

SC2004 now triggers on array indices as well. This was an intentional omission going back to 2014 when ShellCheck struggled with associative arrays. Thanks!

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2022-07-24T21:56:43Z](https://github.com/koalaman/shellcheck/issues/1666#issuecomment-1193401293):

These are redundant parentheses, not a redundant arithmetic context. However, you're definitely onto something: I found a number of instances of `array[$((x))]`, `array[(x)]`, and systematic use of `((..))` as parentheses inside arithmetic contexts presumably due to `$((..))`. These now have stylistic suggestions. Thanks!

Ideally it would also apply to `${array[..]}` but these are currently not parsed robustly enough to allow it.
