#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2017-09-09T22:48:27Z](https://github.com/koalaman/shellcheck/issues/981#issuecomment-328308009):

Wow, that's pretty weird.

I'll hopefully fix this at some point, but until then, note that shellcheck comments have to come before the line/structure they apply to, and that you can add additional comments after a second `#`:

```
#!/bin/sh
# shellcheck ignore=SC2034 # for reasons
foo=1 
```

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2017-11-04T22:59:14Z](https://github.com/koalaman/shellcheck/issues/981#issuecomment-341935994):

This was an interesting case. Since annotations are not parsed as comments, it ended up being a literal string instead, similar to  `foo=1 '#' shellcheck` . It appeared to suppress the warning because this is the syntax that passes a variable to a single command.

You should now get a warning and improved behavior in most of these cases. Nice catch.
