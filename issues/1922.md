#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash

function testit {
  if typeset -F foobar
  then
    foobar "$@"
  else
    echo "$@"
  fi
}

```

#### Here's what shellcheck currently says:
```
Line 4	SC2034: foobar appears unused. Verify use (or export if used externally).
```
#### Here's what I wanted or expected to see:

Since `typeset` and `declare` are really the same command/builtin, shellcheck should not treat them differently. In this case I think the behavior that shellcheck exhibits for `declare` is correct.
