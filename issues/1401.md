#### Here's a snippet or screenshot that shows the problem:

```sh

#!/bin/bash
sudo tee foo < bar  # SC2024
cat bar | sudo tee foo  # SC2002

```

#### Here's what shellcheck currently says:
For SC2024 it says "sudo doesn't affect redirects. Use sudo cat file | ..", but when you change it to use "cat file | ..." SC2002 comes up saying "Useless cat. Consider 'cmd < file | ..' or 'cmd file | ..' instead."


#### Here's what I wanted or expected to see:
I think the problem is that SC2024 assumes you're always reading from a file that requires sudo to read. I think the `sudo tee ... < ...` pattern is the correct pattern (as opposed to `cat ... | sudo tee`), I just think the logic for detecting SC2024 should be tweaked a bit. Maybe only report it for `>`, `>>` and `|` when the right-hand side of the pipe doesn't have sudo, and don't report it for `<`.

I know these are just warnings, and I know there's no way to 100% detect when sudo will be needed on both ends of the pipe/redirect, just thought I'd point out that when you follow the recommendation from SC2024 you are given a different warning about the useless cat (SC2002).
