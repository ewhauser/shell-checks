# [\#529 Issue](https://github.com/koalaman/shellcheck/issues/529) `closed`: detect 'bad' function names.

#### <img src="https://avatars.githubusercontent.com/u/198379?v=4" width="50">[bittorf](https://github.com/bittorf) opened issue at [2015-10-29T10:04:50Z](https://github.com/koalaman/shellcheck/issues/529):

according to http://pubs.opengroup.org/onlinepubs/000095399/basedefs/xbd_chap08.html
a valid function name is like:

```
"consist solely of uppercase letters, digits, and the
'_' (underscore) from the characters defined in Portable
Character Set and do not begin with a digit"
```

the following does not throw an error whith shellsheck, but it should detect it:

``` shell
#!/bin/sh
x-y()
{
  :
}

x-y
```

the same for variables names, but i have not tested it yet.
even not all shells complain about this with 'sh -n $file'.
