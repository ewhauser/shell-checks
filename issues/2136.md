# [\#2136 Issue](https://github.com/koalaman/shellcheck/issues/2136) `open`: SC2005: echo "$(cmd)" Is not the same as cmd

#### For bugs
- Rule Id (if any, e.g. SC1000): SC2005
- My shellcheck: version 0.7.1
- [x] The rule's wiki page does not already cover this (e.g. https://shellcheck.net/wiki/SC2086)
- [x] I tried on https://www.shellcheck.net/ and verified that this is still a problem on the latest commit

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/sh

echo "test start"
echo "$(echo 'test' | base64 -w0)"
echo "test end"

echo "test start"
echo 'test' | base64 -w0
echo "test end"
```

As you can see the result is different:
```
test start                                                                                                                                                                                                                                                 
dGVzdAo=                                                                                                                                                                                                                                                   
test end                                                                                                                                                                                                                                                   
test start                                                                                                                                                                                                                                                 
dGVzdAo=test end
```

As you can see, the first version the echo breaks the line,
and the suggested option doesn't. 

#### Here's what shellcheck currently says:
```
In test.sh line 4:
echo "$(echo 'test' | base64 -w0)"
     ^---------------------------^ SC2005: Useless echo? Instead of 'echo $(cmd)', just use 'cmd'.

For more information:
  https://www.shellcheck.net/wiki/SC2005 -- Useless echo? Instead of 'echo $(...
```

#### Here's what I wanted or expected to see:
Should not exist



#### <img src="https://avatars.githubusercontent.com/u/243351?v=4" width="50">[brother](https://github.com/brother) commented at [2021-02-03 23:16](https://github.com/koalaman/shellcheck/issues/2136#issuecomment-772893061):

In most cases this is a valid help to make code easier to write.
When you see it and think your code is the most appropriate for the case use a directive to ignore the problem for the future.
Or if at all possible, rewrite the code block to eliminate the notice.

For the example above I would have considered a heredoc or a formatted print.

```shell
cat <<EOF
test start
$(echo 'test' | base64 -w0)
test end
EOF

printf "test start\n%s\ntest end\n" $(echo 'test' | base64 -w0)

```
