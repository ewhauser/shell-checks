#### For bugs
- Rule Id (if any, e.g. SC1000): SC2034
- My shellcheck version (`shellcheck --version` or "online"): 0.7.1
- [x ] The rule's wiki page does not already cover this (e.g. https://shellcheck.net/wiki/SC2086)

#### Here's a snippet or screenshot that shows the problem:

```bash
#!/bin/bash
declare -A butter
butter['bubbles']='true'
val='bubbles'
if [[ -v "butter[$val]" ]]; then
	echo "Found $val"
fi
```

#### Here's what shellcheck currently says:

Line 3:
butter['bubbles']='true'
^-- SC2034: butter appears unused. Verify use (or export if used externally).

#### Here's what I wanted or expected to see:

nothing, since the script uses `-v` to check if the associative array `butter` contains the key contained in the variable `$val`

I created a more complex example for three reasons
1. I was getting SC2144 with a simple test.
2. I wanted to see the difference between v, z, and n.
3. Have an example for refence.

```
#!/bin/bash

# FROM https://www.gnu.org/software/bash/manual/html_node/Bash-Conditional-Expressions.html
# -v varname
# True if the shell variable varname is set (has been assigned a value).

# -R varname
# True if the shell variable varname is set and is a name reference.

# -z string
# True if the length of string is zero.

# -n string
# string
# True if the length of string is non-zero.

echo Empty:
x=""  # Same with x=
[[ -z $x ]] && echo z
[[ -n $x ]] && echo n
[[ -v  x ]] && echo v
unset x

echo Unset
[[ -z $x ]] && echo z

[[ -v  x ]] && echo v


printf "%-5s|%-10s|%-3s|%-3s%-3s\n" "index" "value" 'v' 'z' 'n'
declare -a array
array[0]=""  # Same with array[0]=
array[1]=
array[2]=5
array[4]='bar'
declare -a pV
pV[0]='""'
pV[1]='=$'
pV[2]=5
pV[3]='not set'
pV[4]='bar'
for (( i=0; i<5; i++ )); do
  declare -A results
  if [[ -v  array[${i}] ]]; then
    results['v']="1"
  else
    results['v']="0"
  fi
  if [[ -z  "${array[${i}]}" ]]; then
    results['z']="1"
  else
    results['z']="0"
  fi
  if [[ -n  "${array[${i}]}" ]]; then
    results['n']="1"
  else
    results['n']="0"
  fi
  printf "%-5i|%-10s|%-3i|%-2i|%-2i\n" "${i}" "${pV[${i}]}" "${results['v']}" "${results['z']}" \
    "${results['n']}"
done
```
