# [\#1818 Issue](https://github.com/koalaman/shellcheck/issues/1818) `closed`: Specifying external-sources directive in .shellcheckrc file

#### <img src="https://avatars.githubusercontent.com/u/54970?u=11dff30be59beb94b2a13707809a3d892d816fdb&v=4" width="50">[msanders](https://github.com/msanders) opened issue at [2020-02-04 19:11](https://github.com/koalaman/shellcheck/issues/1818):

shellcheck version: 0.7.0

#### For new checks and feature suggestions
- [x] shellcheck.net (i.e. the latest commit) currently gives no useful warnings about this
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related

It would be nice to able to add a directive analogous to the `--external-sources`/`-x` flag in a `.shellcheckrc` file. Currently it's only possible to specify uniformly via the `$SHELLCHECK_OPTS` variable, which works for local use but isn't ideal for shared projects.

This would cut down on manual `# shellcheck source=` comments necessary in scripts and improve things like editor integration, since the `shellcheck` command could just run as-is with an included rc file.

#### Here's a snippet or screenshot that shows the problem:

```
$ cat script/test
#!/bin/sh
. "./sourced.sh"
$ cat .shellcheckrc
source-path=SCRIPTDIR
external-sources=
```

#### Here's what shellcheck currently says:

```
$ shellcheck script/test

In script/test line 2:
. "./sourced.sh"
  ^------------^ SC1091: Not following: ./sourced.sh was not specified as input (see shellcheck -x).

For more information:
  https://www.shellcheck.net/wiki/SC1091 -- Not following: ./sourced.sh was n...
$ shellcheck --external-sources script/test # (returns successfully)
```

#### Here's what I wanted or expected to see:

```
$ shellcheck script/test # (should return successfully)
```
