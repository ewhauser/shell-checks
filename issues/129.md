# [\#129 Issue](https://github.com/koalaman/shellcheck/issues/129) `closed`: SC2120 doesn't take 'set -- ...' into consideration.

#### <img src="https://avatars.githubusercontent.com/u/302055?v=4" width="50">[deryni](https://github.com/deryni) opened issue at [2014-04-04T01:00:22Z](https://github.com/koalaman/shellcheck/issues/129):

``` bash
f() {
    echo "foo"

    set -- blah bar

    echo "$@"
}

f
```

Running shellcheck on the above script results in:

```
In repro.sh line 1:
f() {
^-- SC2120: f references arguments, but none are ever passed.
```

But set is setting the positional arguments.
