#### Here's a snippet or screenshot that shows the problem:

```bash
#!/bin/bash

OVERRIDE="FAILED"
RES_OVERRIDE=$(curl -m 2 -s ${URL}/override)
if [[ $? -eq 0 ]]; then
    OVERRIDE=$(echo "$RES_OVERRIDE" | jq -cr '.[0].override' 2>&1)
fi
```

#### Here's what shellcheck currently says:
`SC2181: Check exit code directly with e.g. 'if mycmd;', not indirectly with $?.`

#### Here's what I wanted or expected to see:

I need to check first whether the `curl` request succeeded and then work on the result.

I tried replacing it with

```bash
OVERRIDE="FAILED"
if RES_OVERRIDE=$(curl -m 2 -s ${URL}/override); then
    OVERRIDE=$(echo "$RES_OVERRIDE" | jq -cr '.[0].override' 2>&1)
fi
```

but accessing invalid URLs will still enter the `if / else` then.
