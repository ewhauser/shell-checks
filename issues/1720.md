# [\#1720 Issue](https://github.com/koalaman/shellcheck/issues/1720) `closed`: alpine image does not behave like "regular"

#### <img src="https://avatars.githubusercontent.com/u/15611634?v=4" width="50">[BeyondEvil](https://github.com/BeyondEvil) opened issue at [2019-10-15T17:21:13Z](https://github.com/koalaman/shellcheck/issues/1720):

With the regular image I simply do:

`docker run --rm -v "$PWD:/mnt:ro" koalaman/shellcheck:stable *.sh`

But with the alpine version I have to be more explicit:

`docker run --rm -v "$PWD:/mnt:ro" -w /mnt --entrypoint /bin/shellcheck koalaman/shellcheck-alpine:stable *.sh`

Is that by design?

If so, why? (just curious).

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2019-11-03T19:25:20Z](https://github.com/koalaman/shellcheck/issues/1720#issuecomment-549169833):

Yes, this is by design.

The `shellcheck-alpine` image is exactly like `alpine` but with ShellCheck pre-installed. The entrypoint is set to `sh` and requires you to specify a full shell command (`shellcheck` or otherwise) just like any other `alpine` based image.

Meanwhile, the `shellcheck` image does not contain a shell or any other userspace utilities. The entrypoint is therefore just ShellCheck itself.

They used to work the same, but the `shellcheck-alpine` image was changed to behave more like the `alpine` image in commit dc1e7c1bd4f.
