# [\#585 Issue](https://github.com/koalaman/shellcheck/issues/585) `closed`: fall positive when using null command `:` for default values

#### <img src="https://avatars.githubusercontent.com/u/176950?v=4" width="50">[vapier](https://github.com/vapier) opened issue at [2016-01-26T01:16:41Z](https://github.com/koalaman/shellcheck/issues/585):

using the null command to initialize a default value for variables triggers quoting warnings when it shouldn't.  the various expansions that happen in this case are largely irrelevant.

for example:

```
#!/bin/sh
: ${FOO:=value}
```

shellcheck shows:

```
In ./test.sh line 2:
: ${FOO:=value}
  ^-- SC2086: Double quote to prevent globbing and word splitting.
```
