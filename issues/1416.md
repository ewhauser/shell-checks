# [\#1416 Issue](https://github.com/koalaman/shellcheck/issues/1416) `closed`: Encourage use of `[[ -n string ]]` rather than `[[ string ]]`

#### <img src="https://avatars.githubusercontent.com/u/1752606?v=4" width="50">[lollipopman](https://github.com/lollipopman) opened issue at [2018-12-14T17:57:08Z](https://github.com/koalaman/shellcheck/issues/1416):

#### For new checks and feature suggestions
- [x] shellcheck.net (i.e. the latest commit) currently gives no useful warnings about this
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related

#### Here's a snippet or screenshot that shows the problem:
```sh
#!/bin/bash

if [[ $(true) ]]; then
  echo true
else
  echo false
fi 
```
#### Here's what shellcheck currently says:

No issues detected!

#### Here's what I wanted or expected to see:

```
SCXXXX: If you did intend to check that the output of the command is
not an empty string, use the longer form of the `-n` conditional expression
to clarify the intent `[[ -n $butter ]]`. However, if you intend to check the 
exit code of the command then remove the conditional expression:

  if true; then
    echo true
  else
    echo false
  fi
```
The rationale for this feature request is that one arg version of -n, i.e.
`[[ $butter ]]` is not well known and some users confuse it with how you test
the exit code of a command. Encourage people to use `-n` which is easier
to search for in documentation, and makes the code easier to grok.
