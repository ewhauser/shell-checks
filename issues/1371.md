#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2018-12-02T23:25:26Z](https://github.com/koalaman/shellcheck/issues/1371#issuecomment-443552375):

You could have used e.g.  `pgrep "^git-(gui|citool)$" | grep -q "^$PPID$"` , but there's nothing especially wrong about what you're doing. Maybe you'd want to use `comm=` instead of `args=` to only match the executable name, but I don't know how your process is called.

It's always annoying to get false positives for perfectly good code, but I don't see a good way of recognizing these kinds of cases in general. It would be very sad if ShellCheck made you write *worse* code, so please don't hesitate to ignore issues that you've evaluated and found to be fine as is.
