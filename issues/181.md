# [\#181 Issue](https://github.com/koalaman/shellcheck/issues/181) `closed`: support for associative arrays

#### <img src="https://avatars.githubusercontent.com/u/62794?v=4" width="50">[comotion](https://github.com/comotion) opened issue at [2014-06-18T13:18:48Z](https://github.com/koalaman/shellcheck/issues/181):

shellcheck needs support for associative arrays

``` bash
declare -A fruit
fruit=(
 [apple]="sweet" \
 [orange]="sour" \
 [grapefruit]="bitter" \
 [tomato]="umame" \
 [ss]="snake" \
)
```

The last declaration gives:
SC2102: Ranges can only match single chars (mentioned due to duplicates)
due to the `[ss]="snake"`

Furthermore:

``` bash
zool=$(echo bool)
$fruit[$zool]="ninja"
```

Gives SC2086: Double quote to prevent globbing and word splitting
