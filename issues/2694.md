#### Here's a snippet or screenshot that shows the problem:

```sh

#!/bin/bash
# set counter 'c' to 1 and condition 
# c is less than or equal to 5
i=5
for (( c=1; c<=$i; c++ ))
do 
   echo "Welcome $c times"
done

```

#### Here's what shellcheck currently says:
```conf
[Line 5:](javascript:setPosition(5, 16))
for (( c=1; c<=$i; c++ ))
               ^-- [SC2004](https://www.shellcheck.net/wiki/SC2004) (style): $/${} is unnecessary on arithmetic variables.
```

#### Here's what I wanted or expected to see:
Nothing, as it's an acceptable way to perform a FOR loop



#### <img src="https://avatars.githubusercontent.com/u/213372?u=772d0d51cdde83cf6508e32165898f462a5392ff&v=4" width="50">[thedward](https://github.com/thedward) commented at [2023-02-14 03:54](https://github.com/koalaman/shellcheck/issues/2694#issuecomment-1429074204):

The provided link for [SC2004](https://www.shellcheck.net/wiki/SC2004) explains the rationale for the rule. If you'd prefer not to get that warning, you can disable that rule with a shellcheck [directive ](https://www.shellcheck.net/wiki/Directive):

```
#!/bin/bash
# set counter 'c' to 1 and condition 
# c is less than or equal to 5
i=5
# shellcheck disable=SC2004
for (( c=1; c<=$i; c++ ))
do 
   echo "Welcome $c times"
done
```
