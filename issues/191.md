# [\#191 Issue](https://github.com/koalaman/shellcheck/issues/191) `closed`:  backtick'd comment is not parsed correctly

#### <img src="https://avatars.githubusercontent.com/u/7633626?v=4" width="50">[aw-was-here](https://github.com/aw-was-here) opened issue at [2014-07-10T17:33:55Z](https://github.com/koalaman/shellcheck/issues/191):

One of the tricks in commenting multiple lines (at least in bash) is to use backticks:

command \
option `#comment` \
option

ie,
ls \
-d `#directory` \
~

is completely valid bash.

shellcheck throws two fatal errors when confronted with this construction: SC1072 and SC1073.
