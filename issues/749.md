# [\#749 Issue](https://github.com/koalaman/shellcheck/issues/749) `closed`: shellcheck fails to parse explicit subshell with heredoc

#### <img src="https://avatars.githubusercontent.com/u/1410954?v=4" width="50">[allanlw](https://github.com/allanlw) opened issue at [2016-10-18T17:01:40Z](https://github.com/koalaman/shellcheck/issues/749):

The following is valid in bash:

``` bash
( cat <<FOO )
blah
FOO
```

shellcheck fails to parse this:

```

In test_heredoc.sh line 1:
( cat <<FOO )
^-- SC1073: Couldn't parse this explicit subshell.


In test_heredoc.sh line 4:

^-- SC1072: Unexpected eof. Fix any mentioned problems and try again.
```

#### <img src="https://avatars.githubusercontent.com/u/243351?v=4" width="50">[brother](https://github.com/brother) commented at [2016-10-19T12:30:18Z](https://github.com/koalaman/shellcheck/issues/749#issuecomment-254798064):

The example executes fine in shellcheck.net.

And locally at version 0.4.4 built from git some days ago.

```
brother ~$ shellcheck -s sh /tmp/test_heredoc.sh
brother ~$ shellcheck -s dash /tmp/test_heredoc.sh
brother ~$ shellcheck -s ksh /tmp/test_heredoc.sh
brother ~$ shellcheck -s bash /tmp/test_heredoc.sh
brother ~$ cat /tmp/test_heredoc.sh
( cat <<FOO )
blah
FOO
```
