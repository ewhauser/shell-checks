# [\#2359 Issue](https://github.com/koalaman/shellcheck/issues/2359) `closed`: Add `cat <<EOF` exception to `check-extra-masked-returns`

#### <img src="https://avatars.githubusercontent.com/u/29582865?u=0c57eebb21e1922b4a0345f7a830924cc295998b&v=4" width="50">[felipecrs](https://github.com/felipecrs) opened issue at [2021-10-19 16:59](https://github.com/koalaman/shellcheck/issues/2359):

As suggested by @DoxasticFox at [this comment](https://github.com/koalaman/shellcheck/issues/2338#issuecomment-946881546), this issue is to discuss if we should add this known exception or not.

I believe it's reasonable: `cat` with a heredoc will never fail, or at least I can't think of when it would fail.

#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2021-10-31 05:23](https://github.com/koalaman/shellcheck/issues/2359#issuecomment-955639467):

Any command can fail with the right (or wrong) ulimit, though it's not unreasonable to ignore that possibility. The more likely failure in this case is if the shell implements heredocs via files and the tmpdir is out of space: 

    bash: line 1: cannot create temp file for here-document: No space left on device
