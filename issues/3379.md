# [\#3379 Issue](https://github.com/koalaman/shellcheck/issues/3379) `open`: False positive for SC2034; triggers when variable only used in "unset" command

#### <img src="https://avatars.githubusercontent.com/u/68940237?v=4" width="50">[slycordinator](https://github.com/slycordinator) opened issue at [2025-12-24 03:59](https://github.com/koalaman/shellcheck/issues/3379):

#### For bugs with existing features

- Rule Id (if any, e.g. ): SC2034
- My shellcheck version (`shellcheck --version` or "online"): online
- [X] The rule's wiki page does not already cover this (e.g. https://shellcheck.net/wiki/SC2086)
- [X] I tried on https://www.shellcheck.net/ and verified that this is still a problem on the latest commit

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash

array=( foo bar baz )
del_index=1
unset 'array[del_index]'
for index in "${!array[@]}"; do
    echo "$index: ${array[$index]}"
done
```

#### Here's what shellcheck currently says:
Line 4:
del_index=1
^-- SC2034 (warning): del_index appears unused. Verify use (or export if used externally).

#### Here's what I wanted or expected to see:
(no output)

Note:
The erroneous warning is shown when I use either of:
`unset 'array[del_index]'`
`unset 'array[$del_index]'`

#### <img src="https://avatars.githubusercontent.com/u/6006766?u=709c2e58b6ed1cf1fde74c48f8877dc5207ff216&v=4" width="50">[nexushoratio](https://github.com/nexushoratio) commented at [2025-12-29 07:17](https://github.com/koalaman/shellcheck/issues/3379#issuecomment-3695669184):

As a work around, the following appears to both make shellcheck happy and function correctly:

```
unset "array[${del_index}]"                                                     |
```

That is, use double quotes and braces.
