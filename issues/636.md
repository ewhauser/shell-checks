# [\#636 Issue](https://github.com/koalaman/shellcheck/issues/636) `closed`: Feature Request: Produce Suggested Fixes

#### <img src="https://avatars.githubusercontent.com/u/17551?v=4" width="50">[eatnumber1](https://github.com/eatnumber1) opened issue at [2016-03-29T17:19:11Z](https://github.com/koalaman/shellcheck/issues/636):

It would be extremely useful if ShellCheck produced suggested fixes for the findings it produces.

One possible way this could be accomplished is by making checks edit the AST (or generate a new one) that they are traversing after they've decided that there's an issue, then new code could be generated. It would then generate JSON looking like the following:

``` json
{"file": "foo.sh",
 "line": 42,
 "startColumn": 42,
 "endColumn": 128,
 "code": 2086,
 "suggestedFix": "\"$foo\""}
```

What kinds of changes would need to happen within ShellCheck to make this happen? I'm happy to contribute code, but my Haskell skills may not be up to this challenge without it being broken down into smaller pieces.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2016-04-09T17:19:16Z](https://github.com/koalaman/shellcheck/issues/636#issuecomment-207819141):

This would primarily require start and end positions for all nodes. At that point one could output character based fixes.

For other than small fixes, it would be helpful to have the ability to serialize chunks of the AST into well formatted bash code. 

You would also need a list of fix deltas at possibly different positions than the main error, and a strategy for merging multiple fixes around the same positions.

Which errors would this be most helpful for?
