Is there a reason that `-v` or `-h` are not supported?

```console

  -S SEVERITY         --severity=SEVERITY        Minimum severity of errors to consider (error, warning, info, style)
  -V                  --version                  Print version information
  -W NUM              --wiki-link-count=NUM      The number of wiki links to show, when applicable
```

I don't see an obvious flag collision here.

Also maybe show short version info on `shellcheck -h`? Like:

```console
$ shellcheck -h
shellcheck v0.7.0
Usage: shellcheck [OPTIONS...] FILES...

[...]
```

#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2020-03-17 05:19](https://github.com/koalaman/shellcheck/issues/1855#issuecomment-599882971):

`-v` was originally saved for a "verbose" mode, later implemented using "optional" checks. `-h` is not the same as `--help` simply because that didn't seem like a useful thing to have a shortcut for. It seemed better to be a little conservative with single letters, so that if a better fitting or more canonical use came along, it wouldn't be taken by something less useful (see `rsync -h`).

As for adding the version to help text, most of the tools on my GNU system don't do that, the help text makes it obvious where to find it, and it seems like noise if what you're looking for is help with the flags.
