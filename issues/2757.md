- Rule Id: `SC2030`
- My shellcheck version: `0.9.0`
- [x] The rule's wiki page does not already cover this at [Wiki](https://shellcheck.net/wiki/SC2030)
- [x] I tried on https://www.shellcheck.net/ and verified that this is still a problem on the latest commit

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/sh
counter=0
found=false
while ! $found; do
  echo "$counter"
  (
    counter=$((counter + 1))
    found=true
  )
done
echo "$counter"
```

#### Here's what shellcheck currently says:

```sh
Line 7:
    counter=$((counter + 1))
    ^-- SC2030 (info): Modification of counter is local (to subshell caused by (..) group).
 
Line 11:
echo "$counter"
      ^-- SC2031 (info): counter was modified in a subshell. That change might be lost.
      
```

#### Here's what I wanted or expected to see:

```sh
Line 4:
while ! $found; do
        ^-- SC2031 (info): found was modified in a subshell. That change might be lost.

Line 5:
  echo "$counter"
        ^-- SC2031 (info): counter was modified in a subshell. That change might be lost.

Line 7:
    counter=$((counter + 1))
    ^-- SC2030 (info): Modification of counter is local (to subshell caused by (..) group).
 
Line 8:
    found=true
    ^-- SC2030 (info): Modification of found is local (to subshell caused by (..) group).
 
Line 11:
echo "$counter"
      ^-- SC2031 (info): counter was modified in a subshell. That change might be lost.
```
