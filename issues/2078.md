- Rule Id SC2005
- My shellcheck version "online" and 0.7.0
- [until my edit] The rule's wiki page does not already cover this (e.g. https://shellcheck.net/wiki/SC2086)
- [X] I tried on https://www.shellcheck.net/ and verified that this is still a problem on the latest commit


#### Here's a snippet or screenshot that shows the problem:

```sh

#!/bin/bash
set -e
echo "$(make)"

```

#### Here's what shellcheck currently says:
```
Line 3:
echo "$(make)"
     ^-- SC2005: Useless echo? Instead of 'echo $(cmd)', just use 'cmd'.
```

#### Here's what I wanted or expected to see:

```
Line 3:
echo "$(make)"
     ^-- SC2005: Useless echo? Instead of 'echo $(cmd)', just use 'cmd'. If using set -e or relying on exit code, use 'cmd || true'.
```

The command `echo "$(false)` will return true, and the script will keep going even when `set -e` or followed by `&&`, so it's not a suggestion that you can blindly apply.
