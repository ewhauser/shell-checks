# [\#2658 Issue](https://github.com/koalaman/shellcheck/issues/2658) `closed`: Enhancing source-path directive

#### <img src="https://avatars.githubusercontent.com/u/14263613?v=4" width="50">[tomaszdrozdz](https://github.com/tomaszdrozdz) opened issue at [2023-01-09 09:31](https://github.com/koalaman/shellcheck/issues/2658):

I have project structure:

```
my_project\
|- .shellcheckrc
|- lib\
    |- l.sh
|-bin\
    |- A\
        |- a.sh
            --------------------
            #!/bin/bash
            . "${PROJECT_DIR}/lib/l.sh"
            some code
            --------------------
    |- b.sh
            --------------------
            #!/bin/bash
            . "${PROJECT_DIR}/lib/l.sh"
            some code
            --------------------
```

.shellcheckrc file is at the "root" of my_project.  

PROJECT_DIR variable is "auto generated" to point to my_project dir.  


I can put multiple source-path directive to .shellcheckrc:
```
source-path=SCRIPTDIR/../../
source-path=SCRIPTDIR/../
```

But it would be nice if I could in .shellcheckrc just put something like:
```
source-path=SHELLCHECKRC
```
so shellcheck interprets it as relative to .shellcheckrc file.
