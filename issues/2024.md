#### Here's a snippet or screenshot that shows the problem:
The issue is the following:
- a file contains shell snippets (e.g. a CI definition file)
- a tool pulls these out and creates a combined file that can have `shellcheck` run over it
- the `shellcheck` results are helpful but difficult to relate back to the original CI file.

#### Here's what shellcheck currently says:
No useful support

#### Here's what I wanted or expected to see:
Some preprocessors etc permit a directive that "sets" the filename and line number that is then reported afterwards e.g.
```
#shellcheck line original.yml 24
```

An example of an existing function can be found here [https://gcc.gnu.org/onlinedocs/cpp/Line-Control.html](https://gcc.gnu.org/onlinedocs/cpp/Line-Control.html)

I'm not expecting a perfect match but being able to periodically resync the filename and line number would make writing such preproc/shellcheck tools produce somewhat more useful output.
