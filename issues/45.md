# [\#45 Issue](https://github.com/koalaman/shellcheck/issues/45) `closed`: "for arg do" is reported as syntax error despite being valid

#### <img src="https://avatars.githubusercontent.com/u/660663?v=4" width="50">[cdown](https://github.com/cdown) opened issue at [2013-11-29T18:19:42Z](https://github.com/koalaman/shellcheck/issues/45):

The shorthand for `for arg in "$@"; do`, `for arg do` (note: without the requirement of a semicolon), is reported as a violation of  SC107{3,2}, but it is explicitly permitted by POSIX, and in fact is _more_ portable than `for arg; do` (with semicolon).

A script that triggers this:

```
for arg do
    echo "$arg"
done
```
