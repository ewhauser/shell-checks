# [\#2306 Issue](https://github.com/koalaman/shellcheck/issues/2306) `open`: detect failure due to processing asterisks due to not accounting for empty globs

#### <img src="https://avatars.githubusercontent.com/u/328602?u=2cdce234c9430b60e57789040be403a50057fa67&v=4" width="50">[pabs3](https://github.com/pabs3) opened issue at [2021-08-24 03:18](https://github.com/koalaman/shellcheck/issues/2306):

In empty directories, globs evaluate to an asterisk `*`, which can cause failure if not handled in the script. This is also mentioned in ShellCheck [SC2045](https://www.shellcheck.net/wiki/SC2045) about iterating over ls output but doesn't appear to be checked for by ShellCheck yet.

#### For new checks and feature suggestions
- [x] I was unable to check https://www.shellcheck.net/ because it requires JavaScript
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/sh
for f in *; do
    cp -- "$f" /target
done
```

#### Here's what shellcheck currently says:

Nothing

#### Here's what I wanted or expected to see:

A warning about failures due to not handling the empty glob case.

#### Here are some correct ways to do it:

```sh
#!/bin/sh
for f in *; do
    [ "$f" != '*' ] || [ -e "$f" ] || break
    cp -- "$f" /target
done
```

```sh
#!/bin/bash
for f in *; do
    [[ "$f" = '*' && -e "$f" ]] || break
    cp -- "$f" /target
done
```

```sh
#!/bin/bash
shopt -s nullglob
for f in *; do
    cp -- "$f" /target
done
```
