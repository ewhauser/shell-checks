# [\#978 Issue](https://github.com/koalaman/shellcheck/issues/978) `closed`: Bad syntax recommendation

#### <img src="https://avatars.githubusercontent.com/u/5316?v=4" width="50">[mcandre](https://github.com/mcandre) opened issue at [2017-08-20T19:58:45Z](https://github.com/koalaman/shellcheck/issues/978):

When ShellCheck scans the content `:() { : | : &; }`, ShellCheck mistakenly reports that the code should be simplified to `:() { : | : & }` without the semicolon. However, this is actually invalid for single line function declarations.

#### <img src="https://avatars.githubusercontent.com/u/197253?v=4" width="50">[kurahaupo](https://github.com/kurahaupo) commented at [2017-08-23T06:42:19Z](https://github.com/koalaman/shellcheck/issues/978#issuecomment-324236562):

Hi Andrew

*How* are you sending it inside the Docker container?

As Koalaman says, &; is forbidden by most shells, so this is very strange.

You won't get a forkbomb unless and until you call the function; simply
defining it is safe. Or change the name of the function so that it's not
recursive, like

    ::() { : | : & }
    ::() { : | : &; }

Commands can be terminated by newlines, semicolons, and ampersands.
Additional newlines are permitted in some places, but other command
terminators are not permitted on empty commands.

So I suspect there's a fault in your testing methodology somewhere.

-Martin


#### <img src="https://avatars.githubusercontent.com/u/5316?v=4" width="50">[mcandre](https://github.com/mcandre) commented at [2017-08-23T16:28:22Z](https://github.com/koalaman/shellcheck/issues/978#issuecomment-324389875):

A semicolon is required for POSIX-style shell scripts that express a code block, such as a function declaration, as a single line. See https://askubuntu.com/questions/521937/write-function-in-one-line-into-bashrc

#### <img src="https://avatars.githubusercontent.com/u/3999221?v=4" width="50">[pawamoy](https://github.com/pawamoy) commented at [2017-08-24T11:47:11Z](https://github.com/koalaman/shellcheck/issues/978#issuecomment-324612607):

From bash manual:

> A list is a sequence of one or more pipelines separated by one of the  operators  ;,  &,
       &&, or ||, and optionally terminated by one of ;, &, or <newline>.

> If  a command is terminated by the control operator &, the shell executes the command in
       the background in a subshell.  The shell does not wait for the command  to  finish,  and
       the  return status is 0.  Commands separated by a ; are executed sequentially; the shell
       waits for each command to terminate in turn.  The return status is the  exit  status  of
       the last command executed.

These paragraphs implicitly say that a command is either terminated by `&` or `;`, not both.
Also, when trying in `sh`:

```
$ f() { : | : &; }                                             
sh: 1: Syntax error: ";" unexpected
```
