#### Here's a snippet or screenshot that shows the problem:

Let say we have the following directory:
```
.
└── scripts
    ├── test.sh
    └── utils.sh

$ cat scripts/test.sh 
#!/usr/bin/env bash

CURDIR="$( cd "$( dirname "${BASH_SOURCE[0]}" )" &> /dev/null && pwd )"
source "$CURDIR/utils.sh"

echo "$test"

$ cat scripts/utils.sh 
#!/usr/bin/env bash
test="Hello"
```

#### Here's what shellcheck currently says:

```
$ shellcheck -x scripts/test.sh 

In scripts/test.sh line 4:
source "$CURDIR/utils.sh"
       ^----------------^ SC1091: Not following: ./utils.sh: openBinaryFile: does not exist (No such file or directory)


In scripts/test.sh line 6:
echo "$test"
      ^---^ SC2154: test is referenced but not assigned (for output from commands, use "$(test ...)" ).

For more information:
  https://www.shellcheck.net/wiki/SC2154 -- test is referenced but not assign...
  https://www.shellcheck.net/wiki/SC1091 -- Not following: ./utils.sh: openBi...
```
And if I move to `scripts` dir:

```
$ cd scripts
$ shellcheck -x test.sh
$ echo $?
0
```

#### Here's what I wanted or expected to see:

When shellcheck tries to follow the sourced file, it should be done from the current file directory, not from `cwd`, at least not only.

Update: `--source-path=SCRIPTDIR` actually makes it do this... but I think specifically when you're explicitly getting the script dir like in your snippet above it should use the script dir automatically?

#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2021-11-15 01:12](https://github.com/koalaman/shellcheck/issues/2253#issuecomment-968418704):

You can indeed use that, or `source-path=SCRIPTDIR` in a `.shellcheckrc` in the project dir, to get the desired behavior. ShellCheck looks relative to cwd because that's what Bash does for non-$PATH sourcing, but in this case it's confusing because of the additional behavior of stripping one leading dynamic element before looking at the path.
