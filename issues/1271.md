```sh
#!/bin/bash

set -e -u

changed_files="$(find /etc/letsencrypt -newermt "$(date -d "yesterday 00:00" '+%Y%m%d %H:%M:%S')" | grep -v "^/etc/letsencrypt$"| wc -l)"

if [[ "${changed_files}" -gt 0 ]]; then

```

#### <img src="https://avatars.githubusercontent.com/u/3999221?v=4" width="50">[pawamoy](https://github.com/pawamoy) commented at [2018-06-27T16:03:30Z](https://github.com/koalaman/shellcheck/issues/1271#issuecomment-400731379):

And you can achieve that with `"$(... | grep -vc "^/etc/letsencrypt$")"`. So the `wc -l` pipe is indeed not needed if you use grep's `-c` option.

Edit: the warning is not telling you to replace your whole `grep` command by `grep -c`, but rather to consider **adding** the `-c` option to it.
