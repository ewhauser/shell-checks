# [\#1677 Issue](https://github.com/koalaman/shellcheck/issues/1677) `open`: syntax: {varname[idx]}< redirection unrecognized

#### <img src="https://avatars.githubusercontent.com/u/141079?v=4" width="50">[dcsobral](https://github.com/dcsobral) opened issue at [2019-08-21T22:24:31Z](https://github.com/koalaman/shellcheck/issues/1677):

The `{varname[idx]}<` / `{varname[idx]>` syntax for redirections is not recognized by shellcheck, causing it to display error SC1083. According to bash hackers' [bashchanges](https://wiki.bash-hackers.org/scripting/bashchanges), this was introduced on 4.3-alpha, and it has a notation to indicate it came from ksh93.

#### For bugs
- SC1083
- 0.7.0
- [X] The rule's wiki page does not already cover this (e.g. https://shellcheck.net/wiki/SC2086)
- [X] I tried on shellcheck.net and verified that this is still a problem on the latest commit

#### Here's a snippet or screenshot that shows the problem:

```sh

#!/usr/bin/env bash

declare -a ARR
I=0
J=1
: {FD}<"${BASH_SOURCE[0]}"
: {ARR[$I]}<"${BASH_SOURCE[0]}"   # Not recognized
: {ARR["$J"]}<"${BASH_SOURCE[0]}" # Not recognized
: {ARR[2]}<"${BASH_SOURCE[0]}"    # Not recognized

grep "[d]eclare" <&"${FD}"
grep "[d]eclare" <&"${ARR[$I]}"
grep "[d]eclare" <&"${ARR[$J]}"
grep "[d]eclare" <&"${ARR[2]}"

```
