# [\#712 Issue](https://github.com/koalaman/shellcheck/issues/712) `closed`: Sub-array expansion not recongnized

#### <img src="https://avatars.githubusercontent.com/u/19393173?v=4" width="50">[zhatt](https://github.com/zhatt) opened issue at [2016-07-09T04:48:27Z](https://github.com/koalaman/shellcheck/issues/712):

The subarray expansion below returns one element not the entire array.

declare -a myarray=( a b c)
status=${myarray[@]:(-1)}  
echo "$status"

$ shellcheck myscript

Line 3:
status=${myarray[@]:(-1)}  
       ^-- SC2124: Assigning an array to a string! Assign as array, or use \* instead of @ to concatenate.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2016-09-25T01:32:53Z](https://github.com/koalaman/shellcheck/issues/712#issuecomment-249397216):

`"${myarray[@]:(-1)}"` actually returns a list of zero or one elements, depending on whether `myarray` is empty.

If you want it to be either a single element or an empty string, you can use `"${myarray[*]:(-1)}"`
