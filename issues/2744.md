#### Here's what shellcheck currently says:
https://github.com/koalaman/shellcheck/blob/master/README.md
RC FILES

Unless --norc is used, ShellCheck will look for a file .shellcheckrc or
shellcheckrc in the script's directory and each parent directory. If found,
it will read key=value pairs from it and treat them as file-wide directives.


#### Here's what I wanted or expected to see:
1. make a default path for shellcheckrc such as
```
/usr/share/shellcheck/:/etc/shellcheck/:$HOME/.config/shellcheck/., 
```

so the same shellcheck-setting can be applied e.g. for different $USERs and Projects
I have my source files in /usr/local/bin for all scripts and multiple users, some run as root from cron jobs,
other run as root for maintenance or for users at -X11- autostart

2. allow several source files to be included 
```
# shellcheck source=/usr/local/bin/file1
# shellcheck source=/usr/local/bin/file2
```

and so on
3. have  
```
shellcheck -x 
```
as default setting
