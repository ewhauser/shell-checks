# [\#254 Issue](https://github.com/koalaman/shellcheck/issues/254) `closed`: separate tests from src modules

#### <img src="https://avatars.githubusercontent.com/u/82731?v=4" width="50">[juhp](https://github.com/juhp) opened issue at [2014-11-06T10:02:28Z](https://github.com/koalaman/shellcheck/issues/254):

ShellCheck.Simple, ShellCheck.Analytics, and ShellCheck.Parser currently export QuickCheck tests and so require functioning TemplateHaskell to build.

But ghc only really support TemplateHaskell well on x86\* archs,
so currently it is difficult to build ShellCheck on other archs like arm and ppc.

Could you please separate out the QuickCheck tests from ShellCheck/ into test/
so that it is possible to build ShellCheck itself (without the tests) on architectures
without TemplateHaskell?

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2017-04-02T02:17:59Z](https://github.com/koalaman/shellcheck/issues/254#issuecomment-290960086):

I'm definitely willing to be convinced, but it currently seems like a pain due to the lack of tooling.

Currently, adding a test is a single line of code next to the check.

With a separate file, it seems like you would have to 1.  export the function being tested in the module, 2. open a different file, 3. search for the appropriate place, and 4. add the test. Given that a typical test is something like `verify someFunction "foo --bar=baz"`, that's a fairly large overhead.

Any input on how to improve on this is appreciated.
