# [\#710 Issue](https://github.com/koalaman/shellcheck/issues/710) `closed`: shellcheck incorrectly detects tr character class as range when it starts with dash

#### <img src="https://avatars.githubusercontent.com/u/1566303?v=4" width="50">[kojiromike](https://github.com/kojiromike) opened issue at [2016-07-05T15:38:26Z](https://github.com/koalaman/shellcheck/issues/710):

```
In bin/setup line 10:
    project="$(echo "$project" | tr -d '[-_]')"
                                       ^-- SC2021: Don't use [] around ranges in tr, it replaces literal square brackets.
```

This code is intended to remove dashes and underscores from `$project`. The indicated expression is not a range. It is just a character class with two characters in it. Shellcheck should not flag when a tr character class begins with a dash.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2016-07-05T15:58:49Z](https://github.com/koalaman/shellcheck/issues/710#issuecomment-230521633):

The expression is in fact a range: it's the range from `[` to `_`, and the additional character `]`. This means that it'll remove underscores, backslashes and right square brackets, but not dashes.

The message has been updated to say "classes" rather than "ranges" in 36263fb though, because this message is equally valid for `[abc]` as it is for `[a-c]`. 

To remove dashes and underscores, use `tr -d '_-'`

Common bugs in shell commands due to misused flags are definitely in scope, like `find . -name '*.avi' -o -name '*.ogv' -print` or `time -o file myprogram`.

It's not feasible to compile an up-to-date list of all valid flags for all versions of all commands though, so it's on an individual command+flag level for frequently misused flags. This is the only instance I've been able to find of trying to `tr` a group starting with `-`.

@RobSis  This is a true positive. Your command removes square brackets, which I can't imagine was intentional:

```
$ echo "a string with spaces and [square brackets]" | tr -d '[[:space:]]'

```

This is how you remove just spaces:

```
$ echo "a string with spaces and [square brackets]" | tr -d '[:space:]'
astringwithspacesand[squarebrackets]
```
