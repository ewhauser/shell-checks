#### <img src="https://avatars.githubusercontent.com/u/198379?v=4" width="50">[bittorf](https://github.com/bittorf) opened issue at [2015-10-01T10:16:34Z](https://github.com/koalaman/shellcheck/issues/486):

would'nt it be better to support the special printf "\$data" case directly?
locally we have to workaround this with an eval-hack:

```
int2charNEW()
{
  local integer="$1"
  octal="$( printf '%03o' "$integer" )"
  eval printf "\\\\$octal"
}

int2charOLD()
{
  local integer="$1"
  printf "\\$(printf '%03o' $integer )"
}
```

I realize this is a bit old but I've been meaning to come back to this for a bit.

For bash at least this version works without any shellcheck warning.

```
int2charB() {
    local integer=$1 octal
    octal=$(printf '%03o' "$integer")
    printf %b "\\$octal"
}
```
