# [\#2032 Issue](https://github.com/koalaman/shellcheck/issues/2032) `closed`: v.0.3.7 - shellcheck not recognizing lastpipe

#### <img src="https://avatars.githubusercontent.com/u/5190243?v=4" width="50">[DiagonalArg](https://github.com/DiagonalArg) opened issue at [2020-08-06 06:06](https://github.com/koalaman/shellcheck/issues/2032):

Code:

```
shopt -s lastpipe
echo "foo" | read FooOut
```

`shellcheck` output:

```
echo "foo" | read FooOut
                  ^-- SC2030: Modification of FooOut is local (to subshell caused by pipeline).
```

`man bash`:

>               lastpipe
>                       If  set,  and job control is not active, the shell runs the last command of a pipe-
>                       line not executed in the background in the current shell environment.

Shellcheck should be checking if lastpipe is on or not.
