#### <img src="https://avatars.githubusercontent.com/u/364471?u=4f9ff9c2c474c35fbb6bd87fd310e508c1b568e6&v=4" width="50">[Flupp](https://github.com/Flupp) commented at [2025-07-25 13:32](https://github.com/koalaman/shellcheck/issues/2401#issuecomment-3117840232):

I noticed a similar problem within command substitution within `[[ … ]]` when no quoting for `$(…)` is needed, however, within it, quoting is still necessary.

Consider the following example:
```bash
x='a  b'
[[ $(echo "$x") == 'a  b' ]]; echo $?
```
Note the two spaces between `a` and `b`. This example outputs 0 as expected. However, the following outputs 1 (as expected but probably usually not intended).
```bash
x='a  b'
[[ $(echo $x) == 'a  b' ]]; echo $?
```
I’d expect a warning about the missing quotes around `$x`.
