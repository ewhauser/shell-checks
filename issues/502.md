#### <img src="https://avatars.githubusercontent.com/u/198379?v=4" width="50">[bittorf](https://github.com/bittorf) opened issue at [2015-10-12T14:45:21Z](https://github.com/koalaman/shellcheck/issues/502):

testcase

``` shell
#!/bin/sh

x()
{
  local x=0
  return $x
}

x
```

according to dash https://wiki.ubuntu.com/DashAsBinSh#local and the wikientry SC2039 it is allowed but behaves differently than in bash. so we should only warn if e.g.

``` shell
local a=1 b=2
```

but the testcase should me fine, should'nt it?

#### <img src="https://avatars.githubusercontent.com/u/6459309?v=4" width="50">[Artoria2e5](https://github.com/Artoria2e5) commented at [2015-10-12T17:09:24Z](https://github.com/koalaman/shellcheck/issues/502#issuecomment-147463308):

`local` is NOT in POSIX. However it's supported by most POSIX-compatible shells, making it a _de facto_ behavior.

In this case, it sounds more like a dash bug -- if dash wants to follow the the principle of least astonishment.
