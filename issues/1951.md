# [\#1951 Issue](https://github.com/koalaman/shellcheck/issues/1951) `open`: SC2155 has inconsistent behavior

#### <img src="https://avatars.githubusercontent.com/u/1200046?v=4" width="50">[kirawrath](https://github.com/kirawrath) opened issue at [2020-05-14 11:41](https://github.com/koalaman/shellcheck/issues/1951):

#### For bugs
version: 0.4.6
#### Assumption
I am assuming that `declare -r` and `local -r` are equivalent when inside a function. You may close this ticket if this is false.

#### Here's a snippet or screenshot that shows the problem:

```sh
➜  ~ echo 'f(){ declare -r a=$(ls); }' | shellcheck -e SC2148,SC2034 -

In - line 1:
f(){ declare -r a=$(ls); }
                ^-- SC2155: Declare and assign separately to avoid masking return values.

➜  ~ echo 'f(){ local -r a=$(ls); }' | shellcheck -e SC2148,SC2034 -
➜  ~ 
```

#### Here's what shellcheck currently says:
shellcheck brings up error SC2155 with `declare -r`, but not with `local -r`.


#### Here's what I wanted or expected to see:
I expect that either both `declare -r` and `local -r` return error SC2155, or none.
