#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash

declare a,b

if (( $# ==2 )); then
        a=$1
        b=$2
else
        a=constant
        b=$1
fi

somefunc $a
somefunc $b

```

#### Here's what shellcheck currently says:
```
Line 14	SC2086: Double quote to prevent globbing and word splitting.
```
#### Here's what I wanted or expected to see:

```
Line 13	SC2086: Double quote to prevent globbing and word splitting.
Line 14	SC2086: Double quote to prevent globbing and word splitting.
```

It appears that if the latest assignment is a constant, the check doesn't "remember" that there was a previous non-constant one. Contrast this with the inverse:

```sh
#!/bin/bash

declare a,b

if (( $# ==2 )); then
        a=constant
        b=$2
else
        a=$2
        b=$1
fi

somefunc $a
somefunc $b

```

#### Here's what shellcheck currently says:
```
Line 13	SC2086: Double quote to prevent globbing and word splitting.
Line 14	SC2086: Double quote to prevent globbing and word splitting.
```
