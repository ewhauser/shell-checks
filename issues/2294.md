# [\#2294 Issue](https://github.com/koalaman/shellcheck/issues/2294) `closed`: Check replacing characters in valiable

#### <img src="https://avatars.githubusercontent.com/u/15100604?u=f8f5854f11feb098158beb14c52ca943968ce331&v=4" width="50">[massongit](https://github.com/massongit) opened issue at [2021-08-03 12:01](https://github.com/koalaman/shellcheck/issues/2294):

#### For bugs
```sh
#!/bin/bash

c="aaaabbbb"
echo "${c##b/a}" # Correct: echo "${c//b/a}"
```

If we want to replace `b` to `a` in valiable `${c}`, we write `${c//b/a}`.
If we mistake this for `${c##b/a}`, `shellcheck` is not pointed out.

#### For new checks and feature suggestions

#### Here's what shellcheck currently says:
Check at https://www.shellcheck.net/ :

```shell
$ shellcheck myscript
No issues detected!

$
```

#### Here's what I wanted or expected to see:
I want `shellcheck` to say `${c##b/a} is not corect.` or `${c##b/a} is not correct. You may replace ${c//b/a}.`.

#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2021-08-03 20:37](https://github.com/koalaman/shellcheck/issues/2294#issuecomment-892149089):

I see what you mean, though how would ShellCheck know that you didn't want `${c##b/a}`? This is valid and useful code when you want to strip a directory prefix for example:

```
c="b/a/file.txt"
echo "${c##b/a}" # /file.txt
echo "${c//b/a}" # a/a/file.txt
```

Since it seems difficult to distinguish from the case where `${c##b/a}` is used intentionally, so I will close this Issue.
