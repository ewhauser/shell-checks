# [\#3372 Issue](https://github.com/koalaman/shellcheck/issues/3372) `closed`: Request: Provide non-XZ binary archives (e.g. .tar.gz) for Linux releases to support CI environments without functional xz

Hello and thank you for maintaining ShellCheck.

## Summary
In our CI infrastructure, we are unable to extract the official *.tar.xz binary archives for Linux due to xz sandbox failures. This prevents us from using precompiled ShellCheck releases, even though the binary itself works correctly.

## Problem details
On our runners (Linux, recent kernels with seccomp/Landlock restrictions), calling:
tar -xf shellcheck-v0.10.0.linux.x86_64.tar.xz
results in:

xz: Failed to enable the sandbox
tar: Child returned status 1
tar: Error is not recoverable: exiting now
This occurs even when setting XZ_DISABLE_SANDBOX=1 or similar variables.
The issue affects only the extraction step; ShellCheck itself works once extracted.

Since the official Linux binaries are distributed only as *.tar.xz, and there is no ZIP or .tar.gz equivalent for Linux, this blocks our ability to use official releases in CI.

## Why this matters
Many CI environments run on hardened or containerized kernels where the xz sandbox cannot be initialized. Repacking the archive internally is possible but adds maintenance overhead and makes us diverge from upstream release artifacts.

## Request
Would it be possible to publish, in addition to the existing .tar.xz files, at least one alternative non-XZ format for Linux binaries, such as:
- shellcheck-<version>.linux.x86_64.tar.gz, or
- a ZIP archive containing the Linux binary

This would significantly improve compatibility with restricted CI environments.

## Additional notes
This request is specifically for prebuilt Linux binaries.
We are aware that .tar.xz is the standard format today and that source code ZIPs already exist, but they do not contain the Linux binary.

Thank you for considering this enhancement
