# [\#1392 Issue](https://github.com/koalaman/shellcheck/issues/1392) `closed`: En dash in front of long options in man page

#### <img src="https://avatars.githubusercontent.com/u/168301?v=4" width="50">[l0b0](https://github.com/l0b0) opened issue at [2018-11-17T22:18:05Z](https://github.com/koalaman/shellcheck/issues/1392):

#### For bugs
- My shellcheck version: 0.5.0

#### Here's a snippet that shows the problem:

```sh
$ man shellcheck | grep '–'
       -a, –check-sourced
       -C[WHEN], –color[=WHEN]
              For TTY output, enable colors always, never or auto.  The default is auto.  –color without an argument
              is equivalent to –color=always.
       -e CODE1[,CODE2...], –exclude=CODE1[,CODE2...]
       -f FORMAT, –format=FORMAT
       -s shell, –shell=shell
       -V, –version
       -x, –external-sources
```

#### Here's what shellcheck currently says:

The long options shown above are prefixed with `–` ([en dash](https://www.fileformat.info/info/unicode/char/2013/index.htm)). If I copy one of those options verbatim it's treated as a file:

```sh
$ shellcheck –check-sourced *.sh 
–check-sourced: –check-sourced: openBinaryFile: does not exist (No such file or directory)
```

#### Here's what I wanted or expected to see:

All the long options should be prefixed with `--` (two hyphens).
