# [\#1263 Issue](https://github.com/koalaman/shellcheck/issues/1263) `closed`: SC2142 occurs when parameters are used in a function

#### <img src="https://avatars.githubusercontent.com/u/13854417?v=4" width="50">[HaleTom](https://github.com/HaleTom) opened issue at [2018-06-19T05:29:48Z](https://github.com/koalaman/shellcheck/issues/1263):

#### For bugs
- SC2142
- My shellcheck version: online
- [x] I tried on shellcheck.net and verified that this is still a problem on the latest commit

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash
alias f='f(){ echo You said: "$@"; }; f'
```

#### Here's what shellcheck currently says:
```
$ shellcheck myscript
 
Line 2:
alias f='f(){ echo You said: "$@"; }; f'
      ^-- SC2142: Aliases can't use positional parameters. Use a function.

$ 
```

#### Here's what I wanted or expected to see:

I'd expect to see nothing in this specific case. I am already using a function.

Aliases *can* use positional parameters with a function, see [this answer](https://stackoverflow.com/a/42466441/5353461).

I wrote that and would be happy if you borrowed from it to update the explanation of this check.
