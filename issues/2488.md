```sh

#!/bin/bash

echo    -e      'test'
echo    -e      "test"

```

#### Here's what I wanted or expected to see:

I do not believe the tab '\t' before '-e' is an issue for sure.  The tab after '-e' could be debatable.  Since '-e' is not a potential operand to be printed, as echo uses it internally as a switch, I see no issue.  No  other command I have found yet objects to tabs between operands for readability.  Spaces needing protecting fall between operands to be concatenated for printing.  If one wants a leading space to be protected then it would need to be inside some form of quote and it will be rather obvious when testing, if needed.  In the above statement there is only a single printable and quoted non white space operand.

Single or double quotes change nothing.
