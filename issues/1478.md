# [\#1478 Issue](https://github.com/koalaman/shellcheck/issues/1478) `closed`: SC1008: Add `#!/bin/false` for Sourced scripts

#### <img src="https://avatars.githubusercontent.com/u/42092626?v=4" width="50">[lamyergeier](https://github.com/lamyergeier) opened issue at [2019-01-27T22:05:37Z](https://github.com/koalaman/shellcheck/issues/1478):

From [Shebang (Unix) - Wikipedia](https://en.wikipedia.org/wiki/Shebang_(Unix)#Examples):

> `#!/bin/false` – Do nothing, but return a non-zero [exit status](https://en.wikipedia.org/wiki/Exit_status "Exit status"), indicating failure. Used to prevent stand-alone execution of a script file intended for execution in a specific context, such as by the '.' command from sh/bash, 'source' from csh/tcsh, or as a .profile, .cshrc, or .login file.

Please add `#!/bin/false`.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2019-01-27T23:05:56Z](https://github.com/koalaman/shellcheck/issues/1478#issuecomment-457962914):

This works when you specify the shell in some other way, but this warning was not very helpful, and on top of that it lacked a wiki page! I've updated the message in 2ea229315 to suggest using a directive, and I created [a wiki page](https://github.com/koalaman/shellcheck/wiki/SC1008) demonstrating how.

tl;dr: Just specify the shell you expect will source the file, and the warning will be omitted:

```
#!/bin/false
# shellcheck shell=bash
echo "Hello World"
```

Thanks for pointing this out!

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2019-01-29T04:06:33Z](https://github.com/koalaman/shellcheck/issues/1478#issuecomment-458400107):

Oh, you're right. The directive not working as the second line was issue #974 fixed for v0.6.0. 

Unfortunately there's no great workaround for previous versions, and it can't be silenced with a directive or `-s` because that was exactly the bug being fixed. You could consider upgrading, or maybe use `#!/bin/bash -n` which would be equivalent to `#!/bin/true`
