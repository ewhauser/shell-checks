# [\#851 Issue](https://github.com/koalaman/shellcheck/issues/851) `closed`: Shellcheck annotation parsing should be forward compatible [SC1073]

#### <img src="https://avatars.githubusercontent.com/u/4692022?v=4" width="50">[eldering](https://github.com/eldering) opened issue at [2017-03-19T15:05:29Z](https://github.com/koalaman/shellcheck/issues/851):

In a shell script file I have the lines
```
# shellcheck source=../lib/lib.error.sh disable=SC1091
. "$DJ_LIBDIR/lib.error.sh"
```
With shellcheck version 0.4.4 this is parsed correctly, but with an older version 0.3.4, I get the error
```
../judge/compile.sh:47:1: error: Couldn't parse this shellcheck annotation. [SC1073]
```
I'm assuming this is because the older version doesn't support the `source` directive yet.

It would be nice if unknown directives like this do not generate an error, or at least this could be suppressed to support forward compatibility.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2017-03-22T03:19:00Z](https://github.com/koalaman/shellcheck/issues/851#issuecomment-288287007):

You're right, it should never have worked this way.

You'll be happy to know that in 0.4.5+, this is no longer a parser error. It'll just emit an ignorable [SC1107](https://github.com/koalaman/shellcheck/wiki/SC1107).
