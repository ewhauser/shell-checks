# [\#2197 Issue](https://github.com/koalaman/shellcheck/issues/2197) `open`: Proposal: ability to disable global .shellcheckrc

#### <img src="https://avatars.githubusercontent.com/u/14852634?u=1317981711caafc930b9c828867764dd57d594e8&v=4" width="50">[Eric-Arellano](https://github.com/Eric-Arellano) opened issue at [2021-04-02 16:53](https://github.com/koalaman/shellcheck/issues/2197):

Hello! I'm a maintainer over at https://github.com/pantsbuild/pants and we're adding Shellcheck to Pants so that people can easily install your tool with a pinned version and run it in parallel with other linters like isort/flake8/shfmt. You have an awesome tool, thank you for making this!

A major goal of build tools like Pants and Bazel is to avoid "Works On My Machine"-type errors by making builds reproducible. We're hoping to support Shellcheck config files, but the global `~/.shellcheckrc` breaks reproducibility so we instead are planning to set `--norc` every time.

Would you be interested in a way to disable global config files, but still have a  repository-specific file? I would be happy to implement this, I'd love to hack on Haskell 🤓 

Possibly relates to https://github.com/koalaman/shellcheck/issues/2006. I don't think I like this design, but one possible way to do this is `--norc --rc=.shellcheckrc`.
