#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2019-07-04T23:26:07Z](https://github.com/koalaman/shellcheck/issues/300#issuecomment-508589764):

These have since been fixed:
```
Line 1:
for f in $1/*; do echo "$f"; done
^-- SC2148: Tips depend on target shell and yours is unknown. Add a shebang.
         ^-- SC2231: Quote expansions in this for loop glob to prevent wordsplitting, e.g. "$dir"/*.txt .
 
Line 2:
foo=($1/*)
^-- SC2034: foo appears unused. Verify use (or export if used externally).
     ^-- SC2206: Quote to prevent word splitting, or split robustly with mapfile or read -a.
```
