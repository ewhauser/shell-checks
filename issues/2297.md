#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2021-08-17 05:20](https://github.com/koalaman/shellcheck/issues/2297#issuecomment-900001597):

ShellCheck has avoided strict validation of parameter expansion because there is no specification and an ever-increasing feature set, so it's tricky to recognize something like `${var@@}` as invalid without breaking on every new bash feature.

However, in this case it starts with another parameter substitution which is never valid.

There are now warnings for anything where the first element of a `${..}` is something completely unexpected, such as another parameter expansion, a command substitution, a quoted variable name, and more, and it catches a number of new issues.

Thanks!
