Actually, i take that back. It should be able to handle the case of:
cd $1 -> cd $1 || exit -> cd "$1" || exit
The problem has to do with the way we are merging the fixes, then sorting it.
It starts with 2 fixes, first to add double quotes, second to add `|| exit`.
The first is made up of 2 replacements, and the second has 1. 
And in `applyFix`, we sort in reverse order of end position, and it ends up applying the ending double quote first.
I have an idea of how to fix this.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2018-12-30T21:11:32Z](https://github.com/koalaman/shellcheck/issues/1431#issuecomment-450586886):

This solution is simple, but doesn't support applying both fixes at once. The optimal solution would be to do AST modifications and re-serializing them, but that would be a lot of work to implement.

Maybe a middle ground would be to prioritize fixes based on the AST depth of the node being modified? Anything added to a deeply nested node should be contained within anything added to a more shallow node. 

For example, if you had a ``x=`${cmd[@]}` `` with a simplified AST:

```
   assignment:
       backtick (level 2)
         simplecommand
             ${cmd[@]} (level 4)
```

and you wanted to turn it into `x=$("${cmd[@]}")`, then any `replaceStart` for the level 2 item would be applied before/left-of any overlapping replaceStart for a level 4 item, and vice versa for replaceEnd.

I incorporated this and it appears to work reasonably well! Replacements now have both a depth and an alignment, and get processed from deepest to shallowest node. It didn't even require any changes to how the fixes were built in each check!
