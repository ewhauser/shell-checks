comparison of string variables should allow '='

#### Here's a snippet or screenshot that shows the problem:

```sh

if [[ "{$1,,}" = "foo" ]]; then      
#                        https://www.gnu.org/software/bash/manual/bash.html#Conditional-Constructs
#          string1 = string2
#              True if the strings are equal. When used with the [[ command, this performs pattern matching.
#
#           ‘=’ should be used with the test command for POSIX conformance.
fi

if [ "{$1,,}" = "foo" ]; then      
# same problem here
fi
```

#### Here's what shellcheck currently says:
The arguments to this comparison can never be equal.


#### Here's what I wanted or expected to see:
no porblem with '=' as per bash reference manual



#### <img src="https://avatars.githubusercontent.com/u/243351?v=4" width="50">[brother](https://github.com/brother) commented at [2023-11-07 20:44](https://github.com/koalaman/shellcheck/issues/2856#issuecomment-1800040402):

The left hand string will never be foo because no brace expansion will be performed. If we assume that $1 is set to foo you are comparing `curly brace foo comma comma curly brace` (`{foo,,}` ) with `foo` and they will never match.

```sh
brother ~$ cat /tmp/test.sh
#!/bin/bash

echo "1 is ->$1<-"

if [[ "{$1,,}" = "foo" ]]; then
	echo "Nice. 1 was set to foo."
fi

set -x

if [[ "{$1,,}" = "foo" ]]; then
	echo "Nice. 1 was set to foo."
fi
brother ~$ /tmp/test.sh foo
1 is ->foo<-
+ [[ {foo,,} = \f\o\o ]]
```
