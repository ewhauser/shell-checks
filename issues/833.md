#### <img src="https://avatars.githubusercontent.com/u/197253?v=4" width="50">[kurahaupo](https://github.com/kurahaupo) commented at [2017-02-19T06:45:54Z](https://github.com/koalaman/shellcheck/issues/833#issuecomment-280900225):

That's fairly clearly a feature of OS X; it definitely _won't_ work on most other operating systems:
```
#!/bin/bash -o pipefail

echo OK
```
When run on Linux 4.2, that produces:
    `/bin/bash: /usr/local/bin/x-shbang-test: invalid option name`

Instead, use:
`set -u -o pipefail`

Just avoid `-e` entirely; it has awful corner cases where it doesn't fail where it should, and does fail where it shouldn't, and ignores the fact that non-zero status may simply be that: a status, rather than an exception. You're much better off actually checking the status of any important commands.

And `-u` is redundant after you've run shellcheck -- it should spot any misspelt variables for you.

#### <img src="https://avatars.githubusercontent.com/u/197253?v=4" width="50">[kurahaupo](https://github.com/kurahaupo) commented at [2019-07-02T09:39:28Z](https://github.com/koalaman/shellcheck/issues/833#issuecomment-507601573):

@nihilus The reason for the "non-failure" _is_ OS-specific.
Interpreters - like all programs - rightly expect separate arguments as separate elements in argv. On _most_ OSes, the interpreter gets given the text after the interpreter name as just one element in argv, with embedded spaces. _That_ is why it fails as I showed above.

Any suggestion that the interpreter must parse the text string is deeply flawed.

In my opinion this is an over-use of the shbang to do something that should be controlled using the `set` built-in.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2021-09-18T19:30:36Z](https://github.com/koalaman/shellcheck/issues/833#issuecomment-922361300):

Since this bug was originally filed, ShellCheck has started supporting `env -S` (which is also supported on macOS), and you can state that all your scripts will only ever run on macOS with `echo disable=SC2096 >> ~/.shellcheckrc`.
