#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash

read -rp"(optional): " abc

abc="abc"

echo "${abc}"
```

#### Here's what shellcheck currently says:
```
Line 5	SC2178: Variable was used as an array but is now assigned a string.
Line 7	SC2128: Expanding an array without an index only gives the first element.
```
#### Here's what I wanted or expected to see:
```
No issues detected!
```

This appears to be happening when inserting a parentheses enclosed string with `read`s `-p` (_prompt_) parameter.

#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2020-10-19 06:00](https://github.com/koalaman/shellcheck/issues/2073#issuecomment-711621364):

You're right, this was a bug in the ad-hoc options parser, where `-rpa foo` would be parsed as `-r -p -a foo` instead of `-r -p a foo`, since it did not account for `-p` taking an argument. This has now been fixed.

In previous versions, you can work around it by adding a space: `read -rp "(optional): " abc`
