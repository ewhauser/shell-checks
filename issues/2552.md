#### Here's a snippet or screenshot that shows the problem:

```sh
# shellcheck shell=bash

setup() {
	cd "$BATS_TEST_TMPDIR"
}

teardown() {
	cd "$BATS_SUITE_TMPDIR"
}


```

#### Here's what shellcheck currently says:

```txt
Use 'cd ... || exit' or 'cd ... || return' in case cd fails
```

#### Here's what I wanted or expected to see:

Nothing

In Bats, `set -e` is set, so when `cd` fails, it will abort the process and show a stacktrace. When Bats [special variables](https://bats-core.readthedocs.io/en/stable/writing-tests.html#special-variables) like `BATS_SUITE_TMPDIR` are used, I think it would be nice for that to imply that the current file is running in the context of Bats, so the rule `SC2086` should be ignored
