#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/sh

day="${1-$(date +%A)}"

case "${day,,}" in
    sunday)    echo '日' ;;

    tuesday)   echo '火' ;;
    wednesday) echo '水' ;;
    thursday)  echo '木' ;;
    friday)    echo '金' ;;
    saturday)  echo '土' ;;
    *)         exit 1    ;;
esac
```

#### <img src="https://avatars.githubusercontent.com/u/20175435?u=8749e0c57901f29f964fdb7b1aea648d1370983a&v=4" width="50">[emanuele6](https://github.com/emanuele6) commented at [2020-07-13 12:14](https://github.com/koalaman/shellcheck/issues/2013#issuecomment-657525206):

At least if the shell being checked is `dash`, this should be an "error" since the script will fail with a shell error if run with `dash`:
```shell
$ shellcheck --shell=dash weekdaykanji; echo "$?"
0
$ dash weekdaykanji
weekdaykanji: 5: Bad substitution
```

The message should say that `${var,,}` is an invalid parameter expansion rather than just saying that it is not defined (at least when checking a `dash` script).
