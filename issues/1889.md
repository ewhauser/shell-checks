#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/sh
xargs -I{} sh -c 'echo {}'
```

...or...

```sh
#!/bin/sh
sh -c "echo $1"
```

#### Here's what shellcheck currently says:

```
No issues detected!
```

#### Here's what I wanted or expected to see:

Textually substituting results into the script argument of `sh` runs serious security risks. Keep your script argument constant, and add parameters after that position.

```sh
#!/bin/sh
sh -c 'echo "$1"' _ "$1"
```

```sh
xargs sh -c 'for arg do echo "$arg"; done' _
```
