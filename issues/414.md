# [\#414 Issue](https://github.com/koalaman/shellcheck/issues/414) `closed`: re-assignment of for loop var

#### <img src="https://avatars.githubusercontent.com/u/7633626?v=4" width="50">[aw-was-here](https://github.com/aw-was-here) opened issue at [2015-07-19T01:28:22Z](https://github.com/koalaman/shellcheck/issues/414):

There should probably be a warning or error if the variable in a for loop is re-assigned or used 
again in another for statement.  For example:

A:

``` bash
for i in 1 2; do
   for i in 2 3 ; do
        echo $i
   done
done
```

B:

``` bash
for i in 1 2; do
  for ((i=2; i>=0; i--)); do
    echo $i
  done
done
```

It's extremely unlikely that either case is what the user intended.
