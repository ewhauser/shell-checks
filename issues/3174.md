#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash
# or
#!/bin/ksh
[ ! -a /bin/sh ]
echo $?
```

#### Here's what shellcheck currently says:

nothing

#### Here's what I wanted or expected to see:

A warning is issued against `[ ! -a p ]` since it is not well-defined.
Actually, bash 5.2.37 and ksh 2020.0.0 give different results: 0 and 1, respectively. It seems that bash performs
```bash
[ ! ] && [ /bin/sh ]
# or
[ -n '!' ] && [ -n /bin/sh ]
```
whereas ksh does
```ksh
! [ -a /bin/sh ]
# or
! [ -e /bin/sh ]
```

#### <img src="https://avatars.githubusercontent.com/u/11627708?v=4" width="50">[e-kwsm](https://github.com/e-kwsm) commented at [2025-03-25 04:05](https://github.com/koalaman/shellcheck/issues/3174#issuecomment-2750014391):

```bash
set -B
[ ! -o braceexpand ]
echo $?  # bash: 0, ksh: 1
[[ ! -o braceexpand ]]
echo $?  # bash: 1, ksh: 1

# shellcheck should suggest
! [ -o braceexpand ]
```

ShellCheck now generally recommends `-e` over unary `-a`, warns about `[ -o opt ]` not being POSIX when applicable, and about the pitfall of `[ ! -o braceexpand ]`. Very surprising behavior and a great check, thanks!
