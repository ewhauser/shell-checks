#### For new checks and feature suggestions
- [x] https://www.shellcheck.net/ (i.e. the latest commit) currently gives no useful warnings about this
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/sh
find . -readable
```

#### Here's what shellcheck currently says:
Nothing.

#### Here's what I wanted or expected to see:
```sh
Line 2:
find . -readable
       ^-- SCXXXX Operand "-readable" is not defined in POSIX standard.
```
[The Open Group](https://pubs.opengroup.org/onlinepubs/9699919799/utilities/find.html)

#### <img src="https://avatars.githubusercontent.com/u/3066697?u=b0881521c5aff3cc9084fc44efc27d042fb6047b&v=4" width="50">[danielcft](https://github.com/danielcft) commented at [2024-08-15 22:24](https://github.com/koalaman/shellcheck/issues/2766#issuecomment-2292392187):

If so, then we would need to be consistent and check all other commands for non POSIX switches. 
Out of my head comes recursive grep, ie. `grep -r ` which is widely used but not documented in the POSIX standard. 

I don't believe we should go down this route. There are useful switches that are not described in POSIX.
