#### Here's a snippet or screenshot that shows the problem:

```bash
#!/usr/bin/env bash
set -e
FOO=1
[ "$FOO" -eq 1 ] && echo "yes" || true
```

#### Here's what shellcheck currently says:

```
[Line 4:](javascript:setPosition(4, 18))
[ "$FOO" -eq 1 ] && echo "yes" || true
                 ^-- [SC2015](https://www.shellcheck.net/wiki/SC2015) (info): Note that A && B || C is not if-then-else. C may run when A is true.

```

#### Here's what I wanted or expected to see:

ShellCheck should not throw this warning when C is `true`. The developer just want the return value to be 0, and it doesn't matter if `true` is executed or not.

With `set -e`, the script will fail before no-op.

The purpose of `|| true` is often to explicitly allow the line to fail without stopping the execution of the whole script.
