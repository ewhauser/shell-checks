# [\#1682 Issue](https://github.com/koalaman/shellcheck/issues/1682) `closed`: Warn of unqoted variables in case.

#### <img src="https://avatars.githubusercontent.com/u/24734779?v=4" width="50">[SoptikHa2](https://github.com/SoptikHa2) opened issue at [2019-09-02T18:21:55Z](https://github.com/koalaman/shellcheck/issues/1682):

#### For new checks and feature suggestions
- [x] shellcheck.net (i.e. the latest commit) currently gives no useful warnings about this
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related


#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/zsh # Bash works the same way
$choice="[[]]"

$answer_1="abc"
$answer_2="[[]]"
$answer_3="def"

if [ "$choice" = "$answer_2" ]; then
  echo "They are equal."
fi

case $choice in
  $answer_1)
    echo "1"
  ;;
  $answer_2)
    echo "2"
  ;;
  $answer_3)
    echo "3"
esac
```

Output of this script is:
```
They are equal.
```

While I would expect output:
```
They are equal.
2
```

The solution is to quote variables in `case`, like this:
```sh
case "$choice" in
    "$answer_1")
   #...
```

#### Here's what shellcheck currently says:

Nothing

#### Here's what I wanted or expected to see:

Shellcheck should warn user in the same way as when we use unquoted variables, which should be warning [SC2086](https://github.com/koalaman/shellcheck/wiki/SC2086).

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2019-09-09T03:10:16Z](https://github.com/koalaman/shellcheck/issues/1682#issuecomment-529283764):

You're right, this is the same effect as seen in [SC2053](https://github.com/koalaman/shellcheck/wiki/SC2053), and there's now a [new warning](https://github.com/koalaman/shellcheck/wiki/SC2254) for this. Thanks!
