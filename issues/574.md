# [\#574 Issue](https://github.com/koalaman/shellcheck/issues/574) `closed`: Warn about invisible/weird whitespace runes

#### <img src="https://avatars.githubusercontent.com/u/664779?v=4" width="50">[maxnordlund](https://github.com/maxnordlund) opened issue at [2016-01-10T22:33:12Z](https://github.com/koalaman/shellcheck/issues/574):

On Mac Alt+Space creates a non-breakable space, which in general breaks shells in all sorts of ways. It would be nice if shellcheck could detect that.

Something like for every unicode whitespace, make sure it's in this whitelist of space, newline, tab and maybe something more.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2016-01-12T02:57:12Z](https://github.com/koalaman/shellcheck/issues/574#issuecomment-170767802):

Non-breaking space and zero-width space are already covered by SC1018:

```
In test line 2:
if true; then true; fi
             ^-- SC1018: This is a unicode non-breaking space. Delete and retype it.
```

I haven't seen any others (like "OGHAM SPACE MARK" and "IDEOGRAPHIC SPACE") pop up IRL.
