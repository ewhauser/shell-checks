Shellcheck itself doesn't depend on docker. But if you want to run a docker container on pre-commit, docker needs to be installed. That's not a shellcheck issue.

Alternatively, you can download the binary and run that directly. Or checkout shellcheck and build it from source. Or ask whoever maintains pre-commit to include docker in their image.

#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2023-01-11 17:35](https://github.com/koalaman/shellcheck/issues/2646#issuecomment-1379245703):

pre-commit hooks are expected to be cross-platform. Here's a list of [languages/runners](https://pre-commit.com/#new-hooks) support by pre-commit, which includes things like `python`,  `lua`, and `node`, and `docker_image`. The latter is really the only one that can run a Haskell program in a portable way. pre-commit implements this by assuming you already have Docker installed, which is an understandable concession.

You can have a look at this third-party hook instead, which even predates the official one: https://github.com/shellcheck-py/shellcheck-py  . It uses the Python runner with a wrapper that downloads an appropriate binary for the current platform.
