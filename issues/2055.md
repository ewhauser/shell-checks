# [\#2055 Issue](https://github.com/koalaman/shellcheck/issues/2055) `open`: Using double quotes `"` in `$((...))` is a bash-ism

#### <img src="https://avatars.githubusercontent.com/u/20175435?u=8749e0c57901f29f964fdb7b1aea648d1370983a&v=4" width="50">[emanuele6](https://github.com/emanuele6) opened issue at [2020-09-09 02:53](https://github.com/koalaman/shellcheck/issues/2055):

#### For new checks and feature suggestions
- [x] https://www.shellcheck.net/ (i.e. the latest commit) currently gives no useful warnings about this
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related


#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/sh

echo "$(("$(date +%u)" * 2))"
```
#### Here's what shellcheck currently says:

Nothing

#### Here's what I wanted or expected to see:

`bash` will substitute `"$(date +%u)"` with `3` → `"$((3 * 2))"` that is valid.
```
6
```
`dash` will only substitute `$(date +%u)` with `3` → `"$(("3" * 2))"` which isn't valid:
```
file.sh: 3: arithmetic expression: expecting primary: ""3" * 2"
```

`shellcheck` should, at least, warn me about using `"`s in a `$((...))` substitution if the shell is not `bash`.
It could also suggest me to remove these `"`s.
EDIT: Maybe even an error for `dash` since `"`s in `$((...))` will always cause an error.
