# [\#354 Issue](https://github.com/koalaman/shellcheck/issues/354) `closed`: False positive for HOSTNAME if used as a variable

#### <img src="https://avatars.githubusercontent.com/u/2865?v=4" width="50">[richq](https://github.com/richq) opened issue at [2015-04-20T07:57:10Z](https://github.com/koalaman/shellcheck/issues/354):

The following snippet gives the warning `test.sh:4:7: warning: In POSIX sh, HOSTNAME is not supported. [SC2039]`

```
#!/bin/sh

HOSTNAME=$(hostname)
echo "$HOSTNAME"
```

Seems like this comes from the bashism checks added in cc86aab3f12130589d0d9a4f4cc514513050b5dd. Does shellcheck have the information available to know that HOSTNAME was (re)defined by the script prior to being used? It'd be good to not show that as a warning in this case.
