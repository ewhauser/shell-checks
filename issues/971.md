#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2017-08-30T11:53:03Z](https://github.com/koalaman/shellcheck/issues/971#issuecomment-325967572):

You could do this today with the help of `jq`:

    shellcheck -f json file | jq -e 'any(.[].level; . == "error")'

It's not currently supported natively because it's less useful than it sounds. There's no clear distinction between `error` and `warning`, it's just a loose sum of accuracy and impact. The plan is to add a verbosity level at some point so that you can choose to get only important warnings/errors.
