# [\#735 Issue](https://github.com/koalaman/shellcheck/issues/735) `open`: RFE: Warn about some useless subshell cases involving conditionals

#### <img src="https://avatars.githubusercontent.com/u/1042176?v=4" width="50">[Herst](https://github.com/Herst) opened issue at [2016-09-09T13:23:40Z](https://github.com/koalaman/shellcheck/issues/735):

Assuming you have a shell with support for standalone curly braces blocks for command grouping (haven't found any without support for it yet) there is no reason to use subshells for cascaded conditon tests, which means that e.g.

``` sh
if ! ( ( [ "$1" = foo ] || [ "$1" = bar ] ) && [ "$2" = baz ] ); then
  :
fi
```

could be written as

``` sh
if ! { { [ "$1" = foo ] || [ "$1" = bar ]; } && [ "$2" = baz ]; }; then
  :
fi
```

instead, i.e. no need for subshells here.

Yet on [many](http://www.unix.com/302591897-post3.html) [places](http://www.unix.com/302498536-post4.html) [on](http://stackoverflow.com/a/3827403/2261442) [the](http://stackoverflow.com/a/16203126/2261442) [internet](http://alvinalexander.com/blog/post/page-1/wed-jun-4-2003) you will find code snippets with those useless subshells.

Should the detection not be too hard then it would be neat if `shellcheck` would warn about this just like it does for some of the Useless Use of Cat™ cases.

/edit:

I just learned [that even with `{ }` a subshell might be created](https://unix.stackexchange.com/questions/127334/bash-subshell-creation-with-curly-braces), so I guess when piping is involved this warning wouldn't be correct which I guess makes implementation harder.
