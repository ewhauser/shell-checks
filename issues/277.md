# [\#277 Issue](https://github.com/koalaman/shellcheck/issues/277) `closed`: How to workaround "SC2086" for command-line options

#### Labels: invalid

#### <img src="https://avatars.githubusercontent.com/u/1235108?v=4" width="50">[myint](https://github.com/myint) opened issue at [2015-01-03T16:15:30Z](https://github.com/koalaman/shellcheck/issues/277):

I'm not sure what the workaround is for cases such as the following.

`foo.bash`:

```
blah $FOO
```

```
$ shellcheck foo.bash

In foo.bash line 1:
blah $FOO
   ^-- SC2086: Double quote to prevent globbing and word splitting.
```

If I put `$FOO` in quotes, I will get behavior that I do not intend. For example, if `FOO` is `--something --nothing`, it will pass a single argument `--something --nothing` to `blah` rather than two separate arguments `--something` and `--nothing`.

Should I just explicitly suppress `shellcheck` for such cases?

#### <img src="https://avatars.githubusercontent.com/u/442919?v=4" width="50">[dualbus](https://github.com/dualbus) commented at [2015-01-04T02:06:52Z](https://github.com/koalaman/shellcheck/issues/277#issuecomment-68617711):

> ```
> blah $FOO
> ```

```
foo=(--something --nothing)
blah "${foo[@]}"
```

This was supposed to be documented here: http://mywiki.wooledge.org/BashFAQ/050

But it seems that the array solution was lost in the edits.

#### <img src="https://avatars.githubusercontent.com/u/95290?v=4" width="50">[resuna](https://github.com/resuna) commented at [2015-04-28T00:15:05Z](https://github.com/koalaman/shellcheck/issues/277#issuecomment-96855561):

The array solution is bash-only, it doesn't work for portable shell scripts.

Suggestion - a comment (eg "#pragma safe") for such cases.

EG:

#pragma safe FLAGS
FLAGS="-av --delete --delete-excluded --exclude=.DS_Store"

Rationale: all the world isn't bash (thank god).
