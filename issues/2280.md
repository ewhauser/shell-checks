#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2021-07-24 20:57](https://github.com/koalaman/shellcheck/issues/2280#issuecomment-886110549):

I'm able to successfully build with them on Ubuntu 20.04, here tested via Docker:

```
FROM ubuntu:20.04
RUN apt-get update && apt-get install -y cabal-install git
RUN git clone --branch v0.7.1 https://github.com/koalaman/shellcheck
RUN cabal update
RUN cd shellcheck && cabal install
```

* Your output says there are missing files and suggests running `ghc-pkg check`. Did you try that? 
* You can clear out your per-user Haskell libs and executables with `rm -rf ~/.cabal ~/.ghc` and try again 
* If still no luck, try updating `ghc` and `cabal-install`. The latest versions on Ubuntu 20.04 appear to be 8.8.1 and 2.4.0.0 respectively, while you're on ghc-8.6.5. Cabal especially changes quite rapidly.

Please reopen if this does not help!
