# [\#652 Issue](https://github.com/koalaman/shellcheck/issues/652) `closed`: Variables used as references declared as unused

#### <img src="https://avatars.githubusercontent.com/u/521538?v=4" width="50">[Earnestly](https://github.com/Earnestly) opened issue at [2016-04-17T00:46:58Z](https://github.com/koalaman/shellcheck/issues/652):

Sorry if there's already an issue about this however bash's recent `declare -n` feature can confuse shellcheck when variables are past to them by reference, here is a small test case:

``` bash
#!/bin/bash

printarray() {
    declare -n a=$1
    printf '%s\n' "${a[@]}"
}

foo=(One day through the primeval wood)

printarray foo
```

Here shellcheck warns us that: `shellcheck: foo appears unused. Verify it or export it. [SC2034]`
