# [\#690 Issue](https://github.com/koalaman/shellcheck/issues/690) `closed`: Don’t warn that function references arguments which are not passed if set is used

#### <img src="https://avatars.githubusercontent.com/u/32383?v=4" width="50">[mina86](https://github.com/mina86) opened issue at [2016-06-15T23:31:52Z](https://github.com/koalaman/shellcheck/issues/690):

At times I use function’s positional arguments to store local variables. This is useful since POSIX shells do not have `local` directive. An example code might look something like:

``` shell
foo() {
    set -- 'foo * bar'
    printf '%s\n' "$1"
}
foo
```

Sadly, this leads to noisy shellcheck output:

```
In a.sh line 1:
foo() {
^-- SC2120: foo references arguments, but none are ever passed.


In a.sh line 5:
foo
^-- SC2119: Use foo "$@" if function's $1 should mean script's $1.
```

It would be fantastic, if shellcheck detected that set builtin is used inside of the function and in those cases not issued the warnings.
