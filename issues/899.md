#### For bugs

- Rule Id (if any, e.g. SC1000):  `SC2039`
- My shellcheck version (`shellcheck --version` or "online"):  both `0.4.6` and `online`
- [x] I tried on shellcheck.net and verified that this is still a problem on the latest commit
-  <s>It's not reproducible on shellcheck.net, but I think that's because it's an OS, configuration or encoding issue</s> It is reproducible online.

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/sh
type cd
```

#### Here's what shellcheck currently says:

```
Line 2:
type cd
^-- SC2039: In POSIX sh, 'type' is undefined.
```

#### Here's what I wanted or expected to see:

Nothing, for `type` is indeed defined in POSIX.  See: `man 1p type` (assuming POSIX man pages installed), or [The Open Group documentation](http://pubs.opengroup.org/onlinepubs/9699919799/utilities/type.html#top).
