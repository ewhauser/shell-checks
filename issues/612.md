# [\#612 Issue](https://github.com/koalaman/shellcheck/issues/612) `open`: Statistics / unused code

#### <img src="https://avatars.githubusercontent.com/u/2039950?v=4" width="50">[VanNostrand](https://github.com/VanNostrand) opened issue at [2016-02-19T22:52:11Z](https://github.com/koalaman/shellcheck/issues/612):

Warning about unused functions should be possible - I know, if a script is going to be sourced, it is unknown if a function is really unused, but if I know that it is not sourced and want to throw out unused code, then I need this feature. Example:

```
#!/bin/sh
unused() {
  echo junk
}
echo main
```

That should be enabled via command line parameter or printed into a statistics file because writing _disable shellcheck=...._ in a large library script will surely be unpleasant.

More general: it would be interesting to get usage counts for variables and functions so that the programmer can see code that could be eliminated or written inline - for example, if a script with 1000 lines contains

```
10: GREP=/bin/grep
[...]
972: $GREP foo bar
```

and shellcheck prints statistics like _variable GREP is used 1 times_ then it shows me that there is potential for elimination.

I would also like to see after the normal shellcheck output, how many issues were found without executing shellcheck an additional time, piping the output into grep -c ^In
When going through scripts with several hundred errors, seeing progress is important and an interesting fact for a checkin message for my code changes.
