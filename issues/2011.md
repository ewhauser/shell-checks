Not a safe solution.

The number of command line arguments is bounded. The bound differs across operating systems.

This represents a less performant, batch style operation where 100% of the file paths are identified first, rather than a more scalable, streaming pattern where paths are uncovered during the natural course of scans.

git coupling, grep dependency, hardcoded interpreters (bash only), subshells, pipes, xargs, space quoting, and other shell minutuae... such things are fragile, nonportable, very easy to mess up.

We really need built-in recursive over directory paths, as modern linters implement.
