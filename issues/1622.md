#### Here's a snippet or screenshot that shows the problem:

```bash

#!/usr/bin/env bash

PATH='/Applications/Android Studio.app/Contents/jre/jdk/Contents/Home/bin':$PATH
EXECUTABLE=$(command -v jarsigner)

# $EXECUTABLE is now set to '/Applications/Android Studio.app/Contents/jre/jdk/Contents/Home/bin/jarsigner'

$EXECUTABLE -help 
```

#### Here's what shellcheck currently says:
No output, return code 0.


#### Here's what I wanted or expected to see:
A warning against unquoted usage of $EXECUTABLE; this small script fails on my machine with `bash: line 5: /Applications/Android: No such file or directory`, but works once I change the last line to `"$EXECUTABLE" -help`.
