# [\#3225 Issue](https://github.com/koalaman/shellcheck/issues/3225) `closed`: False positive SC2094 when tee is used with -a

#### <img src="https://avatars.githubusercontent.com/u/11071985?v=4" width="50">[kkmuffme](https://github.com/kkmuffme) opened issue at [2025-06-20 09:29](https://github.com/koalaman/shellcheck/issues/3225):

Minimal reproducer: (which by itself doesn't make any sense, since 2>&1 could be used, but this is just the minimal example)

```
grep "abc" invalid.log 2> foo.log | tee -a foo.log
```

https://www.shellcheck.net/wiki/SC2094

When using tee with -a flag, it appends and works without issues. I think SC2094 is a false positive error in this case?



#### <img src="https://avatars.githubusercontent.com/u/11627708?v=4" width="50">[e-kwsm](https://github.com/e-kwsm) commented at [2025-07-25 00:40](https://github.com/koalaman/shellcheck/issues/3225#issuecomment-3115422019):

No, stdout may be discarded.

```console
$ { echo 1; sleep 1; echo 2 >&2; } 2> foo.log | tee -a foo.log > /dev/null
$ cat foo.log
2
```
