Code: SC2086
Currently shellcheck recommends the simplest/safest replacement for problematic unquoted variables. However, when these are part of a longer path, the result is cumbersome, particularly if there are multiple variables in the path. This often requires manual fixes for all of these instead of being able to fix them automatically. It would be great if there were an option to instead to detect the whole string and quote it unless it contains a glob (with any needed caveats if that is unsafe).

#### Here's a snippet or screenshot that shows the problem:
Case 1
```
#! /bin/sh

dirA=$1

mkdir ${dirA}/b
```

Case 2
```
#! /bin/sh

dirA=$1
dirB=$2

mkdir ${dirA}/${dirB}
```

Case 1
```

Did you mean: 
mkdir "${dirA}"/b

For more information:
  https://www.shellcheck.net/wiki/SC2086 -- Double quote to prevent globbing ...
```

Case 2
```
In shellcheck_test.sh line 6:
mkdir ${dirA}/${dirB}
      ^-----^ SC2086 (info): Double quote to prevent globbing and word splitting.
              ^-----^ SC2086 (info): Double quote to prevent globbing and word splitting.

Did you mean:
```
