`...` and `?` are valid function names in bash, however shellcheck chokes on them and stops parsing

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2013-11-01T21:36:18Z](https://github.com/koalaman/shellcheck/issues/31#issuecomment-27603070):

Interestingly, ? is valid when doing function foo? { ...; } but not with foo?() { ...; }. 


#### <img src="https://avatars.githubusercontent.com/u/183837?v=4" width="50">[worr](https://github.com/worr) commented at [2013-11-01T21:57:56Z](https://github.com/koalaman/shellcheck/issues/31#issuecomment-27604357):

Your first example won't declare a function. You need the `()` after the name if you omit `function` when declaring the function.

Also, contrary to your examples, I've been doing:

``` bash
?() {
    /bin/true
}

...() {
    /bin/true
}
```

Both are valid, however they totally break the parser.
