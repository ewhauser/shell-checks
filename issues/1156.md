#### <img src="https://avatars.githubusercontent.com/u/19507282?v=4" width="50">[automorphism88](https://github.com/automorphism88) commented at [2018-03-26T03:23:41Z](https://github.com/koalaman/shellcheck/issues/1156#issuecomment-376035935):

> Why do you feel that SC2086 should not be raised?

I'm not the OP, but perhaps because globbing and word splitting aren't applicable when the variable is an integer. However, all bash variables are strings, so the -i option to the declare and local builtins doesn't actually guarantee that the variable will always contain an integer. In the OP's example, however, an error will result if the command substitution returns a string with whitespace. Consider the following example script:

```
#!/bin/bash -
declare -i foo
foo="$(echo "string with whitespace")"
printf '%s\n' "$foo"
```
This throws an error, and the variable `foo` remains unset, while something like `foo=bar` or `foo=$(echo bar)` (assignment to a non-integer with no whitespace) will simply evaluate `bar` as an integer, and will output `0`.

However, if the variable is first assigned, and then declared to be an integer - e.g., if we reverse the order of lines 2 and 3 in my example script - then no error will result, and the output will be `string with whitespace`.

So the question is, should shellcheck assume that a variable declared with the -i option is an integer, and therefore doesn't need double quotes, the way it would with a special variable like `$?` that is always an integer? If so, the OP's report is valid. If we're going to err on the side of paranoia, though, there's no reason not to double quote an integer, so perhaps shellcheck's behavior is correct.

#### <img src="https://avatars.githubusercontent.com/u/4387891?v=4" width="50">[mkoskar](https://github.com/mkoskar) commented at [2018-12-29T14:38:50Z](https://github.com/koalaman/shellcheck/issues/1156#issuecomment-450497289):

Consider following 3 assignments which are equivalent (we could argue about which is better style-wise) but aren't treated the same.

```shell
#!/usr/bin/bash

declare -i x=1

#x=$((OPTARG))  # ok
#x=OPTARG       # ok

x=$OPTARG       # triggers SC2086 on echo below

echo $x

```
