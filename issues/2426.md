#!/bin/bash

function ■()
{
    echo "■ $@"
}

```

```
Line 3	SC1036: '(' is invalid here. Did you forget to escape it?

#### <img src="https://avatars.githubusercontent.com/u/293068?v=4" width="50">[grondilu](https://github.com/grondilu) commented at [2023-11-12 09:25](https://github.com/koalaman/shellcheck/issues/2426#issuecomment-1807069018):

I'm not too surprised shellcheck does not accept that.  It should though, as in the bash man page, one can read:

> In default mode, a function name can be any unquoted shell word  that  does not  contain $

I could not quite find what a shell word is, as the definition is a bit vague:

> A sequence of characters considered as  a  single  unit  by  the shell.

I suspect this means pretty much any sequence of unicode characters.

This is confirmed if you try very weird function names:

```bash
[]{}#()  { echo "ok"; }
```

This actually defines a function `[]{}#`, amazingly enough.

Needless to say, shellcheck refuses to parse that.

#### <img src="https://avatars.githubusercontent.com/u/39741460?u=648bec676a6188873fc5b900e0603f084c1758ef&v=4" width="50">[GB609](https://github.com/GB609) commented at [2026-01-28 09:50](https://github.com/koalaman/shellcheck/issues/2426#issuecomment-3810192824):

I'm running into this as well, as i'm trying to name functions with '#' in it, e.g.
```bash
function prefix#functionName { 
   echo TBD: 
}
```
This is a perfectly valid function name for bash in non-POSIX mode, because '#' is ony marking a comment when:
- at the start of a line
- preceeded by whitespace
- preceeded by a special operator, which are several different tokens containing `&`, `;`, `|`, `(`, `)` in unescaped/unquoted form (plus redirection statements like `>&2`).

In all other situations, e.g. when preceeded by any ascii letter, '#' is treated as just another character.

The snipped above leads to 
```
^-- SC1073 (error): Couldn't parse this function. Fix to allow more checks.
```

One can trick shellcheck into continuing to parse the file by a little reformatting:
```bash
function prefix#functionName 
{ 
   echo TBD: 
}
prefix#functionName
```
But this messes up the invocation check, since the function is just named `prefix` for shellcheck.
So, i get SC2329 when using `prefix#functionName`:
```
^-- SC2329 (info): This function is never invoked. Check usage (or ignored if invoked indirectly).
```

using just `prefix` as invocation is detected then for `prefix#functionName`.

Adding a bash-shebang to the file doesn't change anything.
