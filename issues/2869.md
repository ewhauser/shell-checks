#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2023-12-11 03:53](https://github.com/koalaman/shellcheck/issues/2869#issuecomment-1849282957):

This is unfortunately a known problem. You can work around it with `user:is_superuser() { ((!UID)); }`.

ShellCheck has the annoying restriction that all functions bodies must be `{ .. }` or `( .. )`, which is actually more work than the correct approach of allowing any compound command. I've tried to fix it many times over the years, but it keeps leading to strange and misleading error messages for a variety of newbie syntax issues as ShellCheck keeps going and talking about function definitions when the user really just botched an array assignment that happened to be followed by a for loop. Hopefully one day...
