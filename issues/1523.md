# [\#1523 Issue](https://github.com/koalaman/shellcheck/issues/1523) `open`: encourage using 'basename' always with --

#### <img src="https://avatars.githubusercontent.com/u/198379?v=4" width="50">[bittorf](https://github.com/bittorf) opened issue at [2019-03-17T10:07:39Z](https://github.com/koalaman/shellcheck/issues/1523):

proposal:

```
#!/bin/sh
file='-/a/b/c.de'
basename "$file"
```

this throws an error:
```
# basename "$file"
basename: invalid option -- '/'
Try 'basename --help' for more information.
```

so better remind the use to always call with
```
basename -- "$file"
```

maybe related: #9 and
https://unix.stackexchange.com/questions/94295/shellcheck-is-advising-not-to-use-basename-why/381440

#### <img src="https://avatars.githubusercontent.com/u/2160915?v=4" width="50">[tripleee](https://github.com/tripleee) commented at [2019-07-30T07:19:37Z](https://github.com/koalaman/shellcheck/issues/1523#issuecomment-516297151):

As already commented on the PR, this is by no means specific to `basename`, and so the check should probably be extended to any command which accepts a file name parameter. `basename` is a common victim to this because people do

```bash
for filename in *; do
    basename "$filename"
done
```
but the proper fix here is to say
```
for filename in ./*; do
```
