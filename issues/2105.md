#### Here's a snippet or screenshot that shows the problem:

```sh
#!/usr/bin/env -S bash -x
# shellcheck shell=bash
```

#### Here's what shellcheck currently says:

```
$ shellcheck myscript
 
Line 1:
#!/usr/bin/env -S bash -x
^-- SC2096: On most OS, shebangs can only specify a single parameter.
```

#### Here's what I wanted or expected to see:

Shellcheck should be aware of `env -S`, which according to https://man7.org/linux/man-pages/man1/env.1.html

```
       -S, --split-string=S
              process and split S into separate arguments; used to pass
              multiple arguments on shebang lines
```

According to https://ss64.com/osx/env.html, `env -S` also should work on macOS.
