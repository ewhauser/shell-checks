# [\#772 Issue](https://github.com/koalaman/shellcheck/issues/772) `closed`: README: Use camel-case for package name

#### <img src="https://avatars.githubusercontent.com/u/877109?v=4" width="50">[asayers](https://github.com/asayers) opened issue at [2016-11-07T05:16:05Z](https://github.com/koalaman/shellcheck/issues/772):

The README asks users to `cabal install shellcheck`, which works because cabal-install searches hackage case-insensitively. Stack, however, doesn't do this - you have to use the exact package name (ie. `stack install ShellCheck`). Since some users will swap out `cabal` for `stack`, I suggest using camel-case in the README.
