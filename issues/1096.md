# [\#1096 Issue](https://github.com/koalaman/shellcheck/issues/1096) `closed`: ShellCheck 0.4.7 violates PVP versioning standard

#### <img src="https://avatars.githubusercontent.com/u/28323?v=4" width="50">[peti](https://github.com/peti) opened issue at [2018-01-18T15:07:34Z](https://github.com/koalaman/shellcheck/issues/1096):

The 0.4.7 release is a minor release that's supposed to be API compatible with 0.4.6 according to [PVP](https://pvp.haskell.org/). It is not, however, because the field `csCheckSourced` has been added to https://hackage.haskell.org/package/ShellCheck-0.4.7/docs/ShellCheck-Interface.html#t:CheckSpec. This change requires a version bump to (at least) 0.5.

Ah, you're right. 

There's no breakage when using `emptyCheckSpec { .. }` like ShellCheck itself does, but `CheckSpec` *is* public.
