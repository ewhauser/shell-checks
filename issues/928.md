read -sr -n1 action
((action == "q")) && exit 0

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2017-06-09T23:22:47Z](https://github.com/koalaman/shellcheck/issues/928#issuecomment-307523058):

ShellCheck is right. `"q"` refers to the variable named `q` in arithmetic expressions:

```
$ action=1 q=1
$ (( action == "q" )) && echo "same" || echo "different"
same
$ action=1 q=2
$ (( action == "q" )) && echo "same" || echo "different"
different
```

There should probably be a warning saying that quoted strings are variable names though. Strings should be compared with `[` or `[[`. I created #931 for this.
