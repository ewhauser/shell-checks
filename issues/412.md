# [\#412 Issue](https://github.com/koalaman/shellcheck/issues/412) `closed`: Test expression parsing failure shouldn’t be fatal

Script:

``` bash
mode=-f
file=foo
[ "$mode" "$file" ] && echo bar
```

ShellCheck output:

```
In foo.bash line 3:
[ "$mode" "$file" ] && echo bar
^-- SC1073: Couldn't parse this test expression.
          ^-- SC1072: Expected "]". Fix any mentioned problems and try again.
```

There shouldn’t be a “fatal” parser error, since it’s a perfecly legal bash/sh code.
