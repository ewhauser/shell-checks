# [\#877 Issue](https://github.com/koalaman/shellcheck/issues/877) `open`: Error not thrown for non arithmetic context usage with seq command

#### <img src="https://avatars.githubusercontent.com/u/25167560?v=4" width="50">[inianv](https://github.com/inianv) opened issue at [2017-04-18T12:25:12Z](https://github.com/koalaman/shellcheck/issues/877):

For one of the questions I was answering in StackOverflow, found a snippet,

` for i in $(seq 1 {$value/$NUM}); do`

needs to have thrown **a red-highlighted error** for wrong arithmetic evaluation in bash or any other shell I know of, but instead I just saw this yellow warning,

[This {/} is literal. Check expression (missing ;/\n?) or quote it](https://github.com/koalaman/shellcheck/wiki/SC1083)

Tested on [ShellCheck](http://www.shellcheck.net/) online site!

To reproduce:-
```
#!/bin/bash
a=20
b=5
for i in $(seq 1 {$a/$b}); do
    echo "$i"
done
```
