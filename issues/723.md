# [\#723 Issue](https://github.com/koalaman/shellcheck/issues/723) `open`: SC2055:  [[ A != B || A != C ]] is not true if B == C

#### <img src="https://avatars.githubusercontent.com/u/135807?v=4" width="50">[ecksun](https://github.com/ecksun) opened issue at [2016-08-10T20:55:13Z](https://github.com/koalaman/shellcheck/issues/723):

I believe issue #586 is a special-case of this issue.

I ran into this when reading code similar to the following:

``` sh
[ "$A" != "${A% AB}" -o "$A" != "${A#AB }" ]
```

What this expresses is if `$A` either has `"AB"` as a whitespace separated prefix or suffix.

It can be generalized to the following:

``` bash
[[ "$A" != "$B" || "$A" != "$C" ]]
```

The only way for the expression to always be true is if it can be known that `B != C`, which can be very tricky  except for in the simple cases, where both B and C are constants.

I would suggest both making the check more strict to only handle cases where `B != C` and updating the wiki to acknowledge this particular case.

#### <img src="https://avatars.githubusercontent.com/u/197253?v=4" width="50">[kurahaupo](https://github.com/kurahaupo) commented at [2016-08-11T14:45:12Z](https://github.com/koalaman/shellcheck/issues/723#issuecomment-239182786):

Both of those seem fairly contorted when there are clearer ways to express
the intent:

```
[[ "$A" = @("AB "*|*" AB") ]]
```

Or for other shells:

```
case $A in
"AB "* | *" AB") true ;;
*) false ;;
esac
```
