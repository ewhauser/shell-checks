# [\#2771 Issue](https://github.com/koalaman/shellcheck/issues/2771) `open`: [FEATURE REQUEST] Check if variables are defined before other variables that use them

#### <img src="https://avatars.githubusercontent.com/u/118914686?u=a754987f621e794e4f7af52a3836375135c87f9f&v=4" width="50">[gabriel-astapenco-koin](https://github.com/gabriel-astapenco-koin) opened issue at [2023-05-31 18:30](https://github.com/koalaman/shellcheck/issues/2771):

It is easy to define a variable after it is used by other variables (i.e.: because of a copy paste, refactor, organize, etc).
That's why it would be nice to add a check/suggestion when this happens, in order to save us some time of digging/debugging on what is wrong, and sometimes is not easy because of so many variable definitions on the scripts.

An example on this that happened to me twice this month:
```bash
VAULT_TOKEN_HEADER="X-Vault-Token: $VAULT_TOKEN"
# ...
# Many variable definitions
# ...
VAULT_TOKEN="some token"
```

This will not return an error on the script itself (because X-Vault-Token header will be empty, which is valid for bash) but make your request to the vault API fail with 403, forbbiden.

[Bash](https://tldp.org/LDP/Bash-Beginners-Guide/html/sect_03_02.html) 

Checking defined variables can be difficult, because you need to evaluate flows of your script which are also extended by ENV.
Let's say example:

```
f() {
  A=IDK
}
b() {
  echo "$A"
  # Is A defined?
}
# some complex call diagram or even recursive calls with sets and unsets
```

Can you imagine the complexity now?
In compiled languages like a Java its part of semantic analysis.
But Shell is interpreted script language.
