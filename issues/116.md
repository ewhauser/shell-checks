# [\#116 Issue](https://github.com/koalaman/shellcheck/issues/116) `closed`: Detect if variable is set before used in functions and advise making it local

#### <img src="https://avatars.githubusercontent.com/u/29916?v=4" width="50">[pihentagy](https://github.com/pihentagy) opened issue at [2014-03-12T15:17:06Z](https://github.com/koalaman/shellcheck/issues/116):

```
#!/bin/bash

fun1() {
   a=2
   echo $a
}

fun2() {
   b=2
   echo $b
}
```

This one is toughter, if you first assign to a variable before reading it, shellcheck could also advice making it local.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2014-03-16T22:44:07Z](https://github.com/koalaman/shellcheck/issues/116#issuecomment-37773960):

I like this idea because global variables are iffy. However, since this would trigger any time you try to initialize a global variable in a function, I suspect the false positive rate is too high. 


#### <img src="https://avatars.githubusercontent.com/u/29916?v=4" width="50">[pihentagy](https://github.com/pihentagy) commented at [2014-03-17T09:54:01Z](https://github.com/koalaman/shellcheck/issues/116#issuecomment-37798904):

Well, to be more precise: give a warning, if the variable is either
- used in one method
- used in multiple methods, but in **every** method it is initialized before used (in that case, the parameters of the functions are likely to be named the same)

```
fun1() {
   PARAM_NAME=$1
   # actual work using PARAM_NAME
}
fun2() {
   PARAM_NAME=$1  
   # actual work using PARAM_NAME
}
```
