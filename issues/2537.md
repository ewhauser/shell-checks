#### Here's a snippet or screenshot that shows the problem:

```bash
#!/usr/bin/env bash
# shellcheck enable=all

set -e
set -o posix

test_func()
{
  :
}

_test="$(test_func)"
```

#### Here's what shellcheck currently says:
[SC2311](https://www.shellcheck.net/wiki/SC2311) (info): Bash implicitly disabled set -e for this function invocation because it's inside a command substitution. Add set -e; before it or enable inherit_errexit.


#### Here's what I wanted or expected to see:
`set -o posix` has the effect to also enable `inherit_errexit`, so SC2311 shouldn't appear
