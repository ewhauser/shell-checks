# [\#205 Issue](https://github.com/koalaman/shellcheck/issues/205) `closed`: Handle empty folders more gracefully

#### <img src="https://avatars.githubusercontent.com/u/5316?v=4" width="50">[mcandre](https://github.com/mcandre) opened issue at [2014-08-06T02:27:03Z](https://github.com/koalaman/shellcheck/issues/205):

When shellcheck encounters an empty directory, I would prefer that it not complain.

Current behavior:

```
rm -rf lib/

shellcheck lib/*
lib/*: openFile: does not exist (No such file or directory)
```

Desired behavior:

```
rm -rf lib/

shellcheck lib/*
(no output)
```
