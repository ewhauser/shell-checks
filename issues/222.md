# [\#222 Issue](https://github.com/koalaman/shellcheck/issues/222) `open`: shellcheck.net should take a URL

#### <img src="https://avatars.githubusercontent.com/u/7633626?v=4" width="50">[aw-was-here](https://github.com/aw-was-here) opened issue at [2014-09-02T17:38:25Z](https://github.com/koalaman/shellcheck/issues/222):

I'd love to be able to point shellcheck.net at a URL. In other words, give something like https://git-wip-us.apache.org/repos/asf?p=hadoop.git;a=blob_plain;f=hadoop-common-project/hadoop-common/src/main/bin/hadoop;h=64c67587dc6fb868c6dac1b236c2f352e5a22842;hb=HEAD as input and get back a URL that has the code and shellcheck output as if it was pasted in.

#### <img src="https://avatars.githubusercontent.com/u/243351?v=4" width="50">[brother](https://github.com/brother) commented at [2014-09-08T06:18:57Z](https://github.com/koalaman/shellcheck/issues/222#issuecomment-54780299):

Add a alias, function or command to do something like 
curl "https://git-wip-us.apache.org/repos/asf?p=hadoop.git;a=blob_plain;f=hadoop-common-project/hadoop-common/src/main/bin/hadoop;h=64c67587dc6fb868c6dac1b236c2f352e5a22842;hb=HEAD" | shellcheck -

and you are done. can not really see why shellcheck should do web output for you.

If you really need the web stuff I suggest you go overboard and do something that uses shellcheck.net for the parsing and saving of web output.


#### <img src="https://avatars.githubusercontent.com/u/7633626?v=4" width="50">[aw-was-here](https://github.com/aw-was-here) commented at [2014-09-08T16:58:02Z](https://github.com/koalaman/shellcheck/issues/222#issuecomment-54851554):

I think I'm looking at this from a different perspective.

I'd like to be able to generate a URL to give to someone else to look at the output.  Having shellcheck.net do this would mean that the other person wouldn't need to have shellcheck installed on their local machine.  Another alternative is to use something like pastebin, but that also means static content, lack of colorization, etc, etc.
