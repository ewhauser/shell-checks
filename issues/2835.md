# [\#2835 Issue](https://github.com/koalaman/shellcheck/issues/2835) `closed`: Add an optional check for `avoid-double-brackets`

#### <img src="https://avatars.githubusercontent.com/u/11302521?u=649b398094360628fc128d9561a71b3c86c7afcc&v=4" width="50">[Kreyren](https://github.com/Kreyren) opened issue at [2023-09-30 21:52](https://github.com/koalaman/shellcheck/issues/2835):

I am trying to write my scripts to be as portable as possible to use `# shellcheck shell=sh` and try to replace bash-specific declaration with POSIX sh.

To aid that effort i am proposing to add `avoid-double-brackets` as an optional check which is supposed to encourage the removal of bash-specific double brackets for single-brackets that are compatible with POSIX sh.

You are right i didn't realize that i can set `shell=sh` in `.shellcheckrc` at the time to get this runtime so it doesn't make sense to include as an optional check.

FWIW the idea was to have this on `shell=bash` as a constant effort to turn a huge codebase into a more portable one through integrating POSIX sh syntaxes, but yes `shell=sh` works better for that.
