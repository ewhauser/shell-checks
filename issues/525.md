# [\#525 Issue](https://github.com/koalaman/shellcheck/issues/525) `open`: Does not check for the argument number inside a test for it

``` .bash
#!/bin/bash

if [ $# -ne 3 ]
then
  echo "ERROR: Usage $4 <foo> <bar>"
fi
```

$4 will not exist.

#### <img src="https://avatars.githubusercontent.com/u/243351?v=4" width="50">[brother](https://github.com/brother) commented at [2015-10-27T11:58:35Z](https://github.com/koalaman/shellcheck/issues/525#issuecomment-151468450):

The report is correct as far as parsing goes but the example code is not. 

Five arguments to the script would make $4 set and the ERROR text would be displayed as $# equals 5.
(and for bash I would write it if (( $# != 3 )); ... but that's another discussion =))

``` bash
brother ~$ bash /tmp/test.sh 1 2 
ERROR: Usage  <foo> <bar>
brother ~$ bash /tmp/test.sh 1 2 3 4
ERROR: Usage 4 <foo> <bar>
brother ~$ bash /tmp/test.sh 1 2 3 5 4
ERROR: Usage 5 <foo> <bar>
brother ~$ bash /tmp/test.sh 1 2 3
brother ~$ 
```
