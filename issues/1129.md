#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash

declare -A WEBHOOK_VALUE
UUID=1
WEBHOOK_VALUE["$UUID"]=1

if [[ "WEBHOOK_VALUE[$UUID]" -eq 1 ]]
then
	echo ohno
fi
```

#### Here's what shellcheck currently says:

Line 5:
WEBHOOK_VALUE["$UUID"]=1
^-- SC2034: WEBHOOK_VALUE appears unused. Verify it or export it.

#### Here's what I wanted or expected to see:

The above but also an SC2130 for the line:

```
if [[ "WEBHOOK_VALUE[$UUID]" -eq 1 ]]
```

as the LHS is a string and cannot be a number, the LHS  should be:

```
if [[ "${WEBHOOK_VALUE[$UUID]}" -eq 1 ]]
```

On older versions of shellcheck this used to be caught. version 0.3.7 catches it

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2018-02-28T04:45:01Z](https://github.com/koalaman/shellcheck/issues/1129#issuecomment-369120185):

0.3.7 was wrong. Your code correctly reference the array, and does match.

The bug is that ShellCheck doesn't realize the array is being referenced, because it currently only accepts unindexed variables

#### <img src="https://avatars.githubusercontent.com/u/11960814?v=4" width="50">[evanjardineskinner](https://github.com/evanjardineskinner) commented at [2018-02-28T07:41:35Z](https://github.com/koalaman/shellcheck/issues/1129#issuecomment-369148665):

But the LHS isn't an array reference it is a string literal with a var reference in it to $UUID. To be an array reference it has to have the leading $ and  the curly braces around WEBHOOK_VALUE. See the bash output below:

```
[eskinner@eskinnerp50 ~]$ declare WEBHOOK_VALUE
[eskinner@eskinnerp50 ~]$ UUID=1
[eskinner@eskinnerp50 ~]$ WEBHOOK_VALUE["$UUID"]=1
[eskinner@eskinnerp50 ~]$ echo WEBHOOK_VALUE["$UUID"]
WEBHOOK_VALUE[1]
[eskinner@eskinnerp50 ~]$ echo ${WEBHOOK_VALUE["$UUID"]}
1
```
