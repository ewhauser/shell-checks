# [\#2283 Issue](https://github.com/koalaman/shellcheck/issues/2283) `closed`: Disabling rules on a per-line basis does not work inside array constructs

#### <img src="https://avatars.githubusercontent.com/u/111640?u=59b3d23fbf698052df4185ecadd383b18e4d7a0b&v=4" width="50">[zachriggle](https://github.com/zachriggle) opened issue at [2021-07-17 14:21](https://github.com/koalaman/shellcheck/issues/2283):

#### For bugs
- Rule Id SC2206
- My shellcheck version "online"
- [x] The rule's wiki page does not already cover this (e.g. https://shellcheck.net/wiki/SC2086)
- [x] I tried on https://www.shellcheck.net/ and verified that this is still a problem on the latest commit

#### Here's a snippet or screenshot that shows the problem:

There are no errors fitted if the disable line is moved to just after the shebang, or to before the declaration of the array.

*Broken*
```sh
#!/usr/bin/env bash
true
declare -a array=(
    # shellcheck disable=SC2206
    ${FOO}
)
echo "${array[@]}"
```

*Works*
```sh
#!/usr/bin/env bash
true
# shellcheck disable=SC2206
declare -a array=(
    ${FOO}
)
echo "${array[@]}"
```


#### Here's what shellcheck currently says:

```
$ shellcheck myscript
 
Line 5:
    ${FOO}
    ^-- SC2206: Quote to prevent word splitting/globbing, or split robustly with mapfile or read -a.
```

#### Here's what I wanted or expected to see:

Nothing.

#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2021-07-23 02:11](https://github.com/koalaman/shellcheck/issues/2283#issuecomment-885352546):

ShellCheck actually doesn't have any per-line disabling. All directives are per statement, and not per line. 

This has turned out to be a design mistake, so sorry about that, but as of right now this is intended behavior.
