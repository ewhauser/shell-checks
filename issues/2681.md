# [\#2681 Issue](https://github.com/koalaman/shellcheck/issues/2681) `open`: Incorrect warning about undeclared variable inside trap

#### <img src="https://avatars.githubusercontent.com/u/15802528?u=19dcd5369f39f05c7d6bd8272da2f356a1456902&v=4" width="50">[mikhailnov](https://github.com/mikhailnov) opened issue at [2023-02-02 19:05](https://github.com/koalaman/shellcheck/issues/2681):

Shellcheck incorrectly warns that "i is referenced but not assigned" (SC2154) for the following code:

```bash
#!/bin/bash
trap 'for i in $(tac "$PID_LIST"); do kill $i || : ; kill -9 $i || : ; done; rm -fr "$PID_LIST" "$CHROMIUM_PROFILES_DIR" "$SCROT_TMP" "$CDIR"' EXIT ERR
```

Of course it is a badly written code but the warning is incorrect.

The same code not inside trap is parsed correctly:
```bash
#!/bin/bash
for i in $(tac "$PID_LIST"); do kill $i || : ; kill -9 $i || : ; done; rm -fr "$PID_LIST" "$CHROMIUM_PROFILES_DIR" "$SCROT_TMP" "$CDIR"
```
SC2086 is correctly given.
