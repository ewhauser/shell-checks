# [\#956 Issue](https://github.com/koalaman/shellcheck/issues/956) `open`: `echo "$foo" | ...` should become `... <<< "$foo"` in many cases

#### <img src="https://avatars.githubusercontent.com/u/2199511?v=4" width="50">[Nightfirecat](https://github.com/Nightfirecat) opened issue at [2017-07-20T03:22:04Z](https://github.com/koalaman/shellcheck/issues/956):

#### For new checks and feature suggestions
- [x] shellcheck.net (i.e. the latest commit) currently gives no useful warnings about this
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related


#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash
foo='The quick
brown fox
jumps over
the lazy dog'
echo "$foo" | grep 'a'
```

#### Here's what shellcheck currently says:
(nothing)


#### Here's what I wanted or expected to see:
```
In bar.sh line 6:
echo "$foo" | grep 'a'
              ^-- SC20XX: Pass variable contents through stdin using <<<.
```

---

Of note, this will not apply to all builtins... Not all of them will process input through stdin. `grep` is used as an easy example here that I found sprinkled throughout some of my own scripts.

*Edit:*

Some short timing statistics; `... <<< "$foo"` is notably faster than `echo "$foo" | ...`:

```
$ time for((i=0; i<100; i++)); do foo='The quick brown fox jumps over the lazy dog'; echo "$foo" | grep -q 'a'; done

real    0m18.429s
user    0m3.753s
sys     0m6.458s

$ time for((i=0; i<100; i++)); do foo='The quick brown fox jumps over the lazy dog'; grep -q 'a' <<< "$foo"; done

real    0m12.341s
user    0m2.312s
sys     0m5.049s
```

#### <img src="https://avatars.githubusercontent.com/u/2199511?v=4" width="50">[Nightfirecat](https://github.com/Nightfirecat) commented at [2017-07-26T17:56:58Z](https://github.com/koalaman/shellcheck/issues/956#issuecomment-318133054):

Similarly, `cat file | grep ...` or its variations (`head file | ...`, `tail file | ...`, etc.) would be better as `grep ... < file`.

```
$ echo 'The quick brown fox jumps over the lazy dog' > test.txt
$ time for ((i=0; i<100; i++)); do cat test.txt | grep -q 'a'; done

real    0m20.075s
user    0m5.022s
sys     0m9.404s

$ time for ((i=0; i<100; i++)); do grep -q 'a' < test.txt; done

real    0m12.529s
user    0m2.436s
sys     0m4.319s
```

*Edit:* as @Pawamoy points out, SC2002 already covers useless `cat`s.
