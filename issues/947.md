#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2017-07-08T05:26:58Z](https://github.com/koalaman/shellcheck/issues/947#issuecomment-313835110):

Very confusing. This is definitely a parsing bug. 

To give helpful messages on wrong indenting of the end token, ShellCheck skipped spaces before the token. Unfortunately, the particular function that skipped spaces also skipped comments. Such as `#!`. 

This has now been fixed. Thanks for reporting!

There's still a similar problem with tokens containing leading spaces, but that's for another commit.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2017-07-08T05:29:55Z](https://github.com/koalaman/shellcheck/issues/947#issuecomment-313835234):

Oh, and as a workaround until this fix makes it into a release, you can use any token that doesn't start with `#`. `end` and `eof` are common choices.
