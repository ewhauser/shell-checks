# [\#611 Issue](https://github.com/koalaman/shellcheck/issues/611) `closed`: Warn about missing ;; in case

#### <img src="https://avatars.githubusercontent.com/u/2039950?v=4" width="50">[VanNostrand](https://github.com/VanNostrand) opened issue at [2016-02-19T22:20:19Z](https://github.com/koalaman/shellcheck/issues/611):

if you write this, the bash and shellcheck will not bother:

```
#!/bin/sh
case FOO in
1) ;;
*) 
esac
```

but the syntax is not right due to documentation, where each list is terminated by ;; token, which includes the list of the \* pattern. This is from the bash man page:
`case word in [ [(] pattern [ | pattern ] ... ) list ;; ] ... esac`
I don't know how other shells treat this, but the missing ;; in the code above should be detected by shellcheck.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2016-02-27T18:38:08Z](https://github.com/koalaman/shellcheck/issues/611#issuecomment-189700258):

Looks like this is [explicitly allowed in POSIX](http://pubs.opengroup.org/onlinepubs/9699919799/):

> The compound-list for each list of patterns, with the possible exception of the last, shall be terminated with ";;". 
> The ";;" is optional for the last compound-list.
