# [\#483 Issue](https://github.com/koalaman/shellcheck/issues/483) `open`: style: check for constructs like `if blah; then true; else do something; fi`.

#### <img src="https://avatars.githubusercontent.com/u/6459309?v=4" width="50">[Artoria2e5](https://github.com/Artoria2e5) opened issue at [2015-09-30T13:46:46Z](https://github.com/koalaman/shellcheck/issues/483):

Some scripters, especially newbies can forget about using `!` to reverse the boolean status of a command, resulting in code like this:

``` sh
if dpkg_installed_check mpv; then
  true # or possibly :
else
  do_apt_install mpv
fi || die "Cannot get a working mpv package"
```

Per POSIX:

> The **if** compound-list shall be executed; if its exit status is zero, the **then** compound-list shall be executed and the command shall complete. Otherwise, each **elif** compound-list shall be executed, in turn, and if its exit status is zero, the **then** compound-list shall be executed and the command shall complete. Otherwise, the **else** compound-list shall be executed.
> 
> The exit status of the **if** command shall be the exit status of the then or **else** compound-list that was executed, _or zero, if none was executed_.

So changing it to

``` sh
if ! dpkg_installed_check mpv; then
  do_apt_install mpv
fi || die "Cannot get a working mpv package"
```

Or even `dpkg_installed_check mpv || do_apt_install mpv || die "Cannot get a working mpv package"` doesn't hurt, unless `$?` or `PIPESTATUS` is used in the `else` clause (where boolean `!` does destroy the information the scripter wants).
