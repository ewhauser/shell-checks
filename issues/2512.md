I couldn't find anything similar other than #2196 which is a "global" toggle, and not what I'm referring to here.

### Problem
In many static analysers you can do this
````c
// disable 1234
var something = that.Causes(theError);        // error is not reported
and.this.AlsoCausesTheError();                // error is not reported
// enable 1234
foo.causeErrorAgain();                        // error is reported
````

So we can disable a rule for one or more lines, then enable it again for the rest of the file.

### Feature Request
Please consider something like this for shellcheck:
````sh
# shellcheck disable=SC1083
...
# shellcheck enable=SC1083
````

#### <img src="https://avatars.githubusercontent.com/u/1111474?v=4" width="50">[leagris](https://github.com/leagris) commented at [2022-06-07 06:32](https://github.com/koalaman/shellcheck/issues/2512#issuecomment-1148251242):

Alternatively put the SC disable commands into a commands block that shares the same shellcheck disabled entries:

```sh
# shellcheck disable=SCxxx
{
  var something = that.Causes(theError);  // error is not reported
  and.this.AlsoCausesTheError();                // error is not reported
}
foo.causeErrorAgain();                               // error is reported
``

#### <img src="https://avatars.githubusercontent.com/u/140865?v=4" width="50">[prahal](https://github.com/prahal) commented at [2025-10-26 02:26](https://github.com/koalaman/shellcheck/issues/2512#issuecomment-3447948898):

Note that disable does not disable a rule per https://github.com/koalaman/shellcheck/wiki/Directive#disable, it only silences the error, which means parsing stops at the same point in the file. Therefore, enabling the error again after the point where it was silenced is not possible, because parsing has stopped.
