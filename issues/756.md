# [\#756 Issue](https://github.com/koalaman/shellcheck/issues/756) `closed`: SC2034 on arrays used only passed as arguments

#### <img src="https://avatars.githubusercontent.com/u/5803348?v=4" width="50">[OscarAkaElvis](https://github.com/OscarAkaElvis) opened issue at [2016-10-23T13:22:51Z](https://github.com/koalaman/shellcheck/issues/756):

Hi, I have this testing code (an array and a function):

```
#!/bin/bash
test_array=("a" "b")

function testing() {
    declare -a array_for_loop=("${!1}")
    for i in  "${array_for_loop[@]}"; do
        echo "${i}"
    done
}
```

Desired output is the echo of "a" and "b"

Different types of invoking and their output:

`testing test_array[@]`
# It works but shellcheck shows SC2034 warning

`testing "${test_array[@]}"`
# No SC2034 warning but it doesn't work

`testing test_array[@] "${test_array[@]}"`
# Dirty trick. It works and no warning... Not too much elegant because the second argument is not used.... is only to silent the warning :/

Is this a bug? or how can I pass an array as argument avoiding the warning? Thank you.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2017-09-09T23:41:08Z](https://github.com/koalaman/shellcheck/issues/756#issuecomment-328310010):

ShellCheck doesn't try to figure out whether or not a variable is used through indirection. This is generally nondeterministic.

In this case, the better way to do it is to pass the array as multiple arguments, and then loop over the arguments:

```
#!/bin/bash
test_array=("a" "b")

function testing() {
    for i
    do
        echo "${i}"
    done
}

testing "${test_array[@]}"
```

If you have variables that you know will be used but only through indirection, ignore this warning.
