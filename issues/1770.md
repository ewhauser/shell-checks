- Rule Id (if any, e.g. SC1000): SC2120
- My shellcheck version (`shellcheck --version` or 'online'): online
- [x] I tried on shellcheck.net and verified that this is still a problem on the latest commit

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/sh

# shellcheck disable=SC2120
f() {
        echo "${1:-foo}"
}

g() {
        echo "$1"
}

h() {
        f
        g
}
```

#### Here's what shellcheck currently says:
Nothing
#### Here's what I wanted or expected to see:
Report no error for `f` invocation but report an error for `g` invocation.


#### <img src="https://avatars.githubusercontent.com/u/7482557?v=4" width="50">[Gandalf-](https://github.com/Gandalf-) commented at [2019-12-19T00:43:07Z](https://github.com/koalaman/shellcheck/issues/1770#issuecomment-567277208):

Shellcheck is applying your first directive to the first file - it can't tell the difference here.

If you put another statement before your function declaration, this works. It's a work around, but maybe acceptable?

For instance
```shell
#!/bin/sh

: # the colon command has no effect here

# shellcheck disable=SC2120
f() {
  echo "${1:-foo}"
}

g() {
  echo "$1"
}

h() {
  f
  g
}
```
