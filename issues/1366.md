#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2018-11-01T13:11:03Z](https://github.com/koalaman/shellcheck/issues/1366#issuecomment-435036147):

You're right of course. You can work around this limitation with the `function` keyword:

    function ===() { :;}

ShellCheck intentionally allows a wider variety of symbols in function names when using `function` (specifically `[]*=!`).

This design decision was made because bash considers e.g. `if ![var=()` to be the start of a function definition, but it would be confusing and misleading for ShellCheck to talk about missing function bodies and such, and trial runs on a corpus of bad scripts supported this.
