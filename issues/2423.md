# [\#2423 Issue](https://github.com/koalaman/shellcheck/issues/2423) `open`: SC2012 should ignore filenames, which can't contain question marks

#### <img src="https://avatars.githubusercontent.com/u/10757176?v=4" width="50">[mgutt](https://github.com/mgutt) opened issue at [2021-12-28 12:13](https://github.com/koalaman/shellcheck/issues/2423):

The [SC2012](https://github.com/koalaman/shellcheck/wiki/SC2012) is perfectly fine if a filename could contain a control character, but in the following example this isn't possible as it allows only alphanumeric characters:

```
most_recent_syslog=$(ls -t /var/log/syslog{.[0-9],} 2>/dev/null | head -n1)
```
or:
```
most_recent_syslog=$(ls -t /var/log/syslog /var/log/syslog.[0-9] 2>/dev/null | head -n1)
```


Shellcheck should only return a warning if a filename returned by `ls` could contain a question mark as this is used as a replacement for a control character:
https://ftp.gnu.org/old-gnu/Manuals/fileutils-4.1/html_node/fileutils_27.html
>control characters are output as question marks
