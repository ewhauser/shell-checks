# [\#1636 Issue](https://github.com/koalaman/shellcheck/issues/1636) `closed`: SC2128 is not triggered with read -ra stringarray <<<"${linevariable}"; Expanding an array without an index only gives the first element

#### <img src="https://avatars.githubusercontent.com/u/5332158?v=4" width="50">[evandrocoan](https://github.com/evandrocoan) opened issue at [2019-07-04T02:59:16Z](https://github.com/koalaman/shellcheck/issues/1636):

This code should trigger `SC2128: Expanding an array without an index only gives the first element`
```sh
#!/bin/bash
linevariable="var1     var2";
read -ra stringarray <<<"${linevariable}";
printf "first %s\\n" "${stringarray[0]}"
printf "second %s\\n" "${stringarray[1]}"

stringarray="${stringarray}"
printf "stringarray %s\\n" "${stringarray[0]}"
```

But only this triggers it:
```sh
#!/bin/bash
linevariable="var1     var2";
stringarray=(${linevariable});
printf "first %s\\n" "${stringarray[0]}"
printf "second %s\\n" "${stringarray[1]}"

stringarray="${stringarray}"
printf "stringarray %s\\n" "${stringarray[0]}"
```

Related to:
1. https://github.com/koalaman/shellcheck/issues/1630 SC2178 indefinitely triggers SC2128 when it should not, i.e., after overriding an array variable with a single string

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2019-07-04T03:41:31Z](https://github.com/koalaman/shellcheck/issues/1636#issuecomment-508326560):

So it should! This failed because ShellCheck only looked for `-a` without accounting for things like `-ra`. This has now been fixed, thanks!
