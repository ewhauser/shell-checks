#### <img src="https://avatars.githubusercontent.com/u/2491070?v=4" width="50">[nthexwn](https://github.com/nthexwn) commented at [2019-06-18T08:16:41Z](https://github.com/koalaman/shellcheck/issues/1127#issuecomment-502998755):

I inadvertently triggered [SC1073](https://github.com/koalaman/shellcheck/wiki/SC1073), [SC1072](https://github.com/koalaman/shellcheck/wiki/SC1072), and [SC1009](https://github.com/koalaman/shellcheck/wiki/SC1009) by placing a directive in the following code:

    if [[ ! -z "$1" ]]; then
        log_level="$1"
    # LOG_LEVEL is an optional external configuration, so it's okay for it to be unset.
    # shellcheck disable=SC2153
    elif [[ ! -z "$LOG_LEVEL" ]]; then
        log_level="$LOG_LEVEL"
    else
        log_level="info"
    fi

I believe it would have been more helpful to have [SC1123](https://github.com/koalaman/shellcheck/wiki/SC1123) shown here.  After seeing its description in the OP's output I was able to resolve this by moving the comments in my code to before the whole 'if' statement:

    # LOG_LEVEL is an optional external configuration, so it's okay for it to be unset.
    # shellcheck disable=SC2153
    if [[ ! -z "$1" ]]; then
        log_level="$1"
    elif [[ ! -z "$LOG_LEVEL" ]]; then
        log_level="$LOG_LEVEL"
    else
        log_level="info"
    fi

It would be great if Shellcheck could parse directives at arbitrary points within commands, but without diving into the parsing engine I can only conjecture about how difficult this might be.
