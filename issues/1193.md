#### For bugs
- Rule Id: SC2034
- My shellcheck version: online (0.4.6 locally)
- [X] I tried on shellcheck.net and verified that this is still a problem on the latest commit

#### For new checks and feature suggestions
- [ ] shellcheck.net (i.e. the latest commit) currently gives no useful warnings about this
- [X] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related


#### Here's a snippet or screenshot that shows the problem:

```sh

#!/bin/sh
# shellcheck disable=2162
read up _idle </proc/uptime
echo "system up $up"

```

#### Here's what shellcheck currently says:
    $ shellcheck myscript
     
    Line 3:
    read up _idle </proc/uptime
           ^-- SC2034: _idle appears unused. Verify it or export it.

#### Here's what I wanted or expected to see:
Naming the variable with a '_' prefix in many linters is a way of acknowledging it is known as unused.  As in this example, giving it the name _idle indicates what the second field is, and thus provides value to the developer.

Using a single `_` does work.
