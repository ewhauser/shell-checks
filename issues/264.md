I find most of SC2086 comments to be false postive. For instance:

``` bash
blue="\033[34m"
normal="\033[0m"

echo -e $blue" This is a demo!"$normal
```

**$blue** and **$none** will be flagged under **SC2086** comments. I think they will be problematic if the variable had some special characters like -  
Otherwise, they should be just fine.

Any thought?

#### <img src="https://avatars.githubusercontent.com/u/442919?v=4" width="50">[dualbus](https://github.com/dualbus) commented at [2014-12-04T17:56:14Z](https://github.com/koalaman/shellcheck/issues/264#issuecomment-65673426):

The recommendation is to always quote. Whether you think it's necessary or not.
Not only it will make your code more resitant in the case of future changes
(you don't want that a change in a variable's value break your code 500 lines
below), but it's also a _very minor_ speed improvement.

They're not charging you for the quotes, so use them ;)
