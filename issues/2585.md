- [X] I tried on https://www.shellcheck.net/ and verified that this is still a problem on the latest commit

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/sh

grep -A 1000 -c pattern file | wc -l
```

#### Here's what shellcheck currently says:

```
Consider using grep -c instead of grep|wc -l.
```

#### Here's what I wanted or expected to see:

In this case, ```wc``` is not counting the matches lines but also the (max 1000) lines following the match. The same applies for ```-B``` and ```-C```.

The warning should be suppressed if ```grep``` is called with ```-A```, ```-B``` or ```-C```.
