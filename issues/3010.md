# [\#3010 Issue](https://github.com/koalaman/shellcheck/issues/3010) `open`: Shellcheck does not work with big shellscript file, appears to hang?

#### <img src="https://avatars.githubusercontent.com/u/13236774?v=4" width="50">[JannemanDev](https://github.com/JannemanDev) opened issue at [2024-06-28 11:49](https://github.com/koalaman/shellcheck/issues/3010):

I am trying to check a big shellscript ~5200 lines. You can find the script here: https://github.com/dlt-green/node-installer-docker/blob/main/node-installer.sh

I installed the latest version 0.10.0 on Ubuntu server using:
```
scversion="stable" # or "v0.4.7", or "latest"
wget -qO- "https://github.com/koalaman/shellcheck/releases/download/${scversion?}/shellcheck-${scversion?}.linux.aarch64.tar.xz" | tar -xJv
cp "shellcheck-${scversion}/shellcheck" /usr/bin/
shellcheck --version
```

That works. I can also run small test shellscripts.

But when running `curl -sL "https://raw.githubusercontent.com/dlt-green/node-installer-docker/main/node-installer.sh" | shellcheck -`
it appears if nothing is happening. Even waiting for several minutes I see no output and I have to press CTRL-C which takes a few seconds btw.

It does seem to run but a loading/progress indicator would be nice.
![image](https://github.com/koalaman/shellcheck/assets/13236774/a2793db0-7751-4214-9294-de653fde12fb)

In the end it got killed:
```console
jan@odroid-m1-2:~/dlt.green$ ./check_node_installer.sh
./check_node_installer.sh: line 1: 3900354 Done                    curl -sL "https://raw.githubusercontent.com/dlt-green/node-installer-docker/main/node-installer.sh"
     3900355 Killed                  | shellcheck -
```

The same script under VS Code using the ShellCheck extension, which I presume uses the same library, works perfect and takes about 50 seconds to show the hints, warnings and errors.
