- [ X ] https://www.shellcheck.net/ (i.e. the latest commit) currently gives no useful warnings about this

#### Here's a snippet or screenshot that shows the problem:

I found this in some code:

```sh
cat file1 > file2
```

Wouldn't it be a nice feature if shellcheck recommended this instead:

```sh
cp file1 file2
```

#### Here's what shellcheck currently says:

Nothing

#### Here's what I wanted or expected to see:

Don't use 'cat' to copy files. Use cp instead:
```sh
cp file1 file2
```


#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2023-07-31 03:49](https://github.com/koalaman/shellcheck/issues/2792#issuecomment-1657481171):

I see what you mean, this would look odd when a `cp` would do. There's no real harm done though, and redirecting `cat` has some potentially beneficial differences when it comes to propagating permissions, handling directories, bash special filenames, and the noclobber shell option, so a blanket policy doesn't seem worthwhile. Thanks for the suggestion!
