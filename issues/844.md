# [\#844 Issue](https://github.com/koalaman/shellcheck/issues/844) `closed`: add a warning when people put comments above the shebang

#### <img src="https://avatars.githubusercontent.com/u/176950?v=4" width="50">[vapier](https://github.com/vapier) opened issue at [2017-03-07T01:07:04Z](https://github.com/koalaman/shellcheck/issues/844):

Sometimes people don't realize the shebang must be the first line and think it's OK to put comments before it.  Especially when they test it out by running `./foo.sh` and it still works (due to default script behavior in the kernel exec paths).  It would be nice if we warned about this explicitly.

By default, `SC2148` will fire with a warning like:
`Tips depend on target shell and yours is unknown. Add a shebang.`

But if you're already passing `--shell=bash`, that warning is omitted, and we get no warnings at all.

#### Here's a snippet or screenshot that shows the problem:

```sh
# Some comment
#!/bin/bash
```

To avoid false positives, i think we should warn only if comments precede the shebang.  i.e. this one should not warn:
```sh
# Some comment
some_code=''
#!/bin/bash
```

#### Here's what I wanted or expected to see:

`Shebang must be the first line`
