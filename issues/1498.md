#### <img src="https://avatars.githubusercontent.com/u/1857647?v=4" width="50">[FrankHB](https://github.com/FrankHB) commented at [2023-09-20T02:19:43Z](https://github.com/koalaman/shellcheck/issues/1498#issuecomment-1726782638):

As a note, I'd add a bash-specific use case here.

Consider `: {var:=$(command)}` to initialize `var` if not set or empty, since `:` always has a zero exit status, `set -e` will not cause the shell to exit on failure of `command`. This may be quite unexpected. A workaround is `var={var:-$(command)}`, but this is verbose when `var` is a long name. Then there's the workaround: `_={var:=$(command)}`. The special variable `_` in bash is the last argument of the last command, so if the command succeeds, the meaning is perfectly compatible (to one-argument cases of `:` above).

I find it by myself and don't know whether this is an established idiom, but it seems fit to the wiki page to mention. However, there is another issue: assignment to `_` may lead to shell-specific behavior. @koalaman, is this also worth mentioning in the wiki page, or a standalone warning?
