#### <img src="https://avatars.githubusercontent.com/u/2199511?v=4" width="50">[Nightfirecat](https://github.com/Nightfirecat) commented at [2017-06-22T21:34:22Z](https://github.com/koalaman/shellcheck/issues/940#issuecomment-310509157):

I believe it's because that is improper formatting of a conditional... In bash, I find that both of the following commands will not pass the conditional: (nothing is printed to stdout here)

```
if [ ! id -u 2> /dev/null -eq 0 ]; then
  echo 'true'
fi
if [ id -u 2> /dev/null -eq 0 ]; then
  echo 'true'
fi
```

That said, the following passes Shellcheck validation without warning and works correctly:

```
#!/bin/bash
if [ ! "$(id -u 2> /dev/null)" -eq 0 ]; then
  echo "error: this script must be run as root"
  exit 1
fi
```

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2017-06-23T01:20:05Z](https://github.com/koalaman/shellcheck/issues/940#issuecomment-310544608):

This is invalid syntax in both bash and ShellCheck.

In bash, the error message you get is squashed due to the `2> /dev/null` that applies to `[` and not `id`: 

```
$ [ ! id -u -eq 0 ]
bash: [: too many arguments
```

Fortunately ShellCheck is not so easily fooled :P

You can use `$(id -u)` like @Nightfirecat suggests. Since `id -u` shouldn't be giving errors in the first place, you can leave stderr and just use `[ "$(id -u)" -ne 0 ]` to avoid hiding any other relevant errors.
