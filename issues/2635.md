#### Description

When the `$` is escaped with `\$` it should not be considered as a parameter.

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/sh

alias a="awk '{print \$0}'"
```

#### Here's what shellcheck currently says:

```
In example.sh line 3:
alias a="awk '{print \$0}'"
      ^-------------------^ SC2142 (error): Aliases can't use positional parameters. Use a function.

For more information:
  https://www.shellcheck.net/wiki/SC2142 -- Aliases can't use positional para...
```

#### Here's what I wanted or expected to see:

No errors.
