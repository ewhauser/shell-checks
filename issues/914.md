#### <img src="https://avatars.githubusercontent.com/u/197253?v=4" width="50">[kurahaupo](https://github.com/kurahaupo) commented at [2025-03-12T03:48:53Z](https://github.com/koalaman/shellcheck/issues/914#issuecomment-2716352469):

@kwshi 

> As a side note, when is `[]` square brackets used for glob patterns, anyway?

The short answer is "always, except when there's an defined exception".

Right up front it's worth noting that if a glob expansion does not match anything, it will be left unchanged, so it might _seem_ like glob expansions only happen occasionally.

The exceptions include:

* On the right hand side of a bare assignment
* On the right hand side of an assignment that's an argument to an unquoted _special_ built-in command
  * `declare`
  * `export`
  * `local`
  * `readonly`
* In the selector word between `case` and `in`
* Inside `((`...`))`
* Inside `[[`...`]]` other than on the right side of `=` or `==` or `=~`

(and not quoted, of course).

Compare these two uses of the `declare` built-in command:
```
$ touch a=b0
$ declare a=b[0] ; echo "[$a]"    # no globbing because declare is special
[b[0]]
$ 'declare' a=b[0] ; echo "[$a]"  # globbing because 'declare' is not special
[b0]
```

#### <img src="https://avatars.githubusercontent.com/u/197253?v=4" width="50">[kurahaupo](https://github.com/kurahaupo) commented at [2026-02-23T03:20:35Z](https://github.com/koalaman/shellcheck/issues/914#issuecomment-3942346494):

@LLyaudet just to be clear, the `-g` option on `declare` affects _only_ that statement. It does not affect uses of `bar` elsewhere in the function, unless it happens to refer to the top-level global variable. Be aware that `local` creates a scope that extends into any subsidiary functions; it's _not_ limited to just the current function. Also be aware that `-i` only affects assignment, not expansion. If a variable already has a non-numeric value, setting `-i` on it won't change its value.

This combination is demonstrated when you run:
```
#!/dont/use/env/./change/this/line/to/the/path/to/your/bash
 
# shellcheck enable=all
 
foo() {
  declare -gi bar
  bar=7+$1*4
  export bar
  echo "$FUNCNAME:$LINENO:bar=$bar"
  echo "$FUNCNAME:$LINENO:((bar))=$((bar))"
}
 
zot() {
  local bar
  foo 40+2
  echo "$FUNCNAME:$LINENO:bar=$bar"
  echo "$FUNCNAME:$LINENO:((bar))=$((bar))"
}
 
bar=Hello
echo "main:$LINENO:bar=$bar"
zot
echo "main:$LINENO:bar=$bar"
bar=9*3
echo "main:$LINENO:bar=$bar"
```
and get output that looks like:
```
main:21:bar=Hello
foo:9:bar=7+40+2*4
foo:10:((bar))=55
zot:16:bar=7+40+2*4
zot:17:((bar))=55
main:23:bar=Hello
main:25:bar=27
```
