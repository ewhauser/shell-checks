I read the shebang as a lowest common denominator. When it is `sh` it is supposed to be working with anything that is fully POSIX shell - dash, bash and others. If the authir state `bash` I would not count on that it would work when executed with busybox sh or dash and such.

Read the shebang as a recommendation from the author and a hint about how protable the script is supposed to be rather than what shell interpreter you must use.

If you move a fedora sh tagged script into my debian system it will be executed with dash, and probably be fine with regards to shell syntax - other stuff would probably break.

> I read the shebang as a lowest common denominator. When it is sh it is supposed to be working with anything that is fully POSIX shell - dash, bash and others. 

Exactly that. If a script requires bash then it should start with `#!/bin/bash` or `#!/usr/bin/env bash`. Starting a script with `#!/bin/sh` and using bashisms in that script is a bug, don't do that. If Fedora does that then it's a bug in Fedora. You already found the perfect workaround: `shellcheck -s `

By the way bash behaves slightly differently when invoked with `#!/bin/sh`, see `info bash`.
