#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash

HOSTS=(a b c)
declare -n RESOURCES=HOSTS
RESOURCES_SELECT=()

get_max_width "${RESOURCES[@]}"
RESOURCES+=(d)

if [[ " ${RESOURCES[*]} " == *"$1 "* ]] ; then
	# We filter the resource selection
	RESOURCES_SELECT+=("$1")
fi

if [ "${#RESOURCES_SELECT[@]}" -ne 0 ] ; then
	unset -n RESOURCES
	declare -n RESOURCES=RESOURCES_SELECT
fi

```

#### Here's what shellcheck currently says:

```
Line 16:
        declare -n RESOURCES=RESOURCES_SELECT
                   ^-- SC2178: Variable was used as an array but is now assigned a string.
```

#### Here's what I wanted or expected to see:

Nothing. `declare -n` sets an alias. The underlying variable, on both cases, is an array - so the alias remains an array too
