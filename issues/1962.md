#### Here's a snippet or screenshot that shows the problem:

```sh

#!/bin/bash
myfolders="A B"
find ${myfolders[@]} type -f 

```

#### Here's what shellcheck currently says:
Double quote to prevent globbing and word splitting. [SC2086]


#### Here's what I wanted or expected to see:
Nothing

The problem is that if we double quote the variable myfolders, then find does not interpret the folders as multiple folders and says: 
find: ‘A B’: No such file or directory


PS: the same is true for the command tar

#### <img src="https://avatars.githubusercontent.com/u/8713416?u=58f5c2aee89a2c67485eed4e4f380f59a037ec2a&v=4" width="50">[furkanpham](https://github.com/furkanpham) commented at [2020-05-27 09:08](https://github.com/koalaman/shellcheck/issues/1962#issuecomment-634531592):

`"${myfolders[@]}"` expands to a list of words, with each element in the myfolders array as one word. The reason why it does not interpret "A B" as two seperate folders is because `myfolders` isn't an array but a string, so there's nothing to expand since only one element exists. Creating an array using `=()` syntax will work as intended.
```bash
#!/bin/bash
myfolders=("A" "B" "C D")
find "${myfolders[@]}" -type f
```
This also makes it possible to include directories with whitespace (like `C D`) without relying on word splitting.
