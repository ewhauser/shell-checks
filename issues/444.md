# [\#444 Issue](https://github.com/koalaman/shellcheck/issues/444) `closed`: bogus SC1068 for non-assignment

#### <img src="https://avatars.githubusercontent.com/u/278465?v=4" width="50">[mauke](https://github.com/mauke) opened issue at [2015-08-21T18:10:37Z](https://github.com/koalaman/shellcheck/issues/444):

```
echo ===
```

produces

```
 1  echo ===
    ^––SC2034 echo appears unused. Verify it or export it.
         ^––SC1068 Don't put spaces around the = in assignments.
```

But the code is fine and simply outputs `===`. There are no variables involved.

Yes indeed. From d0029ae, ShellCheck will warn about using two equal signs, and ignore three or more assuming it's some kind of ascii art line.
