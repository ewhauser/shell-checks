#### <img src="https://avatars.githubusercontent.com/u/13442533?u=a08190713570b091ab61a8497e6ff9090dc66680&v=4" width="50">[fwdekker](https://github.com/fwdekker) commented at [2025-02-15 12:05](https://github.com/koalaman/shellcheck/issues/2937#issuecomment-2660891195):

The issue seems to occur when the assignment and subsequent usage of `$?` are interrupted by a statement that alters `$?`.

For example, the following gives the warning SC2319 on line 2:
```shell
test "$1" != "$2"
foo="$?"
bar="$(cat "bar.txt")"
echo "$foo" "$bar"
```

but the following does not:
```shell
test "$1" != "$2"
foo="$?"
# bar="$(cat "bar.txt")"
echo "$foo" "$bar"
```

even though in both cases, the value of `$?` is not statically predictable (because `$1` and `$2` are unknown) and the output is properly fixed in `$foo`.
