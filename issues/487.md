# [\#487 Issue](https://github.com/koalaman/shellcheck/issues/487) `closed`: No warning for indirect variable expansion when using POSIX sh

#### <img src="https://avatars.githubusercontent.com/u/2095665?v=4" width="50">[markgollnick](https://github.com/markgollnick) opened issue at [2015-10-02T00:49:07Z](https://github.com/koalaman/shellcheck/issues/487):

###### Bug Description:

shellcheck produces no warning for [indirect variable expansion](https://www.gnu.org/software/bash/manual/bashref.html#Shell-Parameter-Expansion) when using the `#!/bin/sh` shebang, even though indirect variable expansion is functionality that is [not defined for POSIX sh](http://pubs.opengroup.org/onlinepubs/9699919799/utilities/V3_chap02.html#tag_18_06_02).
###### Bug Reproduction:

`test.sh`:

``` bash
#!/bin/sh
export ULTIMATE="hello world"
export PENULTIMATE="ULTIMATE"
echo "${!PENULTIMATE}"
```

When run through bash in sh mode, the above script produces:

```
$ ./test.sh
hello world
```

**BUG:** When run through shellcheck, everything returns clean:

```
$ shellcheck test.sh
$ echo $?
0
```

**EXPECTATION:** When run through shellcheck, I would expect to receive an error (or at least a warning) due to the `#!/bin/sh` shebang, similar to the SC2039 warning that is given when a user attempts to use arrays despite specifying a shebang of `#!/bin/sh`, e.g.:

`test.sh`:

``` bash
#!/bin/sh
ARGV=("one" "two" "three")
for x in "${ARGV[@]}"; do
  echo "$x"
done
```

```
$ ./test.sh
one
two
three
$ shellcheck test.sh

In test.sh line 2:
ARGV=("one" "two" "three")
     ^-- SC2039: In POSIX sh, arrays are not supported.


In test.sh line 3:
for x in "${ARGV[@]}"; do
          ^-- SC2039: In POSIX sh, array references are not supported.

$ echo $?
1
```
###### Environment/Versions:
- shellcheck v4.0.1
