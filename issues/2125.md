#### More details

Shellcheck does not emit warnings for the non-POSIX operators `-nt`, `-ef`, `-k`, `-o`, `<`, and `>` when using a `#!/bin/sh` shebang or `--shell sh`.

Shellcheck already emits warnings for non-POSIX syntax ([example 1](https://github.com/koalaman/shellcheck/wiki/SC2039), [example 2](https://github.com/koalaman/shellcheck/wiki/SC3026)). It would be useful if shellcheck could also warn for these non-POSIX operators, when the `sh` shell is specified.
