```bash
#!/usr/bin/env bash
tagApiPaths=(
        blah
        blah
        blah
)
echo "DEBUG" $'tagApiPaths \n'"${tagApiPaths[@]}"
```
adding a space like so:
```
#!/usr/bin/env bash
tagApiPaths=(
        blah
        blah
        blah
)
echo "DEBUG" $'tagApiPaths \n' "${tagApiPaths[@]}"
```
makes the error message go away but it adds an undesirable space in front of the first entry of the array.

#### Here's what shellcheck currently says:
Argument mixes string and array. Use * or separate argument. [SC2145] 


#### Here's what I wanted or expected to see:
Nothing, there is nothing wrong with the line.
