# [\#688 Issue](https://github.com/koalaman/shellcheck/issues/688) `closed`: SC2154 Doesn't Parse Function Names Containing Colons Correctly

#### <img src="https://avatars.githubusercontent.com/u/7328370?v=4" width="50">[stevekuznetsov](https://github.com/stevekuznetsov) opened issue at [2016-06-15T17:25:38Z](https://github.com/koalaman/shellcheck/issues/688):

The following code is incorrectly flagged with SC2154:

``` sh
$ cat /tmp/exmaple.sh 
#!/bin/bash
function namespace::funcname() {
    echo 'Hello World!'
}
readonly -f namespace::funcname

$ shellcheck /tmp/exmaple.sh 

In /tmp/exmaple.sh line 5:
readonly -f namespace::funcname
            ^-- SC2154: namespace::funcname is referenced but not assigned.

```

@koalaman any thoughts on this?


#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2016-09-24T21:00:52Z](https://github.com/koalaman/shellcheck/issues/688#issuecomment-249387374):

This is indeed a bug, `readonly -f` shouldn't concern variables at all. It's been fixed in 3a7dc86. Thanks!
