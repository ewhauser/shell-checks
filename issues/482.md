# [\#482 Issue](https://github.com/koalaman/shellcheck/issues/482) `open`: oldbash: suggest `case "$foo" in (pat)` instead `pat)` in `$()` when shebang is bash

#### <img src="https://avatars.githubusercontent.com/u/6459309?v=4" width="50">[Artoria2e5](https://github.com/Artoria2e5) opened issue at [2015-09-30T13:27:43Z](https://github.com/koalaman/shellcheck/issues/482):

In old versions of `bash`, like the still popular 3.1.x, the following code throws a syntax error because some mishandling of precedence:

``` Bash
echo "$(case "$f" in a) printf b;; *) printf "$f";; esac)"
```

A workaround looks like this:

``` Bash
echo "$(case "$f" in (a) printf b;; (*) printf "$f";; esac)"
```

This is documented in http://wiki.bash-hackers.org/syntax/expansion/cmdsubst. I remember seeing such discussion in bug-bash, but forgot the exact title. Process substitution seems to suffer from some similar bug, too.

I am not sure how you would want to do with some popular old version bugs. A command line switch or some directive near script header may be nice.
