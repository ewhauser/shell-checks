# [\#1161 Issue](https://github.com/koalaman/shellcheck/issues/1161) `closed`: SC2016 false positive (probably a legit one)

#### <img src="https://avatars.githubusercontent.com/u/135563?v=4" width="50">[dereckson](https://github.com/dereckson) opened issue at [2018-03-27T01:34:04Z](https://github.com/koalaman/shellcheck/issues/1161):

With ShellCheck 0.4.7 or online, the following code triggers SC2016:

```sh
#!/bin/sh
docker run -it --rm --link "$MYSQL_CONTAINER:mysql" "$MYSQL_IMAGE" \
    sh -c 'exec mysql -h"$MYSQL_PORT_3306_TCP_ADDR" ...'
```

Yet, the code intent is to pass as is the expression to sh,
as the environment variables aren't available in the host
but are available in the Docker container running sh.

But I've no idea what's the intent of most people using `sh -c`,
so it's probably best to keep the advice to ask double check.

#### <img src="https://avatars.githubusercontent.com/u/2199511?v=4" width="50">[Nightfirecat](https://github.com/Nightfirecat) commented at [2018-03-27T06:44:44Z](https://github.com/koalaman/shellcheck/issues/1161#issuecomment-376415114):

From [SC2016's wiki page](/koalaman/shellcheck/wiki/SC2016):

> **Exceptions**
> 
> If you want `$stuff` to be a literal dollar sign followed by the characters "stuff", you can [ignore](https://github.com/koalaman/shellcheck/wiki/Ignore) this message.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2018-03-30T02:30:37Z](https://github.com/koalaman/shellcheck/issues/1161#issuecomment-377428393):

`sudo` is already blacklisted for this warning because you frequently need to expand variables in a different context. `docker` should probably be blacklisted for the same reason.

And done.  Thanks!
