#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2015-10-11T17:29:19Z](https://github.com/koalaman/shellcheck/issues/470#issuecomment-147223322):

For a precommit, you can do this today with `shellcheck -f json files... | jq '.[].level' | grep error`

In the near future, shellcheck is likely to get a verbosity option where you can choose from "nitpick on any minor potential issue" to "only if I'm about to [steam someone](https://github.com/ValveSoftware/steam-for-linux/issue%73/%33671#asdf)".


#### <img src="https://avatars.githubusercontent.com/u/1381995?v=4" width="50">[mulle-nat](https://github.com/mulle-nat) commented at [2015-10-11T18:19:08Z](https://github.com/koalaman/shellcheck/issues/470#issuecomment-147232604):

Nice, thx for the idea with json.

I tried it in real life, result: it works; but now I don't see the reason WHY it failed. So I have to rerun it, which is clumsy IMO. 

In a Makefile I "compile" my shell scripts with a rule like this now:

``` Makefile
%.chk:  %.sh
        - shellcheck $(SHELLFLAGS) $<
        (shellcheck -f json $(SHELLFLAGS) $< | jq '.[].level' | grep error > /dev/null ) && exit 1 || touch $@
```
