#!/bin/bash

#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2021-08-27 00:27](https://github.com/koalaman/shellcheck/issues/2310#issuecomment-906831962):

You're right, this is a false positive because you want the backticks literally in the string. You can either ignore it with a directive, or make sure to escape the backticks so they don't expand:

```
#!/bin/bash
aws logs get-log-events \
    --query "join(\`\"\n\"\`, events[].message)" \
    --output text

# shellcheck disable=SC2016
aws logs get-log-events \
    --query 'join(`"\n"`, events[].message)' \
    --output text
```
