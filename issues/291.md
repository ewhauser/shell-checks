Hey,
for this script one comment is reported:

> SC2053 Quote the rhs of = in [[ ]] to prevent glob interpretation.

``` bash
#!/bin/bash
string1="te st"
string2="te st"
if [[ ${string1} == "te st" ]]; then
  echo "pattern match!"
fi

if [[ ${string1} == ${string2} ]]; then
  echo "pattern match!"
fi
```

But http://mywiki.wooledge.org/BashFAQ/031 says: 

> No WordSplitting or glob expansion will be done for [[ (and therefore many arguments need not be quoted)

..Is the wiki wrong?

#### <img src="https://avatars.githubusercontent.com/u/442919?v=4" width="50">[dualbus](https://github.com/dualbus) commented at [2015-01-21T15:33:45Z](https://github.com/koalaman/shellcheck/issues/291#issuecomment-70858263):

> But http://mywiki.wooledge.org/BashFAQ/031 says: 
> 
> > No WordSplitting or glob expansion will be done for [[ (and therefore many arguments need not be quoted)
> 
> ..Is the wiki wrong?
> wiki.wooledge.org is a pretty good source of information, but... it's a wiki.
> Sometimes idiots add useless information to it. The wiki is wrong. There are
> certain cases when quoting matters inside [[

case #1:
 [[ LHS = RHS ]]
          ^- if you don't quote this, it will try to do pattern matching (with
             globs).

|  dualbus@dualbus ~ % bash -c 'string1=yolo string2=_; [[ $string1 = $string2 ]]; echo $?'
|  0
|  dualbus@dualbus ~ % bash -c 'string1=yolo string2=_; [[ $string1 = "$string2" ]]; echo $?'
|  1

case #2:
 [[ STR =~ REGEX ]]
           ^- quoting here is FUBAR. The recommended practice is to:
              re='some.re'; [[ $str =~ $re ]]

case #3:
 Using $\* or any other variant.

|   dualbus@dualbus ~ % bash -c 'IFS=:; set -- 1 2 3; [[ 1:2:3 = $\* ]]; echo $?'
|   1
|   dualbus@dualbus ~ % bash -c 'IFS=:; set -- 1 2 3; [[ 1:2:3 = "$*" ]]; echo $?'
|   0

These are the ones that I remember right now. There are more, probably.
