# [\#926 Issue](https://github.com/koalaman/shellcheck/issues/926) `closed`: SC1058 false positive when `do` keyword is in line continuation

#### <img src="https://avatars.githubusercontent.com/u/13408130?v=4" width="50">[brlin-tw](https://github.com/brlin-tw) opened issue at [2017-06-05T04:07:24Z](https://github.com/koalaman/shellcheck/issues/926):

- Rule Id: SC1058(SC1073)
- My shellcheck version (`shellcheck --version` or "online"): online & 0.4.6
- [x] I tried on shellcheck.net and verified that this is still a problem on the latest commit
- [ ] It's not reproducible on shellcheck.net, but I think that's because it's an OS, configuration or encoding issue

#### Here's a snippet or screenshot that shows the problem:

```bash
#!/usr/bin/env bash
for required_command in\
	mount\
	grep\
	dd\
	do
	if ! command -v "${required_command}" >/dev/null; then
		echo "Error: Command ${required_command} not found."
	fi
done
```

#### Here's what shellcheck currently says:
```
Line 2:
for required_command in\
^-- SC1073: Couldn't parse this for loop.
                     ^-- SC1058: Expected 'do'.
                       ^-- SC1072: Fix any mentioned problems and try again.
```

#### Here's what I wanted or expected to see:
```
No issues detected!
```

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2017-06-05T17:56:38Z](https://github.com/koalaman/shellcheck/issues/926#issuecomment-306258027):

This is indeed not very helpful. This variation is a bit more correctly diagnosed, though it doesn't mention that the escaped linefeed is insufficient: 

```
#!/usr/bin/env bash
for required_command in mount\
  grep\
  dd\
  do
  ^-- SC1063: You need a line feed or semicolon before the 'do'.
  if ! command -v "${required_command}" >/dev/null; then
    echo "Error: Command not found."
  fi
done
```

It's not valid in bash so there should be some error.
