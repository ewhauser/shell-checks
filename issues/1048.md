#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2017-11-14T00:20:19Z](https://github.com/koalaman/shellcheck/issues/1048#issuecomment-344103680):

The formatters follow the convention of the tools they mimic. `gcc` uses a tab size of 1, `checkstyle` uses a tab size of 8. Changing those would break compatibility.

There's no particular reason why the shellcheck JSON format couldn't have used the `gcc` style instead of `checkstyle` and `tty`, but it didn't, and I don't see any value in changing it now since that would break the integrations that rely on it (like Syntastic).

The tab size used by each formatter is documented: 
https://github.com/koalaman/shellcheck/wiki/Integration

#### <img src="https://avatars.githubusercontent.com/u/9766?v=4" width="50">[blueyed](https://github.com/blueyed) commented at [2019-02-07T18:50:05Z](https://github.com/koalaman/shellcheck/issues/1048#issuecomment-461550298):

Just for information: typically linters use the byte/character offset, and do not handle tab in a special way (therefore a tab in gcc has an offset of 1 I assume).

Regarding compatibility: Syntastic uses `-fgcc`, not the json formatter.

It is not trivial to adjust this in Vim, where you would have to look at the actual line and adjust the column offset based on the tabs before the reported column.
