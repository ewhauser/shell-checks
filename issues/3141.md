#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash

yes | cp -r dist/* .
```

#### Here's what shellcheck currently says:
```sh
$ shellcheck myscript
 
[Line 3:](javascript:setPosition(3, 7))
yes | cp -r dist/* .
      ^-- [SC2216](https://www.shellcheck.net/wiki/SC2216) (warning): Piping to 'cp', a command that doesn't read stdin. Wrong command or missing xargs?

```


#### Here's what I wanted or expected to see:
Here `yes` is used to pipe `y` to commands that ask yes/no questions.

Actually `echo` any word begin with `y/n` (case insensitive) should be valid too, but it may be too difficult to be exempted from this rule.

#### <img src="https://avatars.githubusercontent.com/u/104861306?v=4" width="50">[BlacksilverConsulting](https://github.com/BlacksilverConsulting) commented at [2025-04-01 10:59](https://github.com/koalaman/shellcheck/issues/3141#issuecomment-2768980850):

Looking at the docs for cp, ISTM the better way to do this is to use the -f option.

#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2025-04-07 02:28](https://github.com/koalaman/shellcheck/issues/3141#issuecomment-2781870471):

`cp` only asks questions if invoked with `-i`. It's common to have `alias cp='cp -i'` but these do not apply in scripts.

However, shellcheck still warned for `yes | cp -i ...` while handling `cp -i ... < file` correctly. This is now fixed, and `yes | cp -ri dist/* .` should no longer trigger this warning.
