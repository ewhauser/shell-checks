#### <img src="https://avatars.githubusercontent.com/u/215702?u=b9f8c7106217535ccc93e6b7736992b2ef690350&v=4" width="50">[gromgit](https://github.com/gromgit) commented at [2020-03-04 01:49](https://github.com/koalaman/shellcheck/issues/1856#issuecomment-594274298):

> Maybe shellcheck is interpreting the quoted dash as a flag, and so thinks that the file list glob is a pattern when it's not?

That's exactly right. Your screenshot was ambiguous as to where the error was, so I ran it through shellcheck.net, which is a lot clearer:
```
grep '-' "$UKB"/microarray/*.sample > test.out
         ^-- SC2062: Quote the grep pattern so the shell won't interpret it.
```
This is why the `grep` man page says:

> `-e PATTERN, --regexp=PATTERN`
> Use *PATTERN* as the pattern.  If this option is used multiple times or is combined with the -f (--file) option, search for all patterns given.  **This option can be used to protect a pattern beginning with “-”.**

and the pedantically-correct way to write your code is:
```
grep -e - "$UKB"/microarray/*.sample > test.out
```
