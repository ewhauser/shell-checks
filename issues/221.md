# [\#221 Issue](https://github.com/koalaman/shellcheck/issues/221) `closed`: New check: cd without verifying pwd

#### <img src="https://avatars.githubusercontent.com/u/1020624?v=4" width="50">[mjmeehan](https://github.com/mjmeehan) opened issue at [2014-08-24T03:36:17Z](https://github.com/koalaman/shellcheck/issues/221):

Using cd tends to be perilous in beginners' scripts:

``` bash
cd /var/log/oldstuff
for file in `./*`; do
    rm $file
done
```

If /var/log/oldstuff doesn't exist, all files in the pwd of the shell will be removed. In general, cd should be avoided in favor of absolute paths. When it is used, you must check the new pwd matches what you expect.
