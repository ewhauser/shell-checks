# [\#733 Issue](https://github.com/koalaman/shellcheck/issues/733) `closed`: Shellcheck 0.4.4 ignores shebang regarding POSIX errors (SC2039)

#### <img src="https://avatars.githubusercontent.com/u/5325723?v=4" width="50">[zhimsel](https://github.com/zhimsel) opened issue at [2016-09-08T22:34:04Z](https://github.com/koalaman/shellcheck/issues/733):

Happens in both `vim` and `nvim`, using neomake.

Shellcheck will return SC2039 errors, claiming that I've made the script POSIX compliant, regardless of the shebang or any options. Event hardcoding `bash` will result in an SC2039 error for the following script: 

``` shell
#!/bin/bash

if [[ $1 == 'hi' ]]
then
  echo hi
fi
```

The errors are not present while running `shellcheck` from the command line (which I'm still trying to figure out why; it returns no output even if there are errors in neomake).
