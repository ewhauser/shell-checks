# [\#479 Issue](https://github.com/koalaman/shellcheck/issues/479) `open`: `sed -i <command>` is BSD-incompatible

#### <img src="https://avatars.githubusercontent.com/u/572196?v=4" width="50">[stuartpb](https://github.com/stuartpb) opened issue at [2015-09-29T19:09:39Z](https://github.com/koalaman/shellcheck/issues/479):

I was [just bitten by this](https://github.com/stuartpb/rethinkdb-dockerfiles/pull/15#issuecomment-144156059): Shellcheck should check for a number of GNUisms around `sed` and recommend the POSIX-safe equivalent instead. In this case, [it should recommend the expression be specifically prefixed with `-e`](http://unix.stackexchange.com/a/131940/7733).

#### <img src="https://avatars.githubusercontent.com/u/714124?v=4" width="50">[lyda](https://github.com/lyda) commented at [2018-09-19T15:02:59Z](https://github.com/koalaman/shellcheck/issues/479#issuecomment-422838081):

It's also possible shellcheck could just run sed to determine which it is. `sed --version` will return 0 if it's GNU sed and 1 if it's not. Obviously command line args would be needed too, but a --sed-type={bsd,gnu,auto} is possible.

(note you'd want to direct stdout/stderr to /dev/null)
