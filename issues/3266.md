#### <img src="https://avatars.githubusercontent.com/u/2787581?v=4" width="50">[grische](https://github.com/grische) commented at [2025-08-05 10:25](https://github.com/koalaman/shellcheck/issues/3266#issuecomment-3154610082):

That's intended behaviour. 

Variables with only capital letters are considered to be externally defined.
Use normal lowercase variables if you want them to be checked.

#### <img src="https://avatars.githubusercontent.com/u/243351?v=4" width="50">[brother](https://github.com/brother) commented at [2025-08-07 13:42](https://github.com/koalaman/shellcheck/issues/3266#issuecomment-3164259646):

Maybe activate the [optional check](https://github.com/koalaman/shellcheck/wiki/Optional) for unassigned uppercase vars?
```
name:    check-unassigned-uppercase
desc:    Warn when uppercase variables are unassigned
example: echo $VAR
fix:     VAR=hello; echo $VAR
```

```shell
brother ~$ shellcheck -o check-unassigned-uppercase test.sh

In test.sh line 2:
${LS}
^---^ SC2154 (warning): LS is referenced but not assigned.

For more information:
  https://www.shellcheck.net/wiki/SC2154 -- LS is referenced but not assigned.
```
