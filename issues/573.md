# [\#573 Issue](https://github.com/koalaman/shellcheck/issues/573) `closed`: SC2029 does not catch single quotes within double quotes

#### <img src="https://avatars.githubusercontent.com/u/278727?v=4" width="50">[fenollp](https://github.com/fenollp) opened issue at [2016-01-10T16:36:21Z](https://github.com/koalaman/shellcheck/issues/573):

```
In ./kerl line 582:
    ssh $KERL_DEPLOY_SSH_OPTIONS "$host" "cd '$remotepath' && sed -i -e 's#$path#$(pwd)#g' activate"
        ^-- SC2086: Double quote to prevent globbing and word splitting.
                                              ^-- SC2029: Note that, unescaped, this expands on the client side.
```

I don't believe `$remotepath` will actually get expanded on the client side, given the single quotes.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2016-01-10T18:48:50Z](https://github.com/koalaman/shellcheck/issues/573#issuecomment-170381492):

ShellCheck is right. Single quotes are literal in double quotes, and therefore have no quoting behavior.

All the variable expansions in your expression will happen on the client, and the server will never see nor expand `$remotepath`, `$path` or `$(pwd)`. I don't know where these are set, but the latter one definitely looks like it should happen on the server side, so it should be `\$(pwd)`
