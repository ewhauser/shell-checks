# [\#160 Issue](https://github.com/koalaman/shellcheck/issues/160) `closed`: Contradictory advice regarding double quotes

#### <img src="https://avatars.githubusercontent.com/u/985977?v=4" width="50">[Markus00000](https://github.com/Markus00000) opened issue at [2014-05-18T13:59:20Z](https://github.com/koalaman/shellcheck/issues/160):

These give "Double quote to prevent globbing and word splitting." for the unquoted variables:

```
echo $filename.$extension
echo "$filename".$extension
echo $filename."$extension"
```

This one gives "The double quotes around this do nothing. Remove or escape them." for the second variable:

```
echo "$filename"."$extension"
```

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2014-05-18T17:42:25Z](https://github.com/koalaman/shellcheck/issues/160#issuecomment-43446138):

This is a new check aimed to prevent things like

```
echo "INSERT INTO FOO VALUES ("bar"); "
```

where the nested double quotes aren't. You'll notice it's pointing to the period, not the second variable. 

By removing the quotes around the period, you get:

```
echo "$filename.$extension"
```

I can see how this is confusing.
