#### Here's a snippet or screenshot that shows the problem:

I have a comment line (line 5) that mentions "shellcheck" as the first word of the comment line:

```sh

#!/bin/bash
# The purpose of this script is blah blah blah...
#
# To check this script for issues, use shellcheck as follows:
#    shellcheck this_script.sh
#
# And then...
#

echo "foo bar baz"
```

#### Here's what shellcheck currently says:

```sh
$ shellcheck this_script.sh 

In this_script.sh line 5:
#    shellcheck this_script.sh
^-- SC1073: Couldn't parse this shellcheck directive. Fix to allow more checks.
                    ^-- SC1072: Expected '=' after directive key. Fix any mentioned problems and try again.

For more information:
  https://www.shellcheck.net/wiki/SC1072 -- Expected '=' after directive key....
  https://www.shellcheck.net/wiki/SC1073 -- Couldn't parse this shellcheck di...
```

Is there some way to get this to work?  I know I could combine lines 4 and 5 and it works:

```sh

#!/bin/bash
# The purpose of this script is blah blah blah...
#
# To check this script for issues, use shellcheck as follows:  shellcheck this_script.sh
#
# And then...
#

echo "foo bar baz"
```

Are there any better options?

Your second solution reminded me of using a backslash:
https://tldp.org/LDP/GNU-Linux-Tools-Summary/html/c1195.htm
```
#!/bin/bash
# The purpose of this script is blah blah blah...
#
# To check this script for issues, use shellcheck as follows:
#    \shellcheck this_script.sh
#
# And then...
#

echo "foo bar baz"
```
It can be copy/pasted to the command line:

`$ \shellcheck this_script.sh`

I'm leaning towards either that or your first solution (double pound sign).
