#### Feature suggestion
I feel that the `source` directive promotes redundancy and thereby violates the [DRY](https://en.wikipedia.org/wiki/Don%27t_repeat_yourself) principle:

```
# shellcheck source=/dir/subdir/script.sh
. "$SCRIPTPATH/subdir/script.sh"
```

As it is, it doesn't find errors if you rename a script file and forget to change the second line. This is easy to solve by introducing a `sourcedir` directive:

```
# shellcheck sourcedir=/dir/subdir
. "$SCRIPTPATH/subdir/script.sh"
```
Now, if you change a file name, there is only one place to change.

(I also thought of proposing a `value` directive to specify the value of variable path parts such as `SCRIPTPATH`. That would have reduced redundancy even more effectively. However, there are way too many edge cases and I think that the above solution should be clean and easy to implement.)
