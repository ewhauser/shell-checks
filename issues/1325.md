#### Here's a snippet or screenshot that shows the problem:

The docs for SC1090 state that not following sourced files can "potentially result in warnings about unassigned variables and similar."  It would be nice if SC1090 was only emitted if SC is actually about to emit an unassigned variable warning.

Example use case: shell scripts that dynamically create a Python virtualenv in a temporary directory and source-activate it.  This sets some environment variables that are only being read by Python, but won't even be mentioned in the shell script.

```sh
#!/bin/bash
venvpath="$(mktemp -u)"
trap 'rm -rf "$venvpath"' EXIT
python -mvenv "$venvpath"
(
    source "$venvpath/bin/activate"
    python -c 'print("hello world")'
)
```

#### Here's what shellcheck currently says:

SC1090: Can't follow non-constant source. Use a directive to specify location.

#### Here's what I wanted or expected to see:

No warning.
