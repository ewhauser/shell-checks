# [\#649 Issue](https://github.com/koalaman/shellcheck/issues/649) `closed`: shellcheck should warn on bashisms in /bin/sh scripts

#### <img src="https://avatars.githubusercontent.com/u/421543?v=4" width="50">[jschauma](https://github.com/jschauma) opened issue at [2016-04-16T15:36:07Z](https://github.com/koalaman/shellcheck/issues/649):

If you have a script that begins with a "#!/bin/sh" shebang, shellcheck should warn against bashisms, such as '[[' or '==', since not all systems use bash-as-bin-sh.  (bash-as-bin-sh unfortunately still allows bash-specific syntax.)

```
$ cat f.sh
#! /bin/sh

x="x"

if [[ x == "$x" ]]; then
    echo "bash!"
fi
$ cat f.sh | shellcheck -
$ sh /tmp/f.sh
[[: not found
$ 
```

These checks exist but failed to trigger due to a shebang parsing issue. Thanks! Fixed in f835c2d.
