# [\#269 Issue](https://github.com/koalaman/shellcheck/issues/269) `closed`: false positives with SC2140

#### <img src="https://avatars.githubusercontent.com/u/14709?v=4" width="50">[moy](https://github.com/moy) opened issue at [2014-12-23T12:06:27Z](https://github.com/koalaman/shellcheck/issues/269):

This is similar to issue #238.

Shellcheck raises a warning for expressions such as "$dir"/foo/"$file", because it guesses that the intention of the programmer was to nest quotes. Actually, I find this style legitimate (even though the quotes are not really useful), and I think it makes sense to have any sequence of non-special characters between the inner double-quotes (shellcheck already allows a single / or a single = since #238).

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2014-12-24T03:57:21Z](https://github.com/koalaman/shellcheck/issues/269#issuecomment-68024056):

As of 7b70500, ShellCheck will not warn if it suspects you're just trying to quote each variable individually (like in your example). This will hopefully reduce the number of false positives.
