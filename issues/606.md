I think the fastest way for exploration would be, to test the binary with "external program" option. It can link io to eclipse console. Why didn't I test this earlier?

Tested this for windows:
1. Copy shellcheck binary to you eclipse home (I copied it to a new dir "external programs")
2. Create run configuration for external program
3. Set Location relative to eclipse home (portable): ${eclipse_home}\external programs\shellcheck-stable.exe
4. Set arguments to ${resource_loc}
5. open your file to check
6. click run icon for external programs
7. Output is in Console

_Edit1:_ if you want colors, the console doesn't understand the escape codes, you can use this plugin:
https://marketplace.eclipse.org/content/ansi-escape-console
Change arguments to: --color=always ${resource_loc}

_Edit2:_ The above is working, a possible next step - just nice to have - would be to allow something like root/*.sh as file and recursion min/max.
