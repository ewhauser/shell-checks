# [\#753 Issue](https://github.com/koalaman/shellcheck/issues/753) `open`: Style: Add optional directive to notify of double-quotes which could be replaced with single quotes

#### <img src="https://avatars.githubusercontent.com/u/179397?v=4" width="50">[srcshelton](https://github.com/srcshelton) opened issue at [2016-10-21T15:41:32Z](https://github.com/koalaman/shellcheck/issues/753):

It would be nice to have a (default-off?) directive which suggests substituting:

```
a="This is a constant string"
b="This is ${interpolated}"
```

... for:

```
a='This is a constant string'
b="This is ${interpolated}"
```

... which is, IMHO, a nice indicator or expected usage.

#### <img src="https://avatars.githubusercontent.com/u/8325708?v=4" width="50">[alok](https://github.com/alok) commented at [2018-06-05T20:36:39Z](https://github.com/koalaman/shellcheck/issues/753#issuecomment-394851487):

I second this. It's good practice and can be accurately detected in pretty much all practical cases. Shellcheck already warns if a dollar sign is in single quotes, so this seems likes the natural complement.

#### <img src="https://avatars.githubusercontent.com/u/5316?v=4" width="50">[mcandre](https://github.com/mcandre) commented at [2018-11-15T21:09:35Z](https://github.com/koalaman/shellcheck/issues/753#issuecomment-439191001):

Note that constant string values containing escape sequences should include a leading dollar (`$`) immediately preceding the opening single quote (`'`), in order to ensure that the escapes are treated as escapes, as opposed to literals. In other words, `"\n"` = `$'\n'`, whereas `'\n'` = `$'\\n'` `"\\n"`.
