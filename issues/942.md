#### Here's a snippet or screenshot that shows the problem:

```sh

#!/bin/sh
find -O3 ~/git
```

#### Here's what shellcheck currently says:

Some finds don't have a default path. Specify '.' explicitly. [SC2185]

#### Here's what I wanted or expected to see:

Should have no error. When using either -O1 -O2 -O3, the optimization level must come before the directory. Shellcheck should make sure the if -Ox is provided that it checks the term after it for a directory there.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2017-07-08T22:56:28Z](https://github.com/koalaman/shellcheck/issues/942#issuecomment-313885792):

Indeed. 

Various implementations have various pre-path flags. POSIX has `-L` and `-H`. GNU has `-P`. macOS has `-X`.  Instead of trying to enumerating them all, ShellCheck got along by assuming they were all single characters. 

`-O3` is 2, like `-ok`, so it was incorrectly considered a predicate by this naive matcher. This case has been explicitly added, improving the situation. Thanks!

For the existing version, there's no good rewrite, so please disable the message with a directive.
