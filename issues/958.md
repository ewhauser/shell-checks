# [\#958 Issue](https://github.com/koalaman/shellcheck/issues/958) `closed`: Exported variables not recognised as exported, triggering SC2034

#### <img src="https://avatars.githubusercontent.com/u/13365607?v=4" width="50">[mrtimdog](https://github.com/mrtimdog) opened issue at [2017-07-20T06:34:42Z](https://github.com/koalaman/shellcheck/issues/958):

#### For bugs
- Rule Id (if any, e.g. SC1000): SC2034
- My shellcheck version (`shellcheck --version` or "online"): online
- [x] I tried on shellcheck.net and verified that this is still a problem on the latest commit

#### Here's a snippet or screenshot that shows the problem:

```sh

#!/bin/bash -eux

declare -x a="a"
typeset -x b="b"

f()
{
    local -x c="c"
}

```

#### Here's what shellcheck currently says:

```

$ shellcheck myscript
 
Line 4:
typeset -x b="b"
           ^-- SC2034: b appears unused. Verify it or export it.
 
Line 8:
    local -x c="c"
             ^-- SC2034: c appears unused. Verify it or export it.

```

#### Here's what I wanted or expected to see:

No errors or warnings.

These are all exported variables, even the local one within the function scope, but they're only recognised as exported when using 'declare -x' or 'export', not with 'typeset -x' or 'local -x'.
