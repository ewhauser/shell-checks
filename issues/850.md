# [\#850 Issue](https://github.com/koalaman/shellcheck/issues/850) `closed`: Encourage early binding doublequoted strings for safety

#### <img src="https://avatars.githubusercontent.com/u/5316?v=4" width="50">[mcandre](https://github.com/mcandre) opened issue at [2017-03-17T20:59:09Z](https://github.com/koalaman/shellcheck/issues/850):

(ba)sh strings actually behave differently based on the quote style, and I'm not talking about escape characters.

Single-quoted strings (`'$var'`) use late binding, meaning their contents are determined on use, not when the string is declared. This is how aliases actually work.

Double-quoted strings (`"$var"`) use early binding, meaning their value is set when they are declared.

Could shellcheck encourage programmers to use double-quoted strings more often, to avoid accidental late binding? Double quotes should really be used in every case except aliases.
