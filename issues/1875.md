# [\#1875 Issue](https://github.com/koalaman/shellcheck/issues/1875) `open`: Detect obsolete and optional commands

#### <img src="https://avatars.githubusercontent.com/u/18032974?u=d226975d74317471331926c8371b8d8df20e3bb7&v=4" width="50">[habere-et-dispertire](https://github.com/habere-et-dispertire) opened issue at [2020-03-16 21:23](https://github.com/koalaman/shellcheck/issues/1875):

Unix commands can be categorized as either: obsolete, optional or mandatory:
_https://en.wikipedia.org/wiki/List_of_Unix_commands_

Perhaps obsolete commands could be flagged as errors:
_fort77, qalter, qdel, qhold, qmove, qmsg, qrerun, qrls, qselect, qsig, qstat_

While optional commands could be warnings:
_admin, asa, bg, cc, c99, cal, cflow, compress, ctags, cxref, delta, ex, fc, fg, fuser, get, ipcrm, ipcs, jobs, lex, link, make, more, nl, nm, prs, rmdel, sact, sccs, strip, talk, ulimit, uncompress, unget, unlink, uucp, uustat, uux, val, vi, what, yacc, zcat_

The motivation is for better portability by relying only on external commands that are more likely to be present.

Minimal working example:

~~~bash
#! /usr/bin/env bash
set -Eeuo pipefail

# cmp is a mandatory unix command
cmp foo bar

# cal is an optional unix command
cal

# fort77 is an obsolete command
fort77
~~~
