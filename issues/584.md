# [\#584 Issue](https://github.com/koalaman/shellcheck/issues/584) `closed`: False positive SC2016 for sed scripts

#### <img src="https://avatars.githubusercontent.com/u/1823914?v=4" width="50">[arth1](https://github.com/arth1) opened issue at [2016-01-25T20:25:02Z](https://github.com/koalaman/shellcheck/issues/584):

Consider this snippet (which copies everything after a marker, but not the marker itself):

sed -n <foo '/^#MARKER/,${/^#MARKER/d;p}'

This results in a SC2016: Expressions don't expand in single quotes, use double quotes for that.

Could ${ be excluded from sed scripts?

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2016-01-27T04:06:18Z](https://github.com/koalaman/shellcheck/issues/584#issuecomment-175379557):

`sed` is a tricky case because it's so commonly misused, but also uses `$` for many legitimate things. It's not optimal to just ignore `${` since this is a perfectly normal variable expansion. 7e75d12 fixes this particular case and a few others. 

I also updated the wiki to suggest more generous spacing (like `$ { /^#..`) as an alternative to having to ignore with a directive, which is always a hassle.
