#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2019-01-13T18:49:40Z](https://github.com/koalaman/shellcheck/issues/1428#issuecomment-453855334):

ShellCheck is correct. These arguments must be quoted to prevent them from being expanded as globs. 

Currently, it breaks in pretty random ways based on the contents of the current directory:

```
$ cat myscript
#!/bin/sh

FOO=foo
echo $FOO | tr [:lower:] [:upper:]

$ ./myscript
FOO

$ mkdir r && ./myscript
foo

$ touch l && ./myscript
tr: extra operand ‘r’
Try 'tr --help' for more information.
```

A correctly quoted version with `tr '[:lower:]' '[:upper:]'` will work correctly and output FOO in all the cases above, without being affected by unrelated files in the current directory.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2019-01-13T23:28:29Z](https://github.com/koalaman/shellcheck/issues/1428#issuecomment-453875820):

From the shell's point of view, `[:lower:]` is a glob that matches any character in the set `elorw:`. When such a file exists, the executed command instead becomes e.g. `tr l r r`, resulting in the error shown.

Quoting makes sure `[:lower:]` is passed literally every time
