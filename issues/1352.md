#### <img src="https://avatars.githubusercontent.com/u/1241845?v=4" width="50">[hasufell](https://github.com/hasufell) commented at [2018-12-03T02:30:34Z](https://github.com/koalaman/shellcheck/issues/1352#issuecomment-443570342):

>     * How will this affect packagers who need to use their distro's version of certain Hackage packages instead of what I pick?

Cannot be properly answered, because "packaging" is a way too broad term. Haskell packaging is broken on most distros and the distros that do it usually don't invoke `cabal new-install`, because it's fundamentally incompatible with distro packaging. This file has no effect on e.g. `cabal install`.

>     * How will it affect people who are using an older version of the Haskell Platform that comes with older libraries?

If they use stack, it has no effect, if they use `cabal install` (with or without a sandbox), it has no effect. If they use `cabal new-install`, but their ghc is too old, they can just delete the freeze file.

> How will it affect people using GHC versions that don't support the frozen package version?

See above.

> How will it affect e.g. `hadolint` which uses ShellCheck as a library?

I wasn't aware ShellCheck is used as a library. In that case, I consider that a bug and ShellCheck should be split out into two separate packages.

>     * How will it affect conditionally included libraries like QuickCheck which can currently be stripped out to reduce the binary size?

I don't understand this question. The freeze file, as described in the link, are just *constraints*, not dependencies. The freeze file can be deleted, edited, recreated at any given time by the user.
