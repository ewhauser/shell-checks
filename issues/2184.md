#### Here's a snippet or screenshot that shows the problem:

```sh
#!/usr/bin/env bash
python foo.py 2> "$1".txt | less -S - "$1".txt
```

#### Here's what shellcheck currently says:
```
Line 2	SC2094: Make sure not to read and write the same file in the same pipeline.
Line 2	SC2094: Make sure not to read and write the same file in the same pipeline.
```
#### Here's what I wanted or expected to see:

I'd expect there to be no problems reported. 
This is fine because less is lazy when reading `"$1".txt`.

With a target file reader that is not lazy (in the sense of not expecting the whole file to be present immediately), I could envisage a race condition however.
