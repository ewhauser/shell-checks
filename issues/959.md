# [\#959 Issue](https://github.com/koalaman/shellcheck/issues/959) `open`: Feature: Warn on stupid I/O redirect

#### <img src="https://avatars.githubusercontent.com/u/16140513?v=4" width="50">[drosehn](https://github.com/drosehn) opened issue at [2017-07-27T23:42:46Z](https://github.com/koalaman/shellcheck/issues/959):

While writing a rather complicated section of script, I wanted to capture the output of a command which might fail.  It was fine with me if it did fail, and I did not want to see any error-msgs which might come up.  So I did:

```sh
dropcnt_ACT="$(grep -ic ' some-pattern'  some_file   2>&/dev/null )"
```

I was using the current code at www.shellcheck.net, and it made no complaints about this line.  I then ran my script, and got:

```
# dropcnt_ACT="$(grep -ic ' some-pattern'  some_file   2>&/dev/null )"
-bash: /dev/null: ambiguous redirect
```
I will use the excuse of "it has been a long day", and say that it took me awhile to realize that what I really wanted was `2>>/dev/null` instead of `2>&/dev/null`.  (It took time partly because the script continued running after that line, but then `dropcnt_ACT` had the value of a null string instead of the correct count, so rest of the script did the wrong set of commands).

Perhaps this is something shellcheck could notice and warn about?
