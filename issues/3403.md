#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash

func() { return -- "$1"; }
```

#### Here's what shellcheck currently says:

One can have such function in their `~/.bashrc` where `--` prevents injection like `--help` and ensures that anything is passed as return status to `return`, even if it's wrong (not a number).

#### Here's what I wanted or expected to see:

Make an exception so that `return` does not trigger SC2151 warning if it has `--` as first argument, and then some other argument.
