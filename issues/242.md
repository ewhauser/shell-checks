# [\#242 Issue](https://github.com/koalaman/shellcheck/issues/242) `open`: Enhancement: Testing for empty arrays

#### <img src="https://avatars.githubusercontent.com/u/166207?v=4" width="50">[mietek](https://github.com/mietek) opened issue at [2014-10-23T07:55:15Z](https://github.com/koalaman/shellcheck/issues/242):

With `set -u`, testing for empty arrays should not be done like this:

```
$ declare -a arr
$ ! (( ${#arr[@]} )) && echo empty
bash: args: unbound variable
```

Instead, it should be done like this, with `_` being an arbitrary string:

```
$ declare -a arr
$ [ -z "${arr[@]:+_}" ] && echo empty
empty
```

It would be nice if ShellCheck could help here.
