# [\#939 Issue](https://github.com/koalaman/shellcheck/issues/939) `open`: Warn about missing dollar sign near braces within double quotes

#### <img src="https://avatars.githubusercontent.com/u/7881842?v=4" width="50">[dangom](https://github.com/dangom) opened issue at [2017-06-21T16:43:27Z](https://github.com/koalaman/shellcheck/issues/939):

#### Feature Request

- [x] shellcheck.net (i.e. the latest commit) currently gives no useful warnings about this
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related


#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash

foo="bar"
echo "${foo}_{foo}"
```

#### Here's what shellcheck currently says:
$ shellcheck myscript
No issues detected!


#### Here's what I wanted or expected to see:

Similar to [SC1083,](https://github.com/koalaman/shellcheck/wiki/SC1083) but for braces within double quoted expressions.

$ shellcheck myscript
This { is literal. Check expression (missing $?)
