# [\#669 Issue](https://github.com/koalaman/shellcheck/issues/669) `closed`: false positive with using alternate values SC2068

#### <img src="https://avatars.githubusercontent.com/u/176950?v=4" width="50">[vapier](https://github.com/vapier) opened issue at [2016-05-17T20:46:18Z](https://github.com/koalaman/shellcheck/issues/669):

this code snippet is quoted correctly, but SC2068 is triggered anyways.  

``` sh
$ cat test.sh
echo ${@+"$@"}

$ shellcheck test.sh
Line 1:
echo ${@+"$@"}
     ^-- SC2068: Double quote array expansions to avoid re-splitting elements.
```

quoting on the tested var itself doesn't matter ... the quotes on the replacement will handle the expansion properly.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2017-05-25T02:22:47Z](https://github.com/koalaman/shellcheck/issues/669#issuecomment-303904517):

`:+` was already a handled case, and with 4243c6a `+` is as well. 

It's very easy to fool (it doesn't even check that the value is quoted), but it will at least stop warning here.
