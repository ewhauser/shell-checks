#### <img src="https://avatars.githubusercontent.com/u/38788?v=4" width="50">[kepstin](https://github.com/kepstin) commented at [2023-08-22 00:53](https://github.com/koalaman/shellcheck/issues/2791#issuecomment-1687253300):

For bash specifically, the first example given (under the [Correct Code](https://github.com/koalaman/shellcheck/wiki/SC2044#correct-code) header) correctly handles filenames containing line breaks. This works because it is using the find `-print0` option to separate filenames with a 0 byte (which cannot occur in filenames), and it is using `read -d ''` which causes bash to use the 0 byte as the "line" separator.

There is indeed no good solution for the case where you are using a POSIX shell and specifically need to read filenames that find is outputting into the current script process.
