This first:
```
# No warning is shown for the line below
if [[ -n "${var}" ]]; then
  true
fi
```
My guess is that `shellcheck` is trying to be less noisy by not reporting this undeclared variable `var`.  The script is deliberately testing `var`, so there is no need for `shellcheck` to report it.

Then add `set -u`, which does cause the script to fail in `bash`.

The problem appears to be that `shellcheck` is not taking `set -u` into account.

Does `shellcheck` ever account for any shell options ?

If `shellcheck` does account for shell options then, _I would tend to agree_, that it should take `-u` into account, in the manner expressed in this issue.
