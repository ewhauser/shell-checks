# [\#1202 Issue](https://github.com/koalaman/shellcheck/issues/1202) `open`: SC2129: '{ cmd1; cmd2; } >> file' vs 'here document'

#### <img src="https://avatars.githubusercontent.com/u/14194483?v=4" width="50">[mhoes](https://github.com/mhoes) opened issue at [2018-05-04T15:28:59Z](https://github.com/koalaman/shellcheck/issues/1202):

When checking the following (partial) shell script :

```
#!/bin/sh

echo "line1 $var1" > file.txt
echo "line2 $var2" >> file.txt
echo "line3 $var3" >> file.txt
echo "line4 $var4" >> file.txt
echo "line5 $var5" >> file.txt
```

shellcheck currently suggests (SC2129) using '{ cmd1; cmd2; } >> file'. Although in general, I guess that using this construct with other commands is good advise, in the particular case of using 'echo', a suggestion of 'use a here document instead' might be slightly better ?

```shell
#!/bin/sh
# https://www.shellcheck.net/wiki/SC2129
{
    echo "line1 $var1"
    echo "line2 $var2"
    echo "line3 $var3"
    echo "line4 $var4"
    echo "line5 $var5"
} > file.txt
```
