# [\#13 Issue](https://github.com/koalaman/shellcheck/issues/13) `closed`: Omit "No comments for <file>" messages

#### <img src="https://avatars.githubusercontent.com/u/5316?v=4" width="50">[mcandre](https://github.com/mcandre) opened issue at [2013-10-20T20:57:25Z](https://github.com/koalaman/shellcheck/issues/13):

I like to run `shellcheck *.sh` to scan a whole directory at once. But the helpful messages are often hidden inside long lists of `No comments for <file>`. Could we turn this message off by default, so that `shellcheck` only shows warnings, like how [CheckStyle](http://checkstyle.sourceforge.net/) works?
