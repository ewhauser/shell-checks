What parameter should be used to get debug logging? I would like to know why one file from branch X fails on shellcheck while the other that resides in branch Y passes.

#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2021-04-06 17:11](https://github.com/koalaman/shellcheck/issues/2195#issuecomment-814287291):

There's no extended debug logging. To find out why one file triggers while another doesn't, you can check whether the wiki page mentions special cases when the error will or won't be suppressed, browse the source code, ask on #shellcheck@irc.freenode.net, or post here (though I struggle to keep up)

#### <img src="https://avatars.githubusercontent.com/u/25598447?u=73c3f3a041be0ccabd72bb816d5782b30aefe822&v=4" width="50">[TinCanTech](https://github.com/TinCanTech) commented at [2021-04-06 19:28](https://github.com/koalaman/shellcheck/issues/2195#issuecomment-814383078):

Also, some errors are only reported by `shellcheck` on the first occurrence, this can also be a little confusing at first.  But, the point is, `shellcheck` is a first-line of defence. It spots definite errors, probable errors and other likely shell coding problems.  Use it as an indicator, **not** as a last-word. And then investigate any issues highlighted.
