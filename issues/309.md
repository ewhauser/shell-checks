# [\#309 Issue](https://github.com/koalaman/shellcheck/issues/309) `open`: Bash tab-completion for shellcheck

#### <img src="https://avatars.githubusercontent.com/u/352559?v=4" width="50">[gwern](https://github.com/gwern) opened issue at [2015-02-14T16:55:50Z](https://github.com/koalaman/shellcheck/issues/309):

As a shell-oriented tool, it would be nice if shellcheck automatically provided tab-completion which focused on .sh or shell-related files. A good list (sourced from sh/bash/zsh man pages) would be:

```
*.sh
*.bash
*.ksh
*.zsh
.bash_logout
.bash_profile
.bashrc
.env
.inputrc
.profile
.zlogin
.zlogout
.zprofile
.zshenv
.zshrc
bash.bash.logout
bash.bashrc
profile
zlogin
zlogout
zprofile
zshenv
zshrc
```

If any files match these, they should be prioritized by the tab-completion over other files with the same prefix (ie "arbtt-classify.sh" should match 'shellcheck a\t' before matching "arbtt-classified.txt", since it's much less likely that the text file is the one I want to run shellcheck on).

Completion is controlled by the shell and not the executable. If anyone wants to contribute to e.g. the [bash-completion](https://bash-completion.alioth.debian.org/) project, this is up for grabs.
