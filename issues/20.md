# [\#20 Issue](https://github.com/koalaman/shellcheck/issues/20) `closed`: Add options to generate input for other systems

#### <img src="https://avatars.githubusercontent.com/u/288876?v=4" width="50">[scoopex](https://github.com/scoopex) opened issue at [2013-10-31T08:46:43Z](https://github.com/koalaman/shellcheck/issues/20):

Add a option to create shell and html output compareable to http://www.shellcheck.net/
and create checkstyle output.

Example:
$ shellcheck --recursive --outputdir=target/shellcheck --checkstyle-file target/checkstyle.xml  target/ shellscriptfolder 

This example would: 
- recursively search the folder "shellscriptfolder" for shellscripts
- creates html output in the folder "target/shellcheck" 
  (one outputfile for every shellscript, same directory structure like in "shellscriptfolder)
- creates a output file compatible to the well known "checkstyle" tool
  (http://checkstyle.sourceforge.net/)

This might be very useful for automatic build environments.
Jenkins/Hudson provide plugins for publishing html outout or for interpreting checkstyle reports (provide a awesome presentation and statistics from the checkstyle files to the build metrics of the project)

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2013-11-14T06:58:04Z](https://github.com/koalaman/shellcheck/issues/20#issuecomment-28463340):

Great suggestion! As of 473bb66, `shellcheck -f checkstyle` will give you what's hopefully checkstyle compatible xml output. Other options include "gcc", which editors tend to support. 

Thanks!

Generating fully formatted HTML output seems slightly out of scope for this humble tool though. If anyone wants to make this happen, there's now easily parsable XML output in addition to the simple shellcheck library. Also, recursively finding files is something `find` does better than shellcheck ever could.
