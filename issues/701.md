# [\#701 Issue](https://github.com/koalaman/shellcheck/issues/701) `open`: Glob character ranges with variables are not recognized

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) opened issue at [2016-06-26T22:40:17Z](https://github.com/koalaman/shellcheck/issues/701):

`[$var]` is being mistakenly parsed as `[\$var]`. Example effect:

```
integers="1234"
^-- SC2034: integers appears unused. Verify it or export it.
echo vars[$integers]
         ^-- SC2102: Ranges can only match single chars (mentioned due to duplicates).
```
