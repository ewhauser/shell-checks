#### <img src="https://avatars.githubusercontent.com/u/16472988?v=4" width="50">[redianthus](https://github.com/redianthus) commented at [2019-09-04T17:18:30Z](https://github.com/koalaman/shellcheck/issues/1163#issuecomment-527999157):

Actually, [Debian policy](https://www.debian.org/doc/debian-policy/ch-maintainerscripts.html) states that:

> For shell scripts this means that you almost always need to use set -e (this is usually true when writing shell scripts, in fact).

When you don't use `set -e`, shellcheck raise a warning about missing ` || exit`. But having to write this kind of things all the time is laborious. Using `set -e` makes it easier. But keeping `|| exit` and similar things is useless with `set -e` and also makes the script harder to read... This issue is only in case you use `set -e` (and clearly, there's only disadvantages with adding `|| exit` in this case), not about `set -e` should be used or not.
