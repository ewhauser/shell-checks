#### Here's a snippet or screenshot that shows the problem:

This code:
```sh
#!/usr/bin/env sh

variable=${VARIABLE}

set -o nounset

echo "${variable}"
```
produced the error:
```
SC2153: Possible misspelling: VARIABLE may not be assigned, but variable is.
```

I added the disable directive, with a note:
```sh
#!/usr/bin/env sh

# shellcheck disable=SC2153 (variable not a misspelling)
variable=${VARIABLE}

set -o nounset

echo "${variable}"
```
But this produced a further error:
```
SC1125: Invalid key=value pair? Ignoring the rest of this directive starting here.
```
Which doesn't have a wiki page. On reading the rest of the wiki, this page had the useful information about how to comment a directive:
https://github.com/koalaman/shellcheck/wiki/Directive#documenting-directive-use
> The comment can be added on the preceding line. This is the recommended option for long comments.
> The comment can also be added at the end of the directive line. This is the recommended option for short comments.

#### Here's what shellcheck currently says:

No wiki page (option to create new page for SC1125).

#### Here's what I wanted or expected to see:

A wiki page explaining directive formatting / commenting style.
