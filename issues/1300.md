# [\#1300 Issue](https://github.com/koalaman/shellcheck/issues/1300) `closed`: Make ShellCheck.Interface types more opaque and future proof

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) opened issue at [2018-07-22T17:46:52Z](https://github.com/koalaman/shellcheck/issues/1300):

Currently, all the types in ShellCheck.Interface are highly transparent. This means that simply adding a field or optional setting causes an API incompatibility. Specifically, this affects [hadolint](https://github.com/hadolint/hadolint/blob/master/src/Hadolint/Shell.hs). 

These should probably all be record types with hidden constructors, along with functions like `emptyCheckSpec` that initialize them with useful default values.
