#### <img src="https://avatars.githubusercontent.com/u/255938?v=4" width="50">[ormaaj](https://github.com/ormaaj) commented at [2015-01-02T20:58:45Z](https://github.com/koalaman/shellcheck/issues/272#issuecomment-68561721):

It does. Always has afair.

```
$ shellcheck -s sh - <<<'for x in $(ls); do :; done'

In - line 1:
for x in $(ls); do :; done
^-- SC2034: x appears unused. Verify it or export it.
         ^-- SC2045: Iterating over ls output is fragile. Use globs.
```

What shellcheck does not do is detect field splitting and globs resulting from unquoted expansions in general.

```
$ shellcheck -s bash - <<<'y="foo b*ar"; for x in $y; do : "$x"; done'
<no hints>
```

At minimum shellcheck should suggest an array when using `bash`/`ksh`. For sh it should perhaps recommend a function and `for x` iteration. Even better would be a heuristic to try and tailor the warning based upon the `set -f` state and `IFS` state but that would of course only be a guess.

Or, try and detect obviously correct usages:

```
# This person clearly knows what they're doing, so don't suggest anything.
for x in $(printf '%q ' "$@"); do ...
```


#### <img src="https://avatars.githubusercontent.com/u/255938?v=4" width="50">[ormaaj](https://github.com/ormaaj) commented at [2015-01-03T02:39:11Z](https://github.com/koalaman/shellcheck/issues/272#issuecomment-68580463):

Actually it looks like shellcheck does do this in limited cases as discussed in #267. It just doesn't apply the same rule to every context where arguments are relevant.

``` bash
# Works for simple commands.
 ~ $ shellcheck -s bash - <<<'y="foo b*ar"; cmd $y'

In - line 1:
y="foo b*ar"; cmd $y
                  ^-- SC2086: Double quote to prevent globbing and word splitting.

# Doesn't work when the first word isn't literal.
 ~ $ shellcheck -s bash - <<<'y="foo b*ar"; $y'

# Doesn't work for array assignments.
 ~ $ shellcheck -s bash - <<<'y="foo b*ar"; a=($y)'

In - line 1:
y="foo b*ar"; a=($y)
              ^-- SC2034: a appears unused. Verify it or export it.

# Doesn't work in loops.
 ~ $ shellcheck -s bash - <<<'y="foo b*ar"; for x in $y; do : "$x"; done'
```
