# [\#2026 Issue](https://github.com/koalaman/shellcheck/issues/2026) `open`: ShellCheck should warn against "return" outside a function

#### <img src="https://avatars.githubusercontent.com/u/1450918?u=f5815dd52e856897e04dbe4cfe16c0363bf1d129&v=4" width="50">[Screwtapello](https://github.com/Screwtapello) opened issue at [2020-08-02 10:44](https://github.com/koalaman/shellcheck/issues/2026):

#### For new checks and feature suggestions
- [X] https://www.shellcheck.net/ (i.e. the latest commit) currently gives no useful warnings about this
- [X] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related


#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/sh
return 42
```

#### Here's what shellcheck currently says:

(no warning)

#### Here's what I wanted or expected to see:

```
Line 2:
return 42
^-- SCXXXX: return cannot reliably be used outside a function

Did you mean: 
exit 42
```

#### Notes

Although (on my machine) dash 0.5.10.2 and zsh 5.8 both accept top-level "return" quite happily, bash 5.0.16 exits with an error:

    path/to/script: line 2: return: can only `return' from a function or sourced script

(the exit status is 1, not 42, so it's an error and not just a warning)

bash behaves the same way when started in POSIX-compatible mode (`--posix`) so on machines where bash is `/bin/sh` this can be a problem even if other `/bin/sh` alternatives are fine with it.
