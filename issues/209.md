# [\#209 Issue](https://github.com/koalaman/shellcheck/issues/209) `closed`: Support for more shell languages

#### <img src="https://avatars.githubusercontent.com/u/5316?v=4" width="50">[mcandre](https://github.com/mcandre) opened issue at [2014-08-08T17:52:28Z](https://github.com/koalaman/shellcheck/issues/209):

It's awesome how shellcheck supports bash, sh, ksh, and zsh! Could we add other languages as well?
- csh and tcsh
- ash and dash
- fish

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2014-08-09T16:10:45Z](https://github.com/koalaman/shellcheck/issues/209#issuecomment-51691046):

While it would be nice, for fish, tcsh and csh, that basically means starting from scratch. 

Shellcheck's zsh and ksh support are already pretty lacking (except where bash compatible).

Ash and dash are currently recognized as synonyms for sh, but if there are convincing reasons to treat them separately, that's a possibility.


#### <img src="https://avatars.githubusercontent.com/u/7545917?v=4" width="50">[krader1961](https://github.com/krader1961) commented at [2017-03-08T03:13:23Z](https://github.com/koalaman/shellcheck/issues/209#issuecomment-284934095):

FWIW, someone asked about [lint support for fish on stackoverflow](http://stackoverflow.com/questions/42660371/is-there-a-linter-for-fish-like-there-is-for-bash-with-shellcheck). I pointed out that fish has a `fish_indent` command (similar to `clang-format` or `gofmt`) which could be a starting point for a `fish_lint` command. Doing that makes more sense than trying to teach this tool about fish syntax given that it is quite different from POSIX shells.
