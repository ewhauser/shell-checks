#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2020-05-04 05:20](https://github.com/koalaman/shellcheck/issues/1931#issuecomment-623262424):

This happened because the parse errors were appended after parsing had terminated with failure, and therefore weren't subject to the same annotation filtering as errors emitted *during* parsing. This has now been fixed.

Be aware that when SC1073 says "Fix to allow more checks", it refers to the fact that it failed to parse a syntax tree on which it can run the semantic level checks (SC2xxx). This does not change when the error is cosmetically hidden.
