# [\#63 Issue](https://github.com/koalaman/shellcheck/issues/63) `closed`: Should not report SC2090 when reporting SC2086 on unquoted variable in 'eval cmd $var'.

#### <img src="https://avatars.githubusercontent.com/u/302055?v=4" width="50">[deryni](https://github.com/deryni) opened issue at [2014-01-16T03:08:15Z](https://github.com/koalaman/shellcheck/issues/63):

I realize this is a minor point because fixing the SC2086 violation will cause both errors to go away but SC2090 is a false positive in this case as the embedded quotes will work correctly.

Shellcheck will report both SC2086 and SC2090 for cases 1, 2 and 3 below.

``` bash
#!/bin/bash

c() {
    echo "Arg count: $#"
    for arg; do
        echo "\$arg: $arg"
    done
}

args="--foo 'bar baz' 't*'"

echo 1
c "$args"
echo 2
c $args
echo 3
eval c $args
echo 4
eval c "$args"
```
