#### For new checks and feature suggestions

- [ ] I couldn't check shellcheck.net because it requires JavaScript
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/sh
command name args
if [ $? -eq 1 ] || [ $? -eq 2 ] ; then
	echo failed
fi
```

#### Here's what shellcheck currently says:

Nothing.

#### Here's what I wanted or expected to see:

The exit status variable was referenced in two different commands in an if statement, but the first command will overwrite the exit status variable and the second command will thus get the exit status of the first command not the exit status of the original command outside the if statement. Thus the second command will always get the wrong result.
