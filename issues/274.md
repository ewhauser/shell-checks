``` sh
echo "-no" 
```

is treated like:

``` sh
echo -n "o"
```

Which means that we get a warning like:

``` sh
echo "-no"
     ^-- SC2039: In POSIX sh, echo flags are not supported.
```

This is a false alarm.

It isn't a false alarm. The warning could perhaps be more specific but it is true that you can't count on `echo` to behave a certain way when an option is specified.

I'd suggest:

> In POSIX sh, `echo` flags are only supported by shells that implement the XSI extensions. `printf` should almost always be preferred over `echo`'s `-e` and `-n`.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2015-01-03T01:55:30Z](https://github.com/koalaman/shellcheck/issues/274#issuecomment-68579190):

`-no` is canonically the same as `-n -o`. In bash, it happens to print `-no` because `-o` is not a recognized option at this time.

If you instead tried `echo "-ne"`, you'd see that it doesn't print anything because both `-n` and `-e` are currently valid options.

I suggest you just avoid this mess and use `printf`.
