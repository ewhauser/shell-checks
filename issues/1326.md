# [\#1326 Issue](https://github.com/koalaman/shellcheck/issues/1326) `closed`: new style check: disallow !-z and !-n usage

#### <img src="https://avatars.githubusercontent.com/u/176950?v=4" width="50">[vapier](https://github.com/vapier) opened issue at [2018-08-28T20:35:58Z](https://github.com/koalaman/shellcheck/issues/1326):

using `! -z` and `! -n` in string checks are a bit confusing and unnecessary.  there's already a simpler form with `-n` and `-z` respectively.

`[ ! -z "$FOO" ]` should issue a recommendation that they write `[ -n "$FOO" ]` instead.

conversely, `[ ! -n "$FOO" ]` should issue a recommendation that they write `[ -z "$FOO" ]` instead.

Very nice suggestion, thanks! Simple, tons of instances, no false positives.
