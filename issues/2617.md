#### Here's a snippet or screenshot that shows the problem:

Let's say we have two files
 1. test.sh
```sh
#!/bin/bash
source include.sh
```
 2. include.sh
```
#!/bin/bash
aa=NEWS
bb=test2
```

I want shellcheck to be able to find any issue in `include.sh` but also avoiding any false positive. Correct me if I am wrong, I think the right approach is to make shellcheck emit warnings about the included file when checking `test.sh`  but without directly checking the included file.

#### Here's what shellcheck currently says:

```sh
$ shellcheck  -x test.sh 

$ shellcheck  include.sh 

In include.sh line 2:
aa=NEWS
^-- SC2034 (warning): aa appears unused. Verify use (or export if used externally).


In include.sh line 3:
bb=test2
^-- SC2034 (warning): bb appears unused. Verify use (or export if used externally).

For more information:
  https://www.shellcheck.net/wiki/SC2034 -- aa appears unused. Verify use (or...
```

#### Here's what I wanted or expected to see:

```
$ shellcheck -x test.sh 
In include.sh line 3:
bb=test2
^-- SC2034 (warning): bb appears unused. Verify use (or export if used externally).
```
