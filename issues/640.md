# [\#640 Issue](https://github.com/koalaman/shellcheck/issues/640) `closed`: SC1087 check does not allow dynamic variables

#### <img src="https://avatars.githubusercontent.com/u/2208?v=4" width="50">[svenfuchs](https://github.com/svenfuchs) opened issue at [2016-04-02T15:35:20Z](https://github.com/koalaman/shellcheck/issues/640):

E.g.

```
name=foo
foo=(1 2 3)
length=$(eval "echo \${#$name[@]}")
values=$(eval "echo \${$name[@]}")
```

Warns:

```
length=$(eval "echo \${#$name[@]}")
                        ^-- SC1087: Braces are required when expanding arrays, as in ${array[idx]}.

values=$(eval "echo \${$name[@]}")
                       ^-- SC1087: Braces are required when expanding arrays, as in ${array[idx]}.
```

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2016-04-09T17:04:33Z](https://github.com/koalaman/shellcheck/issues/640#issuecomment-207818241):

Your code is correct, but it's hard for shellcheck (and arguably humans) to tell. Consider adding explicit braces to clearly show what you consider the expansion to be:

```
length=$(eval "echo \${#${name}[@]}")
```
