#### Here's a snippet or screenshot that shows the problem:
Relatively simple suggestion, allow for .shellcheckrc in the "current" directory to supercede the HOME directory copy.  Don't do any kind of merge, just use if found in current dir.

Slightly more complex feature is search in all the containing directories for first .shellcheckrc

/Users/fred/projects/git/somerepo/stuff/scripts/some.sh
If the current dir is /Users/fred/projects/git/somerepo/stuff/scripts/ it would use  /Users/fred/projects/git/somerepo/.shellcheckrc if it existed.

This would allow projects to add a .shellcheckrc into the git repository when certain standards should be enforced for the project.  The latter might be preferred depending on how different tools integrate with spellcheck, and/or what the typical usage for the command line version is -- what the effective current dir is when the command gets run.

#### Here's what shellcheck currently says:
Spellcheck will use ~/.shellcheckrc for global settings, or global environment.  Neither of these is something that can be committed and reviewed in a repository.
