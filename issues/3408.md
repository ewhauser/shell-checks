# [\#3408 Issue](https://github.com/koalaman/shellcheck/issues/3408) `closed`: relax SC3045 checks

#### <img src="https://avatars.githubusercontent.com/u/65664393?v=4" width="50">[ashterenli](https://github.com/ashterenli) opened issue at [2026-02-18 16:35](https://github.com/koalaman/shellcheck/issues/3408):

with SC 0.11.0 I get:

```
ulimit -c 0
       ^-- SC3045 (warning): In POSIX sh, ulimit -c is undefined.
```

This is no longer correct,
since The Open Group Base Specifications Issue 8 IEEE Std 1003.1-2024,
see:
https://pubs.opengroup.org/onlinepubs/9799919799/utilities/ulimit.html

```
-c
Report, or set if the newlimit operand is present,
the core image size limit(s) in units of 512 bytes. [RLIMIT_CORE]
```

See, at the bottom:

```
Issue 8

Austin Group Defect 1418 is applied,
adding the -H, -S, -a, -c, -d, -n, -s, -t, and -v options,
and relating the -f option to the RLIMIT_FSIZE resource for
[setrlimit()](https://pubs.opengroup.org/onlinepubs/9799919799/functions/setrlimit.html).
```

So it seems the use of the above flags in `sh` should be allowed without a warning.
