#### Here's a snippet or screenshot that shows the problem:

```sh

#!/bin/bash
set -euo pipefail

while sleep 1; do
  checksum=$(date '+%s' | tail -c 2 | grep -v 7 | md5sum)
  echo "success (${checksum})"
done

```

The wanted code would look like this:

```
checksum=$(date '+%s' | tail -c 2 | { grep -v 7 || true ; } | md5sum)
```

(no output)

#### Here's what I wanted or expected to see:

Warning that when `-o pipefail` is used any non-zero exit codes of `grep` should be caught explicitly, e.g., with `{ grep ... || true ; }`
