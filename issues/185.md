# [\#185 Issue](https://github.com/koalaman/shellcheck/issues/185) `closed`: Unnecessary warning when assigning to special variables (eg PS1)

#### <img src="https://avatars.githubusercontent.com/u/7984?v=4" width="50">[andreaja](https://github.com/andreaja) opened issue at [2014-07-03T19:13:24Z](https://github.com/koalaman/shellcheck/issues/185):

Given the following source:

``` bash
__prompt_command() {
    __pwd="$PWD"
}

PROMPT_COMMAND=__prompt_command

PS1="\$__pwd\$ "
```

Shellcheck will incorrectly generate a warning:

``` bash
$ shellcheck example.sh

In example.sh line 2:
    __pwd="$PWD"
    ^-- SC2034: __pwd appears unused. Verify it or export it.
```

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2014-07-27T16:55:39Z](https://github.com/koalaman/shellcheck/issues/185#issuecomment-50278941):

From dbadca9, ShellCheck will check for the most basic form of variable references in literal assignments in PS1/PROMPT_COMMAND/trap. 

It's a very naive solution but it reduces some obvious false positives, including this one.
