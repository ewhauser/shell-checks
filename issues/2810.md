#### Here's a snippet or screenshot that shows the problem:

ShellCheck warned me about this:
```bash
#!/bin/bash
[ -z "$(echo a | grep a)" ] && echo true
```
```
In test.sh line 1:
[ -z "$(echo a | grep a)" ] && echo true
  ^-- SC2143 (style): Use ! grep -q instead of comparing output with [ -z .. ].
```

so I followed the advice:
```bash
#!/bin/bash
echo a | ! grep a && echo true
```
which doesn't work:
```
Line 2: Syntax error at unexpected symbol »!«
Line 2: `echo a | ! grep a && echo true'
```
however ShellCheck doesn't recognize this issue at all:
```
No issues detected!
```

correct would be (i think, maybe depends on `pipefail` setting too):
```bash
#!/bin/bash
! echo a | grep a && echo true
```


#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2023-07-31 02:25](https://github.com/koalaman/shellcheck/issues/2810#issuecomment-1657402162):

ShellCheck unintentionally interpreted the `!` as a string command name both in the second ! in `! ! cmd` and in `foo | ! bar`. This has now been fixed, and warnings have been added for each shell that doesn't support one or both. Thanks!
