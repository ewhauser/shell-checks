# [\#667 Issue](https://github.com/koalaman/shellcheck/issues/667) `closed`: SC2103 not silenced by `set -e`

#### <img src="https://avatars.githubusercontent.com/u/1397507?v=4" width="50">[me-and](https://github.com/me-and) opened issue at [2016-05-13T14:10:01Z](https://github.com/koalaman/shellcheck/issues/667):

Using `set -e` silences [SC2164](https://github.com/koalaman/shellcheck/wiki/SC2164), on the grounds that with `set -e`, the `cd` command cannot fail. However [SC2103](https://github.com/koalaman/shellcheck/wiki/SC2103) is not silenced by `set -e`, despite the rationale being effectively the same.

#### <img src="https://avatars.githubusercontent.com/u/1397507?v=4" width="50">[me-and](https://github.com/me-and) commented at [2016-05-13T14:14:11Z](https://github.com/koalaman/shellcheck/issues/667#issuecomment-219054966):

Simple example code that generates SC2103 and shouldn't (fixing with a subshell means setting `var` is lost, and fixing by switching to `cd "$1" || exit` or similar introduces redundant code thanks to the `set -e` line):

``` bash
#!/usr/bin/env bash
set -e

cd "$1"
var=$(do_stuff "$2")
cd -

do_other_stuff "$var"
```
