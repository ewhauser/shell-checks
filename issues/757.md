# [\#757 Issue](https://github.com/koalaman/shellcheck/issues/757) `open`: Add optional check for unbound variable usage which could fail with 'set -u'

#### <img src="https://avatars.githubusercontent.com/u/179397?v=4" width="50">[srcshelton](https://github.com/srcshelton) opened issue at [2016-10-24T22:36:04Z](https://github.com/koalaman/shellcheck/issues/757):

Even though [opinions are divided](http://mywiki.wooledge.org/BashFAQ/112) over the use of `set -u`, it would be helpful if it were possible to flag cases of variables being used (potentially) undefined.

(In a trivial implementation, this could be as simple as detecting `set -u` and then warning of `test -n $var`-type constructs instead of `test -n ${var:-}`, etc.)
