As echo is an external command you can not guarantee that it understands -E or that -E behaves the same on different systems.

See for example https://pubs.opengroup.org/onlinepubs/9799919799/utilities/echo.html.

#### <img src="https://avatars.githubusercontent.com/u/11627708?v=4" width="50">[e-kwsm](https://github.com/e-kwsm) commented at [2025-12-09 18:12](https://github.com/koalaman/shellcheck/issues/3370#issuecomment-3633598150):

`echo -E` is not specified in POSIX, and not all implementations support it:

| implementation | `echo -E` |
|----------------|-----------|
| bash 5.3.8 | supported |
| dash 0.5.12 | not supported |
| ksh 2020.0.0 | not supported |
| mksh R59c | supported |
| [GNU coreutils 9.9](https://www.gnu.org/software/coreutils/manual/html_node/echo-invocation.html) | supported |
| [Free BSD 16.0](https://man.freebsd.org/cgi/man.cgi?query=echo) | not supported |

But OP clearly uses bash builtin, so the warning can be suppressed.
