#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2016-04-01T23:55:52Z](https://github.com/koalaman/shellcheck/issues/637#issuecomment-204611936):

This is definitely a useful feature, because it would allow IDEs and such to highlight bad code rather than just point to the start of it.

There is no way currently to get the end of an AST node though, only the starting point is stored. The entire parsing code would have to be refractored to include end positions as well, and then all the checks would have to be updated to emit it. It would be quite an undertaking.
