[SC1087 check does not allow dynamic variables #640](https://github.com/koalaman/shellcheck/issues/640) is related but this seems more reasonable as an exception when using `declare -n`.

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash

declare -A array=()
# shellcheck disable=SC2034 # Used through declare -n
array[key]=value
variable_name=array
declare -n variable="$variable_name[key]"
echo "$variable"
```

#### Here's what shellcheck currently says:
```sh
Line 7:
declare -n variable="$variable_name[key]"
                     ^-- SC1087 (error): Use braces when expanding arrays, e.g. ${array[idx]} (or ${var}[.. to quiet).
```

#### Here's what I wanted or expected to see:

No error.
