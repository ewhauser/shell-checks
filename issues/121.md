# [\#121 Issue](https://github.com/koalaman/shellcheck/issues/121) `closed`: Need to follow include files and check them too

#### <img src="https://avatars.githubusercontent.com/u/7063704?v=4" width="50">[nachotech](https://github.com/nachotech) opened issue at [2014-03-25T22:14:19Z](https://github.com/koalaman/shellcheck/issues/121):

shellcheck does not check included files in bash.  Here's proof:
# cat test.sh
# !/bin/bash

source include.sh

abc=0

exit 0
# cat include.sh

def=0
# shellcheck test.sh

In test.sh line 5:
abc=0
^-- SC2034: abc appears unused. Verify it or export it.
# 

shellcheck should have also warned about "def" being unused, but it did not, because it did not check the included shell script. I use numerous included scripts in my system, and this tool is not very useful since it does not check included files.

#### <img src="https://avatars.githubusercontent.com/u/1300022?v=4" width="50">[sirosen](https://github.com/sirosen) commented at [2014-05-14T18:53:12Z](https://github.com/koalaman/shellcheck/issues/121#issuecomment-43121933):

I want to highlight that this does not only fail to detect problems with included scripts, but also fails to detect that vars are being used, as long as you read the include

For example, I have a common structure in my bash scripts for sharing several functions, including

```
# Always starts unset
unset usage_message
# Send usage output, but don't necessarily exit
function usage () {
    echo "USAGE: $usage_message" >&2
}
```

if I include usage in another script, and set the usage message, as with

```
# contains the usage function
source "shared_funcs.sh"
# set the usage_message
usage_message="foo bar [-h]"
while getopts "h" option;
do
    case "$option" in
        h)
            usage
            exit 0
            ;;
    esac
done
```

I will get warnings that `usage_message` may be unused, even though if you look at what `usage` does, it is clearly using `usage_message`
