#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2022-07-20 23:47](https://github.com/koalaman/shellcheck/issues/2428#issuecomment-1190880240):

I can reproduce what I'm guessing is the same issue on x86_64 linux by running with QEmu usermode emulation. It does not seem to play well with the GHC runtime system. ShellCheck natively needs 3MB, but qemu-x86_64 needs 6300 MB to emulate it:

```
$ /usr/bin/time ./shellcheck --version > /dev/null
0.00user 0.00system 0:00.00elapsed 92%CPU (0avgtext+0avgdata 2640maxresident)k
0inputs+0outputs (0major+143minor)pagefaults 0swaps

$ /usr/bin/time qemu-x86_64-static ./shellcheck --version > /dev/null
4.36user 1.76system 0:06.12elapsed 100%CPU (0avgtext+0avgdata 6307232maxresident)k
0inputs+0outputs (0major+1663829minor)pagefaults 0swaps
```

I don't know GHC or QEmu internals, but I know GHC RTS likes to take advantage of its 64bit memory space by allocating 1TB up front. I can imagine QEmu guessing QEmu would use some proportional amount of physical memory to keep track of its emulated pages.

To test this theory, I wrote this test program:

```
#include <stdlib.h>
#include <sys/mman.h>
#include <stdint.h>

void main(int argc, char** argv) {
  int64_t size = atol(argc > 1 ? argv[1] : "0");
  mmap(0, size, PROT_READ|PROT_WRITE, MAP_PRIVATE|MAP_ANONYMOUS|MAP_NORESERVE, -1, 0);
}
```

and ran

```
/usr/bin/time -f %M qemu-x86_64-static ./foo $((1*1024**3))
```

When reserving 1 byte, the memory usage was 7MB. When reserving 1GB, it was 13MB. 2GB was 19MB, suggesting 6MB memory per 1GB emulated virtual space. Indeed, this held up to 1024GB, causing QEmu to spend 6300MB, which is the same as running ShellCheck with GHC's 1TB allocation.

I'm not sure how to disable this behavior by default, but the RTS will make do with less virtual space if the initial allocation fails. With an `ulimit -v 1000000` for qemu, the GHC  allocates less, and therefore QEmu needs less memory to emulate it:

```
$ ( ulimit -v 1000000; /usr/bin/time -f %M qemu-x86_64-static ./shellcheck  --version )
ShellCheck - shell script analysis tool
version: 0.8.0
license: GNU General Public License, version 3
website: https://www.shellcheck.net
13652
```

Is there any way to use this information to get the x86_64 Docker image working on the M1?
