#### Feature suggestion

Over time, some `shellcheck` issue-IDs have changed scope, or the code following a particular directives in given scripts could have been updated.

It would be helpful if a `# shellcheck disable=` directive appearing above a script-block which wouldn't have triggered the disabled directive in the first place could note the use of an unneeded directive.

This would be great. The Python mypy project has something similar.

They have a command line option:
```
  --warn-unused-ignores     Warn about unneeded '# type: ignore' comments (inverse: --no-warn-unused-ignores)
```
