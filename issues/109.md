# [\#109 Issue](https://github.com/koalaman/shellcheck/issues/109) `closed`: Incorrectly marking redirect?

#### <img src="https://avatars.githubusercontent.com/u/1575546?v=4" width="50">[Sepero](https://github.com/Sepero) opened issue at [2014-03-09T21:52:56Z](https://github.com/koalaman/shellcheck/issues/109):

In my script I have a redirect. Here is shell check's report.

poff 2>&1> /dev/null
                       ^––
SC2069 The order of the 2>&1 and the redirect matters. The 2>&1 has to be last.

I want both stdout and stderr to both go to /dev/null. I think shell check might misunderstand what is happening here?

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2014-03-09T22:00:28Z](https://github.com/koalaman/shellcheck/issues/109#issuecomment-37141225):

If you try it out, you'll find that Shellcheck is right. 

You have to use `> /dev/null 2>&1` to get the effect you want.
