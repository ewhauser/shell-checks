#### <img src="https://avatars.githubusercontent.com/u/302055?v=4" width="50">[deryni](https://github.com/deryni) commented at [2016-01-22T16:09:03Z](https://github.com/koalaman/shellcheck/issues/579#issuecomment-173963741):

How is that better? That leaves the value of `$CONN_STRING` unquoted in its final expansion (so still isn't space or glob safe) and means you _need_ to use `$COMMAND` unquoted because it now contains the command and (at least) two arguments.

You _can't_ use a string for multiple arguments safely. It can't be done. You need an array for this.
