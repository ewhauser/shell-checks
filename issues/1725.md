The existence of the IFS setting in this case is explicitly to split
the PATH variable into its list of directory elements and the

   $(IFS=<character>; echo $VARIABLE) 

idiom is fast and easy to use in shell to split the VARIABLE on
each <character>.  I have also seen it used for directory pathnames
with IFS=/ to get all of the directory elements separated from each
other.

There are a LOT of other circumstances where the SC2086
warning is valuable, so I would rather not just disable it all the
time.

Suggesting use of an external program like this:

for list in  $(echo "$PATH" | sed 's,:, ,g'); do cmd; done

forces reliance on an external program (sed) being run which
is not really desirable when the functionality is already in
all shell implementations.

#### Here's a snippet or screenshot that shows the problem:

```sh

#! /bin/sh

prog=$1
for list in $(IFS=:; echo $PATH)
do
    [ -e $list/"$prog" ] && echo "$list/$prog exists."
done

```

The suggested workaround does not actually work and WILL BREAK
existing functionality if blindly implemented. 
Internal Field Separator (IFS) is used for word splitting.
The job of IFS=: in this case is to cause the sub-shell to
split the words of $PATH on the colon character into
words for the for loop to use.

Consider the following simple test case for yourself:

$ cat /tmp/i.sh ; sh /tmp/i.sh
#!/bin/sh

count=0
for list in $(IFS=:; echo $PATH); do
  count=$((count + 1))
  echo "Non-quoted loop $count has list=$list"
done

count=0
for list in $(IFS=:; echo "$PATH"); do
  count=$((count + 1))
  echo "Quoted loop $count has list=$list"
done

Non-quoted loop 1 has list=/usr/local/bin
Non-quoted loop 2 has list=/usr/local/sbin
Non-quoted loop 3 has list=/opt/local/bin
Non-quoted loop 4 has list=/opt/local/sbin
Non-quoted loop 5 has list=/usr/local/MacGPG2/bin
Non-quoted loop 6 has list=/opt/X11/bin
Non-quoted loop 7 has list=/usr/bin
Non-quoted loop 8 has list=/bin
Non-quoted loop 9 has list=/usr/sbin
Non-quoted loop 10 has list=/sbin
Non-quoted loop 11 has list=/Users/mdb/bin
Quoted loop 1 has list=/usr/local/bin:/usr/local/sbin:/opt/local/bin:/opt/local/sbin:/usr/local/MacGPG2/bin:/opt/X11/bin:/usr/bin:/bin:/usr/sbin:/sbin:/Users/mdb/bin
$ 

As you can see, the Quoted loop is dis-functional for the intended purpose.
