#### Here's a snippet or screenshot that shows the problem:

```sh
#!/usr/bin/sh
echo "Case:\tTrue positive"
echo "Case:\e[41mFalse negative"
```

#### Here's what shellcheck currently says:
```
Line 3:
echo "Case:\tTrue positive"
     ^-- SC2028 (info): echo may not expand escape sequences. Use printf.
```

#### Here's what I wanted or expected to see:
```
Line 3:
echo "Case:\tTrue positive"
     ^-- SC2028 (info): echo may not expand escape sequences. Use printf.

Line 5:
echo "Case:\e[41mFalse negative"
     ^-- SC2028 (info): echo may not expand escape sequences. Use printf.
```

#### Relevant code

The code only checks for `-r`, `-n`, `-t`.
However there are many more escapes like `\e`, `\0xxx`, `\a` etc.
I think shellcheck should warn on any of those escape unless `echo -e` is used as there is a high probability that they are erroneous.
