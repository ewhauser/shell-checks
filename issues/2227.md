#### Here's a snippet or screenshot that shows the problem:

```sh

#!/bin/sh

test_file='foo.bar'
[ -f "$test_file" ] || /
	echo error!
```

#### Here's what shellcheck currently says:
$ shellcheck myscript
No issues detected!


#### Here's what I wanted or expected to see:
The use case above is for wrapping long lines with `\` but I incorrectly used `/` and `shellcheck` accepted it.

In `/bin/sh` this is an error:
```
$ / || echo $? 
sh: 1: /: Permission denied
126
```
AFAIK, in order to use `\` for line wrap the `\` MUST be the last character on the line, so it seems like a reasonably easy check for `shellcheck` to detect.  I may be wrong, of course ..

`shellcheck` should at least warn for this because it took me an hour to spot my stupid mistake.

Thanks ;-)
