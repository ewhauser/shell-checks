#### <img src="https://avatars.githubusercontent.com/u/1509080?v=4" width="50">[jFransham](https://github.com/jFransham) commented at [2016-11-28T16:30:26Z](https://github.com/koalaman/shellcheck/issues/780#issuecomment-263319089):

The parser treats `"${#a}"` as braces containing `#a` and then tries to look that up as a variable name. This is also true of `"${a:-1}"`, for example. This would require adding a fairly complex bash extension parser. You could probably get away with having a shell-specific parse tree transformation function (but keep the tree data structure the same across shells).

This would probably look something like

```haskell
data Token =
    T_Binary Id String Token Token
    -- ...snip...
    -- If Bash/Fish/Zsh/whatever shells have different string length semantics
    -- then we can have seperate data for each
    | T_StringLen Id Token
    | T_Default Id Token Token
    -- plus whatever else
    -- ...snip...

-- Actual implementation will be more complex, obviously
parseBashExts :: Token -> Maybe Token
parseBashExts (T_DollarBraced iOuter (T_Literal iInner ('#':rest))) =
    Just . T_DollarBraced iOuter . T_Literal iInner . T_StringLen $ T_Literal whateverId rest
parseBashExts _ = Nothing
```

You'd then run `parseBashExts` for each leaf in some way if you use `-s bash` or it's detected to be a bash script some other way and pass the result to the analytics functions, which would then handle it somehow.
