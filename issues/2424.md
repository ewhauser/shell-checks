#### Here's a snippet or screenshot that shows the problem:

code snippet.sh
```sh
#!/usr/local/env bash
trap 'echo "trapped"; exit 1' KILL
while true; do { echo "working"; sleep 1; } done
```

output after starting the snippet.sh, and killing it after a few seconds (from another terminal):
```sh
$> ./snippet.sh 
working
working
working
working
trapped
```

#### Here's what shellcheck currently says:

https://github.com/koalaman/shellcheck/wiki/SC2173
In particular: "…SIGKILL and SIGSTOP can not be caught…"

#### Here's what I wanted or expected to see:

According to the rationale of SC2173, I expected the snipped to be killed without the kill being trap'ed. If so, the last line of output should be 'working'. Instead, it is 'trapped' which means the trap is in fact recognized and the trap routine runs.

Is this a new behaviour of bash version 5.1.12(1)-release (x86_64-apple-darwin19.6.0), or is this the result of the wording "Trying to trap this signal has undefined results."?

Thank you.
