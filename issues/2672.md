#### <img src="https://avatars.githubusercontent.com/u/25598447?u=73c3f3a041be0ccabd72bb816d5782b30aefe822&v=4" width="50">[TinCanTech](https://github.com/TinCanTech) commented at [2023-01-24 14:28](https://github.com/koalaman/shellcheck/issues/2672#issuecomment-1402039951):

The way `shellcheck` expect this to be written is:
```
# shellcheck disable=2016
echo "a" && echo "b"
```
You are trying to insert a `shellcheck` directive into an effective single line of code. a.k.a. pipeline. eg:
```
echo "a" && # shellcheck disable=2016 echo "b"
```
Which, as you can see, is simply not going to work.

#### <img src="https://avatars.githubusercontent.com/u/25598447?u=73c3f3a041be0ccabd72bb816d5782b30aefe822&v=4" width="50">[TinCanTech](https://github.com/TinCanTech) commented at [2023-01-24 14:40](https://github.com/koalaman/shellcheck/issues/2672#issuecomment-1402057152):

`shellcheck` analyses the line as it sees it from the shell's perspective.

In your first case, the comment is a directive instructing `shellcheck` how to behave but you are not using `shellcheck` directives correctly. `shellcheck` cannot apply a directive to a partial part of a pipeline.

In your second, the comment is simply a comment with no additional effects.

I guess `shellcheck` could be enhanced for your use case but that would probably be a monstrous challenge.
