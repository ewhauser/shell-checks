# [\#287 Issue](https://github.com/koalaman/shellcheck/issues/287) `open`: Propose -- to separate parameters from options and have all parameters last

#### <img src="https://avatars.githubusercontent.com/u/3432344?v=4" width="50">[Gunstick](https://github.com/Gunstick) opened issue at [2015-01-19T14:37:38Z](https://github.com/koalaman/shellcheck/issues/287):

I did a test calling this:
     exec qrencode -o - "${PATH_INFO#/}${QUERY_STRING}" 
it did not get flagged as insecure. But it is, as qrencode can take some parameters like "-o filename"
So in general, putting parameters after options is a good idea and separate them with --
     exec qrencode -o - -- "${PATH_INFO#/}${QUERY_STRING}" 

For those who are not aware of the problem, try this:
     ls /etc -l
