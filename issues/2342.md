That syntax is valid for `ksh` to access map elements (nested namespaces):
```lang=ksh
t=( foo=bar )
echo "${t.foo}"
```
Output:
> `bar`

It should be a warning on any other shell.

> It isn't valid at least in `bash` and `bysybox ash`

As I already said, it's invalid in anything except `ksh`.

Busybox's built-in shell is so broken that it would be an enormous separate effort to identify all the uses of standard posix features that don't work in it. (Personally I don't think that effort would be worth it.)

Although the busybox shell is sometimes referred to as `ash`, that's misusing the name of the real `ash`, which is a separate shell that is mostly POSIX compliant, though still somewhat limited.
