# [\#732 Issue](https://github.com/koalaman/shellcheck/issues/732) `closed`: `shopt -s lastpipe` should disable SC2030/SC2031 appropriately

#### <img src="https://avatars.githubusercontent.com/u/160894?v=4" width="50">[CyberShadow](https://github.com/CyberShadow) opened issue at [2016-09-06T00:23:46Z](https://github.com/koalaman/shellcheck/issues/732):

ShellCheck correctly warns on this snippet (from the SC2031 wiki):

``` bash
n=0
printf "%s\n" {1..10} | while read i; do (( n+=i )); done
echo $n
```

However, the code will behave as expected if `shopt -s lastpipe` is used, making the warning spurious.

Perhaps ShellCheck should detect this option is in use and stop treating the last command in a pipeline as a subshell, in the same way that `set -e` disables `cd` warnings.
