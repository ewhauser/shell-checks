Got timeline cleared:
In Apr 5, 2020, shellcheck [v0.7.1](https://github.com/koalaman/shellcheck/releases/tag/v0.7.1) released
In Oct 24, 2020, support for `SC2155 readonly`  [added](https://github.com/koalaman/shellcheck/commit/619662adb63e027866a7511e695b87d8ca069654)
In 5 days ago,  [community/shellcheck](https://github.com/archlinux/svntogit-community/blob/packages/shellcheck/trunk/PKGBUILD) using shellcheck v0.7.1 source code, which has **no** support for `SC2155 readonly`.
In today, shellcheck [v0.7.2](https://github.com/koalaman/shellcheck/releases/tag/v0.7.2) released which support `SC2155 readonly`
In today, [aur/shellcheck-bin](https://aur.archlinux.org/cgit/aur.git/tree/PKGBUILD?h=shellcheck-bin) is using v0.7.2 support `SC2155 readonly`.
In today, my editor using v0.7.2 generate different shellcheck output.

To be brief:
**SC2155 readonly support(Oct 24, 2020) is not in v0.7.1(Apr 5, 2020)**

#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2021-04-20 06:33](https://github.com/koalaman/shellcheck/issues/2210#issuecomment-823012630):

v0.7.1 is a specific release with a specific feature set. It's not expected that a v0.7.1 package built later contains additional features. When a distro releases a "new" revision of 0.7.1, it's most likely the same version just rebuilt with updated library dependencies. 

If you are wondering when an official 0.7.2 package will be released for your distro, this is up to various volunteers working on the distro (thanks everyone!). Since 0.7.2 was released to Hackage today, give it at least a few weeks.
