# [\#2571 Issue](https://github.com/koalaman/shellcheck/issues/2571) `open`: SC2319 - confused warning message when checking $? inside a function

#### <img src="https://avatars.githubusercontent.com/u/113245227?u=f54fd65389f67ab29cc88f36b8e66c2bd0795be7&v=4" width="50">[mgmsam](https://github.com/mgmsam) opened issue at [2022-09-10 23:44](https://github.com/koalaman/shellcheck/issues/2571):

Hi all!!

Conducted a test on the site https://www.shellcheck.net
I report that this problem is repeated inside a function:

**Here's a snippet or screenshot that shows the problem:**

```
#!/bin/bash
print_codout() {
	RETURN=$?
	printf '%s\n' "$RETURN"
}
[ -f foo ] || print_codout
```

**Here's what shellcheck currently says:**
```
$ shellcheck myscript

Line 3
RETURN=$?
       ^-- SC2319 (warning): This $? refers to a condition, not a command. Assign to a variable to avoid it being overwritten.
```
