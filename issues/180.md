# [\#180 Issue](https://github.com/koalaman/shellcheck/issues/180) `closed`: Bash improper warning 2068 for left hand side of double bracket

#### <img src="https://avatars.githubusercontent.com/u/54232?v=4" width="50">[ian-kelling](https://github.com/ian-kelling) opened issue at [2014-06-17T06:09:57Z](https://github.com/koalaman/shellcheck/issues/180):

```
[[ ${forward[@]: -1} == foo ]]
      ^-- SC2068: Double quote array expansions, otherwise they're like $* and break on spaces.
```

afaik, it is proper to not quote in this circumstance

#### <img src="https://avatars.githubusercontent.com/u/1693206?v=4" width="50">[knirch](https://github.com/knirch) commented at [2014-06-17T10:47:58Z](https://github.com/koalaman/shellcheck/issues/180#issuecomment-46292237):

nope, it's not proper, see example here:

``` bash
a=(foo "ba  r")
[[ "${a[@]: -1}" == "ba  r" ]] && echo 1
[[ ${a[@]: -1} == "ba  r" ]] && echo 2
```
