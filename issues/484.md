testing for POSIX (not bash!) i get for
line1:
SC2089 Quotes/backslashes will be treated literally. Use an array.
line2:
SC2090 Quotes/backslashes in this variable will not be respected.

```
#!/bin/sh
VAR="A='1'"
test ${#VAR} -eq 32
```

ofcourse i cannot use arrays here, posix has no knowledge of it.
the second error is only because of the first one und should also
be muted. (the reason for this strange but valid var is a later eval)

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2015-10-01T20:07:26Z](https://github.com/koalaman/shellcheck/issues/484#issuecomment-144833424):

This is definitely a bug. shellcheck should realize that `${#VAR}` does not contain quotes even if VAR does.

You could also have quoted `"${#VAR}"` as a workaround.
