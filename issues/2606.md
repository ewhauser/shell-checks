# [\#2606 Issue](https://github.com/koalaman/shellcheck/issues/2606) `open`: SC2009 recommends away from a POSIX way to a non-POSIX way

#### <img src="(unknown)" width="50">[(unknown)]((unknown)) opened issue at [2022-10-14 08:44](https://github.com/koalaman/shellcheck/issues/2606):

#### For bugs
- Rule Id (if any, e.g. SC2009): 
- [x] The rule's wiki page does not already cover this (e.g. https://shellcheck.net/wiki/SC2009)

Shellcheck currently recommends to use `pgrep` instead of `ps | grep`. That's fine... if one's on a system with pgrep, but pgrep is not POSIX and not everyone has pgrep (for example [AIX](https://www.ibm.com/docs/en/ssw_aix_73/pdf/commands2.pdf)).

See also #2449 in which SC2012 does the same.

I'm not sure what the project's stance is recommending the user away from POSIX. I'm inclined to think it should not, particularly if one is targeting /bin/sh or /bin/ksh, but at the very least the wiki should make a note that the error should be ignored if POSIX compliance is important or if the target system lacks the command/option.
