# [\#1887 Issue](https://github.com/koalaman/shellcheck/issues/1887) `open`: Feature Request: detect set -u / nounset and unset variables first used 

#### <img src="https://avatars.githubusercontent.com/u/2211051?u=7747a673d98078d50467ba6e4d4f3a2f17b6db17&v=4" width="50">[HariSekhon](https://github.com/HariSekhon) opened issue at [2020-03-24 16:59](https://github.com/koalaman/shellcheck/issues/1887):

Would it be possible to check when `set -u` (nounset) is enabled and a previously undefined variable is used, for example, this would fail only at runtime and is not currently detected by shellcheck even though this is simple and common error:

```
#!/usr/bin/env bash

set -u

if [ -z "$MYVARIABLE" ]; then
...
```

which fails like so:

```
./test.sh: line 5: MYVARIABLE: unbound variable
```

when it should be written like this for safety:

```
#!/usr/bin/env bash

set -u

if [ -z "${MYVARIABLE:-}" ]; then
...
```
