# [\#1107 Issue](https://github.com/koalaman/shellcheck/issues/1107) `open`: SC2034 is not muted by valid [[ -v ... ]] tests against associative arrays

#### <img src="https://avatars.githubusercontent.com/u/10137?v=4" width="50">[ghost](https://github.com/ghost) opened issue at [2018-02-05T06:51:27Z](https://github.com/koalaman/shellcheck/issues/1107):

The following code triggers SC2034 in version 0.4.7 and the online version.

```sh
#!/bin/bash
declare -A var=([a]="")
for x in a b c; do
        [[ -v "var[$x]" ]]
        echo $?
done
```

My expectation would be that it does not do so, because the test for an existing key is valid, and obviously references the variable in question.
