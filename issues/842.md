# [\#842 Issue](https://github.com/koalaman/shellcheck/issues/842) `closed`: SC2034 false positive when passing array variable names as function parameters

#### <img src="https://avatars.githubusercontent.com/u/2199511?v=4" width="50">[Nightfirecat](https://github.com/Nightfirecat) opened issue at [2017-03-02T18:31:23Z](https://github.com/koalaman/shellcheck/issues/842):

In shellcheck 0.4.5, SC2034 is raised when passing array variable names as function parameters. This is done so an array variable can be expanded within the called function, effectively letting the name (`$1` in the function) be double-expanded.

```bash
cat << EOF > test.sh
#!/bin/bash
function print_array {
  array=("\${!1}")
  echo "\${array[@]}"
}
test_array=( 1   2   3 )
print_array test_array[@]
EOF
bash test.sh
shellcheck test.sh
```

#### Here's the current output:

```
1 2 3

In test.sh line 6:
test_array=( 1   2   3 )
^-- SC2034: test_array appears unused. Verify it or export it.
```

#### Here's what I wanted or expected to see:

```
1 2 3
```

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2018-08-16T02:21:44Z](https://github.com/koalaman/shellcheck/issues/842#issuecomment-413402392):

You're right of course. Figuring out which variables are used through indirection is in general really hard, and even in this simple case, the array is unused if you have a file `test_array@` in the current directory. 

Something like `read a; echo "${!a}"` would technically have to disable all unused warnings to be technically correct, and then you have things like `read a; [[ $a == t*y* ]] && echo "${!a}"` which would technically have to match variables like `test_array` by pattern. 

Rather than going down that rabbit hole, ShellCheck's policy is to simply not try to figure out any indirect references. This includes trivial cases, to avoid giving the impression that it should be working. Please just ignore the warnings when you know the variables will or may be used indirectly.
