#### <img src="https://avatars.githubusercontent.com/u/302055?v=4" width="50">[deryni](https://github.com/deryni) commented at [2015-09-25T17:13:18Z](https://github.com/koalaman/shellcheck/issues/474#issuecomment-143290858):

Are you expecting shellcheck to attempt to divine whether the command in your `$(...)` substitution is going to return a value that is valid for the `[ ...]` test it is in here?

The syntax, and style, here are valid. The problem is that your logic isn't.

You are attempting (I think) to compare the _return code_ of `ifconfig` against the value `0` but you are, in fact, comparing the _output_ of `ifconfig` against `0` (and also directing all output to `/dev/null` so you are, in fact, comparing the empty string against `0`).

The only thing that shellcheck could reasonably do here (and I think there is a request for this) would be to warn you about the fact that your `$(...)` substitution cannot generate any output and will always result in the empty string.
