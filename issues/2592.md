```sh
#!/usr/bin/env bash

```

#### Here's what I wanted or expected to see:

> Split multiple commands into separate lines for clarity.

[Rationale](https://paperless.blog/bash-lints-beyond-shellcheck): Consistently lining commands up vertically makes it easier to understand the context of each command. If two commands belong “strongly” together, why not put them in a function to make that grouping explicit?
