#### <img src="https://avatars.githubusercontent.com/u/1693206?v=4" width="50">[knirch](https://github.com/knirch) commented at [2017-07-13T14:08:09Z](https://github.com/koalaman/shellcheck/issues/952#issuecomment-315088667):

The exit status of the script will be the exit status of the last command.

While mostly it's IMHO cleaner to handle unwanted exit status to with 
command || true an exit 0 is also a way to convey that the script reached a 
good exit point.

Many scripts that have a big if our case with multiple exit points want to 
have a exit 0 at the end, just to make it clearer for the reader that all 
is good.

> While mostly it's IMHO cleaner to handle unwanted exit status to with command || true an exit 0 is also a way to convey that the script reached a good exit point.

Then, we should add a warning and tell the user to use `command || true` instead of `command; exit 0`.

What do you think about the case where `set -e` is present ? IMO, in this case, there's no point at all in adding `exit 0`, so, we should warn the user for sure here.

Moreover, with `set -e`, if the user writes `command; exit 0` when he meant `command || true`, this is an error, and we could say: `remove useless "exit 0" or use "command || true"`.
