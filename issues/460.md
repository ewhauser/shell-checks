For the `echo "foo` example, ShellCheck currently says "Expected end of double quoted string" and also points to the start of the string it's referring to. While not necessarily ideal, it's relatively helpful.

Please reopen with an example where ShellCheck is not helpful (and where it's also obvious that the user doesn't intend to do what kurahaupo suggests).
