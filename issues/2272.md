#### Here's a snippet or screenshot that shows the problem:

These two are functionally identical in ksh/ksh93 and bash:

```sh
#!/usr/bin/ksh93
read -r example?"Text before user input: "
if (( "${#example}" == 0 )); then 
	printf "%s\n" "No user input"
else
	printf "%s\n" "${example}"
fi
```

```sh
#!/usr/bin/bash
printf "%s: " "Text before user input: "
read -r example
if (( "${#example}" == 0 )); then 
	printf "%s\n" "No user input"
else
	printf "%s\n" "${example}"
fi
```

#### Here's what shellcheck currently says:

Marks the line `if (( "${#example}" == 0 )); then` as SC2154, but this is not true in KSH93

#### Here's what I wanted or expected to see:

No error.  The read variable?"Prompt" format is a valid method in ksh/ksh93
