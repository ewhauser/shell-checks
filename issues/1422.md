# [\#1422 Issue](https://github.com/koalaman/shellcheck/issues/1422) `closed`: Suggest warning for unnecessary braces

#### <img src="https://avatars.githubusercontent.com/u/4327840?v=4" width="50">[bje-](https://github.com/bje-) opened issue at [2018-12-19T09:31:02Z](https://github.com/koalaman/shellcheck/issues/1422):

#### For new checks and feature suggestions
- [X ] shellcheck.net (i.e. the latest commit) currently gives no useful warnings about this
- [X ] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related

#### Here's a snippet or screenshot that shows the problem:

```sh

#!/your/interpreter
foo=1
echo ${foo}
```

#### Here's what shellcheck currently says:

Nothing.

#### Here's what I wanted or expected to see:

Unnecessary braces around $foo. Many users place unnecessary variable expansions around variable names that do not require them. For example, if the variable number contains an underscore.

#### <img src="https://avatars.githubusercontent.com/u/4204285?v=4" width="50">[orbea](https://github.com/orbea) commented at [2018-12-27T15:57:23Z](https://github.com/koalaman/shellcheck/issues/1422#issuecomment-450177951):

What is the harm in adding such braces? This is a stylistic choice and shellcheck should not dictate style.

#### <img src="https://avatars.githubusercontent.com/u/4327840?v=4" width="50">[bje-](https://github.com/bje-) commented at [2018-12-29T02:32:30Z](https://github.com/koalaman/shellcheck/issues/1422#issuecomment-450458146):

Agreed, but the extra syntactic sugar clutters the code and makes it harder to read (in most circumstances). eg `${foobar}` versus just `$foobar`. The harm is that novice shell programmers just copy what they see in existing shell code without ever understanding the quoting rules. It would be good if interested users could ask Shellcheck to warn about such style issues.

#### <img src="https://avatars.githubusercontent.com/u/4204285?v=4" width="50">[orbea](https://github.com/orbea) commented at [2018-12-29T03:04:26Z](https://github.com/koalaman/shellcheck/issues/1422#issuecomment-450460581):

Shellcheck already warns a lot about incorrect quoting and the consequence of this proposal would be countless incorrect warnings for correct code, this would not encourage novice users for writing better shell more than it would encourage more experienced users from not using shellcheck.

Additionally based upon the font, editor, personal preference and other stylistic choices made in the script you have no way of actually knowing what would be easier to read for any single person.
