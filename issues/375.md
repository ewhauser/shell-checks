# [\#375 Issue](https://github.com/koalaman/shellcheck/issues/375) `closed`: shellcheck's parser is confused about directives before a done redirect in a while loop

#### Labels: working as intended

#### <img src="https://avatars.githubusercontent.com/u/7633626?v=4" width="50">[aw-was-here](https://github.com/aw-was-here) opened issue at [2015-05-14T16:25:33Z](https://github.com/koalaman/shellcheck/issues/375):

We've hit a case where we need to disable SC2046 in a while/done that takes input from a subshell. I've been able to simplify the case to this:

``` bash
while read line
do
echo bar
 # shellcheck disable=SC2046
done <  foo
```

Running this against shellcheck 0.3.6 & 0.3.7 results in:

```
while read line
^-- SC1073: Couldn't parse this while loop.

In 6 line 5:
done < foo
     ^-- SC1072: Expected "#". Fix any mentioned problems and try again.
```

... which is a little bizarre.  Replacing the shellcheck disable with some other comment, shellcheck is fine.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2015-05-15T00:17:05Z](https://github.com/koalaman/shellcheck/issues/375#issuecomment-102207354):

Directives are per semantic shell script structure and not per line, so ShellCheck is confused when a directive is applied to half a while loop.

Can you expand on the example a bit? I don't see where the SC2046 (unquoted command expansion) would come from. 


#### <img src="https://avatars.githubusercontent.com/u/7633626?v=4" width="50">[aw-was-here](https://github.com/aw-was-here) commented at [2015-05-15T05:02:53Z](https://github.com/koalaman/shellcheck/issues/375#issuecomment-102258435):

Sure.  The original done was:

```
done < <(docker inspect -f '{{.Id}},{{.State.Running}},{{.State.FinishedAt}}' $(docker ps -qa) 2>/dev/null)
```

It complains that $(docker ps -qa)  needs to be quoted.

The content inside the while was obviously working on the output of the inspect.


#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2015-05-15T15:49:06Z](https://github.com/koalaman/shellcheck/issues/375#issuecomment-102442997):

You're right, this is a bit of an edge case. It's tricky to disable the message while also minimizing scope. 

I would just have disabled it for the entire loop:

```
# shellcheck disable=SC2046
while read line
do
  echo bar
done <  <(docker inspect -f '{{.Id}},{{.State.Running}},{{.State.FinishedAt}}' $(docker ps -qa) 2>/dev/null)
```

You can technically apply it to just the command in the process substitution, but it's awkward:

```
while read line
do
  echo bar
done <  <(
    # shellcheck disable=SC2046
    docker inspect -f  $(docker ps -qa) 2>/dev/null)
```

You obviously shouldn't have to restructure working code to please ShellCheck, but writing around it is a third option:

```
get_docker_stats() {
  # shellcheck disable=SC2046
  docker inspect -f "$1" $(docker ps -qa) 2>/dev/null
}

while read line
do
  echo bar
done <  <(get_docker_stats '{{.Id}},{{.State.Running}},{{.State.FinishedAt}}')
```
