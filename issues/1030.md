# [\#1030 Issue](https://github.com/koalaman/shellcheck/issues/1030) `closed`: "echo done" emits SC1010

#### <img src="https://avatars.githubusercontent.com/u/553208?v=4" width="50">[gpakosz](https://github.com/gpakosz) opened issue at [2017-10-18T14:39:23Z](https://github.com/koalaman/shellcheck/issues/1030):

```
$ shellcheck --version
ShellCheck - shell script analysis tool
version: 0.4.6
license: GNU General Public License, version 3
website: http://www.shellcheck.net
```

```
$ cat ./foo.sh

#!/bin/sh

echo done
```

```
$ shellcheck ./foo.sh

In ./foo.sh line 3:
echo done
     ^-- SC1010: Use semicolon or linefeed before 'done' (or quote to make it literal).
```

I know this is trivially silenced by replacing `echo done` with `echo "done"` but I wanted to let you know 😀



#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2017-10-26T16:59:55Z](https://github.com/koalaman/shellcheck/issues/1030#issuecomment-339731735):

This is intended behavior, and I've edited to wiki to clarify. Your code obviously works perfectly fine here, but `while true; do echo foo done` and similar is a pretty common mistake, so ShellCheck warns anyways.
