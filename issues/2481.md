#### <img src="https://avatars.githubusercontent.com/u/243351?v=4" width="50">[brother](https://github.com/brother) commented at [2022-04-08 09:18](https://github.com/koalaman/shellcheck/issues/2481#issuecomment-1092639636):

Either escape the dollar sign or enclose the EOF in single quotes to avoid variable expansion.
Slim example of start, shellcheck complains about this.
```shell
brother ~$ cat /tmp/test.sh
#!/bin/sh
cat <<EOF
string
$another
EOF
brother ~$ /tmp/test.sh 
string

```

Using single quotes.
```shell
brother ~$ cat /tmp/test.sh
#!/bin/sh
cat <<'EOF'
string
$another
EOF
brother ~$ /tmp/test.sh 
string
$another
```

Or escaping the "variable"
```shell
brother ~$ cat /tmp/test.sh
#!/bin/sh
cat <<EOF
string
\$another
EOF
brother ~$ /tmp/test.sh 
string
$another
```

And ofc setting the variable to something will silence the warning but also make some unintentional changes in the process.

```shell
brother ~$ cat /tmp/test.sh
#!/bin/sh
another="some text and stuff"
cat <<EOF
string
$another
EOF
brother ~$ /tmp/test.sh 
string
some text and stuff
```

Just to expand on brother's response: this is not a false positive. Variables within heredocs are expanded unless escaped or the whole heredoc is single quoted.
