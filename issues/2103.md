#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2020-12-31 21:24](https://github.com/koalaman/shellcheck/issues/2103#issuecomment-753182226):

This was a surprisingly interesting set of bugs: 1. The shebang should have given a carriage return warning, but it was being delayed and eventually discarded, 2. Bash allows carriage returns to be part of the token, so this code is actually valid with the carriage returns. ShellCheck did not support this. 3. The SC1042 error should have given a helpful `Close matches include 'EOF\r' (!= 'EOF').` but it failed to escape the carriage return, so the message was unreadable.

All these issues have now been fixed. Thanks!
