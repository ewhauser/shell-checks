# [\#1209 Issue](https://github.com/koalaman/shellcheck/issues/1209) `closed`: declare -x -f my_function is mistaken for an unassigned variable

#### <img src="https://avatars.githubusercontent.com/u/4683437?v=4" width="50">[per-bohlin](https://github.com/per-bohlin) opened issue at [2018-05-09T12:46:38Z](https://github.com/koalaman/shellcheck/issues/1209):

#### For bugs
- Rule Id: SC2154
- version: 0.4.6
- [x ] I tried on shellcheck.net and verified that this is still a problem on the latest commit

#### Here's a snippet or screenshot that shows the problem:

```sh

#!/bin/bash

my_expensive_function() {
    echo "my super expensive long running function"
}

declare -x -f my_expensive_function

nohup /bin/bash -c "my_expensive_function" &>> out.txt < /dev/null &
```

#### Here's what shellcheck currently says:
```
declare -x -f my_expensive_function
              ^-- SC2154: my_expensive_function is referenced but not assigned.
```


#### Here's what I wanted or expected to see:
I would assume exporting an existing function to be valid in bash (running bash 4.3.48).

#### <img src="https://avatars.githubusercontent.com/u/3129206?v=4" width="50">[matthewpersico](https://github.com/matthewpersico) commented at [2018-05-09T13:18:35Z](https://github.com/koalaman/shellcheck/issues/1209#issuecomment-387735402):

s/declare -x -f my_expensive_function/export -f my_expensive_function/

declare by itself shows all variables and functions in the env
declare -f means show only the functions in the env
declare -x marks the variable for export
Used together, not sure what -x and -f so when used together
declare does not export

I think...

#### <img src="https://avatars.githubusercontent.com/u/4683437?v=4" width="50">[per-bohlin](https://github.com/per-bohlin) commented at [2018-05-10T06:25:56Z](https://github.com/koalaman/shellcheck/issues/1209#issuecomment-387964356):

At least in bash 4.3.38 on ubuntu 16.04 it does.

declare -f 
lists all functions, including definition

declare -f my_function
lists my_function only, including definition

declare -x -f my_function
doesn't print anything but exports the symbol "my_function" and hence the function body.

To try it out.

```
$ foo() { echo "foo"; }
$ bash -c "foo"
bash: foo: command not found
$ declare -f foo
foo () 
{ 
    echo foo
}
$ bash -c "foo"
bash: foo: command not found
$ declare -x -f foo
$ bash -c "foo"
foo
```
