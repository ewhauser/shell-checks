# [\#1019 Issue](https://github.com/koalaman/shellcheck/issues/1019) `closed`: SC2034 false positive

#### <img src="https://avatars.githubusercontent.com/u/4327840?v=4" width="50">[bje-](https://github.com/bje-) opened issue at [2017-09-28T20:10:22Z](https://github.com/koalaman/shellcheck/issues/1019):

- Rule SC2034 (online)

### Here's a snippet that shows the problem:

```
#!/bin/bash

declare -A foo
foo[123]=456
arr=foo
var="$arr[123]"
echo "${!var}"
```

#### Here's what shellcheck currently says:

Line 4:
foo[123]=456
^-- SC2034: foo appears unused. Verify it or export it.

#### Here's what I wanted or expected to see:

Nothing. I have several associative arrays and want to choose between them at runtime.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2017-09-29T20:04:48Z](https://github.com/koalaman/shellcheck/issues/1019#issuecomment-333225642):

ShellCheck intentionally does not try to work out whether or not variables are used through indirection. See #1017 for more about this.

For scripts relying heavily on this, consider disabling the warning script wide.
