#### Here's a snippet or screenshot that shows the problem:

```sh

#!/bin/bash
PID="$(ps -o pid,args -ww -p $(pgrep java) | grep com.ibm.is.log4j.SpaceChecker | grep normalcycle)"

```

#### Here's what I wanted or expected to see:
SC2009 is not really an issue: the $(pgrep java) list the pids I want, making the main concern of SC2009 (to get the wrong process type) to be no longer an issue. In this example, I want to look for java processes that run a specific class and have a specific parameter.



#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2022-10-13 03:26](https://github.com/koalaman/shellcheck/issues/2597#issuecomment-1276977514):

It's too bad `pgrep` doesn't have a `-p pidlist` flag. It would allow arbitrarily nested refinements like `pgrep -p "$(pgrep -p "$(pgrep java)" -f com.ibm.is.log4j.SpaceChecker)" -f normalcycle`. Instead, you no longer get a warning for `ps -p | grep`. Thanks!
