#### Here's a snippet or screenshot that shows the problem:

**Note all these files must be in the same dir.**

File: one.sh
```sh
#!/bin/bash
declare -g var1;
var1='abc';
```

File: two.sh
```sh
#!/bin/bash
declare -g var2;
var2='def';
```

File: script.sh
```sh
#!/bin/bash
# shellcheck source=one.sh
# shellcheck source=two.sh
. one.sh
. two.sh
echo "${var1}${var2}";
```

```
             ^-----^ SC2154 (warning): var2 is referenced but not assigned (did you mean 'var1'?).

For more information:
  https://www.shellcheck.net/wiki/SC2154 -- var2 is referenced but not assign...
```

#### Here's what I wanted or expected to see:

No output.

#### Additional notes:

* `script.sh` runs fine and prints "abcdef" as expected.
* If I reverse the `shellcheck source` lines in `script.sh` so that `two.sh` is on top and `one.sh` is on the bottom, I get the reverse warning - now var1 is seen as referenced but not assigned rather than var2. The script still runs correctly and gives the expected output.

#### <img src="https://avatars.githubusercontent.com/u/178350297?v=4" width="50">[nidorhuang](https://github.com/nidorhuang) commented at [2024-08-14 02:46](https://github.com/koalaman/shellcheck/issues/2989#issuecomment-2287731205):

I bumped into the same issue days before. However I figured out that the correct usage is to place the `# shellcheck source=sourced_file` right before each `source sourced_file` command, respectively. The example below works for me.

```
# shellcheck source=sourced_file1
source sourced_file1
# shellcheck source=sourced_file2
source sourced_file2
# shellcheck source=sourced_file3
source sourced_file3
```

@ArrayBolt3
Note that shellcheck will use the line `# shellcheck source=sourced_file1` by looking of the current folder where you are running the shellcheck command.

So if you have the files inside dir1 and run a command like this
```
shellcheck dir1/file1
```
it will fail instead with this will succeed:
```
cd dir1
shellcheck file1
```

To make it less error prone and just ignore the current folder you can put this in the file:
```
# shellcheck source=SCRIPTDIR/sourced_file1
source sourced_file1
```
to use the folder where the files is, instead of the current folder.
