# [\#3243 Issue](https://github.com/koalaman/shellcheck/issues/3243) `closed`: Support Bash 5.3's new alternate form of command substitution

#### <img src="https://avatars.githubusercontent.com/u/11473063?u=5e4318223971036462166a25a5e61386bfbd414a&v=4" width="50">[jansorg](https://github.com/jansorg) opened issue at [2025-07-09 18:07](https://github.com/koalaman/shellcheck/issues/3243):

Bash 5.3 introduced a new, alternate form of command substitution. See https://www.gnu.org/software/bash/manual/bash.html#Command-Substitution for details.

Example:

```bash
${ local X=12345 ; echo $X; }

${| REPLY=12345; }
```

After looking at the commit, I wasn't sure if the allowed whitespace includes tab and newline. According to the Bash page,both are supported.

Quoting from the page linked above:
> The character c following the open brace must be a space, tab, newline, or ‘|’, and the close brace must be in a position where a reserved word may appear (i.e., preceded by a command terminator such as semicolon). Bash allows the close brace to be joined to the remaining characters in the word without being followed by a shell metacharacter as a reserved word would usually require.
