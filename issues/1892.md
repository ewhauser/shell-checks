# [\#1892 Issue](https://github.com/koalaman/shellcheck/issues/1892) `closed`: Do something with the monstrosity that is delve

#### <img src="https://avatars.githubusercontent.com/u/16061266?v=4" width="50">[josephcsible](https://github.com/josephcsible) opened issue at [2020-03-28 21:52](https://github.com/koalaman/shellcheck/issues/1892):

Bugs like #1858 only happen because we handwrite `delve`: https://github.com/koalaman/shellcheck/blob/7963eeab9d51b5210f21de13eaf2140805287cef/src/ShellCheck/AST.hs#L188-L287

That implementation is super error-prone (as that bug illustrated). We should decide on a refactoring that avoids having to write it, since it's basically just a monomorphic `traverse`.

Commit 0bff853 is a proof-of-concept that it can be replaced with generated code. I didn't just PR that because I imagine you'll frown upon that particular implementation due to its use of Template Haskell and bringing in another dependency. If you agree with the idea, though, I can look into other approaches to do it, such as generics.
