#### <img src="https://avatars.githubusercontent.com/u/2199511?v=4" width="50">[Nightfirecat](https://github.com/Nightfirecat) commented at [2018-05-12T05:30:11Z](https://github.com/koalaman/shellcheck/issues/1217#issuecomment-388531385):

From [SC2144's wiki entry](/koalaman/shellcheck/wiki/SC2144):

> `[ -e file* ]` only works if there's 0 or 1 matches. If there are multiple, it becomes `[ -e file1 file2 ]`, and the test fails.

Here is an example, which I ran on my own computer:

```
$ touch /tmp/foo_some.string.here_bar.txt

$ ls /tmp/*_some.string.here_*.txt
/tmp/foo_some.string.here_bar.txt

$ [ -e /tmp/*_some.string.here_*.txt ] && echo $?
0

$ touch /tmp/foobar_some.string.here_foobar.txt

$ ls /tmp/*_some.string.here_*.txt
/tmp/foo_some.string.here_bar.txt  /tmp/foobar_some.string.here_foobar.txt

$ [ -e /tmp/*_some.string.here_*.txt ] && echo $?
bash: [: /tmp/foo_some.string.here_bar.txt: binary operator expected
```

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2018-05-15T05:51:08Z](https://github.com/koalaman/shellcheck/issues/1217#issuecomment-389050641):

You know your expectations better than ShellCheck.  It can only guess the user's intentions based on their script. In this case it guessed that you wanted to check if any matching files exist. 

In this case it may have been wrong, but the heuristic holds up in general. It's more common to want to check if any files exist, than it is to want to show strange errors when you have more than one match.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2022-02-26T04:04:13Z](https://github.com/koalaman/shellcheck/issues/1217#issuecomment-1051558577):

You can definitely do that. You can also consider making the assumption explicit to fail fast if it's ever violated (which the wiki now also mentions):

```
files=( dir/file* )
[ "${#files[@]}" -ge 2 ] && exit 1
if [ -e "${files[0]}" ]
then 
  echo "The file exists"
else
  echo "No such file"
fi
```
