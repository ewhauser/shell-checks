# [\#2740 Issue](https://github.com/koalaman/shellcheck/issues/2740) `open`: SC2096: Detects also non-shebang comments (i.e. `# /...` without `#!`)

#### Here's a snippet or screenshot that shows the problem:

With a `test.sh` file as in:

```sh
$ echo '# /path/to/file --arg1 --arg2' > test.sh

$ cat test.sh
# /path/to/file --arg1 --arg2
```

ShellCheck incorrectly thinks this comment is a shebang.


#### Here's what shellcheck currently says:

ShellCheck incorrectly thinks this comment is a shebang;

```sh
$ shellcheck -s bash test.sh
# /path/to/file --arg1 --arg2
^---------------------------^ SC2096 (error): On most OS, shebangs can only specify a single parameter.
^-- SC1113 (error): Use #!, not just #, for the shebang.
```


#### Here's what I wanted or expected to see:

I would not expect SC2096 here since it's not a shebang (as SC1113 suggests).  So I'd only expect:

```sh
$ shellcheck -s bash test.sh
# /path/to/file --arg1 --arg2
^-- SC1113 (error): Use #!, not just #, for the shebang.
```


#### Addition info

FWIW, this happens only if the path is absolute, i.e. it starts with a slash (`/`).  It does _not_ happen with, say,

```sh
# path/to/file --arg1 --arg2
```

It also doesn't happen if we use `##`, e.g.

```sh
## /path/to/file --arg1 --arg2
```

So, to me it seems like ShellCheck SC2096 uses regular expression `^#(!|)[[:space:]]*/` to match this line, where I argue it should be `^#![[:space:]]*/`.
