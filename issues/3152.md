# [\#3152 Issue](https://github.com/koalaman/shellcheck/issues/3152) `closed`: Warn if scripts or functions end with `[ condition ] && cmd`

Version: 0.10.0 (reproduced with https://www.shellcheck.net)

Related: https://github.com/koalaman/shellcheck/issues/2977

`if [ condition ]; then cmd; fi` and `[ condition ] && cmd` are handled differently in a subtle way; the `&&` syntax causes the exit code of a script or function to be `1` instead of `0` if the condition fails and it's the last line. For example:

```
#!/bin/sh

[ -z "$HOME" ] && echo "Home is not set"
```

The script above will actually return an exit code of `1`, and not `0` like when `if [ -z "$HOME" ]; echo "Home is not set"; fi` is used, which can be especially subtle if this is the last line of a function that's deeply nested. This can be avoided by using an if-statement, or by switching to `[ ! condition ] || cmd` (probably preferred, since using `&&` implies the user wanted to avoid the if-statement to begin with), which both solve this.
