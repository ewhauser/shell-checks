# [\#2645 Issue](https://github.com/koalaman/shellcheck/issues/2645) `open`: Optional check: function declarations should not have `function` prefix

#### <img src="https://avatars.githubusercontent.com/u/29582865?u=0c57eebb21e1922b4a0345f7a830924cc295998b&v=4" width="50">[felipecrs](https://github.com/felipecrs) opened issue at [2022-12-23 19:14](https://github.com/koalaman/shellcheck/issues/2645):

EDIT: I had initially proposed the opposite, but I changed my mind about this. See the conversation below.

This is to ensure a consistent code style. Example:

```bash
function myfunction() {
  true
}
```

Should be:

```bash
myfunction() {
  true
}
```

#### <img src="https://avatars.githubusercontent.com/u/84648683?v=4" width="50">[wileyhy](https://github.com/wileyhy) commented at [2025-07-24 18:50](https://github.com/koalaman/shellcheck/issues/2645#issuecomment-3114507230):

Well, there are some varying levels of "proper" under consideration.

Would 'function word ()...' conform to [pick your favorite corporation's]
Style Guide? Maybe / maybe not. That's up to them.

Is said syntax a shell error, meaning, could such syntax cause, for instance
  - a backup script to silently fail,
  - data to be read or processed incorrectly,
  - unexpected side effect(s) to occur, or
  - for one to "incur the wrath from high atop the thing?"

Case in point:

  <sh>
  ~ $ type -a '!'
  ! is a shell keyword
  ~ $ function '!' () { printf '%d\n' "${LINENO}";}
  bash: `'!'': not a valid identifier
  ~ $
  <\sh>

In my opinion, if people claim some syntax is "wrong," it may be worthwhile
to ask for whom and when.
