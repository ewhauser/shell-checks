# [\#2264 Issue](https://github.com/koalaman/shellcheck/issues/2264) `closed`: SC2086: false positive in Bash here strings

#### <img src="https://avatars.githubusercontent.com/u/20651620?v=4" width="50">[alecov](https://github.com/alecov) opened issue at [2021-06-26 03:37](https://github.com/koalaman/shellcheck/issues/2264):

The following:

```
#!/bin/bash
cat <<<$1
```

gives an SC2086 error.

According to Bash manuals, however, pathname expansion and word splitting are not performed in here strings.

Indeed:
```
# In a non-empty directory:
$ x=*; cat <<<$x
*
```

and
```
$ x='a   b'; cat <<<$x
a   b
```

shellcheck version online.
