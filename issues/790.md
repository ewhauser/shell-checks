#### Here's a snippet or screenshot that shows the problem:
The following script splits on spaces due to an unquoted command substitution (SC2046):
```
#!/bin/bash
array=($(printf %s\\n 'foo bar' baz))
set -- "${array[@]}"
printf %s\\n "$@"
```
Output:
```
foo
bar
baz
```
Which is effectively the same as:
```
#!/bin/bash
set -- $(printf %s\\n 'foo bar' baz)
printf %s\\n "$@"
```
Output:
```
foo
bar
baz
```

#### Here's what shellcheck currently says:
Shellcheck gives no warning on the script using the array, unlike the script using the command substitution directly.
```
archie@thinkpad ~ % shellcheck shellcheck-sc2046.sh
archie@thinkpad ~ % shellcheck shellcheck-sc2046-2.sh

In shellcheck-sc2046-2.sh line 2:
set -- $(printf %s\\n 'foo bar' baz)
       ^-- SC2046: Quote this to prevent word splitting.
```

#### Here's what I wanted or expected to see:

I expect shellcheck to also warn when using unquoted command substitutions in arrays, as the problematic described in SC2046 also applies there.
