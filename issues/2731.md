# [\#2731 Issue](https://github.com/koalaman/shellcheck/issues/2731) `closed`: Show "SCxxxx" error code in warning message

# Solution

Apparently I had some years-old copy-pasta that was disabling the status code in my `vim-ale` config.

The fix:

```vim
let g:ale_echo_msg_format = '[%severity%] %code%: %s [%linter%]'
```

```txt
[Info] SC2016: Expressions don't expand in single quotes, use double quotes for that. [shellcheck]
```

Documentation at `~/.vim/pack/plugins/start/ale/doc/ale.txt`:

```txt
g:ale_echo_msg_format                                   *g:ale_echo_msg_format*
                                                        *b:ale_echo_msg_format*

  Type: |String|
  Default: `'%code: %%s'`

  This variable defines a message format for echoed messages. The following
  sequences of characters will be replaced.

    `%s`           - replaced with the text for the problem
    `%...code...% `- replaced with the error code
    `%linter%`     - replaced with the name of the linter
    `%severity%`   - replaced with the severity of the problem (e.g. `Error`)
    `%type%`       - replaced with the type of the problem (e.g. `E`)
```

# Original

I just want the error code to be in the error message so that it's easy to disable false positive warnings without having to have my "Top 10 shellcheck codes I have to ignore in every script" printout handy. 😃

#### For new checks and feature suggestions

- [x] https://www.shellcheck.net/ (i.e. the latest commit) currently gives no useful warnings about this
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related

I tried to search, but the keywords I have to work with are "SCxxxx error code" so...

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/sh
set -e
set -u

. .env
```

#### Here's what shellcheck currently says:

```text
[shellcheck] ShellCheck can't follow non-constant source. Use a directive to specify location. [W]
```

#### Here's what I wanted or expected to see:

```text
[shellcheck] [SC1090] ShellCheck can't follow non-constant source. Use a directive to specify location. [W]
```
