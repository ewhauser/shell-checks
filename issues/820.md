# [\#820 Issue](https://github.com/koalaman/shellcheck/issues/820) `closed`: Complain if bash-like string appending is used in POSIX scripts

#### <img src="https://avatars.githubusercontent.com/u/11966684?v=4" width="50">[rugk](https://github.com/rugk) opened issue at [2017-01-08T22:42:30Z](https://github.com/koalaman/shellcheck/issues/820):

#### For new checks and feature suggestions
- [x] shellcheck.net (i.e. the latest commit) currently gives no useful warnings about this
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related


#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/sh
# below fails in POSIX
s=test1
s+="test2"

# works below
s=test1
s=$s"test2"
s="${s}test2"
```

#### Here's what shellcheck currently says:

No issues

#### Here's what I wanted or expected to see:

`+=` is not available as a POSIX shell feature, only available in Bash (e.g.)

See also https://stackoverflow.com/questions/2250131/how-do-you-append-to-an-already-existing-string#answer-2250199
