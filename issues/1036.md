# [\#1036 Issue](https://github.com/koalaman/shellcheck/issues/1036) `closed`: False positive for 1072 when there's inline `disable` directives

#### <img src="https://avatars.githubusercontent.com/u/8427257?v=4" width="50">[mechalynx](https://github.com/mechalynx) opened issue at [2017-10-28T00:12:48Z](https://github.com/koalaman/shellcheck/issues/1036):

#### For bugs
- Rule Id: 1072
- Version: local 0.4.6 / also occurs online
- [X] I tried on shellcheck.net and verified that this is still a problem on the latest commit

#### Here's a snippet or screenshot that shows the problem:

```sh

#! /usr/bin/env bash

if [[ "$1" =~ [0-9]+ ]]
then
  echo "$1"
# shellcheck disable=1001
elif [[ "$1" =~ [^0-9] ]]
then
  echo "$2"
fi

```

#### Here's what shellcheck currently says:

```
Line 3:
if [[ "$1" =~ [0-9]+ ]]
^-- SC1009: The mentioned parser error was in this if expression.
 
Line 4:
then
^-- SC1073: Couldn't parse this then clause.
 
Line 7:
elif [[ "$1" =~ [^0-9] ]]
     ^-- SC1072: Unexpected keyword/token. Fix any mentioned problems and try again.
```

#### Here's what I wanted or expected to see:

No error exists in that code. The false positive does not occur if the `disable` directive is missing or at the block level (outside the `if` or a function block).

#### <img src="https://avatars.githubusercontent.com/u/2199511?v=4" width="50">[Nightfirecat](https://github.com/Nightfirecat) commented at [2017-10-28T00:22:21Z](https://github.com/koalaman/shellcheck/issues/1036#issuecomment-340125571):

It's because the disable directive is in the wrong place. [(source)](https://github.com/koalaman/shellcheck/wiki/Directive)

The following raises no errors:

```
#! /usr/bin/env bash

# shellcheck disable=1001
if [[ "$1" =~ [0-9]+ ]]
then
  echo "$1"
elif [[ "$1" =~ [^0-9] ]]
then
  echo "$2"
fi
```
