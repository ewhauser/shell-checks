# [\#457 Issue](https://github.com/koalaman/shellcheck/issues/457) `closed`: warn about if conditions that will never/always be satisfied

#### <img src="https://avatars.githubusercontent.com/u/2182?v=4" width="50">[danp](https://github.com/danp) opened issue at [2015-09-11T20:09:37Z](https://github.com/koalaman/shellcheck/issues/457):

Given something like this, where `-z` was meant to be used with `$name`:

``` bash
name="hello"
if [ -z "name" ]; then
  echo "your name is blank!"
fi
```

it would be great to get a warning that the supplied condition is always satisfied. Same for cases where the condition could never be satisfied, such as `[ -n "name" ]`.
