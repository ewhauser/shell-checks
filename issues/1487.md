- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related


#### Here's a snippet or screenshot that shows the problem:

```sh

#!/bin/bash
lol()
{
echo "${1:-10}
}
lol

```

#### Here's what shellcheck currently says:
lol references arguments but none are ever passed.



#### Here's what I wanted or expected to see:
I would expect it to not throw an error/warning because ${1:-10} makes the variable default to "10" if $1 is unset or zero length, aka no argument is given, otherwise it is whatever is passed into $1.


#### <img src="https://avatars.githubusercontent.com/u/3999221?v=4" width="50">[pawamoy](https://github.com/pawamoy) commented at [2019-02-07T17:33:39Z](https://github.com/koalaman/shellcheck/issues/1487#issuecomment-461523505):

Just a note: `:-` uses default when variable is empty, even if it is set:

```console
$ a=""
$ echo "${a:-10}"
10
```
