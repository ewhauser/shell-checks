# [\#3240 Issue](https://github.com/koalaman/shellcheck/issues/3240) `closed`: Incorrect file separator used in diff output on Windows

#### <img src="https://avatars.githubusercontent.com/u/60605769?v=4" width="50">[zaneduffield](https://github.com/zaneduffield) opened issue at [2025-07-07 01:55](https://github.com/koalaman/shellcheck/issues/3240):

When running with diff output, a backslash is used to separate the `--- a` and `+++ b` diff headers from the name of the file. This backslash is rejected by both `patch` and `git apply` on Windows, because they expect unix-style paths.

I believe this backslash comes from this section of code, with the `</>` operator
https://github.com/koalaman/shellcheck/blob/20d11c1c33df71a375406c981a629269ee5149a2/src/ShellCheck/Formatter/Diff.hs#L194-L201

A slightly trickier problem is the fact that the path to the file itself may contain backslashes on Windows, which works fine when you use the OS primitives to read the file, but will also break when passed to `git apply` or `patch. Ideally, the file path would be normalised to only use forward slashes.

#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2025-07-17 20:41](https://github.com/koalaman/shellcheck/issues/3240#issuecomment-3085409596):

Huh, interesting issue! `\` now normalizes to `/` on Windows.
