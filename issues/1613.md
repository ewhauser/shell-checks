#### <img src="https://avatars.githubusercontent.com/u/1497172?v=4" width="50">[wheelq](https://github.com/wheelq) opened issue at [2019-06-12T12:08:50Z](https://github.com/koalaman/shellcheck/issues/1613):

#### Here's a snippet or screenshot that shows the problem:

```sh

#!/user/bin/env bash
_myfunc(){
  set -u;
  if ! [ -z "${_MY_VAR+set}" ]; then
    [do something]
  else
    exit 1
}
```

#### Here's what shellcheck currently says:
```Use -n instead of ! -z```

#### Here's what I wanted or expected to see:
I am using set -u. Script will fail if run a check against unbound variable
I can't use `${_MY_VAR-unset}` because I need to perform an action only if variable and its value are set

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2019-07-03T03:29:45Z](https://github.com/koalaman/shellcheck/issues/1613#issuecomment-507923444):

ShellCheck is right. You can replace `! -z` with `-n`. The `set -u` does not matter, and there is no suggestion to change anything else in the expression:

```
 if [ -n "${_MY_VAR+set}" ]; then true; fi
```
