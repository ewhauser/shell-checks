#### Here's a snippet or screenshot that shows the problem:

```sh

#!/usr/bin/env bash
tr ':upper' ':lower'

```

#### Here's what shellcheck currently says:

SC2020: tr replaces sets of chars, not words (mentioned due to duplicates).

#### Here's what I wanted or expected to see:

The tr call from the example above will relpace all uppercase letters to lowercase letters using character classes. Shellcheck should not give a warning for translating character classes.


#### <img src="https://avatars.githubusercontent.com/u/243351?v=4" width="50">[brother](https://github.com/brother) commented at [2022-03-10 19:47](https://github.com/koalaman/shellcheck/issues/2457#issuecomment-1064432806):

Shellcheck is correct in complaining about that syntax. The character classes should be in square brackets and colons like so:
```shell
#!/usr/bin/env bash
tr '[:upper:]' '[:lower:]'
```

Shellcheck does not complain about this syntax.
