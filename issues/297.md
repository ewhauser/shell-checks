# [\#297 Issue](https://github.com/koalaman/shellcheck/issues/297) `closed`: No coproc support

#### <img src="https://avatars.githubusercontent.com/u/5017453?v=4" width="50">[agusdallalba](https://github.com/agusdallalba) opened issue at [2015-01-27T03:44:24Z](https://github.com/koalaman/shellcheck/issues/297):

Bash 4.0 introduced the `coproc [VARIABLE] { commands; } [ #>&# ]` keyword that requires braces. Shellcheck first warns about a literal brace (SC1083) and stops processing altogether on the next literal brace (SC1070). Typical example:

```
#!/bin/bash

network_listen() {  #Usage: network_listen port
    coproc NETCAT { netcat -l -p "$1"; }
    #From now on,
    #${NETCAT[0]} = output from netcat
    #${NETCAT[1]} = intput to netcat
    #$NETCAT_PID  = netcat's PID
} #Fails here

zhu_li $do_the_thing #Should warn SC2086
```

This example should warn that the array will only expand to its first element: 

```
#!/bin/bash

network_listen() {  #Usage: network_listen port
    coproc { netcat -l -p "$1"; }
    #From now on,

    #$COPROC_PID  = netcat's PID
```
