# [\#726 Issue](https://github.com/koalaman/shellcheck/issues/726) `closed`: Warn when comparing the return value to zero

#### <img src="https://avatars.githubusercontent.com/u/14331571?v=4" width="50">[KellerFuchs](https://github.com/KellerFuchs) opened issue at [2016-08-18T00:40:02Z](https://github.com/koalaman/shellcheck/issues/726):

Lots of code cargo-cult the following style:

``` sh
do_something
if [ $? -ne 0 ]; then
  handle_error
fi
```

This has several issues, most notably:
- It is easy to accidentally mask the return value, introducing a bug.
- It makes the code non-reuseable under `set -e`.
- It is less clear than using `if ! do_something`.

As such, a warning that suggest the following would make sense:

``` sh
if ! do_something; then
  handle_error
fi
```

I've attempted to implement this, but the codebase is fairly difficult to navigate.
Any advise?
