#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash
foo=
[[ "${foo}" -eq none ]]
```

#### Here's what shellcheck currently says:
```
Line 3	SC2154: none is referenced but not assigned.
```
#### Here's what I wanted or expected to see:

ShellCheck is right: bash will perform arithmetic expansion of the token `none`, which was intended to be interpreted as a string, not a variable name.  But the real issue here is that the wrong comparison operator was used, `-eq` instead of `=`.  That's *why* the arithmetic expansion is performed, as ShellCheck also recognizes.  ShellCheck could have been a lot more helpful for diagnosing that than in fact it was. At minimum, the guidance page for SC2154 could say something about how arithmetic expansion can be involved in an issue of this kind.

#### <img src="https://avatars.githubusercontent.com/u/3674065?v=4" width="50">[jcbollinger](https://github.com/jcbollinger) commented at [2021-06-23 21:40](https://github.com/koalaman/shellcheck/issues/2261#issuecomment-867180292):

What's happening with the script is this, as best I can determine:

 - `[[` is a Bash keyword, as opposed to an external command or even a built-in one, so the evaluation of the associated expression is performed in the current shell environment, during expansion of the command.
 - *after* parameter expansion and quote removal, the expression is subject to arithmetic expansion.  It is at this point that, in the context of an `-eq` operator, the token `none` is interpreted as a variable reference.
   - when evaluated in this context, operands of `-eq` that do not expand to digit sequences are interpreted as 0
   - the secondary variable expansion and interpretation of non-numeric operands as 0 are indeed special behaviors of `[[`.  They are not documented for POSIX `[` and `test` commands, and they are not exhibited by bash's built-in versions of those or by the external versions of the GNU coreutils.  All of these latter reject `none` as an invalid operand for `-eq`, which ShellCheck also knows about and will diagnose where appropriate.
