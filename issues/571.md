# [\#571 Issue](https://github.com/koalaman/shellcheck/issues/571) `open`: Request for improvement:  SC2039 (and others) check POSIXLY_CORRECT

#### <img src="https://avatars.githubusercontent.com/u/1823914?v=4" width="50">[arth1](https://github.com/arth1) opened issue at [2016-01-06T19:44:42Z](https://github.com/koalaman/shellcheck/issues/571):

Example:

export POSIXLY_CORRECT=1
/bin/echo -e "foo\nbar"
( unset POSIXLY_CORRECT && /bin/echo -e "foo\nbar" )

This should return a warning for the first echo, but not the second.
As it is, it returns a warning for both if --shell=sh and none if --shell=bash
