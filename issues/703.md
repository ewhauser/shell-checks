# [\#703 Issue](https://github.com/koalaman/shellcheck/issues/703) `closed`: Annotation to indicate arguments are optional

#### <img src="https://avatars.githubusercontent.com/u/669650?v=4" width="50">[jasom](https://github.com/jasom) opened issue at [2016-06-27T18:31:54Z](https://github.com/koalaman/shellcheck/issues/703):

I have a couple of utility functions that take an optional $1, and read from stdin otherwise.  I'd _really_ like to not have to do a `#shellcheck disable=SC2119` every single time I use them.  If we could have something like `#shellcheck command_info=optional_args` that I could put right above the command definition, it would greatly reduce the noise from shellcheck.

#### <img src="https://avatars.githubusercontent.com/u/633683?v=4" width="50">[FloGa](https://github.com/FloGa) commented at [2017-02-08T08:30:10Z](https://github.com/koalaman/shellcheck/issues/703#issuecomment-278263528):

Since this issue seems not to be solved, I'll try it with a minimal example:

```bash
#!/bin/bash
relax() {
  local duration="${1:-2}"
  echo "Sleeping for $duration seconds ..."
  sleep "$duration"
}

# do something
relax
```

This will trigger SC2119 as well as SC2120. But `$1` in `relax` is clearly optional. If unset, it will be assigned to `2`. It would be nice if one could tell shellcheck to ignore warnings regarding this optional parameter.

If I ignore SC2119 globally, I will miss *actual* mistakes I might have made.
