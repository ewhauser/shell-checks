#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2019-01-20T20:28:02Z](https://github.com/koalaman/shellcheck/issues/614#issuecomment-455899381):

You're right that `set -m` is required to use these in non-interactive scripts.

I was only able to find 2 instances in 10,000 example files where this could be a problem (out of 3 instances where `bg`/`fg` was used at all). Since it's pretty rare and gives a searchable message when misused, it's probably better to avoid potentially flagging legitimate uses in rc files intended to be sourced.

Thanks for the suggestion!
