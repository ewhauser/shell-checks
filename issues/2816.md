# [\#2816 Issue](https://github.com/koalaman/shellcheck/issues/2816) `open`: SC2086, SC2046 unnecessary for `readonly` assignments

#### <img src="https://avatars.githubusercontent.com/u/51103593?u=af570d5b1e05e6e572221bf2f3a06ba297e744a3&v=4" width="50">[climbTheStairs](https://github.com/climbTheStairs) opened issue at [2023-08-22 12:15](https://github.com/koalaman/shellcheck/issues/2816):

#### For bugs
- Rule Id: SC2086
- My shellcheck version (`shellcheck --version` or "online"): 0.8.0, online
- [x] The rule's wiki page does not already cover this (e.g. https://shellcheck.net/wiki/SC2086)
- [x] I tried on https://www.shellcheck.net/ and verified that this is still a problem on the latest commit

#### Here's a snippet that shows the problem:

```sh
#!/bin/sh

a='b c=d'
readonly e=$a

echo "$e" # "b c=d"
echo "$c" # ""
```

#### Here's what shellcheck currently says:

```
In test line 4:
readonly e=$a
           ^-- SC2086 (info): Double quote to prevent globbing and word splitting.

Did you mean:
readonly e="$a"
```

#### Here's what I wanted or expected to see:

Nothing


#### <img src="https://avatars.githubusercontent.com/u/37733333?u=9e84dc7e36b9ca931c87564bfa0c68e1c7cf5dff&v=4" width="50">[Ordoviz](https://github.com/Ordoviz) commented at [2023-08-23 13:58](https://github.com/koalaman/shellcheck/issues/2816#issuecomment-1690017460):

See #2443. shellcheck warns here because Ubuntu 20.04 ships with dash 0.5.10.2, where the quotes are necessary (I tested it). Since then the POSIX spec has been updated so that quotes are no longer necessary.
