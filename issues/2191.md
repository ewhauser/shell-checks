I'd like to add this to [webinstall.dev](https://webinstall.dev), and we prefer in all possible cases to use official, 1st party builds rather than 3rd party or custom builds.

You have Mac and Linux builds right here on Github Releases (and we support Github Releases builds), but you don't have Windows builds.

What would need to happen to get official Windows builds available?

#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2021-04-01 03:52](https://github.com/koalaman/shellcheck/issues/2191#issuecomment-811618341):

#### <img src="https://avatars.githubusercontent.com/u/122831?u=e6736174efb6a5a0cbc843f3fc3a0d44deba70e6&v=4" width="50">[coolaj86](https://github.com/coolaj86) commented at [2021-04-01 04:18](https://github.com/koalaman/shellcheck/issues/2191#issuecomment-811626222):

Ah, I assumed it was source code as typically that's made available in tar.gz and .zip without other identifiers.

Would you consider adding the distro and arch to Windows builds? I can write some custom checks for shellcheck, but it would be nice to have the explicit info.

And I don't know if Haskell will support it anytime soon, but Windows 10 has been running happily on ARM for years (which I'm sure will get a lot of attention soon as things like Parallels and such switch over to Windows 10 ARM for the M1, etc)

Sidenote: Did you know that Windows 10 has always shipped with BSD tar.exe out of the box? And unlike GNU tar, BSD tar also opens zip files, which made supporting windows much less frustrating that I had expected.

Thanks for the reply. Cheers. :D
