# [\#453 Issue](https://github.com/koalaman/shellcheck/issues/453) `closed`: Useless cat when piping through several commands?

#### <img src="https://avatars.githubusercontent.com/u/169550?v=4" width="50">[mosbth](https://github.com/mosbth) opened issue at [2015-09-04T13:33:11Z](https://github.com/koalaman/shellcheck/issues/453):

```
cat ircLog.txt | grep "pansar" | grep "notepad" > b.txt
    ^-- SC2002: Useless cat. Consider 'cmd < file | ..' or 'cmd file | ..' instead.
```

or

```
cat ircLog.txt | grep '<@wasa>' | head -3 | tail -1 > e.txt
    ^-- SC2002: Useless cat. Consider 'cmd < file | ..' or 'cmd file | ..' instead.
```

The examples might look a bit odd, but its exercises in a course to learn the basics of how to combine unix commands with piping, input and output.

If this is useless cat, how should it be done, or perhaps should I just exclude that rule? Whats your advice?

#### <img src="https://avatars.githubusercontent.com/u/1666345?v=4" width="50">[lucy](https://github.com/lucy) commented at [2015-09-04T14:13:07Z](https://github.com/koalaman/shellcheck/issues/453#issuecomment-137747461):

`grep "pansar" ircLog.txt | grep "notepad" > b.txt` or
`grep "pansar" < ircLog.txt | grep "notepad" > b.txt`


#### <img src="https://avatars.githubusercontent.com/u/302055?v=4" width="50">[deryni](https://github.com/deryni) commented at [2015-09-04T14:17:37Z](https://github.com/koalaman/shellcheck/issues/453#issuecomment-137748312):

Or `awk '/pansar/ && /notepad/' ircLog.txt > b.txt` but that avoids the pipeline entirely so that's not necessarily exactly relevant. =)


#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2015-09-04T16:41:38Z](https://github.com/koalaman/shellcheck/issues/453#issuecomment-137787030):

These are indeed [Useless Uses Of Cat](https://en.wikipedia.org/wiki/Cat_%28Unix%29#Useless_use_of_cat) as lucy demonstrates. It doesn't matter if it's single or multiple commands, it's all just piping:

```
grep '<@wasa>' ircLog.txt | head -3 | tail -1 > f.txt
```

It's not necessarily unreasonable to ignore, but if you choose to do so it should be because you're equally comfortable writing it both ways and you're aware of the effect this will have on efficiency and correctness in each particular case.
