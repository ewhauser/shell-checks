# [\#826 Issue](https://github.com/koalaman/shellcheck/issues/826) `closed`: Possible F/P — SC2086 vs setting default value

#### <img src="https://avatars.githubusercontent.com/u/185848?v=4" width="50">[kgadek](https://github.com/kgadek) opened issue at [2017-01-20T18:06:09Z](https://github.com/koalaman/shellcheck/issues/826):

```sh
#!/bin/bash

: ${FOO:=123}
```

Both on shellcheck 0.4.5 and online the result is:

```
Line 3:
: ${FOO:=123}
  ^-- SC2086: Double quote to prevent globbing and word splitting.
```

I believe this is the proper way to set defaults and there is no issue here.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2017-01-20T20:11:02Z](https://github.com/koalaman/shellcheck/issues/826#issuecomment-274168191):

ShellCheck is technically correct. You should quote this due to several weird edge cases. 

Try `export FOO='/*/*/*/*/*/*'` before running that script, and you'll see it trashing the disk for several minutes for no good reason. You can also make it gobble up arbitrary amounts of memory this way.

See #480 for more cases that apply to this construct in general, but not in this particular toy script.
