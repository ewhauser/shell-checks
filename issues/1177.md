#### <img src="https://avatars.githubusercontent.com/u/8651654?v=4" width="50">[jleffler](https://github.com/jleffler) commented at [2018-04-11T05:49:50Z](https://github.com/koalaman/shellcheck/issues/1177#issuecomment-380335131):

There is no syntax error in the code, though it is unusual and probably unintentional shell scripting.  The `if` statement can be followed by any sequence of commands; the exit status of the command prior to the `then` is used to establish the truth condition.  Thus in:

```
if [ "$A" == "0" ]
echo Apache Server is not runing 
echo Restarting Apache Server
service httpd restart
exit
B=$(pgrep httpd | wc -l)
elseif [ "$B" == "0" ]
echo Apache server is not installed
then
    echo Installing Apache Server
else
    yum install httpd -y && service httpd start && service httpd enable
fi
```

there is an invocation of the `test` command via its `[` synonym, followed by two `echo`s, a `service`, an `exit`, an assignment, the `elseif` command and another `echo`; and the result of the final echo (success) controls the `if` — or would if the `exit` hadn't exited the script before that is executed.  The fact that there isn't often an `elseif` command is neither here nor there as far as the official shell syntax is concerned.

Now, ShellCheck might reasonably legitimately warn that the sequence of commands after the `if` is unusual and probably erroneous, but the shell itself doesn't think there's a problem.
