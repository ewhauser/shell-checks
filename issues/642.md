# [\#642 Issue](https://github.com/koalaman/shellcheck/issues/642) `closed`: SC2086 may not be catching up on some variables

#### Labels: working as intended

#### <img src="https://avatars.githubusercontent.com/u/75783?v=4" width="50">[mat813](https://github.com/mat813) opened issue at [2016-04-04T15:59:18Z](https://github.com/koalaman/shellcheck/issues/642):

For the following code snippet:

``` sh
function echo2() {
    echo=$1
    shift
    foo="$*"
    $echo $foo
}
```

it outputs

```

Line 5:
    $echo $foo
          ^-- SC2086: Double quote to prevent globbing and word splitting.
```

I think it should say both should be quoted.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2016-04-09T17:01:06Z](https://github.com/koalaman/shellcheck/issues/642#issuecomment-207818026):

Maybe. This is intentional, with the rationale being that you're way more likely to have flags that shouldn't be quoted than you are to have a command name with special characters that do need to be quoted.
