# [\#559 Issue](https://github.com/koalaman/shellcheck/issues/559) `closed`: SC2086 is incorrectly reported in [ -v variable ] test

#### <img src="https://avatars.githubusercontent.com/u/151714?v=4" width="50">[barnardb](https://github.com/barnardb) opened issue at [2015-12-11T23:07:46Z](https://github.com/koalaman/shellcheck/issues/559):

Given the following contents in `test.sh`:

```
#!/bin/bash

[ $# == 0 ] || variable="$*"

[ -v variable ] && echo "variable=${variable}" || echo "no variable"
```

`shellcheck test.sh` reports:

```

In test.sh line 5:
[ -v variable ] && echo "variable=${variable}" || echo "no variable"
     ^-- SC2086: Double quote to prevent globbing and word splitting.

```

Adding quotes makes no difference:

```

In test.sh line 5:
[ -v "variable" ] && echo "variable=${variable}" || echo "no variable"
     ^-- SC2086: Double quote to prevent globbing and word splitting.

```

```

In test.sh line 5:
[ -v 'variable' ] && echo "variable=${variable}" || echo "no variable"
     ^-- SC2086: Double quote to prevent globbing and word splitting.

```

This was observed using shellcheck 0.4.1.
