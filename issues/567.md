# [\#567 Issue](https://github.com/koalaman/shellcheck/issues/567) `open`: warn about 'grep pattern' without specifying filename

#### <img src="https://avatars.githubusercontent.com/u/198379?v=4" width="50">[bittorf](https://github.com/bittorf) opened issue at [2015-12-22T16:15:40Z](https://github.com/koalaman/shellcheck/issues/567):

like https://github.com/koalaman/shellcheck/issues/452 we should
warn about a forever hanging grep, e.g.

```
grep mypattern
```

or

```
grep -switches mypattern
```

with a forgotten filename.

#### <img src="https://avatars.githubusercontent.com/u/302055?v=4" width="50">[deryni](https://github.com/deryni) commented at [2015-12-24T16:33:25Z](https://github.com/koalaman/shellcheck/issues/567#issuecomment-167133492):

GNU grep as of some version 2.11 ([changelog entry](http://git.savannah.gnu.org/cgit/grep.git/tree/NEWS#n433)) assumes that a missing filename means the current directory when run with the `-r, --recursive` flag.


#### <img src="https://avatars.githubusercontent.com/u/198379?v=4" width="50">[bittorf](https://github.com/bittorf) commented at [2015-12-24T21:01:24Z](https://github.com/koalaman/shellcheck/issues/567#issuecomment-167159504):

@deryni interesting - thanks, the feature is from march 2012 and it is not POSIX 8-) so still a valid case and a warning should be OK. for interactive scripting it's OK to leave the filename blank, but for real you should do a

```
grep -r PATTERN ./
```
