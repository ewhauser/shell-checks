#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/sh

TEXT="this is DEMO text."

echo "${TEXT^}"
echo "${TEXT^^}"
echo "${TEXT,}"
echo "${TEXT,,}"
```

#### Here's what shellcheck currently says:

> No issues detected!

#### Here's what I wanted or expected to see:

This is the "case modification" feature supported by bash v4.x+, result in:
```
This is DEMO text.
THIS IS DEMO TEXT.
this is DEMO text.
this is demo text.
```

POSIX sh should return `bad substitution` without valid output.

So the expected message will be 
```
In POSIX sh, ${VAR^} is undefined.
In POSIX sh, ${VAR^^} is undefined.
In POSIX sh, ${VAR,} is undefined.
In POSIX sh, ${VAR,,} is undefined.
```
