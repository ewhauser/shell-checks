# [\#716 Issue](https://github.com/koalaman/shellcheck/issues/716) `closed`: Redirection placed before subshell triggers SC1036

#### <img src="https://avatars.githubusercontent.com/u/3425234?v=4" width="50">[yrps](https://github.com/yrps) opened issue at [2016-07-24T22:08:47Z](https://github.com/koalaman/shellcheck/issues/716):

``` shell
#!/bin/sh

acommand >somewhere
>somewhere acommand
(commands) >somewhere
>somewhere (commands)
```

First three are OK.
Last one throws `'(' is invalid here. Did you forget to escape it? [SC1036]`

_edit_
Maybe this is related:

``` shell
time acommand
time (commands)
```

The second triggers `Trying to declare parameters? Don't. Use () and refer to params as $1, $2.. [SC1065]`

Seems to be a problem with recognizing subshells that open after the first token.

Shellcheck is correct; that last one is not valid shell syntax.

The rules for placement of redirections on compound statements are rather
more stringent that for simple statements; basically they must always go
after the closing token: ")", "))", "]]", "}", "done", "esac", or "fi".

There are also complications is you use aliases, as they are expanded before
compound statements are recognized (and hence, before redirections).
