# [\#1837 Issue](https://github.com/koalaman/shellcheck/issues/1837) `closed`: Improvement Request: SC1090 / SC10901 - allow relative path in "# shellcheck source=" to work pinned from file's location

#### <img src="https://avatars.githubusercontent.com/u/2211051?u=7747a673d98078d50467ba6e4d4f3a2f17b6db17&v=4" width="50">[HariSekhon](https://github.com/HariSekhon) opened issue at [2020-02-13 12:05](https://github.com/koalaman/shellcheck/issues/1837):

My scripts usually source files anchored to the source directory like so

```
#!/usr/bin/env bash
srcdir="$(dirname "$0")"
source "$srcdir/lib/utils.sh"
```

Shellcheck complains with the usual
```
. "$srcdir/lib/utils.sh"
  ^------------------------^ SC1090: Can't follow non-constant source. Use a directive to specify location.
```

If you do

```
# shellcheck source=lib/utils.sh
```

and then run ```shellcheck -x script.sh``` this works, but if you run ```shellcheck -x directory/script.sh``` (because I am executing shellcheck against a list of shell scripts from a find command) it breaks.

It would be good if shellcheck could check source both for $PWD and if not found then from the same directory the script exists in.

I've worked around this by wrapping shellcheck in a script which pushd's to each script's directory before running ```shellcheck -x``` so that the current limited to relative path ```source=lib/utils.sh``` works but it would be better if shellcheck supported either:

1. ```source=$SOURCE_DIR/lib/utils.sh``` or some standard equivalent variable to use in ```source``` directives
or
2. auto-checked script's source directory if sourcing did not find file in $PWD

#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2021-10-15 19:19](https://github.com/koalaman/shellcheck/issues/1837#issuecomment-944595075):

ShellCheck has since gotten a `source-path` directive that can be used to specify where ShellCheck looks for included files. It can be set to absolute paths, and paths relative to the cwd or script file. This can be set in the script itself, or in a `.shellcheckrc` in the project. Hopefully this will cover all these use cases.

See https://github.com/koalaman/shellcheck/wiki/Directive
