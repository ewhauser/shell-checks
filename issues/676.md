# [\#676 Issue](https://github.com/koalaman/shellcheck/issues/676) `closed`: regression in SC2034 bar=1 ; ((foo=bar)) doesn't count as use of $bar

#### <img src="https://avatars.githubusercontent.com/u/1693206?v=4" width="50">[knirch](https://github.com/knirch) opened issue at [2016-05-24T06:57:47Z](https://github.com/koalaman/shellcheck/issues/676):

``` bash
foo=1
((t=foo))
```

Yields `foo appears unused. Verify it or export it. [SC2034]`

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2016-05-24T17:10:00Z](https://github.com/koalaman/shellcheck/issues/676#issuecomment-221339660):

Great catch! Fixed in 85e69f8

This only happens for 1-on-1 assignments like this, and fortunately not e.g. `(( t = foo + 1 ))`.
