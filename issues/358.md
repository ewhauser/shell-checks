#### <img src="https://avatars.githubusercontent.com/u/5013687?v=4" width="50">[Chogos](https://github.com/Chogos) commented at [2016-02-12T13:04:15Z](https://github.com/koalaman/shellcheck/issues/358#issuecomment-183317225):

When I used the provided checkstyle2junit.xslt only one failure per file was reported since the resulting XML is not (entirely) valid for the JUnit plugin on Jenkins. I've adjusted it to create a failed testcase for each error which enabled Jenkins to report all of them.

The adjusted script:

``` xml
<?xml version="1.0" encoding="UTF-8"?>
<xsl:stylesheet version="1.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">

  <xsl:output encoding="UTF-8" method="xml"></xsl:output>

  <xsl:template match="node() | @*">
    <xsl:copy>
      <xsl:apply-templates select="node() | @*"/>
    </xsl:copy>
  </xsl:template>

  <xsl:template match="/">
    <testsuite>
      <xsl:attribute name="tests">
        <xsl:value-of select="count(.//file)" />
      </xsl:attribute>
      <xsl:attribute name="failures">
        <xsl:value-of select="count(.//error)" />
      </xsl:attribute>
      <xsl:for-each select="//checkstyle">
        <xsl:apply-templates />
      </xsl:for-each>
    </testsuite>
  </xsl:template>

  <xsl:template match="file">
    <xsl:for-each select=".//error">
      <testcase>
        <xsl:attribute name="classname">
          <xsl:value-of select="../@name" />
        </xsl:attribute>
        <xsl:attribute name="name">
          <xsl:value-of select="../@name" />
        </xsl:attribute>
        <failure>
          <xsl:attribute name="type">
            <xsl:value-of select="@source" />
          </xsl:attribute>
          <xsl:text>Line </xsl:text>
          <xsl:value-of select="@line" />
          <xsl:text>: </xsl:text>
          <xsl:value-of select="@message" />
        </failure>
      </testcase>
    </xsl:for-each>
  </xsl:template>

</xsl:stylesheet>
```


#### <img src="https://avatars.githubusercontent.com/u/65675?v=4" width="50">[josegonzalez](https://github.com/josegonzalez) commented at [2019-01-06T01:12:28Z](https://github.com/koalaman/shellcheck/issues/358#issuecomment-451706525):

Just popping in to say that the Dokku project - which uses shellcheck quite a bit, thanks @koalaman and others! - will be using a python script to munge the original shellcheck output. That is available [here](https://github.com/dokku/dokku/blob/master/tests/shellcheck-to-junit). A few notes:

- The linked to XSLT in the wiki works, but is missing a bunch of data. Worse, it doesn't pass the _only_ xsd I could find online for Junit ([here](https://github.com/windyroad/JUnit-Schema). Good attempt though!
- The XSLT from @Chogos sometimes bailed on my failure output.
- The python script doesn't include passing files by default. This can be passed via the `--files` flag, which should be a file that contains all files being processed.
- The python script doesn't know what exclusion rules were included by default. This can be specified via the `--exclude` flag, which is a comma-separated list of exclusion rules.
- usage of the script is available [here](https://github.com/dokku/dokku/blob/master/tests.mk#L80).
- None of these methods allow the original information to be output _as is_. That is to say, you'd need to run shellcheck twice to get human readable output with xmlstarlet, and my version doesn't support colors. In theory, you could use `tee` with my python script, but I don't want to fiddle with redirection in make targets just now.

One thing that [I did for bats-core](https://github.com/bats-core/bats-core/pull/153) and might make sense here is to have a `raw` output that is then passed through a second command for extra formatting. The raw output could have all the data anyone could want, making it quite easy for people to write formatters for whatever output they like. Supporting multiple formatters would allow both human and machine-readable output, making usage in common CI platforms easier to get into. I'd love to help here, but my haskell is _definitely_ weaker than I'd like it to be.

Regardless, thanks all for your help!
