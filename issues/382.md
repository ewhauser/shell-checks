# [\#382 Issue](https://github.com/koalaman/shellcheck/issues/382) `open`: Doesn't understand awk quotes if awk is variable

#### <img src="https://avatars.githubusercontent.com/u/572706?v=4" width="50">[FransUrbo](https://github.com/FransUrbo) opened issue at [2015-05-21T18:18:49Z](https://github.com/koalaman/shellcheck/issues/382):

Consider the following code:

```
#!/bin/bash                                                                                                                                                   

AWK=${AWK:-/usr/bin/awk}
find /etc/{passwd,group,shadow,gshadow} > /tmp/$$
[ "$(wc -l /tmp/$$ | "${AWK}" '{print $1}')" -eq 2 ] && echo fail
```

Gives:

```
In /tmp/testme.sh line 5:
[ "$(wc -l /tmp/$$ | "${AWK}" '{print $1}')" -eq 2 ] && echo fail
                              ^-- SC2016: Expressions don't expand in single quotes, use double quotes for that.
```

Replacing <code>"${AWK}"</code> with <code>awk</code> works though.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2015-05-21T18:29:34Z](https://github.com/koalaman/shellcheck/issues/382#issuecomment-104379198):

shellcheck generally warns when single quoting variables, but has exceptions for a few commands where this is common, like awk, sh and perl. If it's not obvious to shellcheck that it's an awk command, you'll get this message. 

There's no simple bugfix for this. You can either disable it with a directive or flag, or write around it with a function `awk() { command "${AWK:-awk}" "$@"; }`
