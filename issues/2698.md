# [\#2698 Issue](https://github.com/koalaman/shellcheck/issues/2698) `open`: SC2034: Shellcheck seems to parse assignments incorrectly if the value starts with #

#### <img src="https://avatars.githubusercontent.com/u/992428?v=4" width="50">[git1106](https://github.com/git1106) opened issue at [2023-02-22 16:34](https://github.com/koalaman/shellcheck/issues/2698):

#### Bug report

- SC2034
- My shellcheck version (`shellcheck --version` or 'online'): online
- [x] I tried on shellcheck.net and verified that this is still a problem on the latest commit

#### Here's a snippet or screenshot that shows the problem:

```sh
#! /bin/dash
a=12
b=#${a}
echo "${b}"
```

#### Here's what shellcheck currently says:
```
Line 2	SC2034: a appears unused. Verify use (or export if used externally).
```
#### Here's what I wanted or expected to see:

No message.

(Without reading the source I guess the reason might be that `shellcheck` misinterprets that value as a comment.)

#### Current work-around:

```sh
#! /bin/dash
a=12
b="#${a}"
echo "${b}"
```

#### Further observations:

If the `#` is not in the beginning of the value there is no problem

```sh
#! /bin/dash
a=12
b=0#${a}
echo "${b}"
```
