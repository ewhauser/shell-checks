#### <img src="https://avatars.githubusercontent.com/u/4527928?v=4" width="50">[LLyaudet](https://github.com/LLyaudet) commented at [2026-02-17T20:42:35Z](https://github.com/koalaman/shellcheck/issues/623#issuecomment-3916991729):

Yes. I didn't thought that it was that complicated.
But even when read is the last command in a pipe sequence
and that pipe sequence ends a function,
it still depends on shopt -s lastpipe.
Example:
```
all_distinct(){
  printf "%s\n" "$@" \
    | uniq -d\
    | read
}
echo "$REPLY" # may be set or not according to shopt -s lastpipe
```
It seems the only case that is clear cut would be:
```
all_distinct(){
  printf "%s\n" "$@" \
    | uniq -d\
    | ( read )
}
```
