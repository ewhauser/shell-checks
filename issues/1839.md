# [\#1839 Issue](https://github.com/koalaman/shellcheck/issues/1839) `closed`: When || operator in front of $(...) in an assignment statement, SC2091 will be claimed.

#### <img src="https://avatars.githubusercontent.com/u/25837029?u=9ebd52200228a854b5a2f01dad1c1fd6d3df10a2&v=4" width="50">[Sociosarbis](https://github.com/Sociosarbis) opened issue at [2020-02-15 11:19](https://github.com/koalaman/shellcheck/issues/1839):

- RuleID: SC2091: 
- My shellcheck version (0.70): 
- [x] I tried on shellcheck.net and verified that this is still a problem on the latest commit

**My case:**
```shell
remote_repos=$branch_name || $(git branch -a | grep 'remotes/origin/\S*' | sed -E 's/.*remotes\/origin\/(\S*).*/\1/' | grep -wv 'HEAD'| tr '\n' ' ')
```
When $(...) is not placed right after the equal sign in the assignment statement, SC2091 will be claimed.

Although there is a `||` operator in front of it, but I think this line of code is stlll an assignment statement and would not cause the problem showed in SC2091.(May be.I am not sure😄)




#### <img src="https://avatars.githubusercontent.com/u/2199511?u=6cc22d0767078b50e47021699ffbb376251628ad&v=4" width="50">[Nightfirecat](https://github.com/Nightfirecat) commented at [2020-02-15 15:59](https://github.com/koalaman/shellcheck/issues/1839#issuecomment-586612940):

SC2091 is being raised properly here because your code doesn't do what you think it does. If `$branch_name` is empty or unset, `remote_repos` will still be set to its value. Everything after the `||` is not doing anything.
