# [\#1959 Issue](https://github.com/koalaman/shellcheck/issues/1959) `open`: Allow use the dir of the rc file as source-path value when defined in rc files

#### <img src="https://avatars.githubusercontent.com/u/265507?v=4" width="50">[MOZGIII](https://github.com/MOZGIII) opened issue at [2020-05-19 01:05](https://github.com/koalaman/shellcheck/issues/1959):

I think it'd be very useful to have the ability to use the `.shellcheckrc` dir as the value for the `source-path` defined within the file.

Example:

Assume there's a `/home/user/project-root/.shellcheckrc` file with the following content:

```
source-path=RCDIR
```

would mean that `/home/user/project-root` is added to the `source-path` when processing any script that would involve the config from the `/home/user/project-root/.shellcheckrc`.
