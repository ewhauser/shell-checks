Are you sure about that?

When I try this:

```
set -e -o pipefail
count=$(echo $'first\nsecond'| grep -c unknown | wc -l)
```

it fails.

(When you use `set -o pipefail` in combination with `set -e`, if ANY
component of the pipeline fails, the shell will see this as a command
failure and will abort; however this may vary between different versions of
Bash, or with different shells.)

As a separate point, some would argue that `set -e` is the real problem.
Although some shell guides (and even some training courses) suggest always
including this line, the more sophisticated guides recommend avoiding it,
precisely because `set -e` has a bunch of corner cases where valid code
(like this) breaks, and failing code still doesn't get caught.

In short: `set -e` and `set -u` should be treated as linting tools, not for
use in production code.

> Again: that's why I see that the warning is sometimes ... questionable ...
> Except you can say me how to work with "grep -c" without this problem.

You could also write:

```
count=$( somecmd | grep -c somepattern ) || :
```

which would take rather less space that the comment to disable shellcheck.

Or even

```
count=$( somecmd > >( grep -c somepattern ) )
```

which (since you've used pipefail) will trap when somecmd fails, but won't
trap when grep fails -- with or without pipefail.


#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2016-10-01T19:39:54Z](https://github.com/koalaman/shellcheck/issues/741#issuecomment-250932931):

I'd agree with kurahaupo, this is a good stylistic suggestion that rather exposes an existing issue with the code. 

Correctness relies on the fact that `wc` happens to mask the exit status of grep when `pipefail` is disabled. I'm guessing this was through luck more than by design. 

Rather than relying on this somewhat obscure bit of shell knowledge, it's better to explicitly use `|| true` or equivalent on lines where you don't care about the exit status when using `set -e`  (or to [not use `set -e` at all](http://mywiki.wooledge.org/BashFAQ/105)).

I'd agree that this is far from obvious though, and I've updated the wiki to mention this case. Thanks!
