# [\#2437 Issue](https://github.com/koalaman/shellcheck/issues/2437) `open`: pre-commit hook fails on systems using remove container engines

#### <img src="https://avatars.githubusercontent.com/u/102495?u=c7bd9ddf127785286fc939dd18cb02db0a453bce&v=4" width="50">[ssbarnea](https://github.com/ssbarnea) opened issue at [2022-01-30 11:44](https://github.com/koalaman/shellcheck/issues/2437):

I would have reported this bug toward https://github.com/koalaman/shellcheck-precommit repository because is specific to it.

The problem of using docker_image with volume mounts to run shellcheck with that there is no guarantee that the current container engine is able to mount files from disk. This never works when you configured docker/podman to run remotely (see DOCKER_HOST).

One workaround for this problem is to build the source into a temporary image and use that one, as build process does ensure that the container can reach the files that need to be linted.

What happens on a system configured like this:
```
$ pre-commit rub -a shellcheck
ShellCheck v0.8.0........................................................Failed
- hook id: shellcheck
- exit code: 2

Unable to find image 'koalaman/shellcheck:v0.8.0' locally

tools/test-setup.sh: tools/test-setup.sh: openBinaryFile: does not exist (No such file or directory)
```

The same problem can also be caused by an incorrect context set. In my case:

```
% docker context ls
NAME            DESCRIPTION                               DOCKER ENDPOINT               ERROR
default *       Current DOCKER_HOST based configuration   unix:///var/run/docker.sock
remote-docker                                             tcp://dev02.local:2375
```

Since I have DOCKER_HOST not set, I can change context: `docker context use default` and then `shellcheck-precommit` works as needed.
