# [\#3230 Issue](https://github.com/koalaman/shellcheck/issues/3230) `open`: Do not show SC2059 warning if printf has multiple non-option arguments

#### <img src="https://avatars.githubusercontent.com/u/36376767?u=e0c08192d91b211654c1cfa19052934d81139e0a&v=4" width="50">[marveloo](https://github.com/marveloo) opened issue at [2025-06-26 13:29](https://github.com/koalaman/shellcheck/issues/3230):

When `printf` is used with more than one non-option argument, it strongly suggests that the first argument is intended to be used as a format string. Otherwise, the presence of additional arguments would not be meaningful. Therefore, it seems reasonable to suppress the [SC2059](https://www.shellcheck.net/wiki/SC2059) warning in cases where multiple non-option arguments are passed to `printf`.

```bash
# Show SC2059 warning for this:
printf "$pattern"

# However, do not show a warning for these:
printf "$pattern" "$value"
printf "$pattern" "$value1" "$value2" "$value3"
```
