# [\#687 Issue](https://github.com/koalaman/shellcheck/issues/687) `open`: cd *

#### <img src="https://avatars.githubusercontent.com/u/621959?v=4" width="50">[mcsaucy](https://github.com/mcsaucy) opened issue at [2016-06-13T21:21:47Z](https://github.com/koalaman/shellcheck/issues/687):

Some time ago, I came across a use of `cd *`.

At first, I was very pleased that my shell complained if there were too many arguments to `cd`. Then, I was shaken from my blissful trance by colorful reminders of the cold, harsh reality: I use ZSH as my login shell. Bash doesn't not complain about multiple arguments to `cd`, opting instead to act upon  [1] I was surprised that something so simple (after all, it's only 4 characters) could make me so very sad.

Could we get a check for `cd <anything that could expand to multiple values>`?

[1] Example:

```
mkdir  "$HOME/cd-star"
cd "$HOME/cd-star"
mkdir foo bar baz
cd *
echo "$PWD"  # $HOME/cd-start/bar
```
