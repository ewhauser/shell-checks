# [\#2358 Issue](https://github.com/koalaman/shellcheck/issues/2358) `open`: SC2024 in systemd-nspawn container

#### <img src="https://avatars.githubusercontent.com/u/36802396?u=068da9668c571def803a42ba5447e560931de561&v=4" width="50">[Cody-Learner](https://github.com/Cody-Learner) opened issue at [2021-10-17 17:59](https://github.com/koalaman/shellcheck/issues/2358):

bash 5.1.008-1
shellcheck-bin 0.7.2-1
shellcheck-bin build script URL     : https://aur.archlinux.org/packages/shellcheck-bin

#### My interpreter:
```
#!/bin/bash
```
#### My command:
```
sudo systemd-nspawn -q -D "${bdir}/${crName}" -u builduser --chdir="${buHOME}/${package}" --pipe \
awk '/validpgpkeys/ {print $3}' .SRCINFO >key
```

#### Here's what shellcheck currently says: 
```
In /home/jeff/GitHub/Cody-Learner/aurch/aurch line 132:
		awk '/validpgpkeys/ {print $3}' .SRCINFO >key
                                                ^-- SC2024: sudo doesn't affect redirects. Use ..| sudo tee file
```
#### Here's what I wanted or expected to see:
```
sudo systemd-nspawn -q -D "${bdir}/${crName}" -u builduser --chdir="${buHOME}/${package}" --pipe \
awk '/validpgpkeys/ {print $3}' .SRCINFO >key
```

#### Why:
The awk command is not ran as root or with sudo in the chroot, it's ran as user builduser.




#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2021-10-31 00:34](https://github.com/koalaman/shellcheck/issues/2358#issuecomment-955613943):

It shows up this way because ShellCheck associates each such suggestion with a syntax element, which in turn is associated with a line/column position. In its output it shows the source code from that line when it points out the problem. That's a bit confusing in this case where the command spans multiple lines.

Perhaps it should be showing the nearest command instead of the specific line.

#### <img src="https://avatars.githubusercontent.com/u/36802396?u=068da9668c571def803a42ba5447e560931de561&v=4" width="50">[Cody-Learner](https://github.com/Cody-Learner) commented at [2021-11-01 02:17](https://github.com/koalaman/shellcheck/issues/2358#issuecomment-955870767):

Well it is a bit confusing fore sure. The command initiated by sudo is essentially telling the user, builduser, within the chroot to run the command `awk '/validpgpkeys/ {print $3}' .SRCINFO >key` which redirects to a file. So the redirect ran by builduser, is not ran as sudo within the chroot. 

It's possible to have builduser use sudo, ie: 

`sudo awk '/validpgpkeys/ {print $3}' .SRCINFO >key`

but in this case sudo was not needed or wanted.

On the other hand, running `systemd-nspawn` in this case does require sudo.
