#### Here's a snippet or screenshot that shows the problem:

```sh
#!/usr/bin/bash

var=1

if [ ! -z ${var+x} ]; then
        echo "var is set to '$var'"
fi
```

#### Here's what shellcheck currently says:
Use -n instead of ! -z


#### Here's what I wanted or expected to see:
The recommendation is wrong in some cases like this one:

```sh
$ cat ./test.sh

#!/usr/bin/bash

#var=1

if [ -n ${var+x} ]; then
        echo "var is set to '$var'"
else
        echo "var is undefined"
fi

$ ./test.sh
var is set to ''

$ cat test.sh
#!/usr/bin/bash

#var=1

if [ ! -z ${var+x} ]; then
        echo "var is set to '$var'"
else
        echo "var is undefined"
fi

$ ./test.sh
var is undefined
```
