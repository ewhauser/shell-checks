# [\#2157 Issue](https://github.com/koalaman/shellcheck/issues/2157) `open`: Use of -- in dirname invocations

#### <img src="https://avatars.githubusercontent.com/u/489338?u=fcf906ca6911e77a6889accb2fad658d10e90d39&v=4" width="50">[djmattyg007](https://github.com/djmattyg007) opened issue at [2021-02-22 04:20](https://github.com/koalaman/shellcheck/issues/2157):

#### For new checks and feature suggestions
- [x] https://www.shellcheck.net/ (i.e. the latest commit) currently gives no useful warnings about this
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related


#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash -e

cd "$(dirname "$0")/.."
```

#### Here's what shellcheck currently says:

```
No issues detected!
```

#### Here's what I wanted or expected to see:

Shellcheck should warn that there is no `--` between `dirname` and `"$0"`. Failure to do this could unintentionally cause `"$0"` to be treated as a flag to `dirname`, rather than an actual directory name.
