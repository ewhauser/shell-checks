#### <img src="https://avatars.githubusercontent.com/u/8651654?v=4" width="50">[jleffler](https://github.com/jleffler) commented at [2017-11-29T20:53:02Z](https://github.com/koalaman/shellcheck/issues/1064#issuecomment-347992655):

If the file `bar` does not exist, the subshell exits with status 1 (because the test fails and the `fubar` is not executed.  You'd also get a non-zero exit status if `bar` exists but `fubar` fails. So, the proposed warning message is wrong.  You'd get the same exit status from the sub-shell even if the exit were omitted — but that's different from "will only exit with status 0".

IMO, this is at best only loosely related to [#1031](https://github.com/koalaman/shellcheck/issues/1031) — they are quite different in most respects.
