# [\#466 Issue](https://github.com/koalaman/shellcheck/issues/466) `closed`: SC2164 error should not error when using bash -e

#### <img src="https://avatars.githubusercontent.com/u/1312785?v=4" width="50">[gdbtek](https://github.com/gdbtek) opened issue at [2015-09-19T08:19:28Z](https://github.com/koalaman/shellcheck/issues/466):

```
#!/bin/bash -e

cd '/nonexitpath'
echo 'hello'
```

In this case, echo statement will never be called and error will throw so shellcheck should ignore the error **SC2164** in the case where the script has inline **-e**

#### <img src="https://avatars.githubusercontent.com/u/302055?v=4" width="50">[deryni](https://github.com/deryni) commented at [2015-09-25T02:44:03Z](https://github.com/koalaman/shellcheck/issues/466#issuecomment-143107987):

Using `#!/bin/bash -e` isn't exactly the same as `set -e` since it only works if you run the script directly.

Running `bash some_script.sh` will run without the shebang argument being in effect.

I'd argue that that's reason enough to warn about it since it might not be in effect (it might make sense to include that detail in the warning in this case though). Or perhaps just warn against shebang argument setting entirely.

(There's also the whole "`set -e` isn't reliable/composable" issue but that's a different story entirely.)
