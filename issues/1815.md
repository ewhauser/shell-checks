#### Here's a snippet or screenshot that shows the problem:

This is a syntax error in bash (all versions I checked, up to 5.0.7(1)-release. Freenode #bash folks confirms it's a known bug that isn't likely to be fixed (they also say it's the same problem in zsh, but not in dash or ksh). 

```sh
#!/bin/bash
f() { 
  case "$c" in
    }) : ;;
  esac
}
```

However, if the `})` isn't on a new line, bash doesn't seem to mind:

```sh
#!/bin/bash
f() {
  case "$c" in })
    : ;;
  esac
}
```

#### Here's what shellcheck currently says:

Shellcheck merely warns "^-- SC1083: This } is literal. Check expression (missing ;/\n?) or quote it." in both cases.

#### Here's what I wanted or expected to see:

An error saying this is a syntax error if the shell is bash (or zsh, apparently).
