#### Here's a snippet or screenshot that shows the problem:

```sh
#! /bin/bash
output=$(grep -q foo <<<$'foo\nbar')
ech_rc=$?

if [[ ech_rc -eq 0 && -n "$output" ]]; then
  echo OK
fi

output=$(grep -q foo <<<$'foo\nbar')
echo_rc=$?

if [[ echo_rc -eq 0 && -n "$output" ]]; then
  echo OK
fi

output=$(grep -q foo <<<$'foo\nbar')
fb_rc=$?

if [[ fb_rc -eq 0 && -n "$output" ]]; then
  echo OK
fi

output=$(grep -q foo <<<$'foo\nbar')
fc_rc=$?

if [[ fc_rc -eq 0 && -n "$output" ]]; then
  echo OK
fi
```

#### Here's what shellcheck currently says:
```
$ shellcheck myscript
 
Line 12:
if [[ echo_rc -eq 0 && -n "$output" ]]; then
      ^-- SC1014: Use 'if cmd; then ..' to check exit code, or 'if [[ $(cmd) == .. ]]' to check output.
 
Line 26:
if [[ fc_rc -eq 0 && -n "$output" ]]; then
      ^-- SC1014: Use 'if cmd; then ..' to check exit code, or 'if [[ $(cmd) == .. ]]' to check output.
```


#### Here's what I wanted or expected to see:
Nothing.

`echo` and `fc` are both built-ins, but `ech` and `fb` aren't, which is why (I think) shellcheck currently doesn't warn about them.
