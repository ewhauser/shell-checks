```bash
#!/bin/bash
file="$(EDITOR=echo command nix edit "$@")"
vim "$file"
```

#### Here's what shellcheck currently says:

```
file="$(EDITOR=echo command nix edit "$@")"
```

The wiki page of SC2209 should mention under exception that this is a valid working use case to set a environment variable for this one command.


#### <img src="https://avatars.githubusercontent.com/u/382041?u=2d37dc3ad29ff648666cd73d85400098a779267b&v=4" width="50">[kira-bruneau](https://github.com/kira-bruneau) commented at [2022-10-23 21:15](https://github.com/koalaman/shellcheck/issues/2147#issuecomment-1288202909):

I ran into this same problem and it looks like shellcheck recommends wrapping the value in quotes to make it explicit.

From https://www.shellcheck.net/wiki/SC2209:
> If you do want to assign a literal string, use quotes to make this clear to shellcheck and humans alike.

So you should be able to use:
```
#!/bin/bash
file="$(EDITOR='echo' command nix edit "$@")"
vim "$file"
```

Otherwise, it can be ambiguous what you intended.
