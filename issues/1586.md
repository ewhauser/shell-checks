Shellcheck should add support for stdin.

- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related

#### Here's what shellcheck currently says:

```bash
$ cat test.sh | shellcheck
No files specified.
```

#### Here's what I wanted or expected to see:

Basically the same as:

```bash
$ shellcheck <(cat test.sh)
```

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2019-05-23T00:49:42Z](https://github.com/koalaman/shellcheck/issues/1586#issuecomment-495026229):

The man page now mentions this. Also note that this is not a ShellCheck convention, but can be expected to work on basically all Unix commands.

Since ShellCheck is intended to check script files and to help confused newbies, it's probably better that running `shellcheck` without arguments gives a `--help` type summary rather than appearing to hang waiting for stdin.

}
