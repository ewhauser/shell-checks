# [\#218 Issue](https://github.com/koalaman/shellcheck/issues/218) `closed`: SC2005 and SC2116 overlap, giving two warnings for the same issue

#### <img src="https://avatars.githubusercontent.com/u/1823914?v=4" width="50">[arth1](https://github.com/arth1) opened issue at [2014-08-20T14:51:42Z](https://github.com/koalaman/shellcheck/issues/218):

I have the following:

```
# Space separated list of network alias names
ALIASES=$(echo $(grep -v '^#' /etc/sysconfig/network-aliases 2>/dev/null))
        ^-- SC2116: Useless echo? Instead of 'cmd $(echo foo)', just use 'cmd foo'.
               ^-- SC2046: Quote this to prevent word splitting.
               ^-- SC2005: Useless echo? Instead of 'echo $(cmd)', just use 'cmd'.
```

Even if shellcheck doesn't know that echo sometimes is used to strip whitespace, it surely shouldn't report two warnings about useless echo for a single echo?

Is there any way to get the same result as the above without using three # shellcheck disable=  statements?  It seems overkill for a single short line...

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2014-09-04T15:44:13Z](https://github.com/koalaman/shellcheck/issues/218#issuecomment-54497377):

You're right. 2116 shouldn't be overlapping with 2005, and that's fixed in 9f38021.

You can do it without warnings both before and after this fix using an array:

```
aliases=( $(grep -v '^#' /etc/sysconfig/network-aliases 2>/dev/null) )
```
