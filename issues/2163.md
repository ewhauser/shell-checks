Just for someone landing here in the future from some search engine, it was likely caused by the Busybox's wget used by default in Alpine containers:

```shell
docker run --rm -it alpine:latest sh -c 'wget 2>&1| head -1; wget -q https://github.com/koalaman/shellcheck/releases/download/stable/shellcheck-stable.linux.x86_64.tar.xz'
BusyBox v1.32.1 () multi-call binary.
wget: error getting response: Connection reset by peer
```

Whereas using standard wget works:

```shell
docker run --rm -it alpine:latest sh -c 'apk add wget; wget --version | head -1; wget -q https://github.com/koalaman/shellcheck/releases/download/stable/shellcheck-stable.linux.x86_64.tar.xz; sha256sum shell*'
...
GNU Wget 1.21.1 built on linux-musl.
d0e9884d2deb14325e7331ff4b0c2c9771a3d49aad653c38ac489c9d27bf7b00  shellcheck-stable.linux.x86_64.tar.xz
```

Seems to be caused by issue in Alpine's `ssl_client` and unhandled missing `getrandom()` syscall on that host's kernel:

