# [\#494 Issue](https://github.com/koalaman/shellcheck/issues/494) `open`: warnings for trap

#### <img src="https://avatars.githubusercontent.com/u/197253?v=4" width="50">[kurahaupo](https://github.com/kurahaupo) opened issue at [2015-10-04T22:33:32Z](https://github.com/koalaman/shellcheck/issues/494):

In #429 **aw-altiscale** noted

```
tail -n "+${startline}" "${file}"  | head -n ${numlines} > "${outfile}"
```

>  On Ubuntu w/bash 4.x, I've seen tail throw out the broken pipe write error a few times.

However for tail to report write failure due to broken pipe, it would have to have `SIGPIPE` ignored, which would almost certainly be inherited from the invoking shell.

So I would like to suggest a new warning about `trap '' SIGPIPE` along the lines of _This may cause commands in pipelines to report write errors_

Also warn about trapping `SIGKILL` & `SIGSTOP`.

And while at it, warn about the nonportability using numeric codes for signals other than 1, 2, 3, 15.
