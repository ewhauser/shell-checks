Also interesting on macOS sed can not handle invalid utf8. That includes the nullbytes which are commonly used as separators to allow for filenames with newlines.

```
find . -print0 | xargs -0 sed 's/a/b/g' > /dev/null
```

Shows the following error on macOS

```
sed: RE error: illegal byte sequence
```
