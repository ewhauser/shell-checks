#### Here's a snippet or screenshot that shows the problem:

```sh
#!/usr/bin/env bash
set -euo pipefail

determine() {
  # unbound var here is inside subshell; errexit is ignored ▶ subshell dies only
  tmp=$( true "$unset_var" )
  echo inside
}

# call in a subshell via command substitution
out=$( determine )
echo "returned: ›$out‹"
echo passed
```

```

#### Here's what I wanted or expected to see:

```
$ shellcheck myscript
 
[Line 11:](javascript:setPosition(11, 3))
  out=$( determine )
  ^-- [SCXXXX](https://www.shellcheck.net/wiki/SCXXXX) (warning): Note that `determine` is NOT covered by the `set -u` guardrails. _See this https://mywiki.wooledge.org/BashFAQ/this article and/or use `set -E` + `trap 'exit 1' ERR`_

$

https://mywiki.wooledge.org/BashFAQ/112 (`set -u`) + https://mywiki.wooledge.org/BashFAQ/105 (`set -e`) do not seem to mention anything. Even if this feature cannot be completed - I am leaving this as a TIL-posterity.
