# [\#868 Issue](https://github.com/koalaman/shellcheck/issues/868) `closed`: SC2004 : False positive on $(( ${!foo} )) construct in bash

#### <img src="https://avatars.githubusercontent.com/u/6563434?v=4" width="50">[waptaff](https://github.com/waptaff) opened issue at [2017-04-01T15:17:22Z](https://github.com/koalaman/shellcheck/issues/868):

This construct using an [indirect reference](http://www.tldp.org/LDP/abs/html/ivr.html) triggers a warning on shellcheck 0.4.6:

```
#!/bin/bash
foo=bar
bar=1234
echo $(( ${!foo} + 1 ))
# Previous echo outputs 1235
```
shellcheck complains: `SC2004: $/${} is unnecessary on arithmetic variables.`

I *think* there should be an exception in SC2004, allowing indirect references, like in the `${#var}` and `${var%-}` cases.
