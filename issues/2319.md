#### Here's a snippet or screenshot that shows the problem:

```sh

x=toto
any 'qw('"$x"');$z'
```

#### Here's what shellcheck currently says:

```
In x.sh line 4:
any 'qw('"$x"');$z'
             ^----^ SC2016: Expressions don't expand in single quotes, use double quotes for that.

For more information:
  https://www.shellcheck.net/wiki/SC2016 -- Expressions don't expand in singl...
```

#### Here's what I wanted or expected to see:

nothing as there is no error here.

Interesting to note: replace `any` by `perl` and it works...

#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2021-09-17 03:08](https://github.com/koalaman/shellcheck/issues/2319#issuecomment-921419636):

I've not seen the `any` command before, but you're right, it's clearly meant to accept literal dollar signs. ShellCheck has a hard coded list of commands including `perl` to try to avoid false positives, but it's hard to keep track of all of them. Please ignore it with a disable directive, and don't hesitate to permanently ignore this warning in your `.shellcheckrc` since it's meant to help newbies discover the difference between single and double quotes, and not to stop you from using single quotes correctly. 

I've updated the wiki to be more explicit about this. Thanks!
