# [\#267 Issue](https://github.com/koalaman/shellcheck/issues/267) `closed`: SC2086: false positive while passing parameters in variable

#### Labels: working as intended

#### <img src="https://avatars.githubusercontent.com/u/6888121?v=4" width="50">[volans-](https://github.com/volans-) opened issue at [2014-12-21T12:53:56Z](https://github.com/koalaman/shellcheck/issues/267):

``` bash
PARAMS="-a -b -c 'test'"

program ${PARAMS}
```

In this case double quoting will prevent to interpret the parameters and the whole PARAMS string will be passed as the first parameter.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2014-12-24T03:12:44Z](https://github.com/koalaman/shellcheck/issues/267#issuecomment-68022189):

No it won't. Here's an example:

```
program() {
  echo "The first param is $1"
}
PARAMS="-a -b -c 'test'"
program ${PARAMS}
```
