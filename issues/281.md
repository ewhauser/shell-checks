# [\#281 Issue](https://github.com/koalaman/shellcheck/issues/281) `open`: Should trigger for 'return' outside a function

#### <img src="https://avatars.githubusercontent.com/u/316894?v=4" width="50">[pixelb](https://github.com/pixelb) opened issue at [2015-01-10T18:34:33Z](https://github.com/koalaman/shellcheck/issues/281):

This should trigger a failure:

```
#!/bin/sh
usually_ok || return 1
echo "any failure now hidden"
```

Note 'return' is ok if a shell script is being sourced,
so perhaps shellcheck needs an option to specify that's not the case?

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2015-01-10T20:08:01Z](https://github.com/koalaman/shellcheck/issues/281#issuecomment-69469907):

Hmm.. Maybe a new directive to inform shellcheck of how the file is used?

Something like 

```
# shellcheck context=sourced
```

could tell shellcheck not to warn about this, and also avoid warning about things like unused variables.
