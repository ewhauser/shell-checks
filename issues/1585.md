#### Here's a snippet or screenshot that shows the problem:

```sh

#!/bin/bash

function process_repo() {
        :
}

function search_repos() {
    local failed=0
    while read -r line; do
        repo="$line"
        echo "processing ${repo}" >&3
        echo "${repo}"
        process_repo "${repo}" || failed=1
    done < "${ACTIVE_REPOS}"               # << line 14
    return "${failed}"
}


```

#### Here's what shellcheck currently says:
this is ok and reported ok, but if I modify line 14: add a | (pipe):
```sh
 done < "${ACTIVE_REPOS}"  | sort -u
```
shellcheck will report SC2030 and SC2031 on variable failed, which is obviously false

#### Here's what I wanted or expected to see:
do not mix double and single pipes (or ??). 


#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2019-07-03T03:17:29Z](https://github.com/koalaman/shellcheck/issues/1585#issuecomment-507921434):

ShellCheck is right, this is not ok. Here's a simplified example:

```
#!/bin/bash

failed=0
while read -r _; do
  echo "FAILED"
  failed=1
done < <(echo hello) | sort -u
echo "Failed: $failed"
```

Due to the pipeline, the assignment to `failed` is lost:

```
FAILED
Failed: 0
```

if you remove the `| sort -u`, you get 1 as expected.
