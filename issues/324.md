#### <img src="https://avatars.githubusercontent.com/u/148037?v=4" width="50">[nomeata](https://github.com/nomeata) commented at [2015-05-25T15:01:24Z](https://github.com/koalaman/shellcheck/issues/324#issuecomment-105247882):

shellcheck should probably read the files as binary data, and not try to interpret them as text of any encoding. That would imply not using `hGetContents`, though.

This has been reported against the Debian package as well, see http://bugs.debian.org/786781


#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2017-01-22T23:41:29Z](https://github.com/koalaman/shellcheck/issues/324#issuecomment-274370109):

Fixed in 46fb91b. Files are now always interpreted leniently as UTF-8, and will fall back to ISO-8859-1 (Latin1) failing that. The Haskell libraries are no longer involved in the decoding.
