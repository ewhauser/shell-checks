#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2019-07-03T05:19:41Z](https://github.com/koalaman/shellcheck/issues/1635#issuecomment-507942033):

ShellCheck is right, you have an unused variable named `compute_missing_padding` in addition to a global function of the same name. You can see that the `local` does not affect the function, which still leaks out:

```
function hiddenlistgitremote() {
    local compute_missing_padding;
    function compute_missing_padding() {
        printf "I am a doer\\n";
    }
}
hiddenlistgitremote
compute_missing_padding
```

As far as I know, there's no way (short of a subshell) to limit the scope of a function in bash.
