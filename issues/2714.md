#### <img src="https://avatars.githubusercontent.com/u/49727155?u=d1e61dd035020ecc7481b067e89256414ddc191d&v=4" width="50">[katexochen](https://github.com/katexochen) opened issue at [2023-03-16 09:26](https://github.com/koalaman/shellcheck/issues/2714):

Currently, only there are no binaries released for MacOS with arm arch (M1/M2), see related issue https://github.com/koalaman/shellcheck/issues/2680.

https://github.com/tpoechtrager/osxcross is currently used to build the darwin_x86_64 binary, the project also supports building for darwin_aarch64. Would be great if anyone could add this to the pipeline, thanks!

AFAICT to get arm64 builds going, we'd first need to use a maintained osxcross docker image that is up-to date and can target arm64.
This one seems to be maintained properly: [hub.docker.com/r/crazymax/osxcross/tags](https://hub.docker.com/r/crazymax/osxcross/tags)
Once that is tested with amd64 builds, we can probably do a new Dockerfile for the arm64 too.

#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2024-02-26 01:57](https://github.com/koalaman/shellcheck/issues/2714#issuecomment-1963188965):

I'm glad to hear it's working, thanks! The binary is huge (59MB) because A. stripping it breaks the linker default signature (and there's no `codesign` utility in the Docker image), and B. it can't use `-split-sections` + `ld -dead_strip` because the tooling only allows 16 character section names and the only compatible version of GHC I was able to build uses more.

Hopefully this will improve in the future.
