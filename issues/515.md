#### <img src="https://avatars.githubusercontent.com/u/302055?v=4" width="50">[deryni](https://github.com/deryni) commented at [2015-10-21T19:48:33Z](https://github.com/koalaman/shellcheck/issues/515#issuecomment-150006132):

Inside a `$(...)` expression you have an entirely new parsing context. The inner quotes don't interfere with the external quotes and the external quotes don't quote the internal contents.

For a quick illustration run the following and look at the output. Notice how every version with "unquoted" at the end outputs `argc: 5`?

```
c ()
{
    printf 'argc: %s\n' "$#";
    printf 'argv: %s\n' "$@"
}

echo '# unquoted'
c a b c d e
echo '# quoted'
c a "b c d" e
echo '# unquoted'
: $(c a b c d e >&2)
echo '# unquoted quoted'
: $(c a "b c d" e >&2)
echo '# quoted unquoted'
: "$(c a b c d e >&2)"
echo '# quoted quoted'
: "$(c a "b c d" e >&2)"
```
