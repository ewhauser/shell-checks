#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bash

echo "this prevents gloal application of the line-specific disable on line 5"
# Line 5:
# shellcheck disable=SC2207 #https://github.com/koalaman/shellcheck/wiki/SC2207
# shellcheck disable=SC2012 #https://github.com/koalaman/shellcheck/wiki/SC2012
# shellcheck disable=SC2006 #https://github.com/koalaman/shellcheck/wiki/SC2006
ls -la /foo/bar/thing*.deb | tail -1 | sed 's/.* +//'

declare -a packages

packages=(      
        # shellcheck disable=SC2207 #https://github.com/koalaman/shellcheck/wiki/SC2207
        # shellcheck disable=SC2012 #https://github.com/koalaman/shellcheck/wiki/SC2012
        $(ls -la /foo/bar/thing*.deb | tail -1 | sed 's/.* +//')

        # shellcheck disable=SC2207 #https://github.com/koalaman/shellcheck/wiki/SC2207
        # shellcheck disable=SC2012 #https://github.com/koalaman/shellcheck/wiki/SC2012
        # shellcheck disable=SC2006 #https://github.com/koalaman/shellcheck/wiki/SC2006
        `ls -la /foo/bar/thing*.deb | tail -1 | sed 's/.* +//'`

        $(
            ls -la /foo/bar/thing*.deb | tail -1 | sed 's/.* +//'
         )

        $(
            # shellcheck disable=SC2207 #https://github.com/koalaman/shellcheck/wiki/SC2207
            ls -la /foo/bar/thing*.deb | tail -1 | sed 's/.* +//'
         )

        $(
            # shellcheck disable=SC2012 #https://github.com/koalaman/shellcheck/wiki/SC2012
            ls -la /foo/bar/thing*.deb | tail -1 | sed 's/.* +//'
         )

        $(
            # shellcheck disable=SC2207 #https://github.com/koalaman/shellcheck/wiki/SC2207
            # shellcheck disable=SC2012 #https://github.com/koalaman/shellcheck/wiki/SC2012
            ls -la /foo/bar/thing*.deb | tail -1 | sed 's/.* +//'
         )

        $(
            # shellcheck disable=SC2012 #https://github.com/koalaman/shellcheck/wiki/SC2012
            # shellcheck disable=SC2207 #https://github.com/koalaman/shellcheck/wiki/SC2207
            ls -la /foo/bar/thing*.deb | tail -1 | sed 's/.* +//'
         )

)

deb-dpkg "${packages[@]}"
```

```
Line 15	SC2207: Prefer mapfile or read -a to split command output (or quote to avoid splitting).

Line 20	SC2006: Use $(...) notation instead of legacy backticks `...`.
Line 20	SC2012: Use find instead of ls to better handle non-alphanumeric filenames.
Line 22	SC2207: Prefer mapfile or read -a to split command output (or quote to avoid splitting).
Line 23	SC2012: Use find instead of ls to better handle non-alphanumeric filenames.
Line 26	SC2207: Prefer mapfile or read -a to split command output (or quote to avoid splitting).
Line 28	SC2012: Use find instead of ls to better handle non-alphanumeric filenames.
Line 31	SC2207: Prefer mapfile or read -a to split command output (or quote to avoid splitting).
Line 36	SC2207: Prefer mapfile or read -a to split command output (or quote to avoid splitting).
Line 42	SC2207: Prefer mapfile or read -a to split command output (or quote to avoid splitting).
```
#### Here's what I wanted or expected to see:

Nothing. The local pragmas should lexically apply, even with the intervention of a subshell or backtics. However, if we want to go with having to have the local pragmas in the subshell command itself, they still have to work. Notice that the SC2207 disable pragma fails in all of the subshell scenarios - alone, first in a list, last in a list.

Notice also that if you globally apply the pragmas by commenting out line 3, they all work.
