# [\#497 Issue](https://github.com/koalaman/shellcheck/issues/497) `closed`: Feature discussion / request: Recommend "$HOME" over usage of path /home/foo

#### <img src="https://avatars.githubusercontent.com/u/66993?v=4" width="50">[b-long](https://github.com/b-long) opened issue at [2015-10-06T19:20:55Z](https://github.com/koalaman/shellcheck/issues/497):

I'm not sure how strongly I feel about this, but I thought it might be useful to discuss.  Would there be a benefit to the shellcheck community to recommend the usage of : 

```
"$HOME"/bar/baz
# Assume running as 'foo'
```

over the use of

```
/home/foo/bar/baz
```

What are your thoughts? Care to comment @jimhester ?

#### <img src="https://avatars.githubusercontent.com/u/205275?v=4" width="50">[jimhester](https://github.com/jimhester) commented at [2015-10-06T19:26:10Z](https://github.com/koalaman/shellcheck/issues/497#issuecomment-145972030):

Home directory structure is not standardized across distributions or OSs, so I don't see how this detection would work. This also assumes that every usage refers to the current users' home directory, which is also not always true.


#### <img src="https://avatars.githubusercontent.com/u/66993?v=4" width="50">[b-long](https://github.com/b-long) commented at [2015-10-06T19:32:41Z](https://github.com/koalaman/shellcheck/issues/497#issuecomment-145973521):

Suppose only a subset of OS's were supported, for example those mentioned in [Shellcheck's installation instructions](https://github.com/koalaman/shellcheck#installing) (Which I think means Debian-based, RHEL-based, OSX and _Windows_*).  

If we could determine that a script includes something like : 

```
[ "$(whoami)" != "foo" ]
```

Followed by the usage of :

```
/home/foo/bar/baz
```

?

<sub>\* Windows is in italics above because I'm not sure if the support is for Cygwin usage, or something else</sub>
