# [\#2379 Issue](https://github.com/koalaman/shellcheck/issues/2379) `open`: Detect invalid identifiers in local and declare to avoid unintended side-effects

#### <img src="https://avatars.githubusercontent.com/u/2787581?v=4" width="50">[grische](https://github.com/grische) opened issue at [2021-11-15 14:11](https://github.com/koalaman/shellcheck/issues/2379):

#### For new checks and feature suggestions
- [x] https://www.shellcheck.net/ (i.e. the latest commit) currently gives no useful warnings about this
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash

# shellcheck enable=all

function dummy() {
        local a, b
        a=1; b=2;
}
dummy
echo "${a}${b}"
```

#### Here's what shellcheck currently says:
Nothing


#### Here's what I wanted or expected to see:
```
Line 6:
        local a, b
               ^-- SCxxxx (error): invalid identifier
```

Running the script with a bash, it will print the following error:
```sh
$ bash /tmp/script.sh
/tmp/script.sh: line 6: local: `a,': not a valid identifier
12
```
