I am using the [Bash Infinity](https://github.com/niieani/bash-oo-framework) framework which provides some aliases, like `[array]`. I use them in the screenshot and code below. In the screenshot I am using vim with Syntastic that reads the output of shellcheck and annotates the problematic lines. While the code works fine, shellcheck complains as it cannot understand the custom aliases. Is there a way to load the aliases in shellcheck or sth to avoid what happens?

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/usr/bin/env bash

function ansible_provision() {
  [array] tags=(all)
  [array] skip_tags
  [array] limit=()

  . "$_GO_USE_MODULES" 'instance'

  ansible-playbook \
    --tags="${tags[@]}" \
    --skip-tags="${skip_tags[@]}" \
    --limit="${limit[@]}" \
    -- "$PROJECT_DIR"/playbook.yml >&${STDOUT}
}
```

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2018-05-15T22:54:12Z](https://github.com/koalaman/shellcheck/issues/1132#issuecomment-389339282):

There's currently no way to do so, no :/

Aliases are a particular annoyance for ShellCheck because they modify subsequent parsing. For example, `((RANDOM%2)) && alias forever='while true'` followed by `forever; do sleep 1; done` will cause a bash syntax error 50% of the times you run the script. They're also rarely used in scripts at all, being disabled by default and all, so there's not much in the way of support at the moment.
