# [\#245 Issue](https://github.com/koalaman/shellcheck/issues/245) `closed`: Can't parse function named $

#### <img src="https://avatars.githubusercontent.com/u/324632?v=4" width="50">[e0da](https://github.com/e0da) opened issue at [2014-10-28T18:12:26Z](https://github.com/koalaman/shellcheck/issues/245):

I know it's silly, but I have a function called $ to facilitate copying and pasting command line snippets which are prefixed with $.

``` sh
function $ {
  "$@"
}
```

So the $ triggers SC1000, and I just disable that. But no matter what I try with ignoring, quoting, whatever, shellcheck doesn't wan't to parse past this function.

```
In functions.zsh line 8:
}
^-- SC1070: Parsing stopped here because of parsing errors.
```
