# [\#745 Issue](https://github.com/koalaman/shellcheck/issues/745) `closed`: `sort -o` should not trigger SC2094

#### <img src="https://avatars.githubusercontent.com/u/1275053?v=4" width="50">[benhartley](https://github.com/benhartley) opened issue at [2016-10-10T19:16:14Z](https://github.com/koalaman/shellcheck/issues/745):

This command:

``` sh
sort -o x < x
```

throws this error:

```
Make sure not to read and write the same file in the same pipeline. [SC2094]
```

This is incorrect since `-o` enables you to [use the same file](http://pubs.opengroup.org/onlinepubs/9699919799/utilities/sort.html):

```
-o  output
    Specify the name of an output file to be used instead of the standard 
    output. This file can be the same as one of the input files.
```

#### <img src="https://avatars.githubusercontent.com/u/243351?v=4" width="50">[brother](https://github.com/brother) commented at [2016-10-11T07:46:09Z](https://github.com/koalaman/shellcheck/issues/745#issuecomment-252837383):

[SC2094](https://github.com/koalaman/shellcheck/wiki/SC2094) is general purpose not for sort specific. When you have evaluated the use just put a [directive](https://github.com/koalaman/shellcheck/wiki/Directive) before the line and continue on.

#shellcheck disable=SC2094
