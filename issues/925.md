# [\#925 Issue](https://github.com/koalaman/shellcheck/issues/925) `closed`: SC1073: version: 0.3.5: bug with parsing if inside case

#### <img src="https://avatars.githubusercontent.com/u/84627?v=4" width="50">[layeratwork](https://github.com/layeratwork) opened issue at [2017-06-02T17:50:34Z](https://github.com/koalaman/shellcheck/issues/925):

This still fails on shellcheck.net.

```sh

#! /bin/bash

distrule=$1
installdcls=$2

case $distrule in
    alldist)
	die '--distrule alldist is not a good idea (see bug15923)'
	;;
    cleancleandist|cleandist|dist|qdist)
	if [ "$installdcls" -a '(' "$distrule" = "qdist" ')' ]; then 
	    die 'dcl installation is incompatible with --distrule qdist'
	fi
	;;
    *) die invalid dist rule: $distrule ;;
esac

```

```

        ^-- SC1009: The mentioned parser error was in this if expression.

                                   ^-- SC1072: Expected "]". Fix any mentioned problems and try again.
```

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2017-06-02T19:14:27Z](https://github.com/koalaman/shellcheck/issues/925#issuecomment-305884952):

This is a bug. `\(` is recognized but `'('` is not.

The parsing used for `[` is different from other commands to allow for more helpful error messages regarding `>` and such.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2017-06-03T18:56:43Z](https://github.com/koalaman/shellcheck/issues/925#issuecomment-305994539):
