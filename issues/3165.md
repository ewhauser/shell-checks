# [\#3165 Issue](https://github.com/koalaman/shellcheck/issues/3165) `open`: opening a pipe to a `while read` loop flags assignment of `$?` as SC2319

#### <img src="https://avatars.githubusercontent.com/u/3608596?u=b42276d812f60d8cab0b43c3dc78b5aad07280ef&v=4" width="50">[solstice333](https://github.com/solstice333) opened issue at [2025-03-21 17:40](https://github.com/koalaman/shellcheck/issues/3165):

Min repro:

```bash
#!/usr/bin/env bash

while read line; do
    rev=$line

    [[ "${rev#foo}" == "$rev" ]]
    is_foo=$?   # This $? refers to a condition, not a command. Assign to a variable to avoid it being overwritten. See SC2319.

    echo "$rev: $is_foo"
done < $TEMP/foo.lst | sort
```
where `is_foo=$?` is flagged with SC2319. The expectation is for this line *not* to be flagged bc `$?` is being assigned.

Removing the pipe of `while read ...` to `sort` causes the SC2319 warning to disappear:

```bash
#!/usr/bin/env bash

while read line; do
    rev=$line

    [[ "${rev#foo}" == "$rev" ]]
    is_foo=$?

    echo "$rev: $is_foo"
done < $TEMP/foo.lst
```
