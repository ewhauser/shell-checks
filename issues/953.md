# [\#953 Issue](https://github.com/koalaman/shellcheck/issues/953) `closed`: discrepancy between `declare -A` and `local -A`

#### <img src="https://avatars.githubusercontent.com/u/708570?v=4" width="50">[mmlb](https://github.com/mmlb) opened issue at [2017-07-13T18:40:22Z](https://github.com/koalaman/shellcheck/issues/953):

#### For bugs
- Rule Id (if any, e.g. SC1000): SC2154
- My shellcheck version (`shellcheck --version` or "online"): online
- [x] I tried on shellcheck.net and verified that this is still a problem on the latest commit


#### Here's a snippet or screenshot that shows the problem:

```sh

#!/usr/bin/bash

function arrDeclare() {
  declare -A arrd
  arrd[foo]=bar
  echo "${arrd[*]}"
}

function arrLocal() {
  local -A arrl
  arrl[fool]=bar
  echo "${arrl[*]}"
}

```

#### Here's what shellcheck currently says:

```
Line 11:
  arrl[fool]=bar
       ^-- SC2154: fool is referenced but not assigned.
```

#### Here's what I wanted or expected to see:
nothing

#### More info
This is similar to #359, the difference is in how the array was declared. According to [bash docs](https://www.gnu.org/software/bash/manual/html_node/Bash-Builtins.html#index-local) `local` is synonymous with `declare` + local to function.
