# [\#2715 Issue](https://github.com/koalaman/shellcheck/issues/2715) `open`: SC3049 firing when -- arg terminator used with trap

#### For bugs
- Rule Id (if any, e.g. SC1000): **SC3049**
- My shellcheck version (`shellcheck --version` or "online"): **0.9.0**
- [x] The rule's wiki page does not already cover this: **I reviewed [SC3049](https://www.shellcheck.net/wiki/SC3049)**
- [x] I tried on https://www.shellcheck.net/ and verified that this is still a problem on the latest commit

#### Here's a snippet or screenshot that shows the problem:

This has the error:
```sh
#!/bin/sh

# this line triggers SC3049
trap -- 'date' HUP

# this line does not
trap 'date' HUP
```

#### Here's what shellcheck currently says:

```
[Line 3:]
trap -- 'date' HUP
        ^-- [SC3049](https://www.shellcheck.net/wiki/SC3049) (warning): In POSIX sh, using lower/mixed case for signal names is undefined.
```

#### Here's what I wanted or expected to see:

I didn't expect an error.

The opengroup [manpage for trap](https://www.unix.com/man-page/POSIX/1posix/trap/) suggests that running trap without arguments prints the current signal handlers in a format "suitable for reinput to the shell as commands". That format includes the `--`; for example:

```sh
/ $ trap 'date' USR2
/ $ trap
trap -- 'date' USR2
/ $ 
```
