#### <img src="https://avatars.githubusercontent.com/u/1823914?v=4" width="50">[arth1](https://github.com/arth1) commented at [2017-06-12T17:22:21Z](https://github.com/koalaman/shellcheck/issues/865#issuecomment-307857403):

After further consideration, the /* should probably not be warned against.  Depending on your file system, you can have a file named '*' in the root directory:

$ /* this is a comment */
Are you trying to execute a comment?
$ cat /\\\*
#!/bin/sh
echo "Are you trying to execute a comment?"

However, I can see no possible way that // can be a legal command, and I have seen // used in bash scripts which worked fine until the execution path that had // in it failed.
