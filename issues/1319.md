# [\#1319 Issue](https://github.com/koalaman/shellcheck/issues/1319) `closed`: Function definitions should be recognized in terms of any compound command

#### <img src="https://avatars.githubusercontent.com/u/3674065?v=4" width="50">[jcbollinger](https://github.com/jcbollinger) opened issue at [2018-08-22T20:21:28Z](https://github.com/koalaman/shellcheck/issues/1319):

POSIX specifies that shell functions can be defined in terms of *any* compound command.  By far the most common used to define shell functions is a brace-enclosed list, but parentheses-enclosed lists, `for` and `while` loops, `if` statements, and `case` statements are all compound commands, too.  Shellcheck accepts the brace-enclosed and parentheses-enclosed list variations, but rejects the others -- it cannot even parse them.  At least some real-world shells do accept them, per POSIX specifications.  Bash, for example, does so.
