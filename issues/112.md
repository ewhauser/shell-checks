# [\#112 Issue](https://github.com/koalaman/shellcheck/issues/112) `closed`: /bin/bash requires a warning

#### <img src="https://avatars.githubusercontent.com/u/433817?v=4" width="50">[grimreaper](https://github.com/grimreaper) opened issue at [2014-03-10T18:31:34Z](https://github.com/koalaman/shellcheck/issues/112):

Using #!/bin/bash is wrong because:
-     Sometimes bash isn't installed.
-   If it is installed, it may not be in /bin
-    If it is in /bin, the user may have decided to set PATH to use a different installation of bash. Using an absolute path like this overrides the user's choices.
  -   bash shouldn't be used for scripts intended for portability

As such the user ought to use /bin/sh and write POSIX scripts or use /usr/bin/env bash

The problem is bash isn't basic on some systems: e.g., FreeBSD has tcsh(1)
and sh(1) by default (and so bash is installed in /usr/local/bin/bash).  In
addition, on OSX, the default bash is very old so many users install a new
one to /opt/local/bin/bash and would prefer that scripts use that instead
of /bin/bash.
