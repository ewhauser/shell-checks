# [\#719 Issue](https://github.com/koalaman/shellcheck/issues/719) `open`: Return in subshell in a function

#### <img src="https://avatars.githubusercontent.com/u/952007?v=4" width="50">[szepeviktor](https://github.com/szepeviktor) opened issue at [2016-07-31T16:39:22Z](https://github.com/koalaman/shellcheck/issues/719):

``` bash
A_function() {
    find -type f | while read -r FILE; do
        if [ "$FILE" = "./some_name.sh" ]; then
            echo "Got it."
            return 1
        fi
    done

    return 0
}

A_function
echo $?
```

As you can see the return value of the function will always be zero.

Could we have a warning for it? "`return 1` will never set the return value of the function"
