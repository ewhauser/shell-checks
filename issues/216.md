# [\#216 Issue](https://github.com/koalaman/shellcheck/issues/216) `closed`: SC2034 is too zealous?

#### <img src="https://avatars.githubusercontent.com/u/1823914?v=4" width="50">[arth1](https://github.com/arth1) opened issue at [2014-08-18T15:41:21Z](https://github.com/koalaman/shellcheck/issues/216):

I have a backup program that among other things does this:

```
LEVEL="9"   # default to incremental at level 9

while [ $# -ge 1 ]; do
    case $1 in
    -l )    LEVEL=$2
        shift ;;
    -v )    VERBOSE=1
        ;;
    * ) echo "Usage: $0 [-l level]"
        exit 0 ;;
    esac
    shift
done
...
extvols=$(dump -W | grep '^>' | sed -e 's/).*//;s/( .*\///' |\
awk '{print "dump -u$LEVEL -q -y -f ${backupdir}/"$3"_level$LEVEL.dump "$2}')
eval 2>/dev/null "$extvols"
```

And shellcheck complains twice that $LEVEL is unused.
Granted, it's easy enough to exclude that checker, but shouldn't it see that there is a "$LEVEL" string, even if it's used in an argument to a foreign program?

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2014-08-18T19:50:22Z](https://github.com/koalaman/shellcheck/issues/216#issuecomment-52544976):

You shouldn't be getting two complaints, I filed #217 for that. 

ShellCheck doesn't intend to figure out usage through indirection or evaluation, though (and eval shouldn't be used here in the first place).
