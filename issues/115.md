# [\#115 Issue](https://github.com/koalaman/shellcheck/issues/115) `open`: Variables, which are accessed in one function

#### <img src="https://avatars.githubusercontent.com/u/29916?v=4" width="50">[pihentagy](https://github.com/pihentagy) opened issue at [2014-03-12T15:15:31Z](https://github.com/koalaman/shellcheck/issues/115):

This is all kinds of messed up, dude:

```
#!/bin/bash
fun1() {
   a=2
   echo $a
}
fun2() {
   b=2
   echo $b
}
```

How about advicing prefixing the variable with `local`, if it is used just in one function? Yes, there may be valid cases, where it goes to global scope, but that could cause problems, so better double-check...

Of course, there's a [subtlety](http://www.tldp.org/LDP/abs/html/localvar.html#FTN.AEN18568):

> However, as Thomas Braunberger points out, a local variable declared in a function is also visible to functions called by the parent function.
> 
> …
> 
> This is documented in the Bash manual:
> 
> "Local can only be used within a function; it makes the variable name have a visible scope restricted to that function _and its children._" [emphasis added] _The ABS Guide author considers this behavior to be a bug._
