# [\#544 Issue](https://github.com/koalaman/shellcheck/issues/544) `open`: Proposal: warn on errexit and multiple commands in command substitution

#### <img src="https://avatars.githubusercontent.com/u/527924?v=4" width="50">[christophgysin](https://github.com/christophgysin) opened issue at [2015-11-09T11:00:01Z](https://github.com/koalaman/shellcheck/issues/544):

Unless in poix mode, bash strips errexit in command substitutions. This means that if more than one command is put inside command substitution, all but the last can fail silently without being caught:

```
  $ bash -c 'set -e; x=$(false; true)'; echo $?
  0
```

If only one command is placed inside the command substitution, it works as expected, because the return value of the substitution is the return value of the last command:

```
  $ bash -c 'set -e; x=$(false)'; echo $?
  1
```

If the command is a function though, only the return value of the function is checked, all commands inside the function run without errexit:

```
  $ bash -c 'set -e; f() { false; true; }; x=$(f)'; echo $?
  0
```

If errexit is to be used inisde the command substitution, it needs to be enabled explicitly:

```
  $ bash -c 'set -e; x=$(set -e; false; true)'; echo $?
  1
```

```
  $ bash -c 'set -e; f() { false; true; }; x=$(set -e; f)'; echo $?
  1
```

In posix mode, errexit is inherited:

```
  $ bash -c 'set -e; set -o posix; x=$(false; true)'; echo $?
  1
```

```
  $ bash -c 'set -e; set -o posix; f() { false; true; }; x=$(f)'; echo $?
  1
```

It would be great if shellcheck could warn about using multiple commands or funtions inside command substitution if errexit is enable but not posix mode.
