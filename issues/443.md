# [\#443 Issue](https://github.com/koalaman/shellcheck/issues/443) `closed`: SC2154 triggers incorrectly on ${!prefix*} expansions.

#### <img src="https://avatars.githubusercontent.com/u/302055?v=4" width="50">[deryni](https://github.com/deryni) opened issue at [2015-08-21T03:00:51Z](https://github.com/koalaman/shellcheck/issues/443):

This script:

```
#!/bin/bash

# shellcheck disable=SC2034
osfoo=osfoov osbar=osbarv osblah=osblahv

echo "${!os*}"
```

results in this output from the shellcheck site

```
   1  #!/bin/bash
   2  
   3  # shellcheck disable=SC2034
   4  osfoo=osfoov osbar=osbarv osblah=osblahv
   5  
   6  echo "${!os*}"
            ^––SC2154 os is referenced but not assigned.
```

But that's not what `${!os*}` is doing. That expansion is getting the list of variables that match that prefix.

The output from running that script is `osbar osblah osfoo`.
