# [\#1554 Issue](https://github.com/koalaman/shellcheck/issues/1554) `open`: Allow file level and shellcheckrc shell and severity directives

#### <img src="https://avatars.githubusercontent.com/u/109152?v=4" width="50">[scop](https://github.com/scop) opened issue at [2019-04-22T19:32:15Z](https://github.com/koalaman/shellcheck/issues/1554):

#### For new checks and feature suggestions
- [ ] shellcheck.net (i.e. the latest commit) currently gives no useful warnings about this
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related

#### Here's what I wanted or expected to see:

Would be good to be able to use file wide and `shell` and `severity` directives, like:

```
# shellcheck shell=bash severity=error"
```
...as well as in shellcheckrc:
```
shell=bash
severity=error
```
