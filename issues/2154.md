#### Here's what I wanted or expected to see:

I got to this point because shellcheck threw an SC2045 at me (Iterating over ls output is fragile. Use globs.) when I was using
```
for i in $(ls "$THEDIR"/*)
...
```
and told me to use a glob. However a glob does not work on an empty dir. To wit:
```
$ THEDIR=o/b/b

$ ls -la $THEDIR
total 0
drwxrwxr-x 2 mpersico5 users 17 Feb 19 14:44 .
drwxrwxr-x 3 mpersico5 users 15 Feb 19 14:36 ..
-rw-rw-r-- 1 mpersico5 users  0 Feb 19 14:44 foo

$ for i in "$THEDIR"/*; do     echo "$i"; done
o/b/b/foo

$ rm o/b/b/foo
rm: remove regular empty file 'o/b/b/foo'? y

$ for i in "$THEDIR"/*; do echo "$i"; done
o/b/b/*
```
I want the last result to be empty, not an unresolved glob string. I think the glob advice is potentially dangerous.

#### <img src="https://avatars.githubusercontent.com/u/3129206?v=4" width="50">[matthewpersico](https://github.com/matthewpersico) commented at [2021-02-20 15:40](https://github.com/koalaman/shellcheck/issues/2154#issuecomment-782700801):

Update: My last sentence was prophetic. I started reading the bash manpage searching for 'glob' and I found this:
```
$ shopt nullglob
nullglob        off
$ for i in *; do echo $i;done
*
$ shopt -s nullglob
$ shopt nullglob
nullglob        on
$ for i in *; do echo $i;done
$
```
