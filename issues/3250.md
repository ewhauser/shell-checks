# [\#3250 Issue](https://github.com/koalaman/shellcheck/issues/3250) `open`: does man page confuse search-path and source-path?

#### <img src="https://avatars.githubusercontent.com/u/19217?v=4" width="50">[alastair](https://github.com/alastair) opened issue at [2025-07-18 08:24](https://github.com/koalaman/shellcheck/issues/3250):

in `shellcheck(1)`, the documentation for `-P` says

>     -P SOURCEPATH, --source-path=SOURCEPATH
>        Specify paths to search for sourced files, separated by : on Unix and ; on Windows.  This is equivalent to specifying
>        search-path directives.

Note that this describes "search-path directives". In the Directives section, "search-path" doesn't exist, although "source-path" does, and that also matches with the long format of the argument. It seems like "search-path" here is an error here, perhaps since it was introduced in https://github.com/koalaman/shellcheck/commit/d9e419d60fbd1507b573f33590a7f4b61841aa5b
