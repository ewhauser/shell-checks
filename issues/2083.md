# [\#2083 Issue](https://github.com/koalaman/shellcheck/issues/2083) `open`: [Feature Request] Add warning for docker interactive mode

#### <img src="https://avatars.githubusercontent.com/u/8686001?u=b7d947f51cc8b570e54b6256c2166b61c6085aae&v=4" width="50">[Nkmol](https://github.com/Nkmol) opened issue at [2020-11-03 08:37](https://github.com/koalaman/shellcheck/issues/2083):

#### For new checks and feature suggestions
- [x] https://www.shellcheck.net/ (i.e. the latest commit) currently gives no useful warnings about this
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related


#### Here's a snippet or screenshot that shows the problem:

```sh

#!/bin/bash

docker exec container_1 bash << 'HEREDOC'
        ls -l
HEREDOC

```

#### Here's what shellcheck currently says:
"No issues detected!"


#### Here's what I wanted or expected to see:
I want shellcheck to warn about not using docker interactive mode (`-i`). This is simply is not want you would expect. Enabling interactive mode is needed in this case to keep STDIN open.

This would be expected for all of the redirect operators, like:

- `echo "ls -l" | docker exec container_1 bash`
- `docker exec container_1 other_command < myfile.txt`
