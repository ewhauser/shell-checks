#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2021-09-17 02:12](https://github.com/koalaman/shellcheck/issues/2314#issuecomment-921399847):

Neat case. ShellCheck does not warn when `a && b || c` is a condition, such as `if a && b || c; then ...` since then the point is presumably not to act as an if-then-else by itself but rather to evaluate some condition. That is also what happens here, in a more roundabout way, so in that sense it's working as intended.

Arguably ShellCheck should warn when the construct is in a condition but whose exit code is discarded, like `if true; then a && b || c; d; fi && echo "done"` but the infrastructure is currently lacking for such a niche case.
