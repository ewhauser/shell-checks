# [\#854 Issue](https://github.com/koalaman/shellcheck/issues/854) `closed`: SC2174: should not warn about mkdir -p -m 0755 ../foo

#### <img src="https://avatars.githubusercontent.com/u/4692022?v=4" width="50">[eldering](https://github.com/eldering) opened issue at [2017-03-21T02:39:37Z](https://github.com/koalaman/shellcheck/issues/854):

In a shell script I am using
```
#!/bin/sh
mkdir -p -m 0711 ../bin
```
to make sure that the directory `bin` exists with the right permissions. I use `-p` to make sure `mkdir` does not give an error when the directory already exists.

Shellcheck version 0.4.4 generates a warning here:
```
../judge/testcase_run.sh:172:10: warning: When used with -p, -m only applies to the deepest directory. [SC2174]
```
Although I realize this is a corner case, I think that any composite path where all but the last element consists of `.` or `..` should not trigger this warning.
