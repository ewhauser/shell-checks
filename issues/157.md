# [\#157 Issue](https://github.com/koalaman/shellcheck/issues/157) `closed`: SC2062 Parsing of grep options

#### <img src="https://avatars.githubusercontent.com/u/566245?v=4" width="50">[HairyFotr](https://github.com/HairyFotr) opened issue at [2014-05-14T17:39:44Z](https://github.com/koalaman/shellcheck/issues/157):

Shellcheck issues this warning for this line:

```
grep -R --include=*.scala 'case class '
        ^-- SC2062: Quote the grep pattern so the shell won't interpret it.
```

It goes away, if I quote `*.scala`, so I guess the parser understands the --include= option value as the regex pattern.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2014-05-15T16:15:34Z](https://github.com/koalaman/shellcheck/issues/157#issuecomment-43231049):

You're right. While it's true that you should be quoting that, ShellCheck did bug out and considered it your grep pattern. Fixed in 3824e9c
