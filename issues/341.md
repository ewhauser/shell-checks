# [\#341 Issue](https://github.com/koalaman/shellcheck/issues/341) `closed`: trap ERR should throw a warning for /bin/sh

#### <img src="https://avatars.githubusercontent.com/u/3307781?v=4" width="50">[daxtens](https://github.com/daxtens) opened issue at [2015-04-03T06:45:10Z](https://github.com/koalaman/shellcheck/issues/341):

`trap ERR` is not part of the [OpenGroup shell standard](http://pubs.opengroup.org/onlinepubs/9699919799/utilities/V3_chap02.html#tag_18_28). However, when run over a `/bin/sh` script, ShellCheck doesn't throw an error or warning when `ERR` is listed as part of a `trap`.

For example, see https://github.com/torvalds/linux/blob/master/scripts/link-vmlinux.sh: it doesn't throw a warning on the line with `trap ERR`.
