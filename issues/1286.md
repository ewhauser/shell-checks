# [\#1286 Issue](https://github.com/koalaman/shellcheck/issues/1286) `closed`: SC2216: piping to 'du' is legit

#### <img src="https://avatars.githubusercontent.com/u/13075429?v=4" width="50">[cubanpit](https://github.com/cubanpit) opened issue at [2018-07-08T10:03:42Z](https://github.com/koalaman/shellcheck/issues/1286):

#### For bugs
- Rule Id (if any, e.g. SC1000): SC2216
- My shellcheck version (`shellcheck --version` or "online"): version 0.5.0 && online

#### Here's a snippet or screenshot that shows the problem:

```sh

#!/bin/bash
find . -print0 | du --files0-from=-

```

#### Here's what shellcheck currently says:
```SC2216: Piping to 'du', a command that doesn't read stdin. Wrong command or missing xargs?```


#### Here's what I wanted or expected to see:
No warning, because this is legit with `du`, from manual page:
```
--files0-from=F
              summarize disk usage of the NUL-terminated file names specified in file F;
              if F is -, then read names from standard input
```
