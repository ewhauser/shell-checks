#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash

scope_array ()
{
        declare -a foo1
        foo1=(dummy)
        echo "${foo1[@]}"
        declare -a foo2=(dummy)
        echo "${foo2[@]}"
        declare -a bar1
        bar1=(dummy)
        echo "${bar1[@]}"
        declare -a bar2=(dummy)
        echo "${bar2[@]}"
}

scope_string ()
{
        declare foo1
        foo1=dummy
        echo "${foo1}"
        declare foo2=dummy
        echo "${foo2}"
        declare bar1
        bar1=dummy
        echo "${bar1}"
        declare bar2=dummy
        echo "${bar2}"
}



```

#### Here's what shellcheck currently says:
```
Line 22	SC2178: Variable was used as an array but is now assigned a string.
Line 23	SC2128: Expanding an array without an index only gives the first element.
Line 27	SC2178: Variable was used as an array but is now assigned a string.
Line 28	SC2128: Expanding an array without an index only gives the first element.
```
#### Here's what I wanted or expected to see:

Nothing. `foo*` and `bar*` in `scope_array ()` are distinct from `foo*` and `bar*` in `scope_string ()`. Assigning in the declarations does not change that. This is a false flagging of a type-reassignment error.

For kicks I reversed the order of the functions, putting `scope_string` first. No issues reported.
