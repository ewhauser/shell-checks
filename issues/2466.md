#### Here's a snippet or screenshot that shows the problem:

```sh

#!/bin/sh

foo_bar=blah
echo "${foo_bar}"
unset foo-bar
echo hi
```

#### Here's what shellcheck currently says:

Nothing

#### Here's what I wanted or expected to see:

`SCXXXX: foo-bar is an invalid variable name` (contains a hyphen, not underscore)

Note: this syntax error will throw an error on dash (bash happily ignores):
```sh

$ dash foo.sh 
blah
foo.sh: 5: unset: foo-bar: bad variable name

$ bash foo.sh 
blah
hi
```
