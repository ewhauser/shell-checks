#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash

printf "%(%s)T"
```

#### Here's what shellcheck currently says:
```
Line 3	SC2183: This format string has 1 variables, but is passed 0 arguments.
```
#### Here's what I wanted or expected to see:

As noted in [#775](https://github.com/koalaman/shellcheck/issues/775#issuecomment-267785904) `%()T` patterns still require a parameter. At a minimum the documentation and error message should be updated to mention this requirement, though personally ShellCheck should explicitly permit the zero-parameter usage since it's documented behavior:

> If no argument is specified, conversion behaves as if -1 had been given. This is an exception to the usual printf behavior.


#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2018-05-25T04:38:35Z](https://github.com/koalaman/shellcheck/issues/1171#issuecomment-391938960):

I definitely see your point. I'm a fan of the explicit form, but for the case of a single `%T` and no other parameter, it's a bit redundant.

For now, I've just updated the docs to suggest explicitly specifying `-1`. Thanks!
