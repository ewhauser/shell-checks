#### <img src="https://avatars.githubusercontent.com/u/6459309?v=4" width="50">[Artoria2e5](https://github.com/Artoria2e5) opened issue at [2015-10-14T18:09:51Z](https://github.com/koalaman/shellcheck/issues/505):

I have two issues #482 #483, which are both some kind of misc stuffs, either 'it breaks because of a bug in some widespread thing' or 'it looks bad'.

I think for those 'it looks bad'/'that is a bad habit', we should use SC30xx, with examples like these:
- SC3000: #482: `if foo; then true; else do something; fi` or similar `if` logics where the code can be simplified
- SC3001-3003: Use of legacy names that has better alternatives, e.g.:
  - `typeset` in bash scripts;
  - `source` should be replaced by `.` (bad habit since we have a POSIX name)
  - `function foo()` should be replaced by `foo()` (bad habit when it comes to writing POSIX)
- SC3004: bash 4.4 'quoted array assignment' warning?

And for 'it breaks because of a bug in some widespread thing', we have:
- SC3100: #483, `case` in `$()` in bash 3.1.x should use `(patt)` instead of `patt)` or it breaks
- SC3101: #502, `local` with multiple variables in `dash` may surprise you.

Oh, and one more thing about those codes. The original reason I opened #490 was that I found the wiki pages being too trivial with lots of duplicates in them. Symlinking in the wiki's Git repo may work, but sadly I can't do that since I can only 'push' edits on the wiki's webpage.
