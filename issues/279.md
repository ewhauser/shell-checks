# [\#279 Issue](https://github.com/koalaman/shellcheck/issues/279) `closed`: SC2016: Slightly too eager?

#### <img src="https://avatars.githubusercontent.com/u/166207?v=4" width="50">[mietek](https://github.com/mietek) opened issue at [2015-01-09T02:41:08Z](https://github.com/koalaman/shellcheck/issues/279):

```
In setup.sh line 56:
echo 'source <( "${HOME}/halcyon/halcyon" paths )' >>"${HOME}/.bash_profile"
     ^-- SC2016: Expressions don't expand in single quotes, use double quotes for that.
```

I don’t think ShellCheck should be making a suggestion in this case. I am using single quotes _because_ I don’t want this expression to be expanded.

I too am having this issue. I use single-quotes specifically to disable expansion. If I wasn't using characters that triggered expansion in double-quoted strings then I would have just used a double-quoted string to begin with. In my case, I have code like

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2015-01-19T22:53:13Z](https://github.com/koalaman/shellcheck/issues/279#issuecomment-70575505):

@mietek It shouldn't, but how can shellcheck differentiate this from things like people do all the time like `echo '<img src="$IMAGE">' >> public_html/index.html` ? 

@kballard Believe it or not, this warning is actually one of the most frequently useful ones. Unfortunately, once you have a basic grasp of quoting, it becomes completely useless. 

When you're aware of the difference, I'd suggest just using `-e 2016` always, and there should probably be support for a `.shellcheckrc` where you can add these issues once you outgrow them, so to speak.

@kballard That's a good point.

How do you feel about using a top level brace group? It lets you do this right now and doesn't incur an overhead during execution:

```

}
```
