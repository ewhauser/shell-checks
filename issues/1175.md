#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash
printf "Name:\t%s\n" "$value"
```

#### Here's what shellcheck currently says:
```
Line 2	SC1117: Backslash is literal in "\t". Prefer explicit escaping: "\\t".
Line 2	SC1117: Backslash is literal in "\n". Prefer explicit escaping: "\\n".
Line 2	SC2154: value is referenced but not assigned.
```
#### Here's what I wanted or expected to see:

This line (`printf "Name:\t%s\n" "$value"`) is listed as "Correct code" in [SC2028](https://github.com/koalaman/shellcheck/wiki/SC2028), but triggers these SC1117 errors. At a glance the SC1117 errors seem wrong, but it seems that using single-quotes (`printf 'Name:\t%s\n' "$value"`) is sufficient to suppress the error, in which case the SC2028 docs should be updated.
