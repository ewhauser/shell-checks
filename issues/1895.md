# [\#1895 Issue](https://github.com/koalaman/shellcheck/issues/1895) `open`: Automatically detect by script extension or shebang?

#### <img src="https://avatars.githubusercontent.com/u/25164676?u=c14595fbdc5a4a7e6e9673319c149c7306938236&v=4" width="50">[bsolomon1124](https://github.com/bsolomon1124) opened issue at [2020-03-31 20:08](https://github.com/koalaman/shellcheck/issues/1895):

`shfmt` will automatically detect files by both extension and shebang.

(Specifically, via [this regex](https://github.com/mvdan/sh/blob/70a3fab77962172d53ca1d2625ca02e30e078a14/fileutil/file.go#L15).)

It would be nice to have that functionality with `shellcheck` also.  Something like:

```shell
{
    grep -rlE '^#!\s?/(usr/)?bin/(env\s+)?(sh|bash|fish|zsh)' .
    find . -type f -regex '.*.\(sh\|bash\|fish\|zsh\)' -print
} | tr "\n" " " | uniq
```

However, if that seems like something that does not deserve here, maybe it could at least be mentioned in the docs?

The wiki currently uses the `git ls-files` example that will only include `.sh` extensions by default.
