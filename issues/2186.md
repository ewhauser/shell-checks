#### Here's a snippet or screenshot that shows the problem:

```sh
#!/usr/bin/env bash

retryLimit=3

while ! false; do
  echo "Failed! Retry in 20s ..."

  if [[ "((--retryLimit))" -le 0 ]]; then
    break;
  fi
  sleep 2
done

```

#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2021-04-01 03:57](https://github.com/koalaman/shellcheck/issues/2186#issuecomment-811619792):

You're right, this was too dynamic for ShellCheck to figure out. The operands to `-le` are interpreted as arithmetic expressions, but ShellCheck currently does not re-parse the string argument as an expression. 

You can consider doing e.g. `if (( --retrylimit ))` (or more explicitly `(( --retrylimit <= 0 ))`) instead as a workaround.
