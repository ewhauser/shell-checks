# [\#182 Issue](https://github.com/koalaman/shellcheck/issues/182) `closed`: shellcheck no longer compiles with make

Within the last month or so, shellcheck has stopped compiling with make:

trunk $ make
: Conditionally compiling shellcheck
ghc -O9 --make shellcheck

ShellCheck/Data.hs:4:8:
    Could not find module `Paths_ShellCheck':
      it is a hidden module in the package`ShellCheck-0.3.3'
      Use -v to see a list of the files searched for.
make: **\* [shellcheck] Error 1

This would not be a problem as I can do "cabal build", but the cabal built binary is twice the size of the make built binary.  Also, Enterprise Linux 6 does not have a new enough Haskell base to be able to do update all cabal packages, so it takes a lot of fiddling before "cabal build" works as expected.

The Makefile had several nice properties, including lower memory usage and faster build times. Unfortunately, it didn't work well with packaging or the Haskell ecosystem.
