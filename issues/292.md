#### <img src="https://avatars.githubusercontent.com/u/179397?v=4" width="50">[srcshelton](https://github.com/srcshelton) opened issue at [2015-01-21T17:02:58Z](https://github.com/koalaman/shellcheck/issues/292):

It would be neat to be able to parse a file containing, for example:

``` sh
    A && B || C # ~SC2015
```

... to prevent the raising of, in this case, warning SC2015 for this line.  This would be a neat and largely self-documenting way to prevent known issues from being reported - especially where contraindications (... is that really the correct word?  It seems to have only medical/drugs-trials relevance - might "Exceptions" or "Counterposition" be more widely understood alternatives?) exist, and so the warning really might be not a relevant warning.
