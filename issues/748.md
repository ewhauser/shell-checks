# [\#748 Issue](https://github.com/koalaman/shellcheck/issues/748) `closed`: Defining function without curly braces gives "SC1073: Couldn't parse this function"

#### <img src="https://avatars.githubusercontent.com/u/9645426?v=4" width="50">[rasguanabana](https://github.com/rasguanabana) opened issue at [2016-10-18T15:42:17Z](https://github.com/koalaman/shellcheck/issues/748):

According to [documentation](https://www.gnu.org/software/bash/manual/html_node/Shell-Functions.html#Shell-Functions) following function definitions are valid. `for` loop is a compound command, just like any list of commands surrounded by curly braces.

```
function abc()
 for name in "$@"; do
  echo "$name"
 done
```

It could trigger a warning, but it is still a legal syntax.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2016-10-18T21:14:00Z](https://github.com/koalaman/shellcheck/issues/748#issuecomment-254641227):

You're right, of course. Both POSIX and Bash accept any compound command as the function body.

I looked at this for #93, but changing it to accept any compound command muddled more error messages than it fixed. This is a way more common case:

```
function abc()
 for name in "$@"; do
  echo "$name"
 done
}
```

It's sad when valid code fails to parse. I hope it helps to know that having to add `{..}`s helps less advanced users.
