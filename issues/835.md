shellcheck version: 0.4.5, also online
SC2086 Double quote to prevent globbing and word splitting.

I'm currently using code like `: ${TLDR_DESCRIPTION_STYLE:= Space Yellow }` in order to give a default to environment variables when they are empty.
I think when unquoted variables are found after a `:` this flag should not come up, because any globbing/wordsplitting would be irrelevant in principle (unlike after eg. `test`).

Depending on your taste/styleguide another message might be shown of course.
