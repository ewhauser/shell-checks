# [\#187 Issue](https://github.com/koalaman/shellcheck/issues/187) `closed`: Unable to build v0.3.4

#### <img src="https://avatars.githubusercontent.com/u/936320?v=4" width="50">[justin8](https://github.com/justin8) opened issue at [2014-07-09T14:38:04Z](https://github.com/koalaman/shellcheck/issues/187):

Hi,

I'm the current maintainer for the shellcheck package on Arch linux, and I'm having some troubles getting v0.3.4 to build. I'm using the following commands to build shellcheck:
    runhaskell Setup configure --prefix=/usr --docdir=/usr/share/doc/${pkgname} -O
    runhaskell Setup -v build

This worked perfectly fine fo 0.3.3 and previous versions, but now when I build in a clean chroot I get the following when it attempts to compile ShellCheck.Parser:
ShellCheck/Parser.hs:2098:12:
    Exception when trying to run compile-time code:
      ./ShellCheck/Parser.hs: hGetContents: invalid argument (invalid byte sequence)
    Code: quickCheckAll
    In the splice: $quickCheckAll

I'm definitely not great at anything haskell, but I can only guess that it is trying to read something that should/shouldn't be UTF and is being read with the wrong encoding? and hope you might know more about what might be causing this.

#### <img src="https://avatars.githubusercontent.com/u/936320?v=4" width="50">[justin8](https://github.com/justin8) commented at [2014-07-09T14:48:32Z](https://github.com/koalaman/shellcheck/issues/187#issuecomment-48482288):

Further testing shows that setting my locale to en_US.UTF-8 allows it to build. But this is probably a bug if changing the system locale can break builds, and it is a regression from 0.3.3.

```
$ locale
LANG=
```
