#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2022-07-25 20:23](https://github.com/koalaman/shellcheck/issues/2532#issuecomment-1194583365):

It's referring to failures within the function, which no longer cause an exit due to this invocation. For example:

```
set -e
foo() {
  scp file backup:archive/
  rm file
}

# Works: script stops if scp fails
foo
# Fails: script deletes the file if scp fails
foo || exit $?
```

It's possible (and given the name, very likely) that `checkParameterDefinitionIsTriple` does not depend on `set -e` being enabled to function correctly and that this code is therefore fine, but ShellCheck is currently not able to determine this.
