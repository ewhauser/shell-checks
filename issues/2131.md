#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash
_trap_with_args() {
  local func
  func="$1"
  shift
  for sig; do
  trap "${func} ${sig}" "${sig}" # <-- problem is here
  done
}

clean() {
  exit
}

_trap_with_args clean QUIT INT TERM EXIT

```

#### Here's what shellcheck currently says:

```
Use single quotes, otherwise this expands now rather than when signalled.
See SC2064.
```

#### Here's what I wanted or expected to see:

Nothing because if  I follow what shellcheck is currently suggesting by changing this line:

```
 trap "${func} ${sig}" "${sig}"
```
into this

```
 trap '${func} ${sig}' '${sig}'
```

this will suppress the shellcheck warning but the trap command will not work and produces the following error:

```
./test2: line 7: trap: ${sig}: invalid signal specification
./test2: line 7: trap: ${sig}: invalid signal specification
./test2: line 7: trap: ${sig}: invalid signal specification
./test2: line 7: trap: ${sig}: invalid signal specification

```
