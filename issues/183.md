# [\#183 Issue](https://github.com/koalaman/shellcheck/issues/183) `closed`: False positive for SC2004 - $ unnecessary in (( )) when using associative array in calculation

```
brother /tmp$ cat test.sh 
#!/bin/bash

declare -A kaka

banan=foobar

# Doesn't work but doesn't trigger any shellcheck warnings
```

#### <img src="https://avatars.githubusercontent.com/u/255938?v=4" width="50">[ormaaj](https://github.com/ormaaj) commented at [2014-08-08T10:15:01Z](https://github.com/koalaman/shellcheck/issues/183#issuecomment-51584738):

The `(( arr[$var] ))` construct is unsafe when using associative arrays. The most ideal way to write that is `(( arr[\$var] ))`, which should be shellcheck's recommendation. `(( ${arr[$var]} ))` is a slightly worse but still safer alternative.
