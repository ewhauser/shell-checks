# [\#1960 Issue](https://github.com/koalaman/shellcheck/issues/1960) `open`: SC2142: Function in the alias definition

#### <img src="https://avatars.githubusercontent.com/u/56209373?u=3e0aad168bd7614010c2f087864de8f6e27ca361&v=4" width="50">[aeiplatform](https://github.com/aeiplatform) opened issue at [2020-05-21 01:31](https://github.com/koalaman/shellcheck/issues/1960):

#### For bugs
- Rule Id: `SC2142`
- My shellcheck version: `0.7.0`
- [x] The rule's wiki page does not already cover this at [Wiki](https://shellcheck.net/wiki/SC2142)
- [x] I tried on shellcheck.net and verified that this is still a problem on the latest commit

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/sh
alias idc='idc() { echo "$@"; }; idc'
alias idk='i() { echo "$@"; }; i'
```

#### Here's what shellcheck currently says:

```
Line 2:
alias idc='idc() { echo "$@"; }; idc'
      ^-- SC2142: Aliases can't use positional parameters. Use a function.
 
Line 3:
alias idk='i() { echo "$@"; }; i'
      ^-- SC2142: Aliases can't use positional parameters. Use a function.
```

#### Here's what I wanted or expected to see:
Nothing. Function is used in the alias definition.

#### <img src="https://avatars.githubusercontent.com/u/121299?v=4" width="50">[sparr](https://github.com/sparr) commented at [2023-07-01 21:36](https://github.com/koalaman/shellcheck/issues/1960#issuecomment-1616145021):

You can demonstrate this with a shorter anonymous function:

```sh
alias idc='() { echo "$@"; }'
```
