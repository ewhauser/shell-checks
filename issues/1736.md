#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2019-11-03T21:13:13Z](https://github.com/koalaman/shellcheck/issues/1736#issuecomment-549179166):

You're obviously right in this case.

ShellCheck explicitly suppresses the warning when the variable is used as a command name (since 66cebe7c7) , because the more common reason for having spaces in the command name is flags, e.g. `cmd="git -c core.autocrlf=false"`.

These uses should preferably be rewritten as functions or arrays, but it's still common enough that warning about this case would probably be more noisy than helpful.
