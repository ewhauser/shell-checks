#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash

curl -X POST --fail \
--cert "/path/to/some-cert-file.crt" \
--key "/path/to/some-key-file.key" \
-d '{}' -k -H "Content-Type: application/json" https://localhost:port-number/route
    
if [[ "$?" -ne 0 ]]; then
  echo "$(date -u) healthcheck failed"
  exit 1
fi
```

#### Here's what shellcheck currently says:
Check exit code directly with e.g. 'if ! mycmd;', not indirectly with $?.


#### Here's what I wanted or expected to see:
Nothing. Because the command that I could directly check would need to exist on one large line that would then look ugly. At its root, this shellcheck rule is a style guideline, and it may directly conflict with other style guidelines.
