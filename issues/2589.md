# [\#2589 Issue](https://github.com/koalaman/shellcheck/issues/2589) `closed`: Should suggest indenting continuation lines

#### <img src="https://avatars.githubusercontent.com/u/168301?u=b824d9e71723b9993b271767e40426de83aa7263&v=4" width="50">[l0b0](https://github.com/l0b0) opened issue at [2022-09-25 15:42](https://github.com/koalaman/shellcheck/issues/2589):

#### For new checks and feature suggestions
- [x] https://www.shellcheck.net/ (i.e. the latest commit) currently gives no useful warnings about this
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/usr/bin/env bash
foo bar baz
a bunch of arguments broken into \
multiple \
lines
ban bay bal
```

#### Here's what shellcheck currently says:

> No issues detected!

#### Here's what I wanted or expected to see:

Something like this:

> Indent continuation lines by 4 extra spaces for clarity.

[Rationale](https://paperless.blog/bash-lints-beyond-shellcheck): When continuation lines are indented the same as non-continuation lines it can be taxing to tell which lines belong with the previous one.

#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2022-10-13 03:44](https://github.com/koalaman/shellcheck/issues/2589#issuecomment-1276988922):

I definitely agree, but `shfmt` already does this better. ShellCheck does not keep information about whitespace and indenting while building its AST, so it would additionally be hard to compete with an already excellent formatter built for these kinds of issues.
