# [\#311 Issue](https://github.com/koalaman/shellcheck/issues/311) `closed`: SC1056 Expected a '}'

#### <img src="https://avatars.githubusercontent.com/u/1947085?v=4" width="50">[Obsidian-Jackal](https://github.com/Obsidian-Jackal) opened issue at [2015-02-16T23:26:32Z](https://github.com/koalaman/shellcheck/issues/311):

With the code below which has a "}", shellcheck states that I either don't have one or should a ";" or "\n" in front of it, but there's no apparent reason as to why I would need to add that since it works just fine as is.

```
#!/bin/bash
counter=1
#-----------#
#   Dots    #
#-----------#
# This simply prints three Dots. It waits .1 seconds between each dot.
Dots() {
while [ $counter -le 3 ]
do
echo -n "."
sleep .1
((counter++))
done
let counter=1
echo ""
done
} # END Dots
```

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2015-02-25T20:51:43Z](https://github.com/koalaman/shellcheck/issues/311#issuecomment-76053912):

Ideally yes, but it's very hard for a parser to deduce what the underlying human error was when there just happens to be an extra keyword somewhere (is there an extra `done` or missing `while`?). 

In this case, you got advice more suited for situations like `if true; then foo() { date; echo foo}; fi`, which also terminates a syntactic element not started in the brace group. 

As this is not a typical or systematic error, I'll close the bug.
