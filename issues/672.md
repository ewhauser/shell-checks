# [\#672 Issue](https://github.com/koalaman/shellcheck/issues/672) `closed`: ${foo:=bar} not recognized as assignment

#### <img src="https://avatars.githubusercontent.com/u/22370?v=4" width="50">[charles-dyfis-net](https://github.com/charles-dyfis-net) opened issue at [2016-05-20T20:27:49Z](https://github.com/koalaman/shellcheck/issues/672):

```
: "${foo:=bar}"
```

...yields `SC2154: foo is referenced but not assigned.`

However, this particular idiom is explicitly an assignment to `foo` that acknowledges the possibility that it _may not_ have had a value previously set. The warning is thus not appropriate in this case.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2016-09-25T02:02:07Z](https://github.com/koalaman/shellcheck/issues/672#issuecomment-249398017):

Indeed. Fixed in 4fc4899, thanks!
