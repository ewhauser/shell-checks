#### Here's a snippet or screenshot that shows the problem:

```sh

#!/bin/bash 
myFunction()
{
  eval string1="$1"
  eval string2="$2"
  eval string3="$3"

  echo "string1 = ${string1}"
  echo "string2 = ${string2}"
  echo "string3 = ${string3}"
}

var1="firstString"
var2="second string with spaces"
var3="thirdString"

myFunction "\${var1}" "\${var2}" "\${var3}"

exit 0

```

#### Here's what shellcheck currently says:
Line 8:
  echo "string1 = ${string1}"
                  ^-- SC2154: string1 is referenced but not assigned.
 
Line 9:
  echo "string2 = ${string2}"
                  ^-- SC2154: string2 is referenced but not assigned.
 
Line 10:
  echo "string3 = ${string3}"
                  ^-- SC2154: string3 is referenced but not assigned.

#### Here's what I wanted or expected to see:

$ shellcheck myscript
No issues detected!

If I don't use 'eval' shellcheck works fine. But In this case 'eval' is used to correctly refer to the variable.
