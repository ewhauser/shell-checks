#### Here's a snippet or screenshot that shows the problem:

From the documentation for [SC2154](https://github.com/koalaman/shellcheck/wiki/SC2154/49c0ddbdf94e26e257d6692764e41c731aa7fd94) as of the current revision, this should be giving a warning for SC2154:
```sh

#!/bin/bash
target="world"
echo "hello $target"  # misspelled

```

#### Here's what shellcheck currently says:
```
No issues detected!
```

#### Here's what I wanted or expected to see:
```
...
        ^-- [SC2154](https://www.shellcheck.net/wiki/SC2154) (warning): target is referenced but not assigned (did you mean '<CORRECT_VARIABLE_NAME>'?).
```

It looks like there was a regression in the Wiki documentation, [here](https://github.com/koalaman/shellcheck/wiki/SC2154/_compare/7341de17e49560fa0a308e42a6383d41c63ce06e...49c0ddbdf94e26e257d6692764e41c731aa7fd94). Because this is supposed to be giving examples of _problematic_ code that would give a warning for SC2154, the word "target" should actually be misspelled (as it was before), such that it gives a warning.
