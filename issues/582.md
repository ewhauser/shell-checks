# [\#582 Issue](https://github.com/koalaman/shellcheck/issues/582) `closed`: If using 'done' as part of a command SC1010 is triggered.

#### <img src="https://avatars.githubusercontent.com/u/188624?v=4" width="50">[harleypig](https://github.com/harleypig) opened issue at [2016-01-23T14:46:13Z](https://github.com/koalaman/shellcheck/issues/582):

If `done` (and other keywords I presume) is used as part of a command SC1010 is triggered.

```
#!/bin/bash
ls done
```

Running shellcheck on this gives

```
In test_done line 2:
ls done
   ^-- SC1010: Use semicolon or linefeed before 'done' (or quote to make it literal).
```

#### <img src="https://avatars.githubusercontent.com/u/198379?v=4" width="50">[bittorf](https://github.com/bittorf) commented at [2016-01-23T17:35:12Z](https://github.com/koalaman/shellcheck/issues/582#issuecomment-174206246):

see #521 - this is intended and should not trigger when you write

```
ls 'done'
```
