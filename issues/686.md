# [\#686 Issue](https://github.com/koalaman/shellcheck/issues/686) `open`: Detect wrong use of string equality comparison operator

#### <img src="https://avatars.githubusercontent.com/u/3374183?v=4" width="50">[sanmai-NL](https://github.com/sanmai-NL) opened issue at [2016-06-13T13:04:05Z](https://github.com/koalaman/shellcheck/issues/686):

`==` should not be used as string equality comparison operator since it is not portable, in favor of `=`. Shellcheck 0.4.4 does not seem to detect this issue.

In GNU bash 4.3.42(1)

``` bash
test 's' == 's'
printf '%d\n' "$?"
```

```
0
```

``` bash
test 's' = 's'
printf '%d\n' "$?"
```

```
0
```

In `dash` 0.5.8

``` sh
test 's' == 's'    
```

```
dash: 1: test: s: unexpected operator
```

``` sh
test 's' = 's'
printf '%d\n' "$?"
```

```
0
```
