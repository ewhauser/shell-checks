#### <img src="https://avatars.githubusercontent.com/u/13765553?v=4" width="50">[enjunear](https://github.com/enjunear) commented at [2018-07-08T03:46:33Z](https://github.com/koalaman/shellcheck/issues/1270#issuecomment-403260065):

Hi guys, just noticed this issue after managing to get shellcheck working on a raspberry pi 3.

In my case, I'm using shellcheck in GitLab CI. When running on a GitLab hosted runner, using the `koalaman/shellcheck-alpine` works great, but not when running on my raspberry pi 3 based runner.

For my raspberry pi 3, I created a new `.gitlab-ci-arm.yml` file, using the arm32v7 debian image as follows:

```yml
shellcheck:
  image: arm32v7/debian
  script:
    - apt-get -q update && apt-get -q install shellcheck -y
    - shellcheck *.sh
```

It may not be the most minimal solution, but it works. I'm not sure if it helps your use case @NewtownGal 

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2018-07-09T03:58:03Z](https://github.com/koalaman/shellcheck/issues/1270#issuecomment-403353093):

In addition to the ShellCheck packaged for Raspbian Stretch, there is now also an [official armv6hf build](https://storage.googleapis.com/shellcheck/shellcheck-stable.linux.armv6hf.tar.xz).

This build is available with the `latest`, `stable` and `v0.5.0` tags just like the `x86_64` builds.

For anyone interested, this is based on [resin.io Raspbian](https://hub.docker.com/r/resin/rpi-raspbian/) containers using qemu user emulation to run a armv6hf userland that compiles ShellCheck using an armv6hf GHC.

I obviously hoped to do it through cross-compilation, but this was a massive pain that required a pile of hacks for armv7 on the latest git branch of ghc, and even after spending my week on that, I never managed to get it to successfully build an armv6 compatible binary that would work on Raspberry Pi One or Zero.

I've only confirmed this on a Pi3 that i had lying around, plus qemu in `arm1176` mode. Testing on Pi2, Pi1 and Pi0 would be much appreciated.
