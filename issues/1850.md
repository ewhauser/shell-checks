# [\#1850 Issue](https://github.com/koalaman/shellcheck/issues/1850) `open`: Warn when using find without a binary

In order to resolve [`SC2044`](https://github.com/koalaman/shellcheck/wiki/SC2044), our script was changed like this:

```bash
-for lib in $(find $(dirname "${0}")/../utils -name "*.sh"); do source "${lib}"; done
+find "$(dirname "${0}")/../utils" -name "*.sh" -exec source {} \;
```

This is erroneous because `source` is not a binary.

Maybe shellcheck can prevent this kind of mistake? :thinking:
