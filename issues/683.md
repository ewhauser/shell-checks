# [\#683 Issue](https://github.com/koalaman/shellcheck/issues/683) `closed`: Feature: allow documenting explicit variables to treat as assigned

#### <img src="https://avatars.githubusercontent.com/u/119972?v=4" width="50">[jasonkarns](https://github.com/jasonkarns) opened issue at [2016-06-03T21:11:33Z](https://github.com/koalaman/shellcheck/issues/683):

Related to SC2154 (var is referenced but not assigned), if there are known variables that are expected to be set when the script runs, it would be nice to explicitly list them (as exceptions) via  code comment at the top of the script. This serves both to silence the warning _only_ for the known expected variables (not disabling the entire rule). And it also serves as documentation that the script relies on x, y, z env vars.

Example from jslint:

``` js
/*global
  ADSAFE, report, jslint
*/
```
