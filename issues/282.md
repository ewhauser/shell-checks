# [\#282 Issue](https://github.com/koalaman/shellcheck/issues/282) `open`: SC2094: assumes files too often

#### <img src="https://avatars.githubusercontent.com/u/1167421?v=4" width="50">[forthrin](https://github.com/forthrin) commented at [2023-03-02T16:28:19Z](https://github.com/koalaman/shellcheck/issues/282#issuecomment-1452159458):

The following, always run from a directory other than `~` (which the checker obviously cannot know), triggers a warning, though no files will be read and written to simultaneously.

`git=~/git; for f in "$git"/*/; do git -C "$f"... > git`

That said, a good recommendation here would be something like: _"Avoid identically named variables, directories and executables within the same proximity/scope."_ Is there such a check?
