#### <img src="https://avatars.githubusercontent.com/u/2199511?v=4" width="50">[Nightfirecat](https://github.com/Nightfirecat) commented at [2018-02-13T16:50:44Z](https://github.com/koalaman/shellcheck/issues/1117#issuecomment-365328686):

Have you read [the wiki page for this error](https://github.com/koalaman/shellcheck/wiki/SC2163)? It specifically says:

> ## Exceptions:
> 
> If you do want to export the variable's value, e.g. due to indirection, you can disable this message with a directive:
> 
> ```
> # shellcheck disable=SC2163
> export "$MYVAR"
> ```

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2018-02-18T01:15:51Z](https://github.com/koalaman/shellcheck/issues/1117#issuecomment-366484599):

This message was awful. I've improved it, and since no one would expect `export ${foo##*/}` to export the variable `foo`, I've made it skip any expansion with modifiers.

The side effect of this is that you can now use `${function?}` to make ShellCheck quiet without a directive. 

Latest now says 

    This does not export 'function'. Remove $/${} for that, or use ${var?} to quiet.
