#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2023-08-26 18:31](https://github.com/koalaman/shellcheck/issues/2815#issuecomment-1694451373):

ShellCheck doesn't run scripts so it can't really track their runtime behavior such as sourcing files whose names are computed at runtime.

However, you can specify a static filename using `# shellcheck source=myfile` on the line before the `.` command. This could be `/dev/null` to silence the warning and just ignore anything that the files may include, or a representative file that defines functions or variables that you want the rest of the script to consider defined.
