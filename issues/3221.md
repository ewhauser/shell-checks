# [\#3221 Issue](https://github.com/koalaman/shellcheck/issues/3221) `open`: Using a tilde in the rcfile path doesn't work

#### <img src="https://avatars.githubusercontent.com/u/184550036?u=e5b6f517c55f93ea9b06fd71e771e560fb3277d9&v=4" width="50">[cjones-vultr](https://github.com/cjones-vultr) opened issue at [2025-06-16 15:37](https://github.com/koalaman/shellcheck/issues/3221):

When passing in an explicit rc file with a tilde `~` for the home directory, `shellchecker` is unable to find the file.  Replacing the `~` with the variable `$HOME` allows shellcheck to find the config file.

Here's an example:

```
╰─❯ shellcheck --version
ShellCheck - shell script analysis tool
version: 0.10.0
license: GNU General Public License, version 3
website: https://www.shellcheck.net
╰─❯ shellcheck --rcfile=~/shellcheckrc test.sh
Warning: unable to read --rcfile ~/shellcheckrc
```
