#### <img src="https://avatars.githubusercontent.com/u/1067855?v=4" width="50">[umireon](https://github.com/umireon) commented at [2016-11-20T06:35:46Z](https://github.com/koalaman/shellcheck/issues/323#issuecomment-261761982):

FYI Syntastic can provide `-e` via [g:syntastic_sh_shellcheck_args](https://github.com/vim-syntastic/syntastic/blob/7993f7946fc77b7969d961f32233c6dfad361147/doc/syntastic.txt#L802) as well as [SHELLCHECK_OPTS](https://github.com/koalaman/shellcheck/wiki/Ignore#ignoring-one-type-of-error-forever) can do.

in `~/.vimrc`:

``` vim
let g:syntastic_sh_shellcheck_args="-e SC2059,SC2148"
```

#### <img src="https://avatars.githubusercontent.com/u/13438569?v=4" width="50">[rudolphjacksonm](https://github.com/rudolphjacksonm) commented at [2018-04-18T10:16:18Z](https://github.com/koalaman/shellcheck/issues/323#issuecomment-382338238):

I've found the following works for me, although this is more for a project-specific use case.

Create a file in the root of your project (call it `exclusions` or whatever you like), and set the content as the rules you want to exclude, delimited by a comma and a newline so it's easier to read:
```
SC1090,
SC1091,
SC2086
```
`cat` out the contents of the file (with newlines and whitespace trimmed) into a variable:

`exclusions=$(cat exclusions | tr -d " \t\n\r" )`

Then run shellcheck and pass it the variable. It parses it just fine from what I've seen:

`find . -name "*.sh" | xargs -L 1 shellcheck -e $exclusions --format=gcc`

I use `xargs` in this case as we have hundreds of scripts in a large project directory. If I run shellcheck without xargs it quickly maxes out the memory on our instance.
