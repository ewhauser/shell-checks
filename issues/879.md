# [\#879 Issue](https://github.com/koalaman/shellcheck/issues/879) `closed`: Bug: Shellcheck fails its own proposal

#### <img src="https://avatars.githubusercontent.com/u/730342?v=4" width="50">[JensRantil](https://github.com/JensRantil) opened issue at [2017-04-19T07:59:26Z](https://github.com/koalaman/shellcheck/issues/879):

I have the following file:
```
#!/bin/sh

# shellcheck disable=SC2072
if [ "$JVM_VERSION" \< "1.7" ] ; then
    echo hello
fi
```
which yields
```
bash-3.2$ shellcheck test.sh

In test.sh line 4:
if [ "$JVM_VERSION" \< "1.7" ] ; then
                    ^-- SC2039: In POSIX sh, \< is undefined.
```
I then modify my script to
```
#!/bin/sh

# shellcheck disable=SC2072
if [ "$JVM_VERSION" < "1.7" ] ; then
    echo hello
fi
```
and I get
```
bash-3.2$ shellcheck test.sh

In test.sh line 4:
if [ "$JVM_VERSION" < "1.7" ] ; then
                    ^-- SC2073: Can't use < in [ ]. Escape it or use [[..]].
```
I then modify my script to
```
#!/bin/sh

# shellcheck disable=SC2072
if [[ "$JVM_VERSION" < "1.7" ] ]; then
    echo hello
fi
```
And `shellcheck` tells me:
```
bash-3.2$ shellcheck test.sh

In test.sh line 4:
if [[ "$JVM_VERSION" < "1.7" ]] ; then
   ^-- SC2039: In POSIX sh, [[ ]] is undefined.
```

Expected: That `shellcheck` doesn't recommend me to make a change that doesn't pass.

Version:
```
$ shellcheck -V
ShellCheck - shell script analysis tool
version: 0.4.4
license: GNU General Public License, version 3
website: http://www.shellcheck.net
```
