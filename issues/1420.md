#### <img src="https://avatars.githubusercontent.com/u/1749303?v=4" width="50">[ngzhian](https://github.com/ngzhian) commented at [2018-12-21T09:19:03Z](https://github.com/koalaman/shellcheck/issues/1420#issuecomment-449326577):

Hm. See https://hackage.haskell.org/package/parsec3-1.0.1.8/docs/Text-Parsec-Pos.html#v:updatePosChar

> If the character is a newline ('\n') or carriage return ('\r') the line number is incremented by 1. If the character is a tab ('t') the column number is incremented to the nearest 8'th column, ie. column + 8 - ((column-1) `mod` 8). In all other cases, the column is incremented by 1.

This ends up into the token position map, which the fixe uses.
