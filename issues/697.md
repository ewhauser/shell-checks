# [\#697 Issue](https://github.com/koalaman/shellcheck/issues/697) `closed`: SC2093 doesn't recognize redirections w/ auto-allocated FD numbers

#### <img src="https://avatars.githubusercontent.com/u/22370?v=4" width="50">[charles-dyfis-net](https://github.com/charles-dyfis-net) opened issue at [2016-06-22T20:05:22Z](https://github.com/koalaman/shellcheck/issues/697):

```
        exec {dest_fd}>>"$dest_file"
        ^-- SC2093: Remove "exec " if script should continue after this command.
```

However, in 4.1 or newer this is a redirection of the same form as `exec 3>>"$dest_file"`, but using an auto-allocated FD and storing that FD number to a variable named `dest_fd`.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2016-06-25T22:59:21Z](https://github.com/koalaman/shellcheck/issues/697#issuecomment-228574747):

I'm not getting this error on the latest commit (conveniently testable on shellcheck.net), so it appears to have been fixed.

Please reopen if you can reproduce it on the latest version.
