# [\#2443 Issue](https://github.com/koalaman/shellcheck/issues/2443) `open`: SC2086: False positive for "export" and "local" built-ins (POSIX Shell)

#### <img src="https://avatars.githubusercontent.com/u/11463919?u=ab9b26d787c143fc2f239c6139843eac573eab4d&v=4" width="50">[itaranto](https://github.com/itaranto) opened issue at [2022-02-07 20:27](https://github.com/koalaman/shellcheck/issues/2443):

#### For bugs
- Rule ID:  SC2086
- My shellcheck version (`shellcheck --version` or "online"): 0.8.0
- [x] The rule's wiki page does not already cover this (e.g. https://shellcheck.net/wiki/SC2086)
- [x] I tried on https://www.shellcheck.net/ and verified that this is still a problem on the latest commit

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/sh
export FOO=${BAR}
```

This shouldn't be a problem, it should expand the variable in the same way that `FOO=$BAR` does, i.e. quotes are not required here.

Here's another one:

```sh
#!/bin/sh

# shellcheck disable=SC3043

do_something() {
    local foo=${1}
    echo "${foo}"
}
```

I *know* that in POSIX `local` is not defined, but since almost every POSIX shell supports it, I make an exception.

However, the last example works fine on Bash:

```sh
#!/bin/bash

do_something() {
    local foo=${1}
    echo "${foo}"
}
```

#### Here's what shellcheck currently says:

`SC2086 (info): Double quote to prevent globbing and word splitting.`

#### Here's what I wanted or expected to see:

*Nothing*



#### <img src="https://avatars.githubusercontent.com/u/141546?v=4" width="50">[jwilk](https://github.com/jwilk) commented at [2022-03-14 12:36](https://github.com/koalaman/shellcheck/issues/2443#issuecomment-1066734608):

Double quotes _are_ need in some shells. From example, this happens in Ubuntu 20.04 LTS, which ships with dash 0.5.10.2:
```console
$ dash -c 'v="hello world"; export w=$v; echo "$w"'
hello

$ dash -c 'v="hello world"; export w="$v"; echo "$w"'
hello world
```

This was fixed only in dash 0.5.11:
https://git.kernel.org/pub/scm/utils/dash/dash.git/commit/?id=cbb71a836874d176

Relevant POSIX bug:
https://www.austingroupbugs.net/view.php?id=351
