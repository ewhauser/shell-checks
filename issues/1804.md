# [\#1804 Issue](https://github.com/koalaman/shellcheck/issues/1804) `open`: SC2206 fix wiki concerning splitting by lines and spaces

#### <img src="https://avatars.githubusercontent.com/u/7345761?u=a2355f38b1b7e3b7baae621c4551cfae35202d73&v=4" width="50">[Yanpas](https://github.com/Yanpas) opened issue at [2020-01-22 16:30](https://github.com/koalaman/shellcheck/issues/1804):

`array=( $var )` splits by both newlines and spaces, which is used in my code.

The alternatives mentioned split either by space, either by newline. Demo:
```bash
yan@yPC:~$ v=$(echo a; echo b; echo c)
yan@yPC:~$ echo $v
a b c
yan@yPC:~$ a1=( $v )
yan@yPC:~$ echo ${a1[*]}
a b c
yan@yPC:~$ echo "$v"    
a
b
c
yan@yPC:~$ read -ra a2 <<< "$v"
yan@yPC:~$ echo ${a2[*]}       
a
yan@yPC:~$ read -ra a3 <<< $v 
yan@yPC:~$ echo ${a3[*]}     
a
yan@yPC:~$ v2=$(echo "a x"; echo b; echo c)
yan@yPC:~$ a21=( $v2 )
yan@yPC:~$ echo ${a21[*]}
a x b c
yan@yPC:~$ echo ${a21[1]}
x
yan@yPC:~$ echo ${a21[2]}
b
yan@yPC:~$ mapfile -t a22 <<< "$v2"
yan@yPC:~$ echo ${a22[*]}
a x b c
yan@yPC:~$ echo ${a21[1]}
x
yan@yPC:~$ echo ${a22[1]}
b
yan@yPC:~$ echo ${a22[0]}
a x

```
