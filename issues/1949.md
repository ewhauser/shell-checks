# [\#1949 Issue](https://github.com/koalaman/shellcheck/issues/1949) `open`: False positive warning SC2016 on jq

#### <img src="https://avatars.githubusercontent.com/u/29582865?u=0c57eebb21e1922b4a0345f7a830924cc295998b&v=4" width="50">[felipecrs](https://github.com/felipecrs) opened issue at [2020-05-12 17:19](https://github.com/koalaman/shellcheck/issues/1949):

The following statement works fine:
```sh
jq --arg v "$PRJNAME" '.dev.projects[$v]'
```
Shellcheck doesn't complain about the `$v` because it is a `jq` argument and not from shell.

However, if I use:

```sh
JQ="jq"
$JQ --arg v "$PRJNAME" '.dev.projects[$v]'
``` 

It complains about [SC2016](https://github.com/koalaman/shellcheck/wiki/SC2016) (Expressions don't expand in single quotes, use double quotes for that.), because it probably doesn't recognize the `jq` command.
