# [\#1334 Issue](https://github.com/koalaman/shellcheck/issues/1334) `closed`: Warn on missing newline character at EOF

#### <img src="https://avatars.githubusercontent.com/u/1245341?v=4" width="50">[oliv3r](https://github.com/oliv3r) opened issue at [2018-09-10T09:11:29Z](https://github.com/koalaman/shellcheck/issues/1334):

#### For new checks and feature suggestions
- [x] shellcheck.net (i.e. the latest commit) currently gives no useful warnings about this
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related

Shellcheck currently does not note that a file is missing a new line character at the EOF. While the content is probably accurate, technically, the last 'line' is not an actual line due to the missing new line character. This is almost always a result of badly configured editors and the like.

Shell check should offer a suggestion this is happening as some smarter tools also do: "No newline at end of file".

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2018-09-16T17:43:48Z](https://github.com/koalaman/shellcheck/issues/1334#issuecomment-421802444):

Missing trailing linefeeds are awful! Check out [shfmt](https://github.com/mvdan/sh) for any issues relating to whitespace and formatting.

ShellCheck currently doesn't intend to enforce any particular whitespace usage except for where it's required for the script to work.
