# [\#1678 Issue](https://github.com/koalaman/shellcheck/issues/1678) `closed`: SC3003 wrongly trigged on $'..' C-like strings

#### <img src="https://avatars.githubusercontent.com/u/1111474?v=4" width="50">[leagris](https://github.com/leagris) opened issue at [2019-08-22T14:00:16Z](https://github.com/koalaman/shellcheck/issues/1678):

#### For bugs

- Rule Id: SC2039
- My shellcheck version: online
- [x] I tried on shellcheck.net and verified that this is still a problem on the latest commit

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/usr/bin/env sh
printf $'%s\n' 'Hello world!'
```

#### Here's what shellcheck currently says:

```
Line 2:
printf $'%s\n' 'Hello world!'
       ^-- SC2039: In POSIX sh, $'..' is undefined.
```

#### Here's what I wanted or expected to see:

shellcheck should not complain on C-like strings `$'..'` because it is an accepted POSIX standard, and is supported by `ash`, `dash`, `ksh93`, `zsh` and probably a whole lot if not all modern POSIX-compliant shells.

#### Other legitimate use:

```sh
#!/usr/bin/env sh

IFS=$'\n'
```

See: [SC2141](https://github.com/koalaman/shellcheck/wiki/SC2141)
