I searched issues for "auto fix" and didn't find anything related.
Is there any interest in an auto fix feature?

When first discovering shellcheck, and when inheriting code, and when upgrading to a new version of shellcheck there can be an overwhelming number of simple fixes that can be solved by `find . -iname '*.sh' | xargs -I {} perl -pi -e "$RE" {}` followed by `git diff-highlight` review of course. Am I the outlier here or is anyone else accumulating a list of Regular Expressions to help fix issues identified by shellcheck?

#### <img src="https://avatars.githubusercontent.com/u/122831?v=4" width="50">[coolaj86](https://github.com/coolaj86) commented at [2022-08-15T19:21:32Z](https://github.com/koalaman/shellcheck/issues/1220#issuecomment-1215641681):

@stdedos See <https://prettier.io/docs/en/precommit.html#option-6-shell-script>

Example (untested):

```sh
# List files that have been Added, Changed, Modified
FILES=$(
    git diff --cached --name-only --diff-filter=ACMR | sed 's| |\\ |g'
)
# If there are no such files, end
if [ -z "$FILES" ]; then
    exit 0
fi

# Get just the files that end in '.sh'
SH_FILES=$(
    echo "$FILES" | grep '\.sh$' || true
)
# If there are no such files, end
if [ -z "$SH_FILES" ]; then
    exit 0
fi

echo "$SH_FILES" | xargs -I '{}' -P 4 bash -c "shellcheck -f diff \"{}\" | git apply"
echo "$SH_FILES" | xargs git add
```

#### <img src="https://avatars.githubusercontent.com/u/4197823?v=4" width="50">[TSMMark](https://github.com/TSMMark) commented at [2023-01-05T20:37:59Z](https://github.com/koalaman/shellcheck/issues/1220#issuecomment-1372717048):

I was getting an error from `git apply` due to leading `./` in the filenames: `error: invalid path './path/to/file.sh'`

Using `find` flag `-printf '%P\0'` I was able to remove the leading `./`

Full command:

```shell
find . \
  -path '*/node_modules' -prune -o \
  -path '*/bundle' -prune -o \
  -path '*/vendor' -prune -o \
  -iname '*.sh' \
  -printf '%P\0' \
  | xargs -0 shellcheck -f diff | git apply
```

#### <img src="https://avatars.githubusercontent.com/u/2265048?v=4" width="50">[GongT](https://github.com/GongT) commented at [2024-09-10T02:47:58Z](https://github.com/koalaman/shellcheck/issues/1220#issuecomment-2339502493):

I write a script to fix single error.

Save this to any folder in `$PATH`, `chmod a+x`, run `THE_SCRIPT_NAME <code want to fix> file1.sh file2.sh`

```bash
#!/usr/bin/env bash

set -Eeuo pipefail
shopt -s lastpipe

x() {
	echo -e "\e[2m + $*\e[0m" >&2
	"$@"
}
die() {
	echo -e "\e[38;5;9m$*\e[0m" >&2
	exit 1
}
congratulations() {
	echo -e "\e[38;5;10m$*\e[0m" >&2
	exit 0
}

if [[ $# -lt 2 || ! $1 =~ ^[0-9]+$ ]]; then
	die "Usage: $0 <NUMBER to fix> <...files>"
fi
if ! git diff --exit-code &>/dev/null; then
	echo "refuse run. not inside git repo, or has unstaged file. you must run git add before continue."
	exit 1
fi

declare -i WANT_CODE="$1"
shift

CHECK=(
	shellcheck "$@" --norc --enable=all --check-sourced --external-sources
)

ERRORS=()
WANT_IS_HAPPE=0
(x "${CHECK[@]}" --wiki-link-count=9999 | grep -E '/wiki/SC[0-9]{4}' | grep -oE '[0-9]{4}' || true) | while read -r line; do
	if [[ ${line} -eq ${WANT_CODE} ]]; then
		WANT_IS_HAPPE=1
	else
		ERRORS+=("${line}")
	fi
done

if [[ ${WANT_IS_HAPPE} -eq 0 ]]; then
	congratulations "no error with code ${WANT_CODE}!"
fi

ESTRARG=()
if [[ ${#ERRORS[@]} -ne 0 ]]; then
	ESTR=$(printf ',%s' "${ERRORS[@]}")
	ESTRARG+=("--exclude=${ESTR:1}")
fi

echo "fixing errors: ${WANT_CODE}"

set +Eeo pipefail

x "${CHECK[@]}" "--include=${WANT_CODE}" "${ESTRARG[@]}" --format=diff  | patch --reject-file=/tmp/rej --forward -p 1
rm -f /tmp/rej

congratulations "all files fixed!"
```
