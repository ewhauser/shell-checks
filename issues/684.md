# [\#684 Issue](https://github.com/koalaman/shellcheck/issues/684) `closed`: $ is sometimes necessary in (( … ))

#### <img src="https://avatars.githubusercontent.com/u/32383?v=4" width="50">[mina86](https://github.com/mina86) opened issue at [2016-06-10T14:12:52Z](https://github.com/koalaman/shellcheck/issues/684):

Superfluous warning when `${var-default}` is used inside of `$(( … ))` as seen in:

```
$ cat a.sh
test=$(( ${COLUMNS-80} - 10))
echo $test
$ shellcheck a.sh

In a.sh line 1:
test=$(( ${COLUMNS-80} - 10))
         ^-- SC2004: $ on variables in (( )) is unnecessary.
$ shellcheck --version
ShellCheck - shell script analysis tool
version: 0.3.3
license: GNU Affero General Public License, version 3
website: http://www.shellcheck.net
$
```

#### <img src="https://avatars.githubusercontent.com/u/243351?v=4" width="50">[brother](https://github.com/brother) commented at [2016-06-10T14:22:40Z](https://github.com/koalaman/shellcheck/issues/684#issuecomment-225195487):

The warning is indeed misleading. Having the colon present makes it behave as expected. 

> ${parameter-default} and ${parameter:-default} are almost equivalent. The extra : makes a difference only when parameter has been declared, but is null.
