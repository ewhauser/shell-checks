#### <img src="https://avatars.githubusercontent.com/u/197253?v=4" width="50">[kurahaupo](https://github.com/kurahaupo) commented at [2017-05-03T13:23:35Z](https://github.com/koalaman/shellcheck/issues/888#issuecomment-298909242):

I suspect this request is hopelessly optimistic. There's a lot of groundwork needed to add detection of the divergent usage; it isn't a trivial addition.

And you can't merge the last one with the other two:

    foo=00
    [[ $foo = 0 ]] # false
    [[ $foo == 0 ]] # false
    [[ $foo -eq 0 ]] # true

(And the omission of {} and "" makes no difference there.)

I'd prefer to get a recommendation that it be replaced by:

    (( foo == 0 ))

As for the others, using `=` vs `==` inside `[[  ]]` seems to be developing into a holy war. 

The arguments boil down to:
* `==` is the comparison operator all most C-derived languages, not to be confused with `=` assignment;
* there is historical precedent for `=`_ so don't change it; and
* since they're pattern matches rather than comparisons, having a different operator is a _good_ thing.
