# [\#362 Issue](https://github.com/koalaman/shellcheck/issues/362) `closed`: How to workaround "SC2086" for shells other than bash.

#### <img src="https://avatars.githubusercontent.com/u/95290?v=4" width="50">[resuna](https://github.com/resuna) opened issue at [2015-04-28T00:20:13Z](https://github.com/koalaman/shellcheck/issues/362):

All the world isn't bash, and scripts that are intended to be portable across shells can't depend on bash features. Therefore the workaround on https://github.com/koalaman/shellcheck/wiki/Sc2086 is incomplete.

Suggestion: a tagged comment allowing variables to be flagged safe for unquoted expansion.

Eg:

```
#!/bin/sh

FLAGS="-av --delete --delete-excluded --exclude=.DS_Store"

[...]

rsync $FLAGS ~/Calibre\ Library/. "$D/ebooks/Calibre/."
           ^––SC2086 Double quote to prevent globbing and word splitting.
```

Workaround:

```
#!/bin/sh

#pragma safequote FLAGS
FLAGS="-av --delete --delete-excluded --exclude=.DS_Store"

[...]

rsync $FLAGS ~/Calibre\ Library/. "$D/ebooks/Calibre/."
```
