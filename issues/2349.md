```sh
#!/usr/bin/dumb-init /bin/sh
exit 0
```

#### Here's what shellcheck currently says:
```
^-- SC1008: This shebang was unrecognized. ShellCheck only supports sh/bash/dash/ksh. Add a 'shell' directive to specify.
```

#### Here's what I wanted or expected to see:
Shellcheck doesn't have to know and understand every wrapper in the world, SC1008 rightfully states to just add a `shell=` line. But for docker init scripts, where `#!/.*bin/tini (.*)` and/or  `#!/.*bin/dumb-init (.*)` can be used to help identify the shell quite easily are quote common I'd say, common enough, like the `env` wrapper to make an exception and understand these as well?
