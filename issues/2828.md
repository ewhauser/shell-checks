# [\#2828 Issue](https://github.com/koalaman/shellcheck/issues/2828) `closed`: SC2059: Wiki page implies `printf %b` is not POSIX

#### <img src="https://avatars.githubusercontent.com/u/6653392?v=4" width="50">[J-M0](https://github.com/J-M0) opened issue at [2023-09-14 17:26](https://github.com/koalaman/shellcheck/issues/2828):

Currently, the wiki page for [SC2059](https://github.com/koalaman/shellcheck/wiki/SC2059) says this under Exceptions

>Sometimes you may actually want to interpret data as a format string, like in:
>
>```sh
>octToAscii() { printf "\\$1"; }
>octToAscii 130
>```
>
>In Bash, Ksh and BusyBox, there's a `%b` format specifier that expands escape sequences without interpreting other format specifiers: `printf '%b' "\\$1"`. In POSIX, you can instead [[ignore]] this warning.

According to the oldest version I could find[^1], POSIX has required `printf %b` since 1994.

I wanted to raise this issue here before I went and updated the wiki. Is there a reason `printf %b` isn't said to be POSIX?

[^1]: https://pubs.opengroup.org/onlinepubs/009656399/toc.pdf, page 583
