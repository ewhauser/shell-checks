#### Description
At the moment there doesn't seem to be a way to tell shellcheck to read a specific configuration file instead of looking for the global one. 

For example I've got a [lint](https://github.com/mohkale/dotfiles/blob/cdd4ecaeb6c1fbc6e6390035c79d55a6e7a202aa/setup/lint) script on my dotfiles and I'd like to setup a github action that uses my shellcheckrc file at `core/scrc`, how would you do this?
