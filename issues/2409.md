#### Here's a snippet or screenshot that shows the problem:

```sh
#!/usr/bin/env bash

# register

# Wrapper for dpkg-register

declare datafile
datafile=$(realpath "$(realpath "$(dirname "$0")")/../data/REGISTER.data")

if [[ ! -s $datafile ]]; then
    cat <<EOD > "$datafile"
repo='--dev-repo bbgithub:foo/bar'
dq='--drqs-group GROUP#'
c='--contact UUID --contact UUID'
b='--branch BRANCH'
dr='--dry-run'
pno='--pkg-name-override notbar'
extra="$b $dr $pno"
EOD
    fi

if x-is-available; then
    xterm -geometry "+${XTERMGEO_X}+${XTERMGEO_Y}" -e bash  -c "vi $datafile"
else
    vi "$datafile"
fi

# shellcheck disable=SC1090
. "$datafile"


# shellcheck disable=SC2154 ## variables are sourced in via $datafile
resp=$(yesno "dpkg-register $repo $dq $c $extra" n)
if [ "$resp" = 'y' ]
then
    set -o pipefail
    # shellcheck disable=SC2086 #https://github.com/koalaman/shellcheck/wiki/SC2086
    dpkg-register $repo $dq $c $extra
    exit $?
fi

```

#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2021-12-12 00:24](https://github.com/koalaman/shellcheck/issues/2409#issuecomment-991810309):

ShellCheck is right. The variables in the here doc are being expanded, and none of them are assigned, so your text file will end up with empty strings in their place.

To avoid this, quote the here doc delimiter like TinCanTech says. This will cause the shell not to expand values, and instead write them verbatim.
