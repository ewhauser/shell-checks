# [\#403 Issue](https://github.com/koalaman/shellcheck/issues/403) `closed`: Incorrectly warns about 'done' on RHS of assignment in a loop

#### <img src="https://avatars.githubusercontent.com/u/377248?v=4" width="50">[sorpigal](https://github.com/sorpigal) opened issue at [2015-06-27T12:01:53Z](https://github.com/koalaman/shellcheck/issues/403):

This is a minor issue. Test case:

`tick=0 ; for n in $(seq 1 5) ; do let tick=$((tick+1)) ; check=done ; done ; echo "$tick $n $check"`

Shellecheck will warn that "done" should be quoted to be made literal, but bash and dash have no problem with this usage and work as expected.
