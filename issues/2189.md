# [\#2189 Issue](https://github.com/koalaman/shellcheck/issues/2189) `closed`: Warning for "check exit code directly" is misleading when checking for non-zero exit code

#### <img src="https://avatars.githubusercontent.com/u/23638587?u=6d1f4ed9b2481fd4af374eb927f9a73602c53ba5&v=4" width="50">[jyn514](https://github.com/jyn514) opened issue at [2021-03-26 22:43](https://github.com/koalaman/shellcheck/issues/2189):

#### For new checks and feature suggestions
- [x] https://www.shellcheck.net/ (i.e. the latest commit) currently gives a misleading warning about this
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/sh
false
if [ $? -ne 0 ]; then
        echo 1;
fi
```

#### Here's what shellcheck currently says:

Line 3:
if [ $? -ne 0 ]; then
     ^-- SC2181: Check exit code directly with e.g. 'if mycmd;', not indirectly with $?.

This is very misleading - `if false` will have the *opposite* effect from the current code. I had to run `false; echo $?` a few times to convince myself that I was right and shellcheck was wrong. Shellcheck should only suggest a fix if preserves the same behavior.

#### Here's what I wanted or expected to see:

Line 3:
if [ $? -ne 0 ]; then
     ^-- SC2181: Check exit code directly, not indirectly with $?.

Did you mean: 
if ! false; then
