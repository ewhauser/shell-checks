#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2017-09-12T17:55:59Z](https://github.com/koalaman/shellcheck/issues/1003#issuecomment-328932585):

Given your example, it looks like ShellCheck is right.

`009` is not a valid octal number, and bash will fail too:

```
$ var='0403-009'; (( var ))
bash: ((: 0403-009: value too great for base (error token is "009")
```

However, given a valid arithmetic expression, like `var=0403-007`, it works fine. I imagine the same is true in Ksh88f.

I don't think you actually intended to subtract the two numbers. However, I do see that ksh93 will fall back to base 10 and therefore hide the fact that this is what's going on. 

I was unable to find a version of ksh88f to verify this on, so please reopen if I guessed wrong.
