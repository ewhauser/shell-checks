# [\#2102 Issue](https://github.com/koalaman/shellcheck/issues/2102) `closed`: Expansion the left side of =

#### <img src="https://avatars.githubusercontent.com/u/1413467?v=4" width="50">[marmalodak](https://github.com/marmalodak) opened issue at [2020-12-07 20:01](https://github.com/koalaman/shellcheck/issues/2102):

I saw code like this recently:

```
${FOO}=${something}
```

I'm convinced the code is supposed to say

```
FOO=${something}
```

#### Here's what shellcheck currently says:

I do not see any warning from shellcheck about this.

#### Here's what I wanted or expected to see:

Text expansion on the left side of an = doesn't do what you think it does.

Not sure what the text should really say.


#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2020-12-31 04:34](https://github.com/koalaman/shellcheck/issues/2102#issuecomment-752840163):

There were some rudimentary parse-time warnings for e.g. `$FOO=${something}`, but these were fairly picky. A whole new set of warnings have been added that should do a much better job at detecting a wide variety of issues, including `${0}=foo`, `411toppm=true`, `var${n}=42` and `"x"==y`. 

Thanks!
