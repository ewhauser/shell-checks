- Rule Id SC1089
- My shellcheck version 0.4.6

#### Here's a snippet or screenshot that shows the problem:

```sh
# can we really do funky crap like this?
function [}{*(){
   echo "Let me take you to, funkytown!"
}
[}{*    # why yes, we can!
# though editor auto-indent modes may punish us

```

#### Here's what shellcheck currently says:
emits an SC1089 on the trailing '}' of function definition


#### Here's what I wanted or expected to see:
Oh, I don't know. Maybe something like "bash allows nearly every character (printable or not) as a function name, but I'm just going to quit now if you're going to write that sort of code."

Inspired by seeing actual function names containing '?' in the wild. Documented more here:

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2017-09-09T22:39:12Z](https://github.com/koalaman/shellcheck/issues/909#issuecomment-328307637):

Actually that was just when the () happened to turn into part of an extglob, e.g. `foo+() { bar; }`

I've added most of these characters, but `[]!` is only recognized when using `function` to avoid misdiagnosis of `[` issues, and `{}` is not recognized at all because `function foo{bar` is far more likely to be missing spaces than a valid name.
