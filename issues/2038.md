# [\#2038 Issue](https://github.com/koalaman/shellcheck/issues/2038) `open`: SC1090: How to make it work using functions to load scripts?

#### <img src="https://avatars.githubusercontent.com/u/29582865?u=0c57eebb21e1922b4a0345f7a830924cc295998b&v=4" width="50">[felipecrs](https://github.com/felipecrs) opened issue at [2020-08-12 18:29](https://github.com/koalaman/shellcheck/issues/2038):

Let's say I have: 

```bash
function load_lib() {
  local lib_file=$1
  source "$script_dir/$lib_file" || die "[ERROR] Not able to load the library $lib_file"
}

load_lib "common.sh"
```

As you can expect, shellcheck warns about [SC1090](https://github.com/koalaman/shellcheck/wiki/SC1090).

I would like to do something like:

```bash
function load_lib() {
  local lib_file=$1
  # shellcheck source=./$libfile
  source "$script_dir/$lib_file" || die "[ERROR] Not able to load the library $lib_file"
}

load_lib "common.sh"
```

Or:

```bash
function load_lib() {
  local lib_file=$1
  source "$script_dir/$lib_file" || die "[ERROR] Not able to load the library $lib_file"
}

# shellcheck source=./common.sh
load_lib "common.sh"
```

Or better yet, follow non-constants variables, so I could use a `readonly` argument for the function. (https://github.com/koalaman/shellcheck/issues/1944)

#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2020-08-26 03:38](https://github.com/koalaman/shellcheck/issues/2038#issuecomment-680513830):

Unfortunately there's no great way to do this. Maybe there should be some functionality for letting shellcheck source a file without the script doing so. In your second case, ShellCheck should probably include the file from the directive regardless.

As a workaround, you can use `# shellcheck source=/dev/null` to ignore the file entirely, and then add a 

```
true || source ./common.sh
```

if you want the variable declarations and such from the file. The shell will ignore it, but ShellCheck will still include it.

But if shellcheck could support something as bellow, it would be very good:

```sh
function load_lib() {
  local lib_file=$1
  # shellcheck disable=SC1090
  source "$script_dir/$lib_file" || die "[ERROR] Not able to load the library $lib_file"
}

# shellcheck source=common.sh
load_lib "common.sh"
```

The only difference from my previous second example is that I disable the directive inside the function. 

This use case is especially useful as now [Argbash provides such functionality (scroll up to `DEFINE_LOAD_LIBRARY`)](https://argbash.readthedocs.io/en/latest/guide.html#convenience-macros).

This simplifies the following feature request for shellcheck:

References https://github.com/matejak/argbash/issues/149.
