#### <img src="https://avatars.githubusercontent.com/u/8651654?v=4" width="50">[jleffler](https://github.com/jleffler) commented at [2017-11-18T21:38:52Z](https://github.com/koalaman/shellcheck/issues/1046#issuecomment-345473212):

I think you've misunderstood what's going on.  The `<<` operator is an input redirection operator; it provides standard input to a command.

Consider this script — and adaptation of your example:

    #!/bin/bash

    echo "Part 1"
    variable=<<EOF
    line1
    line2
    EOF

    echo "variable=[$variable]"

    echo "Part 2"
    variable=<<EOF sed 's/^/--: /'
    line1
    line2
    EOF

    echo "variable=[$variable]"

    echo "Part 3"
    <<EOF bash -c 'set -u; echo variable=$variable' xxx
    line1
    line2
    EOF

    variable=<<EOF bash -c 'set -u; echo variable=$variable' xxx
    line1
    line2
    EOF

    echo "variable=[$variable]"

When I run it on a Mac running macOS High Sierra 10.13.1, and run it with either Bash 3.25 (as provided by Apple) or Bash 4.3 (compiled by myself) or Korn shell (`version sh (AT&T Research) 93u+ 2012-08-01`) or Zsh (`zsh 5.3 (x86_64-apple-darwin17.0)`), I get:

    Part 1
    variable=[]
    Part 2
    --: line1
    --: line2
    variable=[]
    Part 3
    xxx: variable: unbound variable
    variable=
    variable=[]

Part 1 shows that nothing is set in the variable.  Part 2 shows that the here document is available as standard input to the command.  Part 3 shows that `variable` is set as an empty environment variable to the command that's executed.  Nothing would change in any of the examples if you used:

    variable=  <<EOF

with space between the `=` and the redirection `<<`.

Yes, I/O redirection can precede the command.  It's aconventional, but permissible.  I recently found a 'stellar' example at work:

    <something> another sed 's/<abc>/<def>/'

The `<something` redirects standard input; the `> another` redirects standard output; the command follows both redirections.  No; it wasn't what I wrote; I just spotted it and had a giggle.
