#### <img src="https://avatars.githubusercontent.com/u/12102230?u=5dd1a55f492ef2053dba5db0a5eb299cea5cf532&v=4" width="50">[rogerxxxx](https://github.com/rogerxxxx) opened issue at [2023-02-28 00:59](https://github.com/koalaman/shellcheck/issues/2703):

declare -i debug=0

if [ ${debug_val} -ne 1 ] ; then

^^ SC2086 (info): Double quote to prevent globbing and word splitting.

I do believe globbing and word splitting is not typically used with variables declared as integer.

Simple fix, re-word the warning,

SC2086 (info): Double quote to prevent string globbing and string word splitting.

Anybody will knowingly realize, if their variable is declared as an integer, then this warning is likely not pertinent for their script.  Or further clarify:

SC2086 (info): If this is a string, double quote to prevent string globbing and string word splitting.
