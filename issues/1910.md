# [\#1910 Issue](https://github.com/koalaman/shellcheck/issues/1910) `closed`: SC2003 expr as octal to decimal converter

#### <img src="https://avatars.githubusercontent.com/u/20344300?u=69df0dc582d8beda63a4fca3868e3d928ed34d9b&v=4" width="50">[ChillerDragon](https://github.com/ChillerDragon) opened issue at [2020-04-07 12:51](https://github.com/koalaman/shellcheck/issues/1910):

I try to increment numbers prefixed with zeros. I used ``expr`` to convert them to decimal. Because bash thinks they are octal:
```
$ x="0008"
$ echo "$((x + 1))"
-bash: 0008: value too great for base (error token is "0008")

$ echo "$(expr "$x" + 1)" # <- works but throws SC2003
9
```



#### <img src="https://avatars.githubusercontent.com/u/20344300?u=69df0dc582d8beda63a4fca3868e3d928ed34d9b&v=4" width="50">[ChillerDragon](https://github.com/ChillerDragon) commented at [2020-04-07 15:03](https://github.com/koalaman/shellcheck/issues/1910#issuecomment-610439789):

Okay I solved it using:
```
$ shopt -s extglob
$ x="0008"
$ x="${x##+(0)}"
$ echo "$((x + 1))"
9
```

Maybe it is worth mentioning it at https://github.com/koalaman/shellcheck/wiki/SC2003

#### <img src="https://avatars.githubusercontent.com/u/54855303?v=4" width="50">[mnuccioarpae](https://github.com/mnuccioarpae) commented at [2021-10-14 10:13](https://github.com/koalaman/shellcheck/issues/1910#issuecomment-943215155):

As an alternative to removing the leading zeroes, you can specify the base for the number:

```sh
$ x=0008
$ echo $((10#$x + 1))
9
```

I have documented it on the wiki: https://github.com/koalaman/shellcheck/wiki/SC2003
