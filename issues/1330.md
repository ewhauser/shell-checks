#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2018-09-11T21:37:16Z](https://github.com/koalaman/shellcheck/issues/1330#issuecomment-420434155):

This is an interesting case.

In POSIX, the command works perfectly fine because no options (not even `--`) shall be recognized. This means that it shouldn't really be silenced by `--`.  The advice should probably be to use `printf` whenever arguments start with a `-`.

In Bash, it works because `"-e hello"` is still arguably a set of flags, but since it contains currently unsupported options (`-hlo `) it's being printed verbatim:

    $ echo "-Eeenenenenenenne" # BATMAN
    (no output)

    $ echo "-Eeenenenenenennex"
    -Eeenenenenenennex

Though of course there is no warning there.
