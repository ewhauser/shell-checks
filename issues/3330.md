#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/sh
find . -maxdepth 1 -type d -exec echo {}        # SC2067
find . -maxdepth 1 -type d -execdir echo {}     # SC2067

find . -maxdepth 1 -type d -exec echo \;        # allowed but {} seems to be missed
find . -maxdepth 1 -type d -execdir echo \;     # allowed but {} seems to be missed

find . -maxdepth 1 -type d -exec echo +         # find: missing argument to `-exec'
find . -maxdepth 1 -type d -execdir echo +      # find: missing argument to `-execdir'
```

`-maxdepth` is not defined in POSIX but used so that find exits immediately.

#### Here's what shellcheck currently says:

```
myscript:2:34: error: Missing ';' or + terminating -exec. You can't use |/||/&&, and ';' has to be a separate, quoted argument. [SC2067]
myscript:3:37: error: Missing ';' or + terminating -exec. You can't use |/||/&&, and ';' has to be a separate, quoted argument. [SC2067]
```

#### Here's what I wanted or expected to see:

As per [POSIX](https://pubs.opengroup.org/onlinepubs/9699919799/utilities/find.html) (and as implemented in GNU findutils and BSD):
- `-exec command ;` does not require `{}`, but does a user intentionally omit `{}`?
- `-exec command +` requires `{}`
