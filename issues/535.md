# [\#535 Issue](https://github.com/koalaman/shellcheck/issues/535) `open`: SC2086 False Positive with ${!PREFIX*} / ${!PREFIX@}

#### <img src="https://avatars.githubusercontent.com/u/6459309?v=4" width="50">[Artoria2e5](https://github.com/Artoria2e5) opened issue at [2015-11-03T13:26:46Z](https://github.com/koalaman/shellcheck/issues/535):

> `${!prefix*}`
> `${!prefix@}`
> Expands to the names of variables whose names begin with prefix, separated by the first character of the IFS special variable. When ‘@’ is used and the expansion appears within double quotes, each variable name expands to a separate word.

Due to restrictions on shell varnames, using `${!prefix*}` is basically safe.

Related bash manpage: https://www.gnu.org/software/bash/manual/bash.html#Shell-Parameter-Expansion

This is a bashism.
