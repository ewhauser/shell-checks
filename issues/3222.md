# [\#3222 Issue](https://github.com/koalaman/shellcheck/issues/3222) `open`: [BATS] Inconsistent SC2016 when using `run`

#### <img src="https://avatars.githubusercontent.com/u/2794994?u=4ff64a9365e865bcaeb40ff2f79c2a40eaeb575f&v=4" width="50">[jakub-bochenski](https://github.com/jakub-bochenski) opened issue at [2025-06-17 15:42](https://github.com/koalaman/shellcheck/issues/3222):

MRE: 
```
#!/usr/bin/env bats
 

@test "reports" { 
  run --separate-stderr \
    bash -c '
      source "$project_root/src/helpers.bash" 
    ' 
}


@test "does not report" { 
  run bash -c '
      source "$project_root/src/helpers.bash" 
    ' 
}
```

https://www.shellcheck.net/ reports:
```
[Line 6:](javascript:setPosition(6, 13))
    bash -c '
            ^-- [SC2016](https://www.shellcheck.net/wiki/SC2016) (info): Expressions don't expand in single quotes, use double quotes for that.
```

One of those must be wrong (reporting or not reporting).

In my case the single quote is intentional and one might argue this could be a common case with `run`. On the other hand the safe option is to always report.
