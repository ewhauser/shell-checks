# [\#450 Issue](https://github.com/koalaman/shellcheck/issues/450) `closed`: `local let` warns about unused variable `let`.

#### <img src="https://avatars.githubusercontent.com/u/3335984?v=4" width="50">[cjwelborn](https://github.com/cjwelborn) opened issue at [2015-08-31T18:38:09Z](https://github.com/koalaman/shellcheck/issues/450):

I was messing around trying to declare a `local` variable using `let` at the same time. It seems there are two ways to do this, and `shellcheck` gives a false positive for one of them. I don't know what the 'best-practice' is, but `local let` seems to work as it should.

With a line like this:

``` bash
local let x=1
```

`shellcheck` should be checking `x`, not `let`.

My test script:

``` bash
# Setup some globals, to see if the 'local' works.
let x=2600
let y=1337
let z=808

function testfunc() {
    # This variable will not be local.
    let local x=0
    # Shellcheck complains about this line !!
    # shellcheck disable=SC2034
    local let y=1
    # Workaround, to ease the complaint.
    local z=$(( 2 ))
}
testfunc

function check_global {
    printf "Outer %s: %4s - " "$3" "$1"
    if [[ "$1" == "$2" ]]; then
        echo "Correct, using \`$4\`"
    else
        echo "Global was modified using \`$4\`!"
    fi
}

check_global "$x" 2600 "x" "let local"
check_global "$y" 1337 "y" "local let"
check_global "$z" 808 "z" "\$(())"
```

The output:

```
Outer x:    0 - Global was modified using `let local`!
Outer y: 1337 - Correct, using `local let`
Outer z:  808 - Correct, using `$(())`
```

`local let a=1` does in fact define a `let` variable though and doesn't evaluate arguments arithmetically.

Compare the output of

```
$ f() { declare -p let a; local a=5+1; declare -p let a; local; }
$ f
-bash: declare: let: not found
declare -- a="6"
-bash: declare: let: not found
declare -- a="5+1"
a=5+1
```

to the output of

```
$ f() { declare -p let a; local let a=5+1; declare -p let a; local; }
$ f
-bash: declare: let: not found
declare -- a="6"
declare -- let=""
declare -- a="5+1"
a=5+1
```

and the output of

```
$ f() { declare -p let a; let a=5+1; declare -p let a; local; }
$ f
-bash: declare: let: not found
declare -- a="6"
-bash: declare: let: not found
declare -- a="6"
```

and

```
$ f() { local a; declare -p let a; let a=5+1; declare -p let a; local; }
$ f
-bash: declare: let: not found
declare -- a=""
-bash: declare: let: not found
declare -- a="6"
a=6
```

I was just investigating this. `let` is a valid variable name, and you're right. I should've tested whether the `let` operation was doing what it was supposed to.

``` bash
function locallet {
    local let x=1+1
    echo " local let x=1+1: $x"
    local y=$((1+1))
    echo "local x=\$((1+1)): $y"
}
locallet
```

Output:

```
 local let x=1+1: 1+1
local x=$((1+1)): 2
```
