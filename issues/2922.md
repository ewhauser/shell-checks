#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/sh

printf '-> %s\n' "$1"
```

#### Here's what shellcheck currently says:
```
[Line 3:]
printf '-> %s\n' "$1"
       ^-- SC3045(warning): In POSIX sh, printf -> is undefined.
```

#### Here's what I wanted or expected to see:

The problem is that the wiki page is missing information on this case. In the example I am printing a literal `-` instead of passing an undefined flag to `printf` as explained in the wiki page.

However the example is still wrong because some shell implementations may actually support `-` flags for `printf` which will then become an undefined option when the `printf` format string starts with a `-`. The wiki could add information that the script could use the ascii code `\055` instead when printing a literal `-`. The following script works and shows no warnings with [shellcheck.net](https://www.shellcheck.net/).
```sh
#!/bin/sh

printf '\055> %s\n' "$1"
```
