#### Here's a snippet or screenshot that shows the problem:

```sh
#!/usr/bin/env bash

demo_by_ref_args()
{
    local -n joined="$1"; shift
    joined="hi there!"
}

declare by_ref
demo_by_ref_args by_ref
echo "$by_ref"
```

#### Here's what shellcheck currently says:
```
Line 6	SC2034: joined appears unused. Verify use (or export if used externally).
```
#### Here's what I wanted or expected to see:

Nothing. The output of this script is:

```
hi there!
```

because the `local -n` declaration passes in the argument by reference. When `joined` is assigned, it has effectively been used. Maybe there is another check to create, namely that that `joined` is assigned to something at some point, but 2034 isn't it.
