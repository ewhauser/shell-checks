```

```
and told me to use a glob. However a glob does not work on an empty dir. To wit:
```
$ THEDIR=o/b/b

$ ls -la $THEDIR
total 0
drwxrwxr-x 2 mpersico5 users 17 Feb 19 14:44 .
drwxrwxr-x 3 mpersico5 users 15 Feb 19 14:36 ..
-rw-rw-r-- 1 mpersico5 users  0 Feb 19 14:44 foo

o/b/b/foo

$ rm o/b/b/foo
rm: remove regular empty file 'o/b/b/foo'? y

o/b/b/*
```
I want the last result to be empty, not an unresolved glob string. I think the glob advice is potentially dangerous.
