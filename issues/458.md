#### <img src="https://avatars.githubusercontent.com/u/197253?v=4" width="50">[kurahaupo](https://github.com/kurahaupo) commented at [2015-09-13T23:46:02Z](https://github.com/koalaman/shellcheck/issues/458#issuecomment-139928976):

"Something else works fine" is not the point; it is valid for the start-mark of a here-doc to be followed by other words on the same line before the contents of the here-doc start at the beginning of the next line. In particular, multiple heredocs can be specified on the same line:

```
if ((somecondition)) ; then cat <&3; cat <<E1 ; else cat <<E2 ; cat <&3 ; fi 3<<E3
stuff for when somecondition is true
E1
stuff for when somecondition is false
E2
stuff for both cases
E3
```

So shellcheck should support it.

(There is an exception where `$( ... <<EOF ... )` does not have to scan for the closing `EOF` beyond the closing bracket because it may occur inside a quoted string.)
