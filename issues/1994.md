#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2020-07-28 05:34](https://github.com/koalaman/shellcheck/issues/1994#issuecomment-664787629):

I've done this a few times as well. There were zero occurrences in my corpus of 10k scripts, but much of it is from GitHub and this kind of error wouldn't make it past development. Due to the high potential destruction and low false positive rate when approached conservatively, a test seems worth it. The current implementation only looks at the first pipeline of the first command to catch wrapper functions like this, while avoiding false positives on any other kind of recursion.

As a hilarious side effect, ShellCheck now suggests replacing `:(){ : | :& }` with `:() { command : | command : & }`

Thanks!

PS: With bash and zsh you can use `FUNCNEST=10` to limit these kinds of things
