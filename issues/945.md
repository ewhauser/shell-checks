# [\#945 Issue](https://github.com/koalaman/shellcheck/issues/945) `open`: Missing space reported in if [[ $var == \[*] ]]

#### <img src="https://avatars.githubusercontent.com/u/170895?v=4" width="50">[twouters](https://github.com/twouters) opened issue at [2017-06-27T08:39:44Z](https://github.com/koalaman/shellcheck/issues/945):

#### For bugs
- Rule Id (if any, e.g. SC1000): SC1020
- My shellcheck version (`shellcheck --version` or "online"): 0.4.6
- [x] I tried on shellcheck.net and verified that this is still a problem on the latest commit
- [ ] It's not reproducible on shellcheck.net, but I think that's because it's an OS, configuration or encoding issue

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash
var="[test]"
if [[ $var == \[*] ]]; then
    echo "square brackets around string"
fi
```

#### Here's what shellcheck currently says:

```
^-- SC1009: The mentioned parser error was in this if expression.
   ^-- SC1073: Couldn't parse this test expression.
                  ^-- SC1020: You need a space before the ]].
                  ^-- SC1072: Missing space before ]. Fix any mentioned problems and try again.
```

#### Here's what I wanted or expected to see:

No errors should be reported, because there is no missing space here.
Can be worked around by using e.g. `\[*']'` or `'['*']'`.
