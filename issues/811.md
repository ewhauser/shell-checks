# [\#811 Issue](https://github.com/koalaman/shellcheck/issues/811) `closed`: Warning about egrep/fgrep being deprecated

#### <img src="https://avatars.githubusercontent.com/u/5762120?v=4" width="50">[facutuesca](https://github.com/facutuesca) opened issue at [2016-12-30T12:33:43Z](https://github.com/koalaman/shellcheck/issues/811):

Currently, egrep and fgrep are deprecated, but most distributions provide them as aliases for grep to allow historical applications that rely on them to run unmodified [1]. Maybe a new rule could be added that warns the user about this, and suggest the equivalent use of grep

[1] http://manpages.ubuntu.com/manpages/precise/man1/grep.1.html

I'd like to suggest recommending against using -P too, as PCRE doesn't have
the "regular" (ie linear) performance that was originally meant by "regular
expression". Indeed PCRE has some awful worst-case performance issues,
and very few
searches actually *need* it.

(Anyone remember Facebook having an outage because someone posted an entry
with like a megabyte of trailing whitespace, which conspired with a naïve
"trim whitespace" function that used PCRE in a way that took quadratic
time?)
