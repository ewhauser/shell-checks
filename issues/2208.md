#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2021-04-23 06:02](https://github.com/koalaman/shellcheck/issues/2208#issuecomment-825410932):

I also had to read it a couple of times to see what the purpose of that assignment was, but yes, you're doing it right and can safely disable it.

You can also consider using prefix assignments to explicitly set the variable just for that function call, which eliminates the need for disable directives entirely:

```
if ! ( cd "${in_dir}" && FORCE_DIR="${in_temp_exp_disable}" extract "$n" ) ; then
```
