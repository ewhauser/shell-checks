# [\#2583 Issue](https://github.com/koalaman/shellcheck/issues/2583) `open`: Question: Is there a chance to detect a missing $ for a variable?

#### <img src="https://avatars.githubusercontent.com/u/192842?u=2c93d559304e381569e7c7787b788be87128448c&v=4" width="50">[manfredlotz](https://github.com/manfredlotz) opened issue at [2022-09-23 08:15](https://github.com/koalaman/shellcheck/issues/2583):

Let us say I have

```
something=0

this="some content"

echo "some actions..."

#this is wrong and shellcheck complains
something=this

echo "$something"
```
This works fine as `shellcheck` complains that `this` appears unused.

But if I add a usage of variable `this` like follows then `shellcheck` has no chance to detect anything.

```
something=0

this="some content"

#this usage of `$this`...
echo "$this"
echo "some actions..."

# ... hides the following mistake
something=this
echo "$something"
```

Question: Is there a possibility to tell `shellcheck` that it has to assume that all strings have to be quoted? Then above error would be detected.
