# [\#2148 Issue](https://github.com/koalaman/shellcheck/issues/2148) `open`: SC2015 (A && B || C) does not take into account B being a break

#### For bugs
- Rule Id: SC2015
- Shellcheck version: online
- [x] The rule's wiki page does not already cover this
- [x] I tried on https://www.shellcheck.net/ and verified that this is still a problem on the latest commit

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash

for _ in 1 2 3; do true && break || sleep 15; done
```

This seems to be the most concise expression for a trivial retry, see [this](https://unix.stackexchange.com/a/82610/455740) answer. 

#### Here's what shellcheck currently says:
SC2015: Note that A && B || C is not if-then-else. C may run when A is true.

#### Here's what I wanted or expected to see:
Verify. This is a special case due to the break builtin.
