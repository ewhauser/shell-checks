#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2015-10-02T01:24:56Z](https://github.com/koalaman/shellcheck/issues/472#issuecomment-144894486):

ShellCheck is indeed overenthusiastic here. Fixed in ee181cf, by not emitting this error when redirecting stderr or when using non->/< redirection operators. 

If you want something that doesn't trigger in the latest stable release without ignoring the error, you can use `[ "$1" -eq "$1" ] 2>/dev/null`
