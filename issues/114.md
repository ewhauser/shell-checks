#### <img src="https://avatars.githubusercontent.com/u/352559?v=4" width="50">[gwern](https://github.com/gwern) commented at [2015-12-19T17:15:10Z](https://github.com/koalaman/shellcheck/issues/114#issuecomment-166004907):

grimreaper: the problem with that is that the GNU `sort -R` is not a random sort or shuffle as one would expect. It's a sort of keys ie, identical lines get sorted to be contiguous. Quick example:

```
$ echo -e 'foo\nfoo\nbar1\nbar2\nbar3\nbar4\nbar5' | sort --random-sort
bar3
bar1
bar2
foo
foo
bar5
bar4

$ echo -e 'foo\nfoo\nbar1\nbar2\nbar3\nbar4\nbar5' | shuf
bar2
bar4
bar1
bar5
foo
bar3
foo
```

Personally, I'd say shellcheck ought to warn on _any_ use of `sort -R`/`sort --random-sort`, nevermind this specific use-case where `shuf` is more efficient. I'm not sure I've ever seen a use of `sort -R` where the author expected, intended, or wanted that difference from a shuffle, and everyone I've mentioned this to is surprised, in part because the documentation began to mention this only [very recently](https://bugs.debian.org/cgi-bin/bugreport.cgi?bug=641166). (I know it may seem unlikely that this bug would bite anyone, but it's the sort of subtle brokenness that would drive a developer nuts and cost them many hours if it does ever become a problem - because who would ever suspect that their problem might lie in the reliable, trusty old `sort` utility, in use on millions of machines and battle-tested for decades?)
