# [\#661 Issue](https://github.com/koalaman/shellcheck/issues/661) `open`: "[" vs. "test": "[ a == a ]" is detected as error, "test a == a" is not

#### <img src="https://avatars.githubusercontent.com/u/7539154?v=4" width="50">[johnLate](https://github.com/johnLate) opened issue at [2016-04-28T19:53:00Z](https://github.com/koalaman/shellcheck/issues/661):

The following error (SC2039):

```
In foo.sh line 3:
[ abc == abc ] ; echo $?
      ^-- SC2039: In POSIX sh, == in place of = is undefined.
      ^-- SC2050: This expression is constant. Did you forget the $ on a variable?
```

is not raised for the last line of this script:

```
#!/bin/sh
[ abc = abc ] ; echo $?
[ abc == abc ] ; echo $?
test abc = abc ; echo $?
test abc == abc ; echo $?
```

However, both lines fail with `dash`.

This suggests that `test` should be treated like `[`.
