You might have intepreted the issue #777 a tad bit wrong. It's about disabling checking for an error for the entire file not for the next line or closure as is the normal behaviour.

```
brother ~/git/other/gleshys (main *%)$ head -n2 attic/functions
#!/bin/bash

brother ~/git/other/gleshys (main *%)$ shellcheck -f gcc attic/functions 
attic/functions:23:9: note: Double quote to prevent globbing and word splitting. [SC2086]
attic/functions:27:6: note: $/${} is unnecessary on arithmetic variables. [SC2004]
attic/functions:35:3: note: read without -r will mangle backslashes. [SC2162]
brother ~/git/other/gleshys (main *%)$ head -n2 attic/functions
#!/bin/bash
# shellcheck disable=SC2004
brother ~/git/other/gleshys (main *%)$ shellcheck -f gcc attic/functions 
attic/functions:23:9: note: Double quote to prevent globbing and word splitting. [SC2086]
attic/functions:35:3: note: read without -r will mangle backslashes. [SC2162]
```

Disabling ALL checks for the file is not possible, maybe you could avoid executing shellcheck with the problematic (?) file of yours? 

Maybe only search for errors and not all severities? (`-S error`)

Piggybacking here _(I'll open a fresh issue, if preferred)_ to suggest/request a mechanism to entirely disable shellcheck for a given file.

My use case is that I have many reference files in a VS Code project that are littered with (legitimate) shellcheck complaints that I'd like to silence; since I'm using an IDE+shellcheck plugin, it is challenging/impractical to selectively silence these files any other way without also losing syntax highlighting.

Optimally, I'd like to see something like:
```bash
#!/bin/bash

```
or
```bash
#!/bin/bash
# shellcheck disable=*
```
simply silence shellcheck for an entire file.

#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2021-07-26 19:53](https://github.com/koalaman/shellcheck/issues/2252#issuecomment-886981920):

Since 0.7.2 you've been able to specify a range, so you can technically use:

```
#!/bin/bash
# shellcheck disable=SC1000-SC9999
```

There should really be a better way though.
