# [\#208 Issue](https://github.com/koalaman/shellcheck/issues/208) `closed`: Encourage dot (`.`) over `source`

#### <img src="https://avatars.githubusercontent.com/u/5316?v=4" width="50">[mcandre](https://github.com/mcandre) opened issue at [2014-08-08T17:47:51Z](https://github.com/koalaman/shellcheck/issues/208):

Though checkbashisms does not report this, I believe `source` is a bashism; POSIX sh (and even modern descendants like ash/dash) only recognizes the dot (`.`) command.

Could shellcheck get an optional rule for this, disabled by default? Something like:

```
In hello.sh line 1:
  source world.sh
  ^-- Replace source with dot (.) for POSIX compatibility
```

#### <img src="https://avatars.githubusercontent.com/u/5316?v=4" width="50">[mcandre](https://github.com/mcandre) commented at [2014-08-08T18:17:09Z](https://github.com/koalaman/shellcheck/issues/208#issuecomment-51638643):

Note: Except for csh/tcsh, which are non-POSIX compliant. csh and tcsh _only_ recognize `source`, not dot (`.`).


#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2014-08-09T17:56:41Z](https://github.com/koalaman/shellcheck/issues/208#issuecomment-51693781):

When specifying `#!/bin/sh` or `-s sh`, you get

```
source foo
^-- SC2039: #!/bin/sh was specified, but 'source' in place of '.' is not standard.
```
