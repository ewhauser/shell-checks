#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2018-11-08T00:16:04Z](https://github.com/koalaman/shellcheck/issues/1356#issuecomment-436825857):

When the docs say:

```
# shellcheck source=src/lib.sh
. "$(find_install_dir)/lib.sh"
```

That's a script with a comment directive and not a command prompt with a result. Running the command `shellcheck source=src/lib.sh` is intended to fail with "No such file or directory".

Add the line `# shellcheck source=src/lib.sh` to the script instead.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2019-01-14T03:49:32Z](https://github.com/koalaman/shellcheck/issues/1356#issuecomment-453896652):

That was indeed it. I can reproduce it in a hidden directory. According to the [snap docs](https://docs.snapcraft.io/the-home-interface/7838?_ga=2.24410105.2111026388.1547436586-19277213.1487906982), this is intentional behavior from their side:
>home allows access to non-hidden files in the user’s home ($HOME) directory.

There's not much I can do to change the sandboxing, but the workarounds I've found are:
* `sudo snap install --classic shellcheck` to install without a sandbox
* `mount --bind` in such a way that the file is available without going through a hidden dir.
* Not using `snap`, such as by compiling or downloading a binary

Please reopen or file another issue if this does not work or is not sufficient

I got this error when [executing `shellcheck` with `globstar`](https://travis-ci.org/yakshaveinc/linux/jobs/573358708#L213). The [solution](https://github.com/yakshaveinc/linux/commit/307286785420df64d89f708c7a1c7b2ddf53eb65) was to add `nullglob`.

    $ bash -c 'shopt -s globstar nullglob; shellcheck **/*.{sh,ksh,bash}'
