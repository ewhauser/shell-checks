#### <img src="https://avatars.githubusercontent.com/u/2818758?u=40d36e49d938fda80abd46d6347b9bace1f73dd4&v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2022-08-12 03:25](https://github.com/koalaman/shellcheck/issues/2553#issuecomment-1212693156):

The command is fine as written, since `$(..)` starts a new quoting context. However, if you were to copy-paste it from the script into your crontab, cron will terminate the command at the first unescaped `%` and treat the rest as stdin. (For example, `0 0 * * * nl > file %foo%bar%baz` will run `nl > file` and feed the lines `foo`, `bar`, and `baz` on stdin).

It's certainly a problem, but specific to `cron`'s processing of the command before giving it to the shell. If you put the line in a script and invoke the script via crontab, it will work as expected.
