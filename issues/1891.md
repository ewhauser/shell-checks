#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash
if [ 0 -ne "${FOO}" ] || [ 0 -ne "${BAR}" ]; then
    echo hello
fi
```

#### Here's what shellcheck currently says:

In docker/external/minimal.sh line 2:

#### Here's what I wanted or expected to see:

Not an error.
The wiki page mentions the special case when when the variables foo and bar is the same.
But in this case, I cannot see that this could apply.

Using FOO=0, BAR=0, respectivly FOO=1, BAR=1 must give different outcome for this if statement.

The example below works. So maybe there is some strange thing when I am using yoda-statements (backwards, always write: <constant> <compare-operator> <variable>).
Is this a bug?, I think so, or am I missing something.

Great tool BTW!

```sh
#!/bin/bash
if [ "${FOO}" -ne 0 ] || [ "${BAR}" -ne 0 ]; then
    echo hello
fi
```
