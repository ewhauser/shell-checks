# [\#1295 Issue](https://github.com/koalaman/shellcheck/issues/1295) `open`: Feature request: Allow directives in trailing comments

#### <img src="https://avatars.githubusercontent.com/u/1682574?v=4" width="50">[reidpr](https://github.com/reidpr) opened issue at [2018-07-17T16:36:11Z](https://github.com/koalaman/shellcheck/issues/1295):

I would like to be able to specify directives on the same line as the command in order to save vertical space and reduce clutter.

This is similar to, but not a duplicate of, #1267.

(Thanks for your hard work on ShellCheck. I am finding tons of bugs in my code with it.)

#### For new checks and feature suggestions
- [x] shellcheck.net (i.e. the latest commit) currently gives no useful warnings about this
- [x] I searched through https://github.com/koalaman/shellcheck/issues and didn't find anything related

#### Here's a snippet or screenshot that shows the problem:

```sh
#!/bin/bash

CH_RUN_FILE="$(command -v ch-run)"       # shellcheck disable=SC2034
CH_LIBEXEC="$(ch-build --libexec-path)"  # shellcheck disable=SC2034
```

#### Here's what shellcheck currently says:

(`shellcheck.net` as of right now.)

```sh
$ shellcheck myscript
 
Line 3:
CH_RUN_FILE="$(command -v ch-run)"       # shellcheck disable=SC2034
^-- SC2034: CH_RUN_FILE appears unused. Verify use (or export if used externally).
                                         ^-- SC1126: Place shellcheck directives before commands, not after.
 
Line 4:
CH_LIBEXEC="$(ch-build --libexec-path)"  # shellcheck disable=SC2034
^-- SC2034: CH_LIBEXEC appears unused. Verify use (or export if used externally).
                                         ^-- SC1126: Place shellcheck directives before commands, not after.
```

#### Here's what I wanted or expected to see:

```sh
$ shellcheck myscript
No issues detected!
```

#### <img src="https://avatars.githubusercontent.com/u/13248616?v=4" width="50">[tomasohara](https://github.com/tomasohara) commented at [2021-10-16T16:49:51Z](https://github.com/koalaman/shellcheck/issues/1295#issuecomment-944952393):

Hi, is this hard to implement? Besides clutter, it avoids the checks from applying to subsequent lines. This is what [pylint](https://pylint.pycqa.org/en/latest/user_guide/message-control.html) does.

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2021-10-17T02:48:34Z](https://github.com/koalaman/shellcheck/issues/1295#issuecomment-945039339):

@tomasohara Are you aware that directives already only apply to the immediate next command, and not any subsequent ones? (The exception is directives between the shebang and first command, which are considered filewide)

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) commented at [2021-10-31T20:48:46Z](https://github.com/koalaman/shellcheck/issues/1295#issuecomment-955789131):

The current system seemed like a good approach at the time, and still has some benefits, but in retrospect it was a mistake. It would be tricky to support end-of-line comments right now, especially if they can come after something that's not even a command, such as `do`. I would definitely like to fix this at some point, though there are several other significant rewrites I'd prioritize first.
