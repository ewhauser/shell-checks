# [\#262 Issue](https://github.com/koalaman/shellcheck/issues/262) `open`: Enhancement: Disallow assigning while declaring

#### <img src="https://avatars.githubusercontent.com/u/166207?v=4" width="50">[mietek](https://github.com/mietek) opened issue at [2014-12-01T21:31:23Z](https://github.com/koalaman/shellcheck/issues/262):

Freshly debugged:

``` diff
@@ -570,7 +570,8 @@ restore_updated_cabal_layer () {

    log 'Locating Cabal layers'

-   local updated_name=$(
+   local updated_name
+   updated_name=$(
        list_stored_files "${platform}/${archive_prefix}" |
        sed "s:^${platform}/::" |
        match_updated_cabal_archive_name "${tag}"
```

It would be great if ShellCheck could flag any case where a variable has a value assigned at the same time it is declared.  Doing this loses the exit status of any command used to generate the value, as `$?` will now refer to the successful exit status of `local`.
