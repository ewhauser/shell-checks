Are you accessing GitHub with a Subversion client through the [GitHub Subversion bridge](https://help.github.com/articles/support-for-subversion-clients/)? Most GitHub project maintainers will not be expecting that, and you be sure to should point it explicitly out in bug reports. I recommend [switching to a real Git client](https://help.github.com/articles/set-up-git/), though.

That said, I cannot reproduce this problem. The commit that GitHub calls r874 is b1486ec1e96345b00600ef7d2b17dc3d93a539d4, which has `Version: 0.4.6`.

If you are accessing the shellcheck repository through Subversion somewhere *other* than the GitHub Subversion bridge, that’s even more unexpected, and you should probably report a bug to wherever you found this strange mirror. (But really, you want to just use Git.)
