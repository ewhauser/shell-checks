Hi
In ksh you can use "set --" construct (I am not sure about other shells) as explained here:

set -- 
    parameters which follow this option do not set shell flags, but are assigned to positional parameters.

See:
https://www.mkssoftware.com/docs/man1/set.1.asp

In this case shell check tool incorrectly gives a warning "SC2120 f references arguments, but none are ever passed".
The script looks like this:
# !ksh

function f
{
test="a b"
set -- "$test"
echo "$1"
}

f
