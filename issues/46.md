# [\#46 Issue](https://github.com/koalaman/shellcheck/issues/46) `closed`: Reduce number of false positives for variables in printf strings

#### <img src="https://avatars.githubusercontent.com/u/2818758?v=4" width="50">[koalaman](https://github.com/koalaman) opened issue at [2013-12-03T18:49:59Z](https://github.com/koalaman/shellcheck/issues/46):

There are some legitimate use cases for variables in printf strings, like

```
printf "%${pad_len}s\n" "pad me"
```

We should reduce false positives, e.g. by looking for %s or valid format codes in the printf string, indicating that the user is familiar with them
